# 一条 CPU 核心轨，8 颗电源并肩抬——输出看着很稳，拆开看每一相，才知道谁在偷懒、谁快烧

<div class="article-meta"><span class="badge">📘 模块二 · 电源完整性 PI · 深化篇 · 多相供电与均流</span><span class="badge">⏱ 约 27 分钟</span><span class="badge qr">🔧 含工程师速查</span></div>


一位工程师接手一块笔记本主板，测 CPU 核心轨 VCORE。他把探头挂在 VCORE 的输出去耦电容上，看纹波——**几毫伏，漂亮，合格**；看精度——**落点准，合格**；看瞬态——**过冲在窗口内，合格**。一条输出轨该测的都测了，全绿。

可这块板子跑满负载压力测试，**十几分钟后热保护跳闸、降频**。他上热像仪一看，傻眼了：VCORE 那一圈**八颗功率级里，有一颗明显比旁边热十几度**，其余七颗温温的，就那一颗红得发烫。再拿电流探头**逐相**去抓电感电流，真相出来了——那颗热的相**扛的电流比别人多了快一倍**，其余几相在「偷懒」，全靠它一个人顶。

问题从头到尾，**在输出那一个点上根本看不出来**。因为八颗功率级的输出是**并在一起**的，你在合成点上看到的电压、纹波、精度，是八相**平均**之后的结果——某一相扛得多、某一相扛得少，在合成点上互相填平，看着风平浪静。**只有拆开、逐相去看，才知道这八个人抬得均不均匀。** 那颗先烧的 MOS，就是被「均流不到位」这件合成点照不出来的事，一点点烤糊的。

**这就是本篇要建立的视角：多相供电，体检的不是「输出那一条轨」，而是「组成它的每一相」。** 输出稳只是必要条件，远不是充分条件。真正决定「会不会有一相先烧、能不能扛住大电流瞬态、效率高不高」的，是**逐相均流、相位交错、负载线 AVP** 这几件在合成点上看不见的事。这一篇，我们就把多相 VRM 的这套体检，从原理到实测，一次讲透。

**读完这篇，你能搞懂：**

- ✅ 为什么 CPU/GPU 核心轨要拆成多相——单相扛不动的到底是什么（电流、损耗、电感饱和）；
- ✅ 多相的命门**均流**：逐相电流怎么测、判据卡多少（±5~10%）、均流采样方式（DCR / Rsense / 控制器均流环）差在哪；
- ✅ **相位交错**怎么把输出纹波抵消掉——为什么在占空比 D=k/N 处抵消最好、之间还有「缺口」、**为什么抵消不到零**；
- ✅ 交错还顺带降了**输入电容的 RMS 电流应力**，这也是要量的一项；
- ✅ Intel VR 的**负载线 AVP / Vdroop**（原方案里最容易漏的一项）：**DC 负载线**扫 Vout–Iout 求斜率、**AC/瞬态负载线**快阶跃测过冲下冲，判据是设计 Vdroop（mΩ）；
- ✅ **轻载减相**（phase shedding）怎么验——减相阈值、切换平不平滑；
- ✅ **相间热均衡**怎么用热像仪看；
- ✅ 怎么测——**逐相电感电流探头**（DC 霍尔式测均值+相位、别用只看纹波的 Rogowski）、测前 degauss 归零、math 合成总电流、SW 节点验错相，以及这里也要 **DC 耦合**；
- ✅ 这套多相体检比普通单相 buck，**多出了哪些普通单相根本没有的测试项**。

## 一、先破假设：输出稳 ≠ 每一相都健康

普通单路 buck 的测试，隐含一个默认假设：**一颗稳压器出一条输出，测这条输出就代表了这颗电源的全部。** 对单相 buck，这没错——就它一个人干活，输出好就是它好。

但多相打破了这个假设。它的本质是**N 颗 buck 并联、共同抬一条输出轨**，而这 N 相之间是**有分工的**：

- **有分担**——总电流被拆成 N 份，理想情况每相扛 1/N；谁扛多了谁先热、先老化、先烧；
- **有相位**——N 相不是同时开关，而是**错开 360°/N**（8 相就是每 45° 一相）依次动作；
- **有合成**——你在输出点看到的电压/纹波/精度，是 N 相电流**叠加平均**后的结果，单相的不均衡在这里被填平、看不见；
- **有取舍**——负载轻时可以关掉几相（减相）省电，负载重时全开。

所以「输出那条轨测着合格」只是**必要条件**。这颗多相电源健不健康，取决于组成它的 N 相**各自**扛得均不均、错得齐不齐、热得平不平。开篇那块板子就是活例子：输出的电压、纹波、精度单看全对，但**八相里有一相扛了近两倍电流**——均流违规，那一相被自己烤糊。

![图1：翻车现场——八相 VCORE 输出合成点看着全绿（纹波/精度/瞬态都合格），但热像仪拆开看，八颗功率级里有一颗红得发烫、扛了近两倍电流，其余七相在偷懒。合成点照不出单相的不均衡。](/images/pi02/ill01_stack.png)

**打破的默认假设**：不再是「一颗稳压器、一条输出、测输出即可」，而是「**N 相 × 各自的电流/相位/温度**，且要**拆到每一相**才测得出来」。

## 二、为什么要拆多相：一个人扛不动，喊来八个人接力

给这个视角一个好记的比方：**多相供电，就是一群搬运工接力抬一件超重的货。**

CPU 核心要的电流有多大？高性能桌面/笔记本 CPU 睿频时，VCORE 电流轻松**上百安培**。这么大的电流，交给**单相** buck 会怎样？

- **电感扛不住**：电流全走一颗电感，磁芯容易**饱和**（饱和后电感量骤降、电流失控），得用又大又贵的电感；
- **MOS 和铜损太大**：损耗大致随电流平方走，电流集中在一相，那颗功率级（DrMOS）发热爆表，效率掉、还得堆散热；
- **瞬态跟不上**：单相的电流变化率（dI/dt）受一颗电感限制，大负载阶跃时回不稳。

于是工程上把这一条大电流轨，拆给 **N 个功率级并联**接力：每人只扛总量的 1/N，电感小了、每相发热降了、N 相合力的等效电流变化率也快了。这就是多相。

![图2：搬运工接力抬重物类比——①每人扛一样重（均流）否则有人累趴；②步调错开 360°/N（交错）合力才平稳；③活少让几个先歇（轻载减相）。对应本篇三个命门。](/images/pi02/ill02_porters.png)

类比里三件事对应本篇三个命门：

1. **每人必须扛得一样重**——否则有人累趴（**均流**，命门一）；
2. **步调要错开**——八个人不能同时使劲，错开节奏合力才平稳（**相位交错 / 纹波抵消**，命门二）；
3. **活少时让几个先歇**——轻载关掉几相更省电（**减相**）。

外加大电流轨特有的一项：**负载越重、故意把电压往下压一点**（负载线 AVP / Vdroop，命门三）——留给瞬态更多余量。下面逐项讲透。

## 三、多相怎么工作：错相并联，360°/N 依次动作

先把拓扑说清。N 相 buck **共用一个控制器**、**共用输入和输出**，但每相有自己的**上下桥 MOS（DrMOS）+ 电感**。控制器给 N 相的开关信号**在时间上均匀错开 360°/N**：8 相就是相邻两相错开 45°，一圈 360° 正好八相依次各动一次。

![图3：N 相 buck 错相并联拓扑——共用控制器/输入/输出，每相独立 DrMOS+电感，开关信号错开 360°/N（8 相=每 45°）。下方各相 SW 节点波形依次错开。](/images/pi02/ill03_interleave.png)

错相带来两个直接好处，正是后面要测的：

- **输出纹波互相抵消**：每相电感电流是个三角波纹波，八个三角波错开 45° 叠加，波峰填波谷，合成到输出的**总纹波远小于单相**（命门二细讲）；
- **输入电流更平滑**：八相错开轮流从输入电容取电，输入电容看到的**脉冲电流被摊薄**，RMS 应力下降，输入电容可以用得更小。

**判据形态在这里就变了**：单相你只有「一条输出」一个测量对象；多相你有「**N 相各自的电流波形 + 相位 + 温度**」外加「合成输出」——测量对象从 1 个变成 N+1 个，判据从一行限值变成**逐相平衡表 + 相位表 + 负载线斜率**。

## 四、命门一：逐相均流——谁偷懒，谁的邻居先烧

**均流（Current Sharing / Current Balance）**：N 相里每一相实际扛的电流，应当**尽量相等**，都接近总电流的 1/N。这是多相的头号命门，也是开篇那块板子翻车的地方。

**为什么均流这么要命？** 因为损耗随电流平方走。假设八相里有一相扛了 1.8×平均电流，其余分摊剩下的——那一相的发热是平均相的 **3 倍多**（1.8²≈3.24）。它会更热、结温更高、老化更快，最终**那一相的 MOS 或电感先烧**；烧了之后剩下七相分担更大电流，**雪崩式**接连挂。你在输出点永远看不到这件事，因为八相电流一并联就平均了。

![图4：逐相均流好 vs 坏——左：八相电感电流均值几乎齐平（均流好）；右：一相高高突起扛近两倍、其余偏低（均流坏），突起相发热≈平均相的 3 倍多、先烧。判据：不平衡 (Imax−Iavg)/Iavg ≤±5~10%。](/images/pi02/ill04_sharing.png)

**均流判据**：稳态下逐相电流的不平衡度

```
不平衡度 = (I_max − I_avg) / I_avg
判据：≤ ±5~10%（具体看控制器/平台规格）
```

方案里原来写的 ±10~15% 偏松，现代控制器均流环做得好，**收紧到 ±5~10%** 更贴合实际；究竟卡多少，以控制器 datasheet / 平台 VR 规格为准。

**均流是怎么实现的（决定你怎么测、怎么判）**：控制器要先**感知每一相的电流**，才能把偷懒的那相往上调。感知方式有三种，各有坑：

- **DCR sensing（电感直流电阻采样）**：用一个 RC 网络「读」电感自身的直流电阻上的压降来推算电流。**便宜、无额外损耗，但电感 DCR 随温度漂**（铜温度系数约 +0.39%/℃），温度一高读数就偏，**需要温度补偿**，否则热起来均流就走样——这本身就是一个要留意的测试点；
- **Rsense（串联采样电阻）**：串一颗精密电阻测压降，**准，但有额外损耗和成本**；
- **控制器内部均流环**：有些数字控制器直接用 RDS(on) 采样 + 片内均流算法，读数可通过 PMBus/SVID 遥测读回。

**怎么测均流**：给 VCORE 加一个**稳定的大负载**（比如 60A、100A），用电流探头**逐相**抓每一相的电感电流，读出各相的直流均值，代入上面公式算不平衡度。重点：要在**不同负载点**（轻/中/满）都测——有些设计轻载均流很好、满载才劣化。

## 五、命门二：相位交错与纹波抵消——为什么抵消不到零

**相位交错（Interleaving）**的第一个作用前面说了：N 相开关错开 360°/N。它带来的**纹波抵消**，是多相最漂亮、也最容易被讲错的一件事。

**为什么能抵消**：每相电感电流是围绕其直流均值上下起伏的三角波。N 相错开相位后，某一相电流正往上冲时，另一相正好在往下走，叠加到输出节点时**波峰填了波谷**，合成的总纹波电流远小于单相纹波。八相里这种填补是八重的，所以合成输出纹波可以只有单相的几分之一到几十分之一。

![图5：纹波抵消——上方八相三角波电流错开 45° 叠加，合成总纹波（红）远小于单相（灰）；下方「纹波抵消系数 vs 占空比 D」曲线在 D=1/8、2/8…这些整数点处触底（最小），之间回升成「缺口」，且始终不为零。](/images/pi02/ill05_ripple.png)

**但——关键的技术点，别讲错**：纹波**不是永远抵消到零**。抵消效果和**占空比 D**强相关：

- 当 **D = k/N**（k 为整数，即占空比正好落在 1/8、2/8、3/8…这些点）时，各相纹波在输出处**几何上抵消得最彻底**，合成纹波达到**最小**（理论最小、接近零但工程上仍有残余）；
- 当 D **落在这些整数点之间**时，抵消不完全，合成纹波会**回升**，形成一个个「**缺口**」。

所以正确的说法是：「**交错后输出纹波远小于单相，且在 D=k/N 处最小、其间有回升**」，**绝不能说「多相纹波永远是零」**。VCORE 电压低（0.7~1.1V）、输入 12V，占空比通常很小（~5~9%），一般落在第一个抵消谷附近，但具体要看工作点。测试时值得**扫几个负载/电压点**，看合成纹波随工作点的变化，验证抵消行为符合预期。

**交错的第二个作用：降输入 RMS 电流。** 八相错开轮流从输入取电，输入电容看到的脉冲被摊薄，**输入电容的 RMS 电流应力下降**，可以用更少更小的输入电容。这一项也该量：在输入回路测 RMS 电流，对比单相或全相同步时的差别，确认交错确实降低了输入应力。

## 六、命门三：负载线 AVP / Vdroop——故意让电压随负载往下掉

这是大电流 CPU 核心轨**特有**、普通电源和很多人容易漏的一项（原方案也曾漏，本篇专门补）。

**先破直觉**：你可能以为 VCORE 应该「不管负载多大都死死稳在设定值」。**恰恰相反**——Intel/AMD 的 VR 规范要求 VCORE **故意随负载电流增大而线性下降**一点点，这叫**负载线（Load Line）**，实现手段叫 **AVP（Active Voltage Positioning，主动电压定位）**。

**为什么要故意往下掉？** 为了给**瞬态**留余量。CPU 负载会突然从几安跳到上百安（或反过来）：

- 如果空载就把电压顶在最高，那么**重载瞬间**电压下冲，容易跌破下限 → 死机；
- 如果按负载线，**空载时电压就已经偏高一点、重载时本就该低一点**，于是这条斜线正好把**整个允许的电压窗口**用满，瞬态上冲下冲都落在窗口内，还能**少用输出电容**、省成本省板面。

![图6：DC 负载线——横轴 Iout 从 0 扫到满载，纵轴 Vout 沿一条直线下降，斜率就是负载线电阻 R_LL（=设计 Vdroop，单位 mΩ）。验两件事：斜率对不对、这条线直不直（不能中途弯折）。](/images/pi02/ill06_loadline_dc.png)

**负载线的定义（判据来源）**：

```
V_out = V_ref − I_load × R_LL
```

其中 **R_LL** 就是**负载线电阻 / Vdroop 斜率**，单位毫欧（mΩ），由平台 VR 规格规定（例如某平台 1.3 mΩ）。测试要验的就是**实测斜率 = 设计 Vdroop**。分两种测法：

**① DC 负载线（静态）**：用可编程电子负载，把 Iout **从 0 一档档扫到满载**，每个电流点记录稳定后的 Vout，画一张 **Vout–Iout 图**。这条线应当是**一条直线**，**斜率就是 R_LL**。验两件事：斜率对不对（=设计 Vdroop）、线性好不好（不能中途弯折）。

**② AC / 瞬态负载线（动态）**：用电子负载施加**快速负载阶跃**（数十 A/µs 的 dI/dt），看 Vout 的**过冲和下冲**。理想情况下，加了正确 AVP 的电压瞬态**恰好贴着负载线的目标值落**，过冲/下冲都在规格窗口内；如果 AVP 没调好，瞬态会冲出窗口。这一项直接关系「大负载跳变时会不会死机」，是多相 VRM 的**一等公民**测试项。

![图7：AC/瞬态负载线——负载电流数十 A/µs 快阶跃（上），Vout（下）在跳变瞬间过冲/下冲后，稳态落回负载线目标值。AVP 调好时上冲下冲都在规格窗口（绿带）内；调不好会冲出窗口（红）。](/images/pi02/ill07_loadline_ac.png)

> **合规提示**：Intel VR/SVID 规范属 **CNDA 受限文档**，真实平台的 Vdroop 数值、VID 码表不进公开篇；本篇只讲**通用原理与测法**，具体 mΩ 值以你手上的平台规格为准、只留内部。

## 七、轻载减相（Phase Shedding）：活少了，让几个先歇

多相全开虽然扛得动大电流，但**轻载时**每相都开着并不划算——开关损耗、驱动损耗白白摊在很小的负载上，效率反而低。于是控制器会在**负载低于某阈值**时**关掉几相**（8 相可能只留 1~2 相干活），叫**减相 / phase shedding**；负载回升再**加相**。

![图8：轻载减相——横轴负载电流，纵轴工作相数呈阶梯：轻载只留 1~2 相、负载升高逐级加相到满相。切换点（阶梯边沿）处输出电压要平滑、无毛刺跌落。](/images/pi02/ill08_shedding.png)

**要验的点**：

- **减相/加相阈值**：在什么负载点减到几相、什么点加回来，符不符合设计；
- **切换是否平滑**：减相/加相的那一瞬间，**输出电压不能有明显扰动**（毛刺、跌落）——切换没做好会在负载临界点反复横跳（抖动）或产生电压尖峰；
- **减相后仍要均流**：剩下的几相之间同样要均流，别以为减了相就不用管。

**怎么测**：用电子负载**缓慢扫负载**、同时逐相看电感电流（哪些相在减相点熄火/点亮）、并盯输出电压有没有在切换点抖动。

## 八、各相热均衡：热像仪一照，偷懒的相无处遁形

均流的最终体现是**温度**。八相扛得均，八颗功率级就**温度接近**；某相偷懒扛得多，它就是**热点**。所以**热像仪看相间温差**，是均流最直观的交叉验证——开篇那块板子，就是热像仪先发现那颗红得发烫的相。

![图9：相间热均衡——左：八颗功率级热像温度接近（均流好，全青绿）；右：一颗明显发红热点、其余偏冷（均流坏）。热点相通常就是电流最大的相，与逐相电流数据互相印证。](/images/pi02/ill09_thermal.png)

**要验的点**：

- 满载稳态下，**相间温差**在设计范围内（几℃ 内，具体看规格）；
- 没有单独某相是明显热点；
- 结合逐相电流：热点相是不是也是电流最大的相（互相印证均流数据）。

**注意**：热还受布局、散热、气流影响，不能只凭温差下均流结论——**热像仪是均流的旁证，电流探头逐相实测才是主证**，两者对上才踏实。

## 九、怎么测：逐相电感电流探头 + math 合成 + DC 耦合

多相体检的测法，有几个和普通单相测试**很不一样**的关键点。

![图10：多相测法总图——每相电感端子各挂一个 DC 霍尔式电流探头（测前 degauss），示波器 math 把 N 相电流相加得合成总电流；叠加各相 SW 节点验错相时序；电压侧用有源轨探头 DC 耦合测负载线；热像仪看相间温差。](/images/pi02/ill10_measure.png)

**关键点一：逐相电流，要用对探头。**
测相电流的核心是**每一相的电感电流**（幅值+相位+直流均值）。探头选型是本篇最大的坑：

- 要读**含直流均值**的相电流（算均流必须有 DC 分量）→ 必须用 **DC 耦合的霍尔式电流探头**（如 Tek TCP0030A、Keysight N2783 类），它能测到从直流到几十 MHz 的完整电流；
- 只想看**开关纹波/边沿**（不关心 DC）→ 可用 **Rogowski 线圈**（高带宽、不饱和，但**天生测不到直流**，需积分器）——**测均流千万别用它**，因为它读不出直流均值，均流数据无从谈起。
- 探头要**夹在电感的端子/引脚**上（不是随便夹一段走线），**测前务必 degauss（消磁）归零**，否则直流读数带偏置、均流全错。

**关键点二：用示波器 math 合成总电流、验相位交错。**
逐相抓下 N 相电感电流后，用示波器的 **math 功能把它们相加**，得到合成的总输出电流——对照它和各相的关系，能直接看出：各相相位是不是均匀错开 360°/N、有没有哪一相幅值明显偏大（偷懒的反面：过载相）、合成纹波是不是远小于单相。再叠加各相 **SW 节点**（开关节点）电压，能直观验证**错相**时序对不对。

**关键点三：这里也要 DC 耦合——和第 08 篇测纹波恰好相反。**
和深化 01、03 一样，这里要特别说清，避免和第 08 篇打架：

- **第 08 篇测输出纹波**：VCORE 是「大直流 + 叠在上面的几毫伏小交流」，为看清小交流用 **AC 耦合**挡掉大直流——**第 08 篇这么做完全正确，不用改**；
- **本篇测负载线 / 相电流均值**：我们要测的恰恰是**直流电平本身**（Vout 随 Iout 怎么降、每相直流电流是多少）。用 AC 耦合，它的 ~10Hz 高通会把这些**直流信息滤掉**，负载线和均流就都没了。
- 所以本篇**电压侧用 DC 耦合 + 电源轨探头偏置**（看 Vout–Iout 直流关系），**电流侧用 DC 耦合霍尔探头**（读相电流直流均值）。

> **一句话记牢**：第 08 篇测「大直流上的小交流」→ 用 AC 挡直流；本篇测「直流本身（负载线、相电流均值）」→ 必须 DC 耦合。**目的相反，耦合方式就相反，两篇都没错。**

**仪器与探头小结**：多通道示波器（够通道逐相抓，通道不足则逐相挪测、共时基拼）；**DC 霍尔式电流探头**（测相电流均值+相位，测前 degauss）；有源电源轨探头（DC 耦合+偏置，测负载线电压侧）；可编程电子负载（DC 扫负载 + 数十 A/µs 快阶跃）；热像仪（相间热均衡）；瞬态看 ≥20MHz 带宽。

## 十、和普通单相 buck 比，多出了哪些活

把多相 VRM 和普通单相 buck 摆一起，看它打破了普通电源测试的哪个默认假设、多出了哪些活：

| 维度 | 普通单相 buck | 多相 VRM（新增） |
|---|---|---|
| 被测本质 | 一颗稳压器出一条输出 | **N 颗 buck 并联抬一条轨** |
| 测量对象 | 1 个（输出） | **N+1 个（N 相各自 + 合成输出）** |
| 判据形态 | 一行固定限值 | **逐相平衡表 + 相位表 + 负载线斜率** |
| 均流 | 无（就一相） | **逐相不平衡 ≤±5~10%**（相对判据） |
| 相位 | 无 | **N 相交错 360°/N**，验纹波抵消（D=k/N 最佳、有缺口） |
| 负载线 | 一般不强调 | **DC + AC 负载线，斜率 = 设计 Vdroop（mΩ）** |
| 减相 | 无 | **减相阈值 + 切换平滑** |
| 热 | 单点 | **相间热均衡（热像仪）** |
| 探头 | 普通电压/电流探头 | **DC 霍尔式逐相电流探头（degauss）+ 有源轨探头** |
| 典型失效 | 纹波/瞬态超标 | **一相偷懒 → 过热先烧、AVP 没调好 → 瞬态冲出窗口、减相点抖动** |

**多出的活一句话**：**逐相均流 + 相位交错/纹波抵消 + 负载线 AVP（DC+AC）+ 轻载减相 + 相间热均衡** ——这些都是「N 相并联抬一条轨」才有、单相 buck 里根本不存在的测试项。这就是你从「测一条输出」过渡到「体检一组多相」，要补上的那份增量清单。

**放到三篇一起看**：深化 01 破的是「电压是定值」这条假设（电压其实是**动态变量**），深化 02 破的是「测输出就够了」这条假设（要**拆到每一相**），深化 03 破的是「一颗管一条轨」这条假设（一颗芯片/一套 VRM 管**几十条轨的系统调度**）。三篇合起来，就是从「单点、定值、单相」的旧框架，升级到「矩阵、动态、多相、系统」的现代供电测试观。

---

## 一句话总结

**多相供电，体检的是「组成输出的每一相」，不是「输出那一条轨」。** 输出稳只是必要条件——真正决定会不会有一相先烧、能不能扛住大电流瞬态的，是**逐相均流（≤±5~10%）、相位交错与纹波抵消（D=k/N 最佳、抵消不到零）、负载线 AVP/Vdroop（DC+AC，斜率=设计 mΩ）、轻载减相、相间热均衡**。测法上要**逐相 DC 霍尔电流探头**抓相电流均值（测前 degauss）、math 合成验交错、有源轨探头 DC 耦合扫负载线（第 08 篇测纹波才用 AC，目的相反）。主角是 x86 VCORE 8 相 VRM，高通 PMIC 少相集成、原理相同。涉密的真实 Vdroop/VID 只留内部，公开只讲通用原理与方法。

## 🔧 工程师速查（附录）

### 核心概念一句话

| 概念 | 一句话 | 备注 |
|---|---|---|
| **多相 VRM** | N 颗 buck 并联共抬一条大电流轨 | 测的是每一相，不是只测输出 |
| **均流** | 每相扛的电流应接近总电流 1/N | 不平衡 ≤±5~10%；偷懒相过热先烧 |
| **相位交错** | N 相开关错开 360°/N | 8 相 = 每 45° 一相 |
| **纹波抵消** | 交错使合成输出纹波 ≪ 单相 | **D=k/N 处最小、之间有缺口，抵消不到零** |
| **输入 RMS 电流** | 交错摊薄输入脉冲、降输入电容应力 | 也要量 |
| **负载线 / AVP** | Vout 故意随 Iout 线性下降 | 给瞬态留余量、省输出电容 |
| **Vdroop 斜率 R_LL** | Vout = Vref − Iload×R_LL | 单位 mΩ，=设计值；DC+AC 都验 |
| **减相 phase shedding** | 轻载关几相省电 | 验阈值 + 切换平滑 |
| **DCR sensing** | 用电感直流电阻推算相电流 | 便宜但需**温度补偿**（DCR 随温漂） |
| **相间热均衡** | 满载各相温度接近 | 热像仪；均流的旁证 |

### 多相体检判据矩阵（速查层皇冠）

| 测试项 | 测什么 | 判据 | 测法/探头 |
|---|---|---|---|
| 逐相均流 | 各相电感电流直流均值 | 不平衡 (Imax−Iavg)/Iavg ≤ ±5~10% | DC 霍尔电流探头逐相（degauss），轻/中/满载都测 |
| 相位交错 | 各相开关相位 | 均匀错开 360°/N | 逐相电流 + SW 节点，math 看时序 |
| 纹波抵消 | 合成输出纹波 vs 单相 | 远小于单相；D=k/N 最小、有缺口（非零） | 合成电流 math + 输出纹波（AC 耦合，20MHz 限带） |
| 输入 RMS 电流 | 输入电容应力 | 交错后下降 | 输入回路电流探头 |
| DC 负载线 | Vout–Iout 曲线 | 直线，斜率 = 设计 Vdroop（mΩ），线性 | 电子负载 DC 扫，有源轨探头 DC 耦合 |
| AC/瞬态负载线 | 快阶跃过冲/下冲 | 落在规格窗口、贴负载线目标 | 电子负载 数十 A/µs，≥20MHz 带宽 |
| 轻载减相 | 减相/加相阈值、切换 | 阈值符合设计、切换平滑无抖动 | 缓扫负载，逐相看熄火/点亮 + 输出电压 |
| 相间热均衡 | 满载相间温差 | 在设计范围（几℃）内 | 热像仪，与逐相电流互印证 |

### 测量设置速记

- **电压侧**：有源电源轨探头，**DC 耦合 + 偏置**（测负载线/均值），瞬态看 ≥20MHz。
- **电流侧**：**DC 霍尔式**电流探头逐相（测均值+相位，**测前 degauss 归零**），夹电感端子；只看纹波边沿才用 Rogowski（无 DC，测均流不可用）。
- **激励**：可编程电子负载，DC 逐档扫（画负载线）+ 数十 A/µs 快阶跃（瞬态负载线）。
- **合成**：示波器 math 把 N 相电流相加 → 验交错/抵消/过载相。
- **热**：热像仪看相间温差，作均流旁证。

### 用例清单（拿去套）

1. 逐相均流 @轻/中/满载：算不平衡度，≤±5~10%。
2. 相位交错：验 360°/N 均匀，无相幅值异常。
3. 纹波抵消：扫工作点，合成纹波 ≪ 单相，验 D=k/N 行为。
4. 输入 RMS 电流：确认交错降低输入应力。
5. DC 负载线：扫 Vout–Iout，斜率=设计 Vdroop、线性。
6. AC 瞬态负载线：数十 A/µs 阶跃，过冲/下冲在窗口。
7. 轻载减相：阈值 + 切换平滑 + 减相后仍均流。
8. 相间热均衡：满载温差 + 与电流互印证。

### vs 普通单相 buck

- 普通单相：测 1 条输出、一行限值就完事。
- 多相多出：**逐相均流 + 相位/纹波抵消 + 负载线 AVP(DC+AC) + 减相 + 相间热** ——单相根本没有的项；测量对象从 1 个变 N+1 个。

### 常见坑

- ❌ 只测输出、不拆相 → 一相偷懒过热先烧，输出点照不出来。
- ❌ 用 Rogowski 测均流 → 读不到直流均值，均流数据无效。
- ❌ 电流探头没 degauss → 直流偏置，均流全错。
- ❌ 说「多相纹波抵消到零」→ 错，D=k/N 最小、之间有缺口、始终有残余。
- ❌ 漏测负载线 AVP → 大负载瞬态冲出窗口、死机查不到根因。
- ❌ 用 AC 耦合测负载线/相电流均值 → 直流被滤掉，全测不到。
- ❌ DCR sensing 不做温补 → 热起来均流走样。
