先说个几乎人人都遇到过、却很少深究的场景。

你的笔记本、主板明明写着「千兆网口」，插上网线一看，**只协商到了 100 Mbps**，甚至掉到 10M；或者能上千兆，但**一拷大文件网速就抖、丢包、延迟飙高**；再或者，**换了根网线、或者把水晶头重新压一下，一切又好了**。很多人第一反应是「网卡坏了」「运营商限速」。

但做硬件的人清楚：**这类「千兆只跑百兆、网速上不去、换根线就好」里，有一大半根本不是网卡坏、不是限速，而是以太网这条物理链路——PHY 的发送波形、那颗变压器、RJ45 连接器、还有那段双绞线的回损和串扰——没测好、没匹配好。** 网线质量差一点、水晶头压得松一点、走线阻抗不连续一点，**回损一大、串扰一大，自协商就只能往下退档保命**——千兆退百兆、百兆退十兆，正是这么来的。

这条链路怎么从 10M 一路翻到 10G、为什么它走双绞线还要先过变压器、为什么测它的发送端非得让芯片进「测试模式」——这，就是这一篇要讲的，也是我们**从机内接口走向机外接口**的第一篇。

## 一、先认清主角：以太网 BASE-T 是什么、它和机内高速接口到底不一样在哪

**以太网 BASE-T** 就是你用 **RJ45 水晶头 + 双绞线（网线）** 连的那种有线网络；「BASE」指基带传输、「T」指双绞线（Twisted pair）。它连接的两端是：设备里的 **以太网 PHY 芯片**（网卡 / 主板 / 交换机里），和另一端的**对端 PHY**（路由器、交换机、另一台电脑）。中间的链路是：**PHY → 一颗网络变压器（magnetics）→ RJ45 连接器 → 双绞线 → 对端**。

它和前面所有机内高速接口，有**四个根本区别**，正好是这一篇的测试主线：

1. **走出机箱、上双绞线，中间还隔着一颗变压器**：机内接口是板上差分走线直连；以太网的信号要先**经过一颗网络变压器做磁耦合**，才上网线。这颗变压器带来**直流隔离、共模抑制、防雷防浪涌**，但也意味着你**测量点在变压器之后的 MDI（介质相关接口，就是 RJ45 那一侧）**，测的是**过了磁耦合、准备上线**的信号。
2. **基带多电平 PAM，不是串行 NRZ 差分**：机内高速接口大多是一对差分线跑 NRZ（两电平）、靠提高单线速率堆带宽。以太网走的是**基带 + 多电平 PAM**——**Manchester（10M）→ MLT-3 三电平（100M）→ PAM5 五电平（千兆）→ PAM16 十六电平（2.5G 及以上）**，靠「一个符号塞更多电平」来提速率。这条路和 PCIe Gen6、USB4 v2 后来冲进 PAM 是**殊途同归**，但以太网**早在千兆时代就 PAM5** 了。
3. **同一根线上同时双向收发（千兆起）**：百兆及以下是「一对发、一对收」分开的；但**千兆及以上，四对线每一对都同时既发又收**——线上是**你的发送信号 + 对端来的接收信号 + 自己的回声 + 相邻对的串扰**全叠在一起。设备靠 **hybrid（混合电路）+ 回声抵消 + 近端串扰（NEXT）抵消** 这套 **DSP** 把它们分开。
4. **测发送端靠「测试模式」，不是插上就看眼**：正因为线上是双向叠加、看不到干净的发送信号，IEEE 802.3 专门定义了**发送测试模式**——让 PHY 停止正常收发、**发一段规定的已知码型**，你才能在 MDI 上抓到干净波形、去套模板量指标。**这是以太网和机内接口在测试方法上最大的不同。**

一句话：**以太网 BASE-T = PHY 经变压器磁耦合、上双绞线、跑基带多电平 PAM、千兆起同线双向的机外网络链路**；它测的是**波形模板 + 下垂 + 抖动 + 失真 + 回损**，而且**必须先让芯片进测试模式**——把你前面机内那套「插上看眼图靠 CDR」的思路，又换了一遍。

## 二、原理与演进：基带 PAM 多电平 + 同线双向 + hybrid 抵消，为什么非要测试模式

### 2.1 基带多电平 PAM：靠「电平数」提速率，不是靠串行拉单线速率

以太网从一开始就走**基带 + 多电平**这条路：不把信号搬到某个高频载波上，直接在双绞线上跑**多个电压电平**的方波信号。

- **10BASE-T**：**Manchester 编码**，两电平，每比特中间强制一次跳变（自带时钟），简单可靠。
- **100BASE-TX**：**4B/5B + MLT-3**，MLT-3 是**三电平**（-1 / 0 / +1）、边沿平缓的编码，把频谱压低、好过 Cat5 线，符号率 125 Mbaud。
- **1000BASE-T**：**PAM5**（五电平），四对线并行、每对 125 Mbaud，还带 **Trellis 网格编码**做前向纠错，靠电平数和四对并行把带宽做到千兆。
- **2.5G / 5G / 10GBASE-T**：**PAM16**（十六电平），配 **LDPC 前向纠错 + THP（汤姆林森-哈拉希玛预编码）** 对抗码间干扰，符号率 200 / 400 / 800 Mbaud。

**一条主线：以太网提速率靠的是「加电平 + 加纠错 + 用满四对」，而不是像机内串行那样把单线 NRZ 速率无限往上拉。** 所以它的信号从来不是「一只干净的两电平眼」，而是**三电平、五电平、十六电平**——测起来天生就是「多电平」那种复杂。

### 2.2 同线双向 + hybrid + 回声 / 串扰抵消：线上是一锅粥

千兆及以上最关键的一点：**四对线，每一对都同时收发**。发送信号出去的同时，对端的信号也从同一对线回来，加上自己发出去在阻抗不连续处反射回来的**回声**、还有相邻三对线耦合过来的**近端串扰（NEXT）**——**线上电压是这四样叠加的一锅粥**。

设备靠一套 **DSP** 把它拆开：**hybrid（混合电路）**先大致分离收发，**回声抵消器**减掉自己的回声，**NEXT / FEXT 抵消器**减掉串扰，**均衡器**补偿线缆损耗。**这套 DSP 抵消，是千兆以太网能在一对线上双向跑起来的核心**，也是为什么它对**回损、串扰**这么敏感——抵消器是按理想信道设计的，**回损 / 串扰一旦超标，抵消不干净，误码就上来、只能退档。**

### 2.3 为什么非要「测试模式」：正常工作时你看不到干净的发送眼

正因为线上是双向叠加的一锅粥，**你在正常通信时，根本没法在 MDI 上抓到一只干净的发送波形**——抓到的是收发混在一起的东西。所以 **IEEE 802.3 为发送端一致性专门定义了「测试模式」**：命令 PHY **停止正常收发、单向发出一段规定的已知码型**（比如 1000BASE-T 的 Test Mode 1～4，分别用来测波形、抖动主 / 从、失真）。只有进了测试模式，你才能在 MDI 上抓到**确定、可重复**的波形，去套模板、量下垂、量抖动。**「进测试模式」是以太网发送端测试绕不开的第一步**，这一点和机内接口的「进 compliance 模式」思路相通，但以太网**更是非它不可**。

![图3：以太网三大原理。①基带多电平 PAM：Manchester(10M,2 电平)→MLT-3(100M,3 电平)→PAM5(千兆,5 电平,四对+Trellis 纠错)→PAM16(2.5/5/10G,16 电平,LDPC+THP)，符号率 125/125/200/400/800 Mbaud；提速率靠加电平+加纠错+用满四对，不是拉单线 NRZ 速率。②同线双向一锅粥：千兆起四对每对同时收发→线上=发送+接收+回声+NEXT 串扰叠加，靠 hybrid+回声抵消+NEXT/FEXT 抵消+均衡这套 DSP 拆开；回损/串扰超标→抵消不干净→误码退档。③为什么要测试模式：正常通信线上双向叠加抓不到干净发送眼→IEEE 802.3 定义发送测试模式(如 1000BASE-T Test Mode 1-4)让 PHY 停收发单向发已知码型，才能套模板量下垂/抖动/失真](/images/50/ill03_arch.png)

## 三、版本与速率：10BASE-T 到 10GBASE-T

以太网双绞线家族都归 **IEEE 802.3** 管，一代一个 Clause，一张表理清：

| 标准 | 速率 | 编码 / 电平 | 符号率 | 线对 | 线缆 | IEEE Clause |
|---|---|---|---|---|---|---|
| **10BASE-T** | 10 Mbps | Manchester（2 电平） | 10 MHz | 2 对（1 收 1 发） | Cat3 | Clause 14 |
| **100BASE-TX** | 100 Mbps | 4B/5B + MLT-3（3 电平） | 125 Mbaud | 2 对（1 收 1 发） | Cat5 | Clause 25 |
| **1000BASE-T** | 1 Gbps | PAM5（5 电平）+ Trellis | 125 Mbaud | **4 对同线双向** | Cat5e | Clause 40 |
| **2.5GBASE-T** | 2.5 Gbps | PAM16 + LDPC | 200 Mbaud | 4 对同线双向 | Cat5e | Clause 126 |
| **5GBASE-T** | 5 Gbps | PAM16 + LDPC | 400 Mbaud | 4 对同线双向 | Cat6 | Clause 127 |
| **10GBASE-T** | 10 Gbps | PAM16 + LDPC + THP | 800 Mbaud | 4 对同线双向 | Cat6a | Clause 55 |

几个测试上必须记住的点：

- **千兆是分水岭**：百兆及以下是**两对线、收发分开**，测起来相对简单（一只三电平 / 两电平的发送波形）；**千兆起变成四对线、同线双向、上 DSP 抵消**——从此发送端测试**非进测试模式不可**，还多出**回损、串扰**这些「线上一锅粥」带来的判据。
- **2.5G / 5G 是「降频版 10G」**：2.5GBASE-T 和 5GBASE-T（合称 NBASE-T，IEEE 802.3bz）就是把 **10GBASE-T 的 PHY 降频**——符号率取 10G 的 **1/4（200M）和 1/2（400M）**，编码同为 PAM16。它们的意义是**让 Cat5e / Cat6 这些老线也能翻上 2.5G / 5G**，正好卡在笔记本、Wi-Fi 6/7 AP 这些场景。
- **线缆等级越往上越关键**：10M 用 Cat3 就行，千兆要 Cat5e，10G 要 Cat6a——**速率越高，对线缆的回损、串扰、损耗要求越苛刻**，这也是「换根好线就好了」的物理原因。
- **符号率 ≠ 速率**：像 1000BASE-T，每对只有 125 Mbaud，靠**五电平 + 四对并行 + 双向**才做到千兆；选示波器带宽、算指标要按**符号率和电平数**来，别被「千兆」这个数字带偏。

![图4：以太网 BASE-T 版本速率阶梯。10BASE-T(10M·Manchester 2 电平·10MHz·2 对·Cat3·C14)→100BASE-TX(100M·4B5B+MLT-3 3 电平·125Mbaud·2 对·Cat5·C25)→1000BASE-T(1G·PAM5 5 电平+Trellis·125Mbaud·4 对同线双向·Cat5e·C40)→2.5GBASE-T(2.5G·PAM16+LDPC·200Mbaud·Cat5e·C126)→5GBASE-T(5G·PAM16·400Mbaud·Cat6·C127)→10GBASE-T(10G·PAM16+LDPC+THP·800Mbaud·Cat6a·C55)。底部三条：①千兆是分水岭（百兆及下 2 对收发分开;千兆起 4 对同线双向+DSP 抵消,发送测试非进测试模式不可,多出回损/串扰判据）②2.5G/5G=降频版 10G(NBASE-T/802.3bz,符号率取 10G 的 1/4、1/2,让 Cat5e/6 老线翻上去）③符号率≠速率(1000BASE-T 每对仅 125Mbaud,靠 5 电平×4 对×双向做到千兆)。都归 IEEE 802.3](/images/50/ill04_versions.png)

## 四、独特之处：变压器磁耦合 + 测试模式 + 回损 / 共模 + 自协商 / MDI-X

以太网 BASE-T 有四处机内接口没有、直接影响测试的独特设计：

- **① 网络变压器（magnetics）磁耦合**：信号上线前先过一颗变压器，做**直流隔离、共模抑制、防雷击浪涌**。它让测量点定在**变压器之后的 MDI**；也带来一个独有指标——**低频下垂（droop）**：变压器传不了直流，一段持续同电平的码型电压会**慢慢往下掉**，掉太多就判不对电平，所以 IEEE 专门规定了**下垂上限**。
- **② 发送测试模式（Test Mode）**：前面讲过，测发送端**非进测试模式不可**。以 1000BASE-T 为例，Clause 40 定义了 **Test Mode 1（发送波形 / 模板）、Test Mode 2（主时钟抖动）、Test Mode 3（从时钟抖动）、Test Mode 4（发送失真 distortion）**——四种模式各测一项，通过 PHY 寄存器切换。
- **③ 回损（return loss）+ 共模 / EMI**：因为要在一对线上双向跑、还要过长网线，**MDI 的回损（阻抗匹配得好不好、反射大不大）** 是核心判据之一——回损差，DSP 的回声抵消就吃力；同时网口是 EMI 大户，**共模噪声、辐射发射**也要压住。这两项在机内接口里没这么突出。
- **④ 自协商（Auto-Negotiation）+ 自动翻转（MDI/MDI-X）**：上电后两端先用**低速链路脉冲（FLP）互报能力**，协商出双方都支持的最高速率和双工——**「千兆口只协商到百兆」就是自协商在链路质量不行时主动退档**；再加 **Auto MDI-X**，自动判断该用直连还是交叉，所以现在网线不用分交叉线了。

一句话：**变压器带来下垂这个独有指标、测试模式是发送端测试的前提、回损 / 共模是「上线 + 双向」的核心判据、自协商则是「只跑到百兆」这类现象的直接源头。**

![图5：以太网 BASE-T 四大独特。①网络变压器磁耦合：上线前过变压器做直流隔离/共模抑制/防浪涌→测点定在变压器后 MDI；带来独有指标"低频下垂 droop"（变压器传不了直流,持续同电平会慢慢掉压,IEEE 规定下垂上限）。②发送测试模式：测发送非进测试模式不可,1000BASE-T Clause 40 定义 Test Mode 1 波形/模板、2 主抖动、3 从抖动、4 发送失真,PHY 寄存器切换。③回损 return loss+共模/EMI：一对线双向+过长线→MDI 回损(阻抗匹配/反射)是核心判据,回损差则回声抵消吃力;网口 EMI 大户,共模/辐射也要压。④自协商 Auto-Neg+Auto MDI-X：上电用低速 FLP 脉冲互报能力协商最高速率/双工("千兆只协商到百兆"=链路差主动退档),自动翻转直连/交叉(不用分交叉线)](/images/50/ill05_unique.png)

## 五、测什么 · 三类：发送端波形（靠测试模式）、回损 / 共模、线缆通道

以太网 BASE-T 的一致性测试，也能归成三大类，**但主角是波形模板和回损，不是机内那套 CDR 抖动**：

1. **发送端波形（Transmitter，靠测试模式）**：让 PHY 进测试模式发已知码型，在 MDI 抓波形，套 IEEE 802.3 的判据。重点量：**波形模板 / 模板（peak 电压、波形落在模板内）、低频下垂（droop）、发送抖动（jitter，主 / 从时钟）、发送失真（distortion）、上升 / 下降时间、电平对称性**。不同速率用不同 Clause 的模板：100M 看 Clause 25 的眼图 / 模板，千兆看 Clause 40 的 Test Mode 波形，10G 看 Clause 55 的发送指标。
2. **回损 + 共模 / EMI（MDI 端口特性）**：用**网络分析仪**测 **MDI 回损（return loss）**——反射有多大、阻抗匹配好不好，这直接决定 DSP 回声抵消干不干净；再测 **共模抑制、共模噪声、辐射 / 传导发射（EMI）**。这一类机内接口里没这么重，以太网里却是**退档、丢包的常见根因**。
3. **线缆 / 通道（Channel）**：网线本身要按 **TIA / ISO 布线标准**测**插入损耗、回损、近端 / 远端串扰（NEXT / FEXT）、传播时延、时延偏斜**——这归线缆认证，但和 PHY 一致性**共用一套「信道预算」思想**（↔第 38 篇）：PHY 的模板和限值，正是**给线缆和连接器留够余量后**反推出来的。**「换根好线就好了」，本质就是把这一类的余量补回来了。**

贯穿三类的共同度量，全是**机外双绞线的东西**：波形模板、下垂、失真、回损、串扰、共模、EMI。**抖动只是发送端里的一项，不是主角**——这和机内串行「抖动是天王山」正好调了个位置。

![图6：以太网测什么三类（主角是波形模板+回损,不是 CDR 抖动）。①发送端波形（靠测试模式）：PHY 进测试模式发已知码型,MDI 抓波形套 IEEE 802.3 判据,量波形模板/peak 电压/低频下垂 droop/发送抖动(主从时钟)/发送失真/上升下降/电平对称;100M 看 C25 模板、千兆看 C40 Test Mode 波形、10G 看 C55。②回损+共模/EMI（MDI 端口）：网分测 MDI 回损(反射/阻抗匹配→决定回声抵消干不干净)+共模抑制/共模噪声/辐射传导发射;退档丢包常见根因。③线缆/通道：网线按 TIA/ISO 测插损/回损/NEXT/FEXT/时延偏斜,与 PHY 共用信道预算思想(↔38),"换根好线就好"=补回余量。共同度量:模板/下垂/失真/回损/串扰/共模/EMI——抖动只是其一,和机内正好调个位置](/images/50/ill06_test3.png)

## 六、怎么测 · 台面：MDI / RJ45 夹具 + 进测试模式 + 示波器套模板 + 网分测回损

以太网 BASE-T 的测试台面，和机内那套「测试点 + 探头 + 一致性软件」思路相通，但器件和入口都换成了双绞线世界的：

- **① MDI / RJ45 测试夹具**：用专用的 **RJ45 测试夹具 / 转接板**，从**变压器之后的 MDI**（就是网口那一侧）把四对差分线引出来，接示波器或网络分析仪。测的是**过了磁耦合、准备上线**的信号，而不是芯片管脚。
- **② 让 PHY 进测试模式**：通过 **MDIO / MII 管理接口写 PHY 寄存器**，命令它进入规定的测试模式（如 1000BASE-T 的 Test Mode 1～4），持续输出对应的已知码型，锁定在目标速率档。**这一步是发送端测试的前提**，没有它就只能抓到双向叠加的一锅粥。
- **③ 示波器抓波形、套模板**：示波器带宽按**符号率**选够（10GBASE-T 800 Mbaud 需要较高带宽），抓 MDI 波形，用示波器的**以太网一致性软件**自动套 IEEE 802.3 模板，量**峰值、下垂、抖动、失真、上升下降**，出通过 / 不通过 + 裕量。
- **④ 网络分析仪测回损 / 阻抗**：换上**矢量网络分析仪（VNA）**，测 MDI 的**回损（S 参数）、阻抗匹配**；EMI 则上**EMI 接收机 / 频谱仪**测辐射和传导。
- **⑤ 自协商 / 互操作**：用**协议 / 链路测试仪 + 各种对端设备（交换机、不同长度和等级的网线）**，验自协商能不能稳定协商到目标速率、长距离和劣质线下会不会退档、丢包。

一句话：**以太网台面 = RJ45 夹具从变压器后的 MDI 引出信号 + 写寄存器让 PHY 进测试模式 + 示波器套模板量发送波形 + 网分量回损 + 自协商互操作**；示波器和「套模板」的概念和机内相通，但入口是 MDI、主角是模板和回损，且**非进测试模式不可**。

![图7：测以太网的台面。①MDI/RJ45 夹具：专用 RJ45 夹具/转接板从变压器后的 MDI（网口侧）引出四对差分线接示波器/网分,测过磁耦合准备上线的信号非芯片管脚。②让 PHY 进测试模式：MDIO/MII 管理接口写 PHY 寄存器进规定测试模式(如 1000BASE-T Test Mode 1-4)持续发已知码型锁速率档——发送测试前提。③示波器抓波形套模板：带宽按符号率选够(10G 800Mbaud 要高带宽),以太网一致性软件自动套 IEEE 802.3 模板量峰值/下垂/抖动/失真/上升下降出 pass/fail+裕量。④网分测回损/阻抗：VNA 测 MDI 回损 S 参数/阻抗匹配,EMI 上接收机/频谱仪测辐射传导。⑤自协商/互操作：链路测试仪+各种对端设备/网线验协商到目标速率、长距离劣线会不会退档丢包。一句话=RJ45 夹具引 MDI+写寄存器进测试模式+示波器套模板+网分量回损+互操作](/images/50/ill07_bench.png)

## 七、判据：IEEE 802.3 一致性 —— 波形模板 + 下垂 + 抖动 + 失真 + 回损，从线缆信道预算反推

以太网 BASE-T 的判据，全部来自 **IEEE 802.3 标准**，一代一个 Clause（10M = Clause 14，100M = Clause 25，千兆 = Clause 40，2.5G / 5G = Clause 126 / 127，10G = Clause 55），核心是这几块：

- **发送波形模板（Transmit template / mask）**：在时间 - 电压平面上画一个模板，发送波形必须**落在模板里**；同时规定**峰值电压、电平对称性、上升 / 下降时间**。
- **低频下垂（droop）**：变压器磁耦合带来的独有判据——持续同电平码型的电压**下掉不许超过规定百分比**。
- **发送抖动（jitter）与发送失真（distortion）**：如 1000BASE-T 的 Test Mode 2 / 3 量主 / 从时钟抖动、Test Mode 4 量发送失真，都有明确限值。
- **MDI 回损（return loss）**：在工作频段内**反射不许超过一条限值曲线**——这是保证 DSP 回声 / 串扰抵消能工作的前提。

这些模板和限值同样是**从系统信道预算反推**出来的：把**双绞线的插入损耗、回损、NEXT / FEXT 串扰、连接器、变压器**这些机外通道的损伤全算进一个预算，扣掉这些余量，剩下的就是 PHY 发送端必须守住的窗口。**和机内不同的是：机内串行预算里的大头是抖动和均衡；以太网预算里的大头是回损、串扰、和线缆损耗。** 这跟前面 DP 分物理层 / 链路层两本、UFS 分 M-PHY / 协议两本，是同一个「电气一本、系统 / 协议一本」的套路——只不过以太网的「电气那本」，判据换成了**双绞线世界的模板 + 下垂 + 回损**。

## 八、常见翻车 → 分层揪真凶（别一上来就换网卡）

网口跑不满、丢包、掉速，最忌讳一上来就换网卡、换主板。正确姿势还是分层：

- **① 自协商 / 速率层**：先看**协商到了几速、有没有退档**——「千兆口只跑百兆 / 十兆」绝大多数死在这里，多半是**链路质量（线缆 / 连接器 / 回损）不行，自协商主动退档保命**。先查协商结果和链路两端能力，别急着拆机。
- **② 线缆 / 通道层**：协商上了但**丢包、时快时慢、长距离才犯**。去测网线的**插损、回损、串扰、长度**——水晶头压得松、用了劣质线或超长线，回损 / 串扰一超，DSP 抵消不干净就丢包。**「换根好线、重压水晶头就好」，命中的就是这一层。**
- **③ 发送信号 / 模板层**：线也好、也协商上了，但**误码率偏高、偶发**。让 PHY 进测试模式，量**发送模板、下垂、抖动、失真**——发送波形本身压线、下垂超标，接收端裕量就薄。
- **④ 变压器 / 共模 / EMI 层**：**变压器选型不对、共模抑制差、辐射超标**，会带来偶发误码、也过不了 EMC。高温、长时间、特定负载才复现的，重点怀疑这一层。
- **⑤ 供电 / PoE 层**：如果走 **PoE（网线供电）**，供电协商不对、大负载掉压也会导致链路不稳、设备偶发重连——呼应模块三 PI。

口诀一句：**先看协商到几速、有没有退档 → 再测线缆回损 / 串扰 → 再进测试模式量发送模板 / 下垂 → 再查变压器 / 共模 / EMI → 别忘 PoE 供电。**

![图8：以太网常见翻车分层揪真凶。左五层：①自协商/速率层（协商到几速/有没有退档——"千兆只跑百兆/十兆"绝大多数死这层,多半链路质量差自协商主动退档,先查协商结果别拆机）②线缆/通道层（协商上但丢包/时快时慢/长距离才犯→测网线插损/回损/串扰/长度,水晶头松/劣线/超长→回损串扰超 DSP 抵消不干净丢包;"换好线重压水晶头就好"命中这层）③发送信号/模板层（线也好也协商上但误码偏高偶发→进测试模式量发送模板/下垂/抖动/失真,波形压线/下垂超标裕量薄）④变压器/共模/EMI 层（变压器选型不对/共模抑制差/辐射超标→偶发误码+过不了 EMC,高温长时特定负载才复现）⑤供电/PoE 层（PoE 供电协商不对/大负载掉压→链路不稳偶发重连,呼应模块三 PI）。右现象→指向哪层对照。口诀:先协商退档→线缆回损串扰→测试模式量模板下垂→变压器共模 EMI→别忘 PoE](/images/50/ill08_debug.png)

## 一句话总结

**一句话：以太网 BASE-T 是我们从机内接口走到机外的第一站——PHY 的信号先过一颗变压器做磁耦合才上双绞线，走的是基带多电平 PAM，靠加电平、加纠错、用满四对来提速率，而不是像机内串行那样拉单线 NRZ。**

- **编码演进**：10M 的 Manchester → 百兆 MLT-3 三电平 → 千兆 PAM5 五电平 → 2.5G 以上 PAM16 十六电平。
- **千兆起同线双向**：四对线在同一根线上同时双向收发，靠 hybrid、回声抵消、串扰抵消这套 DSP 把一锅粥拆开。
- **最反直觉的一点**：正常工作时抓不到干净的发送眼图——测发送端必须先让 PHY 进「测试模式」发已知码型，再套 IEEE 802.3 的波形模板。
- **测的主角变了**：从机内的「眼图 + 抖动 + 均衡」，换成波形模板 + 下垂 + 抖动 + 失真 + 回损；变压器带来独有的下垂指标，同线双向和长网线让回损、串扰成了核心判据。
- **判据来自 IEEE 802.3**：10M 到 10G 一代一个 Clause，模板和限值从线缆信道预算反推。
- **调试先分层**：先看自协商退没退档 → 再测线缆回损和串扰 → 再进测试模式量发送模板和下垂 → 再查变压器、共模、EMI → 最后别忘 PoE 供电。
