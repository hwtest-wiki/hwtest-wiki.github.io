# SPI 读出来全是乱码、错开一位、一提速就错、换个模式又对了——SPI：全双工、推挽、片选、更快，测边沿信号完整性 + 时序 + 模式 + 协议，判据看器件手册

先说个写驱动、调 flash / 传感器的人几乎都撞过的南墙。

你在板子上挂了一颗 SPI 的 NOR Flash、一片 ADC、或者一个屏幕控制器，用 SPI 去读写，结果 **读回来全是乱码、或者每个字节都错开了一位（bit）、或者读出来的值忽对忽错**；又或者，**低速（比如 1MHz）跑得好好的，一把时钟提到 20MHz、40MHz 就开始出错**；再或者，你对着数据手册反复核对接线都没错，**最后只是在初始化里把 SPI 模式从 Mode 0 改成 Mode 3，一切又全对了**。

做硬件、写驱动的人慢慢会明白：**SPI 这类「读出乱码、错开一位、提速就错、换个模式就好」，绝大多数根本不是芯片坏、也不是接线错，而是这几件物理 / 协议层的事没配好——CPOL/CPHA 模式对不对、快边沿在你这段走线上过冲振铃到什么程度、采样点的 setup/hold 满不满足那颗芯片手册、片选（CS）的时序和拓扑对不对。** 模式选错，采样沿落在数据还没稳的地方，读出来自然错位或乱码；走线一长、时钟一快，推挽的陡边沿在没端接的线上来回反射、振铃过零，接收端就可能把一个沿当成两个、多采一位；片选没拉稳、或者拉低时间不够，从机的移位寄存器根本没对齐。

SPI 到底怎么工作、为什么 CPOL/CPHA 这么爱坑人、它的信号和时序该怎么测、为什么判据要翻器件手册而不是查统一规范——这，就是这一篇要讲的，也是我们**低速板级总线这条线的第二站**。

## 一、先认清主角：SPI 是什么、它和 I2C 到底不一样在哪

**SPI**（Serial Peripheral Interface，串行外设接口）是摩托罗拉（Motorola）提出的**板级、同步、全双工**串行总线，和 I2C 一样满主板都是它：**SPI NOR / NAND Flash、ADC / DAC、小屏幕（LCD/OLED 控制器）、以太网 / CAN 控制器、SD 卡的 SPI 模式、各种传感器**——凡是「要比 I2C 快、要连续搬数据」的外设，很多都走 SPI。

它用**四根线**（外加地和电源）：

- **SCLK（或写 SCK，串行时钟）**：主机发出的时钟，节拍。
- **MOSI（Master Out Slave In）**：主机发、从机收的数据线。
- **MISO（Master In Slave Out）**：从机发、主机收的数据线。
- **CS / SS（Chip Select / Slave Select，片选，通常低有效）**：主机把它拉低，点名「这一颗，听我的」。

一个**主机（Master，通常是 SoC / MCU）** 出时钟、出片选、发起传输；**从机（Slave）** 被片选点中后，跟着 SCLK 的节拍一位一位地收发。它和上一篇 I2C，有五个根本区别——**每一个都直接决定了它怎么测**：

1. **推挽，不是开漏；没有上拉，上升时间不再是命门**：I2C 是开漏，芯片只能拉低、高电平靠上拉电阻慢慢充，所以上升时间 = Rp × Cb 是它的命门。**SPI 是推挽（push-pull）**——芯片里是**上下两颗管子（图腾柱）**，**能主动拉高、也能主动拉低**，上下沿都是有源驱动、又快又陡，**根本不需要上拉电阻**。好处是快；代价是——**陡边沿在长走线上会反射、振铃、过冲**，I2C 慢悠悠的 RC 边沿没有的信号完整性问题，在 SPI 这儿又回来了。

2. **全双工，不是半双工**：I2C 就一根 SDA，同一时刻要么主机说、要么从机说，是**半双工**。SPI 有 **MOSI 和 MISO 两根独立数据线**，**每一个时钟沿，主机往 MOSI 送一位、同时从 MISO 收一位**——收发同时进行，是**全双工**。物理上它就是主从两个**移位寄存器首尾相接转成一个环**：转一圈，两边的数据就交换了。

3. **片选选设备，不是地址寻址、没有 ACK**：I2C 靠**地址**点名、被点中的从机在第 9 拍给 **ACK** 应答。SPI **没有地址、没有 ACK**——主机**把哪颗的 CS 拉低，就是选中哪颗**。挂多颗从机有两种接法：**独立片选（每颗一根 CS，星形）** 或 **菊花链（daisy-chain，数据串着穿过每一颗、共用一根 CS）**。因为没有 ACK，**从机在不在、收没收到，SPI 自己不告诉你**，得靠读回的数据或更上层协议判断。

4. **没有时钟拉伸、没有仲裁**：I2C 从机能摁住 SCL 让主机等（时钟拉伸），还能多主仲裁。SPI **从机不能拖时钟、通常也是单主机**——时钟完全由主机说了算，从机必须跟得上，跟不上就得靠降时钟或上层握手，总线本身不管。

5. **更快，但判据没有总线级统一表、全翻器件手册**：I2C 有 **NXP UM10204** 一张统一规范表，各档速率、电平、时序的**电气/时序判据印得死死的、跨器件通用**（你还是要看器件手册确认它支持到哪个速率档、有没有特殊项，但判据本身有一张权威表可对）。**SPI 是事实标准，压根没有这样一张总线级规范表**——能跑多快、setup/hold 要多少、CS 提前多久，**全部只能翻你挂的那颗芯片自己的 datasheet**，没有可对照的统一判据。这是 SPI 测试判据上和 I2C 最不一样的一点。

一句话：**SPI = 四线（SCLK/MOSI/MISO/CS）、推挽驱动、全双工、片选选设备、更快、判据看器件手册的板级同步总线**；它测的是**边沿的信号完整性 + setup/hold 时序 + CPOL/CPHA 模式 + 协议解码**，把 I2C「开漏 / 上拉 / 上升时间 / 地址 / ACK / 拉伸」那套，整个换掉了。

### 1.1 I2C ↔ SPI 一张表看清（顺便记住这条线的脉络）

把上面五条对照，用一张表收口——**SPI 基本是把 I2C 的每一条都改了个方向**：

| 维度 | I2C（第 51 篇） | SPI（本篇） |
|---|---|---|
| 线数 | 2 线（SCL / SDA） | 4 线（SCLK / MOSI / MISO / CS） |
| 驱动方式 | 开漏 + 上拉（高电平靠 RC 充） | 推挽（上下沿都主动驱动） |
| 边沿 / 命门 | 上升时间 t_r = Rp × Cb | 边沿本就快，反愁过冲振铃反射 |
| 双工 | 半双工（一根 SDA 分时） | 全双工（MOSI + MISO 同时收发） |
| 选设备 | 地址寻址 + ACK 应答 | 片选 CS，无地址、无 ACK |
| 流控 / 仲裁 | 时钟拉伸 + 多主仲裁 | 无（通常单主机） |
| 速率 | 100k~3.4M（UFm 5M） | 几 M~数十 M，Flash 100M+ |
| 判据 | UM10204 统一表判电气时序 + 手册确认档位 | 无总线级规范，全翻各芯片 datasheet |
| 测什么 | 上升时间 + 电平 + 时序 + 协议 | 边沿 SI + setup/hold + 模式 + 协议 |
| 最常翻车 | 上拉 / 电平域 / 地址 ACK | CPOL/CPHA / 振铃过冲 / MISO 往返 |

![图3：I2C ↔ SPI 一张图看清。左列 I2C（第 51 篇，2 线·开漏·共享寻址）、右列 SPI（本篇，4 线·推挽·全双工·片选），逐维度对照：线数（2 线 SCL/SDA vs 4 线 SCLK/MOSI/MISO/CS）、驱动（开漏+上拉 RC 充高 vs 推挽上下沿主动）、边沿（t_r=Rp×Cb 是命门 vs 边沿本就快愁过冲振铃）、双工（半双工分时 vs 全双工同时收发）、选设备（地址寻址+ACK vs 片选 CS 无地址无 ACK）、流控（时钟拉伸+多主仲裁 vs 无通常单主机）、速率（100k~3.4M/UFm 5M vs 几 M~数十 M/Flash 100M+）、判据（NXP UM10204 统一表 vs 各芯片 datasheet 无统一）、测什么（上升时间+电平+时序+协议 vs 边沿 SI+setup/hold+模式+协议）、最常翻车（上拉/电平域/地址 ACK vs CPOL/CPHA/振铃/MISO 往返）。一句话：SPI 把 I2C 每一条都改了方向，下一篇 UART 再把三者放一起对比](/images/52/ill02b_vs_i2c.png)

> 🧵 这也顺出了「低速总线」这条线的脉络：**I2C（最省线、慢速、开漏共享）→ SPI（快、全双工、推挽片选）→ 下一篇 UART（无时钟、异步收发）**。到 UART 那一篇，我们会把 **I2C / SPI / UART 三者放进同一张表**一起对比，一眼看清「板级三大低速总线」各自的取舍。

## 二、原理：推挽全双工的四线移位、CPOL/CPHA 四模式、片选与拓扑

SPI 的原理，可以拆成三块，每一块都直接对应一类要测的东西。

### 2.1 推挽 + 四线全双工：主动驱动、边沿快，收发同时进行

SPI 的每一根输出线（SCLK、MOSI，以及从机侧的 MISO），芯片里都是**推挽输出级**：一颗**上管（拉高到 Vdd）** 加一颗**下管（拉低到地）**，像个**图腾柱（totem-pole）**——要拉高就上管导通、要拉低就下管导通，**两个方向都是芯片主动驱动**。所以：

- **上升沿、下降沿都又快又利索**（有源驱动，不像 I2C 上升沿要靠电阻慢慢充电）。这就是 SPI 能比 I2C 快很多的根本——**它的边沿不受 Rp × Cb 那条 RC 曲线限制**。
- **代价是**：推挽的边沿太陡（上升 / 下降时间可能只有一两纳秒），走线一长、又没做端接，这种**陡边沿会在传输线上反射、叠加成振铃和过冲**——这正是我们在模块一 SI 里讲的那套。所以 **I2C 不用操心的信号完整性，SPI 快起来、线一长就得操心了**。

数据怎么走？SPI 的收发，本质是**两个移位寄存器接成一个环**：主机的移位寄存器经 **MOSI** 接到从机的移位寄存器，从机的移位寄存器经 **MISO** 接回主机。**每来一个 SCLK 时钟沿，两个寄存器一起移一位**——主机把最高位从 MOSI 推出去、同时从 MISO 收进来一位；转满 8 个时钟，主从两边的一个字节就**互换**完成了。所以 SPI 天生**全双工**：你想只读不写，也得往 MOSI 送 8 位「废数据」把从机的数据「摇」出来（这就是为什么读 flash 也要发 dummy 字节）。

![图4：推挽输出级 器件级原理图（自绘简化，对应 §2.1）。一根输出线（SCLK / MOSI / MISO 都一样）：芯片内部是图腾柱推挽——顶上 Vdd 接一颗上管（P 管，导通则把线主动拉高）、底下接地接一颗下管（N 管，导通则把线主动拉低），两管栅极由内部逻辑互补控制；输出接到走线，走线有特征阻抗 Z0、末端可加一颗串联端接电阻 Rs（22~33Ω）压反射。两个动作：①要送 1→上管导通把线拉到 Vdd（有源、快）；②要送 0→下管导通把线拉到地（有源、快）。对照 I2C 开漏：SPI 上下沿都主动驱动、无需上拉、边沿快，所以能高速；代价——边沿太陡在长走线上反射振铃过冲，需串联端接/控制压摆率。一句话：高电平不再靠松手+上拉充电，是上管主动顶上去的，所以没有 t_r=RC 的命门，取而代之是快边沿的信号完整性](/images/52/ill02c_push_pull.png)

### 2.2 CPOL / CPHA：四种模式，最爱坑人的一件事

SPI 只有一根时钟 SCLK，但**在哪个沿采样、空闲时时钟是高还是低**，可以配置——这就是 **CPOL（时钟极性）** 和 **CPHA（时钟相位）** 两个开关，组合出**四种模式（Mode 0~3）**：

| 模式 | CPOL | CPHA | 空闲时 SCLK | 采样（读数据）沿 | 移位（换数据）沿 |
|---|---|---|---|---|---|
| **Mode 0** | 0 | 0 | 低 | 第 1 个沿（上升） | 第 2 个沿（下降） |
| **Mode 1** | 0 | 1 | 低 | 第 2 个沿（下降） | 第 1 个沿（上升） |
| **Mode 2** | 1 | 0 | 高 | 第 1 个沿（下降） | 第 2 个沿（上升） |
| **Mode 3** | 1 | 1 | 高 | 第 2 个沿（上升） | 第 1 个沿（下降） |

- **CPOL** 决定**时钟空闲电平**：0 = 空闲低、1 = 空闲高。
- **CPHA** 决定**在哪个时钟沿采样**：0 = 每个时钟周期的**第一个沿（前沿）** 采样、1 = **第二个沿（后沿）** 采样。

**Mode 0（0,0）和 Mode 3（1,1）用得最多**。要命的是：**主机和从机的模式必须一致**——如果主机 Mode 0（上升沿采样）、从机却按 Mode 3（也是上升沿采样，但空闲电平不同，实际数据对齐关系差半拍），采样点就落在数据还没稳、或者刚好在跳变的地方，读出来**要么整片乱码、要么每个字节都错开一位**。这就是本篇开头「换个模式就全对了」的真相——**十有八九是 CPOL/CPHA 配错了**。测 / 调 SPI，**第一件事就是把两端手册的模式对上，并在示波器 / 逻辑分析仪的解码器里设成同一个模式**。

### 2.3 片选与拓扑：CS 选谁、独立片选还是菊花链

SPI 靠 **CS（片选，低有效）** 选设备，没有地址：

- **一次传输的边界，就是 CS 的一段低电平**：主机把某颗从机的 **CS 拉低 → 传输若干字节 → 拉高**，这一整段就是一「帧」。CS 拉低到第一个 SCLK 之间要留够**建立时间**，最后一个 SCLK 到 CS 拉高也要留够**保持时间**（具体多少看那颗芯片手册）。**CS 中途被拉高，从机的移位寄存器就复位**，这一帧就废了。
- **挂多颗从机，两种拓扑**：
  - **独立片选（星形）**：每颗从机**各拉一根 CS** 回主机，SCLK / MOSI / MISO 三线共享。要跟谁说话就拉低谁的 CS。**挂 N 颗就要 N 根 CS 线**——省了地址，费了引脚。
  - **菊花链（daisy-chain）**：所有从机**共用一根 CS**，数据**从第一颗的输出串到第二颗的输入**，一路穿下去、再回到主机。省了片选线，但**时序和数据组织复杂**，且一颗出问题整链受影响，用得相对少。

因为**没有地址、没有 ACK**，SPI 的协议层比 I2C 简单得多——**没有「点名 - 应答」这一环**，主机拉低 CS 就默认从机在听。好处是快、简单；坏处是**从机在不在、有没有收到，总线层面无从得知**，只能靠读回内容或上层协议自己判断。

![图5：SPI 三大原理。①推挽+四线全双工：芯片推挽输出级（上管拉高/下管拉低都主动驱动、边沿快、无需上拉，代价是快边沿长走线反射振铃需端接）；主从两个移位寄存器经 MOSI/MISO 接成环，每个 SCLK 沿同时移一位=全双工，读也要发 dummy 把数据摇出来。②CPOL/CPHA 四模式：CPOL 定空闲时钟高低、CPHA 定前沿还是后沿采样；Mode 0(0,0 空闲低前沿采)/Mode 1(0,1 空闲低后沿采)/Mode 2(1,0 空闲高前沿采)/Mode 3(1,1 空闲高后沿采)，Mode 0/3 最常用；主从模式必须一致，配错→采样落在数据没稳处→乱码/错一位（换模式就好的真相）。③片选与拓扑：CS 低有效选设备无地址无 ACK，一段 CS 低电平=一帧，CS 拉低到首 SCLK 留建立、末 SCLK 到拉高留保持，中途拉高复位废帧；独立片选(每颗一根 CS 星形，N 颗 N 线)或菊花链(共用一根 CS 数据串行穿过)](/images/52/ill03_arch.png)

## 三、速率与信号方式：推挽快边沿，几 MHz 到上百 MHz，判据看手册

SPI **没有像 I2C 那样一张印死档位的统一规范表**。它是事实标准，速率上限由**两端芯片手册的最高时钟、以及你这段走线的信号完整性**共同决定：

| 类型 | 常见时钟 | 信号方式 | 数据线 | 备注 |
|---|---|---|---|---|
| **普通 SPI（单线数据）** | 几 MHz ~ 数十 MHz | 推挽、全双工 | MOSI + MISO 各 1 | 最常见；具体上限查从机手册 |
| **高速 SPI（如 NOR Flash）** | 数十 MHz ~ 100+ MHz | 推挽 | 单线或多线 | 走线短、常需串联端接 |
| **Dual / Quad SPI（QSPI）** | 上百 MHz（含 DDR） | 推挽 | 2 / 4 根数据线 | Flash 提吞吐；不再是全双工收发，是多线并行 |

几个必须记住的点：

- **上升时间不再是判据主角**：I2C 要卡 t_r ≤ 1000/300/120ns，因为它靠 RC 上升、慢了就认不出高电平。SPI 推挽驱动，**边沿本来就快（常常一两纳秒）**，t_r 一般不用愁「太慢」——反而要担心**「太快」带来的过冲、振铃、反射**。测试关注点从 I2C 的「上升够不够快」翻成了 SPI 的「边沿会不会冲过头、抖出双沿」。
- **电平是普通推挽单端电平**：高电平接近 Vdd、低电平接近地，门槛按器件的逻辑电平（如 LVCMOS）判，**不像 I2C 那样卡死 0.3/0.7Vdd 的开漏门槛**。同样要注意**电平域匹配**——1.8V 主控接 3.3V flash，一样要电平转换。
- **速度上限靠两头卡**：一头是**从机手册的最高 SCLK**（超了从机跟不上、采样出错）；另一头是**你这段走线的信号完整性和往返时序**——线越长、边沿越陡，反射振铃越凶，能跑的时钟越受限（下一节细讲）。
- **QSPI / Dual 等多线变体**：用 2 / 4 / 8 根数据线并行搬数据（多见于 Flash），**牺牲「全双工同时收发」换吞吐**，本质还是 SPI 家族——这一大家子（标准 / 3 线 / Dual / Quad / Octal、SDR / DDR）的区别，下面 §3.1 单独摆一张对比。

![图6：SPI 速率与信号方式（没有 I2C 那张统一规范表，判据看器件手册）。普通 SPI（几 MHz~数十 MHz·推挽全双工·MOSI/MISO 各一根·上限查从机手册）→高速 SPI 如 NOR Flash（数十~100+MHz·走线短常需串联端接）→Dual/Quad QSPI（上百 MHz 含 DDR·2/4 根数据线并行提吞吐·不再全双工）。底部：①上升时间不再是判据主角——推挽边沿本就快(一两纳秒)，不愁太慢、反愁太快带来的过冲/振铃/反射②电平是普通推挽单端(近 Vdd/近地，按 LVCMOS 等逻辑门槛)，不卡 0.3/0.7Vdd 开漏门槛，但一样要电平域匹配③速度上限两头卡：从机手册最高 SCLK + 走线 SI 和往返时序④QSPI/Dual 用多数据线换吞吐、牺牲全双工。SPI 是事实标准无统一规范，判据一律翻 datasheet](/images/52/ill04_versions.png)

### 3.1 SPI 家族：标准 / 3 线 / Dual / Quad / Octal、SDR / DDR 的区别

我们前面讲的「四线全双工 SPI」是最经典的一支，但 SPI 其实是**一大家子**——为了在**省线**和**提吞吐**两个方向上各走极端，衍生出好几种变体。它们的区别主要在**数据线几根、还全不全双工、单沿还是双沿采样**：

| 变体 | 数据线 | 双工 | 采样沿 | 典型用途 |
|---|---|---|---|---|
| **标准 SPI（4 线）** | MOSI + MISO 各 1 根 | **全双工** | 单沿（SDR） | 通用外设：ADC / 屏 / 传感器 / 小容量 Flash |
| **3 线 SPI（半双工）** | 1 根双向（MOSI/MISO 合成 SDIO） | 半双工 | 单沿 | 省一根线，命令 / 数据分时收发 |
| **Dual SPI** | 2 根（IO0 / IO1） | 半双工（数据阶段 2 线并行） | 单沿 | Flash 提读吞吐 |
| **Quad SPI（QSPI）** | 4 根（IO0~3） | 半双工 | 单 / 双沿（DDR） | NOR Flash 主流、XIP 片上执行 |
| **Octal SPI（OSPI）** | 8 根（IO0~7） | 半双工 | 单 / 双沿（DDR） | 高端 Flash / PSRAM，超高吞吐 |

几个关键分野：

- **全双工的只有标准 SPI**：因为它 **MOSI / MISO 各占一根专线**，一根只发、一根只收，才能同时收发。**从 Dual 开始，数据线就变成「双向共享的 IO」**——发命令 / 地址时常用**单线**，进入数据阶段再**切到多线并行**，一根线一会儿输出、一会儿输入，**全双工就没了，换来的是吞吐**。所以 Dual / Quad / Octal 几乎都是**为 Flash / PSRAM 这种要大带宽连续搬数据的场合**而生，普通传感器 / ADC 用不上。
- **SDR 与 DDR（单沿 / 双沿）**：普通 SPI 是**单沿采样（SDR）**，一个时钟周期传一位。高端 Quad / Octal 常上 **DDR（双沿）**——**上升沿、下降沿都传数据**，同样时钟吞吐翻倍；代价是**两个沿都要满足 setup / hold**，时序裕量更小、更难测。
- **3 线 SPI 是往「省线」方向走**：把 MOSI / MISO 合成**一根双向 SDIO**，省一根引脚、分时收发（一些 ADC / 传感器 / 小屏用），本质是半双工的标准 SPI。
- **命名别被绕晕**：厂商芯片手册里的 **QUADSPI / OSPI 控制器**，本质就是上面说的多线 SPI；**QSPI ≈ Quad SPI**，主要服务 NOR Flash（很多还支持 **XIP 片上执行**，CPU 直接从 Flash 取指跑代码）。

这些变体对**测试**的直接影响：

- **线越多，同抓的通道越多**；而且数据线是**双向共享**，示波器 / 逻辑分析仪的解码器**必须按 phase 配置**——命令 / 地址 / dummy / 数据**各占几根线、是 SDR 还是 DDR**——配错了照样解成乱码（比标准 SPI 的「设对 CPOL/CPHA」又多了一层）。
- **DDR 双沿**：上下沿都要量 setup / hold，裕量更小；再叠加高速 + 多线并行，**走线等长、串扰、信号完整性**比标准 SPI 更吃紧。
- **读波形的方式也变了**：标准 SPI 的 MOSI / MISO 分开看；多线模式下**同一组 IO 上先是命令、后是数据，方向中途翻转**，得跟着帧结构看。

![图6b：SPI 家族对比（标准 / 3线 / Dual / Quad / Octal · 全双工 vs 半双工 · SDR vs DDR）。五行：①标准 SPI(4线)=MOSI+MISO 各 1 根·全双工·单沿 SDR·通用外设 ADC/屏/传感器/小 Flash②3 线 SPI=1 根双向 SDIO(MOSI/MISO 合一)·半双工分时收发·省一根线③Dual SPI=2 根 IO0/1·半双工数据阶段 2 线并行·Flash 提读吞吐④Quad SPI/QSPI=4 根 IO0~3·半双工·单/双沿 DDR·NOR Flash 主流 XIP 片上执行⑤Octal SPI/OSPI=8 根 IO0~7·半双工·单/双沿 DDR·高端 Flash/PSRAM 超高吞吐。底部：全双工只有标准 SPI(MOSI/MISO 专线)；一上多线数据线变双向共享、牺牲全双工换吞吐，几乎专为 Flash/PSRAM；DDR 双沿翻倍吞吐但两沿都要 setup/hold 时序更紧；解码要按 phase(命令/地址/dummy/数据各占几线、SDR/DDR)配置，比标准 SPI 多一层](/images/52/ill04b_family.png)

## 四、独特之处：推挽快边沿把 SI 带回来 + 全双工 + CPOL/CPHA 模式 + 片选无地址

SPI 有四处和 I2C 很不一样、直接影响测试的独特设计：

- **① 推挽快边沿，信号完整性重新变重要**：I2C 慢悠悠的 RC 上升沿，几乎不用担心反射振铃；SPI 推挽驱动、边沿一两纳秒，**一旦走线长、又没端接，陡边沿在传输线上来回反射，叠成振铃和过冲**——轻则电平过冲损伤器件，重则**振铃过零被接收端当成额外的时钟沿 / 数据跳变，多采一位、误码**。这就是「慢速好、一提速就错」的一大来源。对策是**模块一 SI 那套**：源端**串联端接电阻（常见 22~33Ω）**、**控制压摆率（slew-rate）**、**缩短走线**、注意参考地完整。**SPI 是低速总线里最需要回头看信号完整性的一个**。

- **② 全双工，收发同时**：MOSI 和 MISO 两根线，每个时钟沿同时收发。测的时候要**四根线一起抓**（SCLK / MOSI / MISO / CS），才能看清「主机发了什么、从机同时回了什么、CS 框住的是哪一帧」。这和 I2C 只有一根双向 SDA 完全不同。

- **③ CPOL / CPHA 模式必须两端一致**：这是 SPI 独有、也是最爱翻车的测点。**四种模式，主从必须对上**；配错就采样在错误的沿上，**乱码或整体错开一位**。测 / 调时，**先核对两端手册的模式，再把解码器设成同一模式**——这一步 I2C 完全没有（I2C 采样沿是固定的「SCL 高电平期间」）。

- **④ 片选、无地址、无 ACK**：靠 CS 拉低选设备，**一段 CS 低电平就是一帧**；没有地址点名、没有第 9 拍 ACK、没有时钟拉伸。所以协议解码里，**CS 是帧边界、bit 顺序（MSB/LSB first）和字长要设对**，而「读不到 / 读错」也不会像 I2C 那样有个「没 ACK」的明确信号——只能看**读回的字节对不对、CS 时序稳不稳**。

![图7：SPI 四大独特（对照 I2C）。①推挽快边沿把 SI 带回来：I2C 慢 RC 边沿不愁反射，SPI 推挽边沿一两纳秒，走线长+没端接→陡边沿传输线反射叠成振铃过冲→振铃过零被当额外时钟/数据沿→多采一位误码（慢速好一提速就错的大来源），对策=模块一 SI 那套：源端串联端接 22~33Ω/控制压摆率/缩短走线/参考地完整——SPI 是低速总线里最需回看 SI 的。②全双工收发同时：MOSI/MISO 两线每个时钟沿同时收发，必须四线(SCLK/MOSI/MISO/CS)一起抓。③CPOL/CPHA 模式两端必须一致：四模式配错→采样落错沿→乱码/错开一位，测调先核对手册模式再设解码器同模式(I2C 没有这步)。④片选无地址无 ACK：CS 低=选中=帧边界，无点名无应答无拉伸，解码要设对 CS/bit 顺序 MSB-LSB/字长，读不到没有"没 ACK"明确信号只能看读回字节和 CS 时序](/images/52/ill05_unique.png)

## 五、测什么 · 三类：信号电气、时序、协议解码

SPI 的测试，也归成三大类，**但主角从 I2C 的「上升时间 + 电平门槛」换成了「快边沿的信号完整性 + setup/hold 时序 + CPOL/CPHA 模式」**：

1. **信号电气（Signal / Electrical）**：用示波器看 SCLK / MOSI / MISO / CS 四条波形，量：**边沿的过冲 / 下冲 / 振铃、上升 / 下降时间（关心「太快抖出双沿」而非「太慢」）、高低电平 V_OH / V_OL 与逻辑门槛裕量、走线反射**。因为推挽边沿快，**这一类比 I2C 更吃示波器带宽**（要能看清一两纳秒的边沿和振铃），也更常需要**源端串联端接**来救。

2. **时序（Timing）**：量数据相对**采样时钟沿**的关系，对**器件手册**：**SCLK 频率、SCLK 高 / 低时间、数据建立 t_SU / 保持 t_HD（相对采样沿）、CS 建立（CS 拉低到首个 SCLK）/ CS 保持（末个 SCLK 到 CS 拉高）、CS 拉高的空闲间隔**。尤其高速时要算一笔**MISO 往返账**：主机发出时钟 → 时钟走到从机 → 从机驱动 MISO → MISO 走回主机 → 必须在主机的采样沿之前稳住。**线一长、钟一快，这个往返就来不及**——这是 SPI 高速下掉速 / 出错的一个隐蔽根。

3. **协议解码（Protocol）**：用示波器的 **SPI 触发 / 解码**或**逻辑分析仪**，把四条波形翻成**字节帧**：**CS 框出的每一帧、CPOL/CPHA 模式、bit 顺序（MSB / LSB first）、字长（8/16 位）、MOSI 发的命令 / 地址、MISO 回的数据**。这一类回答「读出乱码（模式错？bit 顺序错？）、错开一位（模式 / 相位差半拍）、某颗读不到（CS 拉的是不是它 / 时序够不够）」。

贯穿三类的度量，和 I2C 差别很大：**没有上拉电阻、没有上升时间 = RC、没有地址 / ACK / 时钟拉伸**；换成了**快边沿 SI、相对采样沿的 setup/hold、CPOL/CPHA 模式、CS 帧**。但和高速串行比，它**依然没有眼图、没有抖动分离、不用去嵌、不用 BERT**——还是低速单端那一岸，只是比 I2C 更靠近信号完整性一点。

![图8：SPI 测什么三类（主角=快边沿 SI+setup/hold 时序+CPOL/CPHA 模式，不是上升时间和上拉）。①信号电气：示波器看 SCLK/MOSI/MISO/CS 四波形，量边沿过冲/下冲/振铃、上升下降时间(关心太快抖双沿而非太慢)、V_OH/V_OL 与逻辑门槛裕量、走线反射；推挽边沿快，比 I2C 更吃示波器带宽、更常需源端串联端接。②时序：量数据相对采样时钟沿的关系对器件手册——SCLK 频率、高/低时间、数据建立 t_SU/保持 t_HD、CS 建立(CS 低到首 SCLK)/CS 保持(末 SCLK 到 CS 高)/CS 空闲间隔；高速要算 MISO 往返账(钟出→到从机→驱 MISO→回主机→采样前须稳，线长钟快就来不及)。③协议解码：SPI 触发解码/逻辑分析仪翻成帧——CS 框每帧、CPOL/CPHA 模式、bit 顺序 MSB/LSB、字长、MOSI 命令地址/MISO 数据，回答乱码/错一位/某颗读不到。共同：无上拉无上升时间 RC 无地址 ACK 拉伸；仍无眼图抖动去嵌 BERT](/images/52/ill06_test3.png)

## 六、怎么测 · 台面：够带宽示波器四线同抓 + SPI 解码 / 逻辑分析仪 + 量 setup/hold + 试端接降钟

SPI 的测试台面，比高速接口**朴素**，但比 I2C **多两分讲究**（带宽要够、要会看 SI、要设对解码模式）：

- **① 四线同抓，示波器带宽要够**：SPI 要看**四根线的相互关系**（时钟对数据、CS 对帧），至少两三个通道同抓 SCLK + 一条数据 + CS。因为推挽**边沿快（一两纳秒）**，示波器带宽 / 探头带宽要**比 I2C 高**，才能看清边沿和振铃、量准 setup/hold；带宽不够会把过冲振铃「抹平」，看不出真问题。

- **② 量边沿与信号完整性**：量 **过冲 / 下冲 / 振铃幅度、边沿时间**，看振铃有没有**过零 / 逼近门槛**（那会被误采）。若振铃凶，试**源端串联端接（22~33Ω 起调）**、**降低驱动压摆率**、**缩短走线**——这是 SPI 特有、I2C 基本用不上的一招。

- **③ 量 setup / hold，对器件手册**：两通道同抓 SCLK 和数据，量**数据在采样沿前稳定多久（t_SU）、之后保持多久（t_HD）**，以及 **CS 建立 / 保持**，逐项对**从机 datasheet** 的最小值。**高速下重点核 MISO 往返**：把时钟降一半若问题消失，八成是往返 / setup 不够。

- **④ SPI 解码 / 逻辑分析仪，先设对模式**：打开示波器 **SPI 解码**或接**逻辑分析仪**，**务必先设对 CPOL/CPHA 模式、CS 极性、bit 顺序、字长**——设错了解出来的也是「乱码」，会把你带沟里。设对后看 **MOSI 命令 / 地址、MISO 回读、CS 帧边界**是否符合预期。**读出乱码先怀疑模式**，其次 bit 顺序 / 字长。

- **⑤ 降时钟 / 换模式做二分**：这是 SPI 最快的现场判定法。**把 SCLK 降下来（如 20M→1M）** 若立刻正常 → 指向**信号完整性 / 往返时序**（高速才犯）；**换一个 CPOL/CPHA 模式**若立刻正常 → 就是**模式配错**。两招一试，大半问题当场定位。

- **⑥ 电平域 / 供电检查**：确认主从 Vdd 一致或有电平转换、CS / 时钟的上电与复位时序对（呼应模块二 PI）——上电瞬间 CS 若没被拉稳，从机可能进错状态。

## 七、判据：翻器件手册那几行 setup/hold 与 fmax —— 不是统一规范表，也不是眼图模板

SPI 的判据，和 I2C、和高速串行都不一样，最需要转过弯来：

- **I2C**：查 **NXP UM10204** 一张**统一规范表**——各档速率、电平、时序印死，人人一样。
- **高速串行（模块六）**：从信道损耗**反推一条眼图模板 / 抖动预算**。
- **SPI**：**既没有统一规范表、也没有眼图模板**——判据全在**你挂的那颗芯片自己的 datasheet** 里的**「SPI Timing / AC Characteristics」那几行**：最高 SCLK 频率 **f_SCK(max)**、时钟高 / 低时间、**数据建立 t_SU / 保持 t_HD**、**CS 建立 / 保持 / 空闲**、以及从机**输出有效延迟（SCLK 沿到 MISO 有效）**。

所以测 SPI 的判据动作就一句：**把示波器 / 逻辑分析仪量到的数，逐项去对那颗芯片手册的最小 / 最大值**。要点：

- **同一条 SPI 总线上不同芯片的手册可能不一样**——最高频率、建立保持各不相同，**要按最"娇气"那颗的限值**跑（就近取严）。
- **CPOL/CPHA 模式也在手册里写死**（有的芯片只支持 Mode 0，有的 0/3 都行）——**先对模式，再对时序**。
- **高速时把 MISO 往返延迟算进预算**：手册给的是芯片引脚处的时序，**走线来回的传播延迟要你自己加上去**，否则手册满足、板级照样错。

一句话：**SPI 没有「一张表管所有」，判据是「一颗芯片一本手册」——查对、就近取严、把走线往返补进去。**

## 八、常见翻车 → 分层揪真凶（别一上来就怀疑芯片坏）

SPI 的「乱码、错一位、提速就错、换模式就好」，也最忌一上来换芯片、重写驱动。正确姿势还是**分层**：

- **① 模式 / 协议层（最高频）**：**CPOL/CPHA 配错**——主从模式不一致，采样落错沿，表现为「整片乱码」或「每字节错开一位」，**换个模式就全对**（最经典）；**bit 顺序 / 字长设错**（MSB 当 LSB、8 位当 16 位）也读出乱码；**CS 拉错颗 / CS 时序不够**导致点错设备或帧没对齐。先在解码器里核对模式、bit 顺序、CS，再核两端手册模式是否一致。

- **② 信号完整性 / 边沿层（提速才犯）**：**推挽陡边沿在长走线上反射振铃过冲 → 振铃过零被当额外沿 → 多采一位 / 误码**，典型「1MHz 好好的、上 20MHz / 40MHz 就错」「短线样机好、长线量产错」。量边沿振铃，试**源端串联端接 22~33Ω、降压摆率、缩短走线**；这一层是 SPI 相对 I2C 新增的重点。

- **③ 时序 / 往返层（也是提速才犯）**：**SCLK 超从机手册最高频率**，或**高速下 MISO 往返延迟 + 从机输出延迟 > 采样窗口**，主机采到的是上一位 / 没稳的数据，表现同样是「提速就错、降速就好」。核 datasheet 的 f_SCK(max) 与 setup/hold，把**走线往返补进预算**，必要时**降时钟**或**换在更晚的沿采样**。

- **④ 电平域 / 供电层**：**1.8V 主控接 3.3V 从机没做电平转换 → 门槛够不到 / 灌不动**；**CS / 时钟上电复位时序不对 → 从机上电进错状态**。查两端 Vdd、电平转换、CS 上电默认电平（呼应模块二 PI）。

**口诀**：先在解码器核对模式 / bit 顺序 / CS（协议）→ 提速才犯的看边沿振铃试端接（信号完整性）→ 再核 f_max 与 setup/hold 补走线往返（时序）→ 最后查电平域和上电时序（供电）。**降时钟、换模式两个二分动作，永远先做。**

## 一句话总结

**一句话：SPI 是低速板级总线这条线的第二站——它把 I2C 几乎每一条都改了方向：四线（SCLK/MOSI/MISO/CS）、推挽驱动、全双工、片选选设备、更快，判据看器件手册而不是统一规范表。**

- **推挽换掉开漏，命门也换了**：SPI 上下沿都主动驱动、不用上拉，所以 **I2C「上升时间 = 上拉 × 电容」的命门在这儿没有了**；取而代之，**推挽的陡边沿在长走线上反射振铃过冲**，把模块一的信号完整性又带了回来——**SPI 是低速总线里最需要回看 SI 的一个**，对策是源端串联端接（22~33Ω）、控压摆率、缩短走线。
- **最爱翻车的是 CPOL/CPHA 模式**：四种模式主从必须一致，配错就采样落错沿、乱码或错开一位——「换个模式就全对」十有八九是它。测 / 调第一步：核对两端手册模式，并把解码器设成同一模式。
- **测什么（三类）**：①信号电气（快边沿的过冲 / 振铃 / 反射，比 I2C 更吃带宽、更常要端接）；②时序（数据相对采样沿的 setup/hold、CS 建立保持、fmax，还要把 **MISO 往返**算进高速预算）；③协议（SPI 解码 / 逻辑分析仪看 CS 帧 + 模式 + bit 顺序 + 字长）。
- **判据靠翻手册**：SPI 没有 I2C 那张统一规范表、也没有眼图模板——**一颗芯片一本 datasheet**，量到的数逐项对手册、就近取严、把走线往返补进去。
- **它是一大家子**：标准 SPI（4 线全双工）之外，还有 3 线（省线半双工）、Dual / Quad（QSPI）/ Octal（OSPI）——**一上多线，数据线就变双向共享、牺牲全双工换吞吐**，几乎专为 Flash / PSRAM；高端还上 DDR 双沿翻倍吞吐、时序更紧。多线模式解码要按 phase（命令 / 地址 / dummy / 数据各占几线、SDR/DDR）配，比标准 SPI 又多一层。
- **调试永远分层**：先核对模式 / bit 顺序 / CS → 提速才犯的看边沿振铃试端接 → 再核 fmax 与 setup/hold 补往返 → 最后查电平域和上电时序。**降时钟、换模式两个二分动作，永远先做。**
