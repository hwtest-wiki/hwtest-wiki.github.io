# 眼图 eye diagram：单看每一位波形都合格，几万位叠一起，眼睛怎么就快闭了？

<div class="article-meta"><span class="badge">📘 模块三 · 信号完整性 SI · 眼图 eye diagram</span><span class="badge">⏱ 约 19 分钟</span><span class="badge qr">🔧 含工程师速查</span></div>


一条 5Gbps 的高速串行链路，板子回来上测试台。工程师先按老规矩，用示波器抓单个位的波形挨个量：**上升时间没超、摆幅够、过冲也压在范围内**，一位一位看，条条合格。可一挂上误码率测试仪跑起来，**误码率死活压不到 10⁻¹² 的指标，卡在 10⁻³ 上下**，比要求差了九个数量级。

问题出在哪？他把示波器切到**眼图模式**，用链路的恢复时钟做触发，让几万个 bit 的波形按一个 UI 叠加起来——屏幕上那只本该睁得又大又亮的"眼睛"，**眼皮耷拉、眼角发虚，中间的开口只剩一条缝**。仿真时眼高有 400mV、眼宽有 0.7UI，实测**眼高只剩 80mV、眼宽只剩 0.2UI**。

**单看每一位都合格，几万位叠一起眼睛却快闭了。** 为什么？因为单个 bit 看不出"最坏情况"：反射的余振、隔壁线的串扰、走线的损耗、时钟的抖动，每一样单独摊到某一位上都不算大，可它们**在不同的 bit 上此消彼长、偶尔撞到一起**，就把眼睛从四面八方啃小了。眼图，就是把这些"偶尔撞一起"的最坏情况，全逼到一张图上给你看。这一篇，我们就把"眼图怎么形成、怎么读、谁在啃它、怎么测、怎么把眼睛重新睁开"这条链，从头捋清楚。

**读完这篇，你能搞懂：**

- ✅ 眼图到底是**怎么叠出来的**——UI、恢复时钟触发、成千上万位的叠加；
- ✅ 怎么读眼图——**眼高（垂直余量）、眼宽（水平余量）、交叉点、采样点**分别是什么；
- ✅ SI 三杀手加抖动，**谁压眼高、谁压眼宽**，把前几篇串成一张体检表；
- ✅ 怎么把眼图**量化**——眼高眼宽、Q 因子、**BER 浴盆曲线**、**掩模（mask）测试**；
- ✅ 怎么测——**恢复时钟（CDR）、PRBS 码型、累积够多、实时 vs 采样示波器**；
- ✅ 怎么把眼睛**重新睁开**——前几篇的治法 + **均衡（预加重 / CTLE / DFE）**。

## 一、眼图是什么：把几万位波形按 UI 对齐，叠成一只"眼睛"

先说一个最基础、却最关键的词：**UI（Unit Interval，单位间隔）**。它就是**一个 bit 占用的时间**。5Gbps 的链路，1 秒传 50 亿个 bit，那么

**UI ＝ 1 ÷ 5Gbps ＝ 200ps。**

一个 UI 就是 200 皮秒，一位数据的"地盘"。眼图，说白了就是干一件事：**把这条码流按一个一个 UI 切成段，然后把成千上万段全部平移、对齐、叠在同一个时间窗里画出来。**

**怎么对齐？靠恢复时钟。** 高速串行链路里没有单独的时钟线，接收端用一个叫 **CDR（时钟数据恢复）** 的电路，从数据自己的边沿里"拎"出一个时钟。示波器抓眼图时，就用这个恢复时钟做触发：**每来一个（或两个）UI，就把这一段波形叠画到屏幕上同一个位置**。叠上几万、几十万个 bit，你就得到一张眼图。

**为什么非得叠这么多位？因为单看一位，看不出统计余量。** 一条真实的码流，有 `010101…` 这种连续翻转的，也有 `0001110000` 这种连续不变、憋了好几位才跳一下的。每种前后组合，走过有损耗、有反射、有串扰的链路，到达接收端时的**幅度和相位都略有不同**。你只量其中一位，看到的是运气；把所有组合叠起来，眼睛的开口才是**所有 bit 里最坏的那一撮**——而链路能不能可靠工作，恰恰由最坏情况决定。

![图1：翻车现场——一条5Gbps链路，单个bit波形逐个量（上升时间/摆幅/过冲）条条合格，可切到眼图模式把几万位叠起来，眼睛却快闭了：仿真眼高400mV/眼宽0.7UI、误码率10⁻¹²，实测眼高只剩80mV/眼宽0.2UI、误码率掉到10⁻³。单看每位都合格，叠一起才露出最坏情况](/images/31/ill01_scene.png)

叠加的过程，用动图看最直观：一段码流从左往右流过来，示波器按 UI 一段一段把它折叠、对齐、堆到同一个窗口里。刚开始只有几条轨迹，看不出名堂；随着叠加的 bit 越来越多，各种 `0→1`、`1→0`、`1→1`、`0→0` 的组合轨迹全铺上去，**中间那块谁都不碰的空白区**，就慢慢显出一只"眼睛"的形状。

![图2b动图：一段码流按UI逐段折叠、对齐、叠加，从零星几条轨迹逐渐堆成一只完整的眼睛，中间显出无轨迹穿过的"眼睛"开口](/images/31/ill02b_form_anim.gif)

> **一句话记住：眼图 = 把码流按一个 UI 切段、用恢复时钟对齐、把成千上万位叠画到同一时间窗。UI = 一个 bit 的时间（5Gbps → 200ps）。叠这么多位，是为了看出所有 bit 组合里最坏的那一撮余量。**

![图2：眼图是怎么形成的。上：一段实际码流（含010101连续翻转、和000111连续位）；中：按UI切成一段一段；下：用恢复时钟对齐、全部叠加到同一时间窗，中间浮现"眼睛"开口。旁注：连续翻转的位走一条路径、连续不变的位走另一条，叠起来才现出最坏边界](/images/31/ill02_form.png)

## 二、怎么读眼图：眼高、眼宽、交叉点、采样点

眼图的信息量，全在那只"眼睛"的形状里。学会读它，就四个关键位置：

**眼高（Eye Height）——竖直方向的开口，就是电压噪声余量。** 接收端靠一个**判决电平**（通常在眼睛正中间那条横线）来区分 0 和 1：高于它判 1、低于它判 0。眼高，就是眼睛上眼皮（所有"1"里最低的那条）到下眼皮（所有"0"里最高的那条）之间的**垂直距离**。**眼高越大，判决电平上下能容忍的噪声越多**；眼高被压到接近零，一点噪声就能让 0/1 判错——误码。

**眼宽（Eye Width）——水平方向的开口，就是时序（抖动）余量。** 接收端不是随便挑个时刻采样，而是尽量在**眼睛最宽的正中间**采（这时候上下开口最大、最保险）。眼宽，就是左边一簇过零交叉点到右边一簇交叉点之间的**水平距离**（通常用 UI 的比例表示，如 0.7UI）。**眼宽越大，采样时刻左右能容忍的抖动越多**；眼宽被抖动啃窄，采样点稍微偏一点就可能踩到边沿、判错。

**交叉点（Crossing）——上升沿和下降沿相交的那两簇点。** 它的**厚度**反映边沿的抖动：交叉点越"糊"、越厚，说明边沿在时间上晃得越厉害（抖动大）；交叉点的**高低**如果偏离中线，说明占空比或上升下降不对称。

**采样点（判决点）——眼睛正中心。** 接收端的最佳采样位置，就在眼睛水平中央、判决电平那一点。**一句话：接收端就是站在眼睛中心，往上下看还剩多少电压余量（眼高）、往左右看还剩多少时间余量（眼宽）。**

把这几个量连起来看：**眼睛睁得越大（又高又宽），余量越足、误码率越低；眼睛被啃得越小，余量越薄；眼睛闭合（开口为零），就必然误码。** 这就是为什么一张眼图能顶得上一大堆单项测量——它把"还剩多少余量"直接画成了一只眼睛的大小。

![图3：眼图解剖。一只睁开的眼睛，标注：①眼高=上眼皮到下眼皮的垂直开口=电压噪声余量②眼宽=左右两簇交叉点之间的水平开口=时序/抖动余量（如0.7UI）③交叉点=边沿相交处，厚度=抖动④判决电平=横向中线⑤最佳采样点=眼睛正中心⑥一个UI的宽度标尺⑦上升/下降沿。旁注：接收端站在眼睛中心，上下看眼高、左右看眼宽](/images/31/ill03_anatomy.png)

> **一句话记住：眼高 = 垂直开口 = 电压噪声余量；眼宽 = 水平开口 = 时序/抖动余量；交叉点厚度 = 抖动；采样点在眼睛正中心。眼睛越大余量越足，闭了就误码。**

## 三、谁在啃眼睛：反射串扰压眼高，损耗抖动压眼宽

眼图最妙的地方，是它把前面几篇讲的捣乱因素**全汇到一张图上**——每个杀手从不同方向啃眼睛，一看眼睛哪边瘪了，就知道是谁在作怪。还记得第 27 篇结尾那三个杀手加上抖动吗？现在一一对号入座：

**反射（第 28 篇）——垂直方向压眼高。** 阻抗不连续处的余振（过冲、振铃）叠在电平上，就是一层电压噪声，把上下眼皮往中间挤，**眼高变小**。

**串扰（第 30 篇）——垂直方向压眼高。** 隔壁 aggressor 每次翻转串过来的尖峰，也是叠在电平上的噪声，同样**压眼高**；而且它跟着邻线的数据走、时有时无，让上下眼皮变得毛糙。

**损耗（第 27 篇 / 高频衰减）——眼高眼宽一起压，是最典型的"闭眼元凶"。** 走线对高频的衰减，会让陡边沿**走不到位**：这一位还没爬到满幅，下一位就来了，前一位的"尾巴"拖进后一位，这叫 **ISI（码间干扰，Inter-Symbol Interference）**。连续翻转的位（高频成分多）衰减最狠、幅度最小，连续不变的位（低频）幅度最大——两者一叠，**眼睛既变矮（眼高↓）又变窄、边沿变糊（眼宽↓）**。走线越长、速率越高，损耗吃眼睛越狠。

**抖动（Jitter，下一篇专讲）——水平方向压眼宽。** 边沿在时间轴上左右晃动（时钟不干净、电源噪声、ISI 都会引入抖动），交叉点这一簇就横向摊开、变厚，**把眼宽从两侧啃窄**。

看明白了没有——**反射、串扰主要从上下压（噪声 → 眼高），抖动主要从左右压（时间 → 眼宽），损耗则上下左右一起来（ISI）。** 所以拿到一张病态眼图，先看它是**矮（眼高小）**、还是**窄（眼宽小）**、还是**糊（边沿厚）**，就能快速缩小怀疑范围，回头去查对应的那一篇。眼图不解决问题，但它是最好的"分诊台"。

![图4：谁在啃眼睛。中间一只被啃小的眼睛，四个方向标注杀手：上下（反射余振/串扰尖峰→噪声→压眼高）；左右（抖动→边沿左右晃→压眼宽）；损耗ISI→连续翻转位幅度衰减+边沿拖尾→眼高眼宽一起压。对比：干净大眼 vs 被啃小的眼。旁注：矮=噪声(反射/串扰)、窄=抖动、糊=损耗，据此分诊](/images/31/ill04_killers.png)

> **一句话记住：反射、串扰从上下压（噪声→眼高↓）；抖动从左右压（时间→眼宽↓）；损耗 ISI 上下左右一起压（连续翻转位衰减、边沿拖尾）。眼图矮→查噪声、窄→查抖动、糊→查损耗。**

## 四、把眼图量化：眼高眼宽、Q 因子、BER 浴盆曲线、掩模测试

光看"睁得大不大"还不够，一致性测试要的是**数字和判决**。眼图有一套量化工具：

**眼高、眼宽的数值。** 直接读：眼高多少 mV、眼宽多少 UI（或 ps）。判据来自接口标准或器件手册（如某接口要求接收端眼高 ≥ 某 mV、眼宽 ≥ 0.4UI）。

**Q 因子（Q-factor）——把眼高和噪声折算成误码率。** 定义大致是

**Q ≈ 眼高 ÷ （上下噪声的均方根之和）。**

Q 越大，信号离判决电平越"远、越干净"，误码率越低。它俩有一一对应的换算：**Q ≈ 6 对应 BER ≈ 10⁻⁹，Q ≈ 7 对应 BER ≈ 10⁻¹²**。所以拿到眼高和噪声，就能估出这条链路的误码率够不够指标——不用真的跑到 10⁻¹² 那么久。

**BER 浴盆曲线（Bathtub Curve）——眼宽到底有多宽，得按误码率来量。** 把接收端的**采样点从眼睛左边一路扫到右边**，每个位置测一下误码率画成曲线：采样点在眼睛正中间时误码率最低（曲线底部），越往两边靠近交叉点、误码率越高（曲线两壁陡升）——整条曲线长得像个**浴盆**。**浴盆底部那段够低（比如低于 10⁻¹²）的宽度，才是"某个误码率下的真实眼宽"**。垂直方向同理，扫判决电平能画出**垂直浴盆曲线**，量出某误码率下的真实眼高。浴盆曲线是"眼睛开口"更严谨、更定量的说法。

**掩模 / 眼图模板（Mask）测试——一票否决的合格判据。** 在眼图正中间放一个**规定形状的禁区多边形**（掩模，通常是个六边形加上下两块），外加对幅度上下限的约束。规矩很简单：**任何一条波形轨迹都不许碰到这个禁区**，碰到（mask hit）就判 fail。它把"眼睛必须睁到多大"变成一个客观的、能自动判的几何模板，是 USB、PCIe、HDMI、以太网这些接口一致性测试里的**标准动作**。

![图5：眼图量化三件套。左：BER浴盆曲线——采样点从眼左扫到眼右，误码率画成浴盆状，浴盆底部<10⁻¹²的宽度=某BER下的真实眼宽；中：Q因子=眼高÷噪声，标Q=6→BER10⁻⁹、Q=7→BER10⁻¹²换算；右：掩模mask测试——眼中央六边形禁区+上下限，任何轨迹碰到禁区即fail。旁注：浴盆越宽、Q越大、无mask碰撞=余量足](/images/31/ill05_bathtub.png)

> **一句话记住：眼高眼宽读数值；Q 因子 = 眼高÷噪声，Q≈6→BER10⁻⁹、Q≈7→BER10⁻¹²；BER 浴盆曲线横扫采样点画出某误码率下的真实眼宽；掩模测试放个中央禁区多边形，波形碰到即 fail——一致性测试标准判据。**

## 五、怎么测眼图：恢复时钟、PRBS 码型、累积够多

眼图好看，但要测对、测准，有几个不能省的动作：

**① 触发要用恢复时钟（CDR），不能用固定触发。** 眼图叠加的前提是"每个 UI 对齐同一位置"。链路自己的时钟会有抖动，如果用示波器固定频率去触发，你测出来的是"时钟抖动 + 链路抖动"的混合，还叠不齐。必须用 **CDR 从数据里恢复时钟**再触发，这样测的才是**相对于恢复时钟**的真实眼图（这也是接收机自己看到的眼）。示波器/眼图仪一般内置软/硬件 CDR。

**② 码型要用 PRBS，覆盖所有组合。** 眼图要看"所有 bit 组合的最坏情况"，就得让码流**跑遍各种前后组合**。用 **PRBS（伪随机码，如 PRBS7、PRBS31）** 当激励，它能在有限长度里遍历足够多的 `0/1` 组合，把 ISI 的最坏情况逼出来。用 `1010` 这种规则码测眼图，会**假**——因为它只走一条路径，眼睛看着很大，其实没覆盖最坏组合。

**③ 累积要够多。** 眼图是统计图，叠的 bit 太少，罕见的最坏组合、大抖动的边沿还没出现，眼睛会**偏乐观**。要累积**足够多的 UI**（几万到上百万，甚至更多），尤其想验证到 10⁻¹² 这种低误码率，靠直接叠加是叠不出来的——要么累积到足够久，要么用**浴盆曲线外推**估算。

**④ 实时采样 vs 等效时间采样。** 抓眼图有两类仪器：**实时示波器**（一次抓一长段，软件折叠成眼图，能抓偶发事件、能做抖动分解，但带宽/采样率要求高、贵）；**采样示波器 / 眼图仪**（等效时间采样，靠恢复时钟重复采样点点画出眼图，带宽极高、噪声底极低、最适合看纯净眼图，但**抓不到偶发/单次事件**）。看你是要"高保真的稳态眼"还是"逮偶发问题"来选。

**⑤ 别忘上一篇的老规矩：带宽要够、探头地线要短、接被测点。** 眼图里的边沿又快又细，仪器带宽不够照样把眼睛测得又矮又糊，让你冤枉链路。

![图6：怎么测眼图。①CDR从数据恢复时钟做触发（不用固定触发，否则叠不齐/混入时钟抖动）②PRBS伪随机码激励（覆盖所有bit组合，1010规则码测眼会假偏大）③累积足够多UI（几万~百万，太少偏乐观；低BER靠浴盆外推）④实时示波器（抓偶发/可做抖动分解）vs 采样示波器/眼图仪（等效采样、带宽高噪底低、看纯净眼但抓不到偶发）⑤带宽够、地线短](/images/31/ill06_measure.png)

> **一句话记住：恢复时钟（CDR）触发才叠得齐、才是接收机看到的眼；PRBS 码型覆盖最坏组合（别用 1010 规则码，会假）；累积够多 UI（低 BER 靠浴盆外推）；实时示波器逮偶发、采样示波器/眼图仪看纯净眼；带宽仍要够。**

## 六、怎么把眼睛重新睁开：前几篇的治法 + 均衡

眼图是一张**体检报告**——它告诉你余量还剩多少，但真要"治"，得回到啃它的那些杀手身上。好消息是，**这些治法我们前几篇几乎都讲过了**：

- **眼睛矮（噪声压眼高）**：查反射，做好端接（第 28 篇）；查串扰，拉间距守 3W、加护地线、补完整参考平面（第 30 篇）。
- **眼睛糊、边沿抖（抖动压眼宽）**：净化时钟、稳住电源、降低 ISI（下一篇专讲抖动怎么分解、怎么治）。
- **眼睛又矮又窄（损耗 ISI）**：这就要请出一个新武器——**均衡（Equalization）**。

**均衡，就是专门跟损耗对着干。** 损耗把高频（连续翻转的位）压得比低频（连续不变的位）狠，均衡就**反过来给高频更多增益、或给跳变的位更大力度**，把被损耗压平的频响重新拉平，眼睛就重新睁开。工程上分三种，前几篇的思路一脉相承：

- **发端预加重 / 去加重（Pre-emphasis / De-emphasis）**：发送时**故意把跳变位的幅度加大**（或把连续不变位压低），预先补偿即将到来的高频损耗。
- **接收端 CTLE（连续时间线性均衡）**：一个**高频增益更大的模拟滤波器**，把收到的信号高频抬起来，抵消线损。
- **接收端 DFE（判决反馈均衡）**：根据前几位已经判决的结果，**反算并扣掉它们拖到当前位的 ISI 尾巴**，把拖尾"减"干净。

高速接口（PCIe、USB3/4、SerDes）里，这几种均衡常常一起上——发端预加重 + 收端 CTLE + DFE，硬是把一只在长走线尽头快闭合的眼睛，**在接收芯片内部重新睁开到能可靠判决**。所以你在**芯片输出脚**测眼图可能很难看，但接收端**均衡之后**的内部眼（很多 SerDes 能吐出内部眼图或 eye monitor 数据）才是真正决定误码率的那只眼。

## 🔧 工程师速查（附录）

### 一、基本概念

| 名词 | 含义 |
|---|---|
| UI（单位间隔） | 一个 bit 占用的时间；UI ＝ 1 ÷ 波特率（5Gbps → 200ps） |
| 眼图 | 把码流按 UI 切段、用恢复时钟对齐、成千上万位叠加到同一时间窗 |
| 眼高 Eye Height | 垂直开口 = 电压噪声余量（上眼皮最低"1"到下眼皮最高"0"） |
| 眼宽 Eye Width | 水平开口 = 时序/抖动余量（左右两簇交叉点间距，用 UI 表示） |
| 交叉点 Crossing | 上升/下降沿相交处；厚度反映抖动、高低反映占空比对称性 |
| 采样点 | 接收端最佳判决位置 = 眼睛正中心 |

### 二、谁啃眼睛（分诊表）

| 眼图症状 | 主要元凶 | 回查哪篇 |
|---|---|---|
| 眼睛**矮**（眼高小、噪声层厚） | 反射余振、串扰尖峰 | 第 28 篇端接 / 第 30 篇串扰 |
| 眼睛**窄**（眼宽小、交叉点厚） | 抖动（时钟/电源/ISI） | 下一篇 抖动 Jitter |
| 眼睛又矮又窄、边沿**糊/拖尾** | 损耗 ISI（高频衰减、码间干扰） | 第 27 篇 / 均衡 |

### 三、量化工具

- **眼高 / 眼宽**：读数值，对接口标准或手册判据（mV、UI/ps）。
- **Q 因子**：Q ≈ 眼高 ÷ 噪声 RMS 之和；**Q≈6 → BER≈10⁻⁹，Q≈7 → BER≈10⁻¹²**。
- **BER 浴盆曲线**：横扫采样点画误码率浴盆，底部 <目标 BER 的宽度 = 该 BER 下真实眼宽；竖扫判决电平得垂直浴盆 → 真实眼高。
- **掩模 / 眼图模板（Mask）**：眼中央禁区多边形 + 幅度上下限，任何轨迹碰到 = fail；接口一致性标准判据。

### 四、测量要点

- **触发用 CDR 恢复时钟**：才叠得齐、才是接收机看到的眼；固定触发会混入时钟抖动。
- **码型用 PRBS**（PRBS7/PRBS31）：覆盖最坏 bit 组合；`1010` 规则码测眼**偏大失真**。
- **累积足够多 UI**：太少偏乐观；验证 10⁻¹² 级靠浴盆曲线**外推**。
- **仪器选型**：实时示波器（逮偶发、可抖动分解）vs 采样示波器/眼图仪（等效采样、带宽高噪底低、看纯净眼、抓不到偶发）。
- **带宽要够、地线要短**（呼应第 29 篇 5 倍法则），否则把眼睛测矮测糊。

### 五、把眼睛睁开

- **压眼高的（噪声）**：端接治反射（第 28 篇）、3W/护地线/完整参考面治串扰（第 30 篇）。
- **压眼宽的（抖动）**：净化时钟、稳电源、降 ISI（下一篇）。
- **损耗 ISI**：**均衡**——发端**预加重/去加重**、收端 **CTLE**（高频增益）、收端 **DFE**（扣前几位拖尾）。高速链路常三者并用。
- **看内部眼**：很多 SerDes 提供均衡后内部眼 / eye monitor，外部脚测不准时以内部眼为准。

### 六、常见坑

- ☐ 用 **1010 规则码**测眼图 → 没覆盖最坏组合，眼睛假大。
- ☐ 用**固定触发**代替 CDR → 混入时钟抖动、叠不齐。
- ☐ 累积 UI **太少** → 罕见最坏组合没出现，眼睛偏乐观。
- ☐ 仪器**带宽不够 / 地线长** → 眼睛测矮测糊，冤枉链路。
- ☐ 只在**芯片输出脚**测眼、忽略接收端**均衡后内部眼** → 误判链路不合格。

> 说明：本文数值为通用工程近似（UI=1/波特率、Q↔BER 换算、眼高 400/80mV、眼宽 0.7/0.2UI、PRBS、掩模均为业界通用概念与示意值），具体项目以接口标准（PCIe/USB/以太网等）、器件手册与一致性测试规范为准；不涉及任何机密资料。
