# 一根 Type-C 线，怎么同时接显示器、固态、外接显卡、还有一堆 USB 设备——Thunderbolt / USB4：把 PCIe、DisplayPort、USB 全「打包成隧道」塞进一条链路的融合者

先说个几乎人人都体验过、却很少想过背后有多复杂的场景。

你有一台轻薄本，只有一个 Type-C 口。下班回到家，插上一根雷电线到扩展坞，瞬间：**一台 4K 显示器亮了**、**一块外接固态开始每秒几个 GB 地拷贝**、**鼠标键盘 U 盘全能用**、有人甚至**外接了一块显卡打游戏**——而这一切，**只用了那一根线**。

反过来，翻车也很典型：换了根便宜的 Type-C 线，**显示器闪、掉分辨率、外接固态掉速甚至断开**；或者插上某个扩展坞，**雷电设备直接识别不到、或者只跑成了普通 USB**。很多人以为是坞坏了、线是假的。但做硬件的人清楚：**这类「一口通吃却偶发翻车」里，一大半是 Thunderbolt / USB4 这条融合链路——要么是链路本身的信号眼图 / 抖动没测好，要么是里面某一条隧道（PCIe / DP / USB）没协商好、带宽没分对。**

它是怎么把三种协议塞进一根线的、TB3 到 TB5 怎么从 40 Gbps 翻到 80 甚至 120、为什么它的测试要**分两层**看——这，就是这一篇要讲的，也是我们把整个串行家族**收口**的一篇。

## 一、先认清主角：Thunderbolt 和 USB4 是什么、什么关系

**Thunderbolt（雷电）** 是 Intel 主导的高速接口，从 TB3 起统一用 **Type-C 接口**；**USB4** 是 USB-IF 组织在 **Thunderbolt 3 基础上**制定的标准——所以这俩本质上是**同源、高度兼容的孪生兄弟**，可以放在一起讲。它们的定位不是「又一种接口」，而是**一根线通吃**：

- **一端**是电脑上的 Thunderbolt / USB4 **主机口**，**另一端**是显示器、扩展坞、外接固态、外接显卡、雷电硬盘……中间还能**菊花链**串接多个设备。
- 关键在于它**内部同时承载三种协议**：**PCIe**（接外接固态、外接显卡、雷电硬盘）、**DisplayPort**（接显示器）、**USB**（接键鼠、U 盘等普通外设）。

它有三个关键特征，正好是这一篇的测试主线：

1. **融合 / 隧道（tunneling）**：它把 PCIe、DP、USB 这几路数据**各自打包成「隧道」**，一起在同一条链路上传输，由一个 **Thunderbolt / USB4 路由器**动态分配带宽。**这是它最独特的地方，也是它测试多出一整层的原因。**
2. **串行差分 + 嵌入式时钟 → 看眼图**：链路本身是**多对差分 lane**（Type-C 上 2 对发、2 对收），时钟埋在数据里、靠 **CDR** 恢复——**物理层电气还是看眼图 + 抖动**，和 USB3、PCIe 完全同岸。
3. **靠 Retimer 撑长距离**：速率高到 40 甚至 80 Gbps，走过长线缆和板级走线信号会烂，所以主机、线缆、坞里普遍要放 **Retimer（信号重定时器）**——它把信号**接收、恢复、再干净地重发**，是这类接口能过一致性、能拉长线的关键器件。

一句话：**Thunderbolt / USB4 = 一根 Type-C 线，把 PCIe、DP、USB 打包成隧道一起传**；它的电气还是「看眼图」那一岸，但多出「隧道 / 协议」这一整层要测。

## 二、隧道原理：把三种协议「打包」进一条链路，动态分带宽

这是 Thunderbolt / USB4 最核心、也是它区别于前面所有接口的地方——**隧道（tunneling）**。

前面的接口，一条链路就跑一种协议：PCIe 链路只跑 PCIe，DP 链路只跑 DP。而 Thunderbolt / USB4 是**在物理链路之上，架了一层「传输 / 交换」**：

- **打包成隧道**：主机把要传的 PCIe 数据、DP 视频、USB 数据，各自**切成小包、贴上标签**，塞进同一条链路轮流传；到了对端，路由器再**按标签拆开**，还原成一路 PCIe、一路 DP、一路 USB，送给各自的控制器。这就叫**把协议「隧道化」**。
- **一个路由器居中调度**：链路两端各有一个 **Thunderbolt / USB4 路由器**，负责建立链路、协商能力、**动态分配带宽**——你插一台高刷 4K 显示器，它就多分点给 DP 隧道；你拷大文件，它就多分点给 PCIe 隧道。
- **总带宽是共享的**：所有隧道**共用**这条链路的总带宽（TB4 是 40 Gbps、TB5 是 80 Gbps）。所以「显示器一插上，外接固态就慢了一点」不是错觉——它们在**抢同一条隧道链路的带宽**。

对测试的意义很直接：**你不能只测链路的眼图就算完**，还得验**每一条隧道能不能正常建立、带宽分配对不对、极限场景（4K 视频 + PCIe 满速 + USB 同时跑）下会不会有隧道被饿死或掉链**。这就是它比 PCIe、DP 多出的那一层。

## 三、版本与速率：TB3 到 TB5，USB4 到 USB4 v2

Thunderbolt 和 USB4 的版本是**交织**的，一张表理清：

| 家族 | 版本 | 链路速率 | 编码 | 隧道协议 |
|---|---|---|---|---|
| Thunderbolt | **TB3** | 40 Gbps | NRZ | PCIe 3.0 / DP 1.2 / USB 3.1 |
| Thunderbolt | **TB4** | 40 Gbps（同 TB3 物理层，要求更严） | NRZ | PCIe（≥32 Gbps）/ DP 1.4 / USB 3.2 |
| Thunderbolt | **TB5** | 80 Gbps 双向，**可到 120 Gbps 非对称**（给显示） | **PAM3** | PCIe 4.0 / DP 2.1 / USB4 v2 |
| USB4 | **USB4（v1）** | 20 / 40 Gbps（Gen2 / Gen3） | NRZ | PCIe / DP / USB 3.x |
| USB4 | **USB4 v2** | 80 Gbps | **PAM3** | PCIe / DP 2.x / USB 3.x |

几个测试上必须记住的点：

- **TB4 不比 TB3 快，但要求更严**：两者物理层都是 40 Gbps NRZ；TB4 的意义是把「可选项」变成「必选项」——强制支持充电、菊花链、2 米线、最低 PCIe 带宽等。**所有 TB4 都是 USB4，但不是所有 USB4 都是 TB4。**
- **TB5 / USB4 v2 是大跳变，从 NRZ 跳进 PAM3**：为了把单向 lane 从 20 Gbps 提到 40 Gbps，又不想把符号率翻倍（那样通道扛不住），它们**换了 PAM3——三电平编码**（下一节细讲）。这和 PCIe Gen6 跳 PAM4 是同一个思路、不同选择。
- **TB5 的「非对称带宽」**：正常是 80 Gbps 双向（收发各 40）；接超高刷显示器时，可以把带宽**重新分配成一个方向 120 Gbps**（三条 lane 都给下行显示），另一个方向留 40。**测试要覆盖对称和非对称两种配置。**
- **Type-C 是唯一物理接口**：全系列都走 Type-C，2 对差分发、2 对差分收；速率算法要按每 lane 的实际线速率和编码来推，别被「40 / 80 Gbps」这种链路总带宽数字带偏。

![图4：TB/USB4 版本速率阶梯。上排 Thunderbolt：TB3(40G·NRZ·PCIe3/DP1.2/USB3.1)→TB4(40G·NRZ·同 TB3 物理层但要求更严·强制充电/菊花链/2m 线)→TB5(80G 双向·可 120G 非对称·PAM3·PCIe4/DP2.1)。下排 USB4：USB4 v1(20/40G·NRZ·Gen2/Gen3)→USB4 v2(80G·PAM3)。每格标速率+编码+隧道协议。底部三条：①TB4 不比 TB3 快只是要求更严（所有 TB4 都是 USB4，非所有 USB4 都是 TB4）②TB5/USB4v2 从 NRZ 跳 PAM3（同 PCIe Gen6 跳 PAM4 思路不同选择）③TB5 非对称带宽=可把 80G 双向重分成单向 120G 给显示，测试要覆盖对称+非对称。全系列唯一物理口=Type-C（2 发 2 收）](/images/49/ill04_versions.png)

## 四、独特之处：PAM3 三电平 + 非对称带宽 + 隧道分层

Thunderbolt / USB4 有三处别家没有、直接影响测试的独特设计：

### 4.1 PAM3：三电平编码（不是 PAM4）

到 TB5 / USB4 v2，为了把速率翻倍又不把符号率翻倍，链路**从 NRZ 换成了 PAM3——三个电平**（低、中、高），一个符号能携带比 1 bit 更多的信息。

- 和 **NRZ**（两电平、一只眼）比：PAM3 是**三电平、两只眼**，眼睛更小、信噪比更低、测试更复杂。
- 和 **PCIe Gen6 的 PAM4**（四电平、三只眼）比：PAM3 选了**更保守的三电平**——在同样的信噪压力下，PAM3 比 PAM4 更容易做、抗噪更好，代价是每符号带的信息略少。**这是一个很典型的工程折中：PCIe 用 PAM4 冲极限带宽，USB4 v2 用 PAM3 换更稳的信号裕量。**
- 一致性测试因此要**同时看两只眼、评三个电平的分离度**，比 NRZ 的单眼复杂。

### 4.2 非对称带宽

前面讲过，TB5 能把 80 Gbps 双向**重分成 120 Gbps 单向**给显示。这意味着**同一条链路有两种带宽配置**，测试和信号裕量都要分别覆盖。

### 4.3 隧道分层

这是它和所有前面接口最根本的不同：**它的「内容」是别的协议**。所以它的判据体系天然分两层——**链路电气一致性**（管这条 Type-C 链路的眼图、抖动、PAM3 电平），和**隧道 / 协议一致性**（管 PCIe、DP、USB 各隧道建立、路由、带宽分配）。

一句话：**PAM3 让它的眼图测试更像 PCIe Gen6 那种多电平；非对称带宽让它多一种配置要测；而隧道分层，让它的测试永远是「电气 + 协议」两层一起看。**

![图5：TB/USB4 三大独特。①PAM3 三电平（非 PAM4）：TB5/USB4v2 从 NRZ 换 PAM3——NRZ 两电平一只眼 → PAM3 三电平两只眼（眼更小/信噪更低/更复杂）；对比 PCIe Gen6 PAM4 四电平三只眼——PAM3 更保守、抗噪更好、每符号信息略少（典型折中：PCIe 冲带宽用 PAM4，USB4v2 换裕量用 PAM3）。一致性要同时看两只眼+评三电平分离度。②非对称带宽：TB5 可把 80G 双向重分成 120G 单向给显示→同链路两种配置都要测。③隧道分层：它的"内容"是别的协议→判据天然分两层=链路电气一致性(眼图/抖动/PAM3 电平)+隧道协议一致性(PCIe/DP/USB 各隧道建立/路由/带宽)](/images/49/ill05_pam3.png)

## 五、测什么 · 三类：链路 Tx 眼图、链路 Rx 压力、隧道 + 协议

Thunderbolt / USB4 的一致性测试，可以归成三类——**前两类是链路物理层电气（复用 USB3 / PCIe 那套），第三类是它独有的隧道 / 协议层**：

1. **链路 Tx / 发送端眼图**：示波器在 Type-C 测试点抓链路差分眼。TB3 / TB4 / USB4 是 **NRZ 单眼**，套眼图模板、扣抖动、验去加重预设；TB5 / USB4 v2 是 **PAM3，要同时看两只眼、评三电平分离度**。方法和 USB3、PCIe 完全一脉相承。
2. **链路 Rx / 接收端压力**：用码型 + 抖动发生器造校准的**压力信号**（把损耗、抖动、串扰加满）灌进被测端，测**抖动容限 / 误码**（↔第 41 篇 USB3 的 Rx jitter tolerance）；高速下还要考虑接收端均衡和 **Retimer** 的行为。
3. **隧道 / 协议**：这是它独有的一层。要验**链路训练与协商**（协商到几代速率、对称还是非对称）、**各隧道能否正常建立**（PCIe 隧道、DP 隧道、USB 隧道分别拉起来）、**带宽分配是否正确**、以及**极限并发场景**（4K 视频 + 外接固态满速 + USB 设备同时跑）下会不会有隧道掉链、饿死。用 **Thunderbolt / USB4 协议分析仪**看包、看路由、看隧道状态。

贯穿始终的还有**通道**：Type-C 线缆、连接器、板级走线、还有那些 **Retimer**，插损 / 回损 / 串扰都要测——线缆质量差，正是「换根线就翻车」的直接原因。

![图6：TB/USB4 测什么三类（前两类=链路电气复用 USB3/PCIe，第三类=独有隧道协议层）。①链路 Tx 眼图：Type-C 测试点抓链路差分眼，TB3/4/USB4=NRZ 单眼套模板扣抖动验去加重预设；TB5/USB4v2=PAM3 同时看两只眼+评三电平分离度（方法同 USB3/PCIe）。②链路 Rx 压力：码型+抖动发生器造校准压力信号(loss/jitter/串扰加满)灌被测端测抖动容限/误码(↔41)，考虑接收端均衡+Retimer 行为。③隧道/协议(独有)：验链路训练协商(几代速率/对称非对称)+各隧道建立(PCIe/DP/USB 分别拉起)+带宽分配+极限并发(4K 视频+固态满速+USB 同跑)会不会掉链饿死，用 TB/USB4 协议分析仪看包/路由/隧道态。贯穿：Type-C 线缆/连接器/走线/Retimer 插损回损串扰——换根线就翻车的根源](/images/49/ill06_test3.png)

## 六、怎么测 · 台面：Type-C 夹具 + 进 compliance 模式 + Retimer

Thunderbolt / USB4 的测试台面，电气部分和 USB3、PCIe 那套高度一致，但有几处 Type-C / 融合接口独有的讲究：

- **① Type-C 测试夹具 / 转接板**：Type-C 口小、引脚密、还要区分 2 对发 2 对收，必须用专用的 **Type-C 测试夹具（fixture）** 把链路的差分对引出来接示波器；测线缆、坞则用对应的转接板。
- **② 进 compliance / 测试模式**：和 USB3、PCIe 一样，让被测端进入**规定的一致性测试模式**，持续输出测试码型、锁定在目标速率档和编码（NRZ 或 PAM3），方便示波器长时间累积眼图；再通过规定手段在**各速率档、对称 / 非对称配置**间切换。
- **③ 抓眼 + 判读**：NRZ 抓单眼套模板；PAM3 抓两只眼、评三电平分离度；配合接收端均衡和 Retimer 一起判。
- **④ Retimer 单独考量**：链路里那些 Retimer 本身也要过一致性——它的输入眼、输出眼、抖动都要单独测，因为它是「接收 - 恢复 - 重发」，一旦它没调好，整条链路就崩。
- **⑤ 协议 / 隧道验证**：换上 **Thunderbolt / USB4 协议分析仪 + 各种目标设备**（显示器、外接固态、USB 设备、扩展坞），验隧道建立、菊花链、带宽分配、极限并发。

一句话：**Thunderbolt / USB4 台面 = USB3 / PCIe 的眼图 Rx 那套电气测法 + Type-C 夹具 + Retimer 单独测 + 一整套隧道 / 协议验证**；你前面几篇学的电气方法，在这里几乎全部复用。

## 七、判据：USB4 / Thunderbolt 一致性 —— 电气 + 隧道两本，从预算反推

Thunderbolt / USB4 的判据，天然分**两大层**，正好对应它「融合者」的身份：

- **链路电气一致性**：来自 **USB4 电气规范**和 **Intel Thunderbolt 一致性规范**——管这条 Type-C 链路的**发送眼图模板、抖动预算、去加重预设、Rx 抖动容限、PAM3 的三电平分离度、Retimer 指标**。这些模板和限值，和串行家族一样，是**从信道损耗加抖动预算反推**出来的。
- **隧道 / 协议一致性**：管 **PCIe 隧道、DP 隧道、USB 隧道**能否正确建立、路由、分配带宽，以及**互操作**（各种设备、线缆、坞混插能不能认、能不能达标）。
- **认证体系是双轨的**：**USB-IF 负责 USB4 认证**，**Intel 负责 Thunderbolt 认证**——一个设备要打「雷电」标，得过 Intel 那套更严的认证；打「USB4」标，过 USB-IF 的即可。这跟 DP 分物理层 / 链路层两本、UFS 分 M-PHY / JEDEC 两本，是同一个「电气一本、系统 / 协议一本」的套路——只不过 Thunderbolt 的「协议那本」，装的是**别人家的三种协议**。

## 八、常见翻车 → 分层揪真凶（别一上来就换线换坞）

一口通吃翻车，现象往往很唬人（显示器闪、固态掉、设备认不到），但排查还是分层：

- **① 链路协商 / 速率层**：链路训练过没过、协商到了 TB5 还是回落成 USB4、甚至只跑成了普通 USB / DP Alt Mode。**雷电设备「只跑成普通 USB」、掉到低速**，绝大多数死在这一层——多半是线缆 / 坞 / Retimer 不达标，链路只能回落保命。先查这里。
- **② 链路电气层**：协商上了高速但偶发误码、断连。去量链路的**眼图（NRZ 单眼 / PAM3 两眼）、抖动、通道插损、Retimer 输入输出**。**换根便宜线就翻车**，通常就是线缆通道 / Retimer 这一层。
- **③ 隧道 / 带宽层**：链路很稳，但**某个设备时好时坏、并发时掉链**。去看隧道建立、带宽分配——比如 4K 高刷 + 外接固态满速时，DP 隧道和 PCIe 隧道抢带宽，某一路被压。
- **④ 协议 / 互操作层**：特定设备、特定坞、菊花链某一级认不到。去查各隧道协议、路由、设备兼容。
- **⑤ 供电 / PD 层**：Type-C 还要供电（USB PD），**供电协商不对、大负载掉压**会导致设备偶发重连——呼应第 42 篇 USB PD。

口诀一句：**先看协商到几代、有没有回落 → 再量链路眼图 / 抖动 / Retimer → 再看隧道和带宽分配 → 再查协议互操作 → 别忘 Type-C 供电。**

![图8：TB/USB4 常见翻车分层揪真凶。左五层：①链路协商/速率层（训练过没过/协商到 TB5 还是回落 USB4/只跑成普通 USB·DP Alt Mode——雷电设备"只跑成普通 USB"掉速死这层,多半线缆/坞/Retimer 不达标回落,先查）②链路电气层（协商上高速偶发误码/断连→量眼图 NRZ 单眼/PAM3 两眼+抖动+通道插损+Retimer 输入输出;换便宜线翻车常在这层）③隧道/带宽层（链路稳但某设备时好时坏/并发掉链→隧道建立+带宽分配,4K 高刷+固态满速抢带宽）④协议/互操作层（特定设备/坞/菊花链某级认不到→各隧道协议/路由/兼容）⑤供电/PD 层（Type-C 供电协商不对/大负载掉压→偶发重连,呼应 42 USB PD）。右现象→指向哪层对照。口诀：先看协商回落→链路眼图抖动 Retimer→隧道带宽→协议互操作→别忘供电](/images/49/ill08_debug.png)

## 九、工程师速查（一图带走）· 兼模块六串行家族收官

- **主角 / 定位**：一根 Type-C 线，把 **PCIe + DisplayPort + USB 打包成隧道**一起传的**融合者**；Thunderbolt（Intel）与 USB4（USB-IF 在 TB3 基础上）同源孪生。
- **站哪一岸**：链路物理层还是**嵌入式时钟 + CDR + 看眼图**（同 USB3 / PCIe）；但**测试分两层**——链路电气 + 隧道 / 协议。
- **速率 / 编码**：TB3 / TB4 = 40 Gbps **NRZ**；TB5 / USB4 v2 = 80 Gbps **PAM3**（三电平两眼），TB5 可到 **120 Gbps 非对称**给显示。
- **独特**：**隧道分层**（内容是别人家协议）、**PAM3 三电平**（比 PCIe 的 PAM4 保守）、**非对称带宽**、**Retimer 关键器件**。
- **测什么三类**：链路 Tx 眼图（NRZ 单眼 / PAM3 两眼）、链路 Rx 压力（抖动容限）、隧道 + 协议（各隧道建立 / 带宽 / 并发）。
- **怎么测**：Type-C 夹具 + 进 compliance + Retimer 单独测 + 协议分析仪验隧道；电气全复用 USB3 / PCIe。
- **判据 / 调试**：USB4 电气 + Thunderbolt CTS 两层，USB-IF / Intel 双认证；调试分层：协商回落 → 链路电气 / Retimer → 隧道带宽 → 协议互操作 → 供电。
- **收官**：学到这，模块六串行家族（USB / HDMI / DP / UFS / MIPI / PCIe / DDR / Thunderbolt）你就通了——**两把尺子（用途 + 时钟架构）分岸，三大类测法（Tx 眼 / Rx 压力 / 协议）打天下，判据永远从预算反推、调试永远分层。**

![图9：TB/USB4 工程师速查卡（六格）兼模块六收官。主角/定位：一根 Type-C 把 PCIe+DP+USB 打包成隧道的融合者,TB(Intel)与 USB4(USB-IF 基于 TB3)同源孪生。站哪岸：链路物理层嵌入式时钟+CDR 看眼图(同 USB3/PCIe),但测试分两层=链路电气+隧道协议。速率/编码：TB3/4=40G NRZ,TB5/USB4v2=80G PAM3 三电平两眼,TB5 可 120G 非对称给显示。独特：隧道分层/PAM3(比 PAM4 保守)/非对称带宽/Retimer 关键。测什么三类：链路 Tx 眼/Rx 压力/隧道+协议。怎么测：Type-C 夹具+compliance+Retimer 单测+协议分析仪,电气复用 USB3/PCIe。判据/调试：USB4 电气+TB CTS 两层,USB-IF/Intel 双认证;协商回落→电气/Retimer→隧道带宽→协议→供电。底部：↔47 PCIe·44 DP·42 USB PD·41 USB3 Rx·38 一致性总览·速查02 全景地图（模块六串行家族收官，下一篇 以太网）](/images/49/ill09_quickref.png)

## 一句话总结

**Thunderbolt / USB4 不发明新的物理层哲学，它是个「融合者」——一根 Type-C 线，把 PCIe、DisplayPort、USB 这几路数据各自打包成隧道，塞进同一条串行链路，靠一个路由器动态分配带宽。所以它站的还是「嵌入式时钟 + CDR + 看眼图」那一岸（和 USB3 / PCIe 同岸），到 TB5 / USB4 v2 又从 NRZ 跳进 PAM3；但它的测试永远分两层——一层测这条 Type-C 链路自己的眼图 / 抖动 / Retimer，一层测每一条隧道能不能建立、带宽分得对不对。判据是 USB4 电气规范 + Thunderbolt 一致性规范两本、USB-IF 与 Intel 双认证；调试先看协商到几代、有没有回落，再量链路电气 / Retimer，再看隧道和带宽分配。学会了前面 USB3、DP、PCIe，再把它们「叠起来」，模块六整个串行家族你就通了。**
