# 同样一个 USB 口，插 U 盘飞快、插移动硬盘却掉速——USB 2.0 的 480M 到底靠什么保证？一张眼图模板，判它"过没过"

<div class="article-meta"><span class="badge">📘 模块六 · 高速接口一致性测试 · USB 家族第 1 篇 · USB 2.0 高速眼图模板</span><span class="badge">⏱ 约 34 分钟</span><span class="badge qr">🔧 含工程师速查</span></div>


一个再普通不过的场景：同一个笔记本、同一个 USB 口，插上 U 盘，拷文件飞快；换上那块移动硬盘，速度却只有一半，偶尔还弹"设备无法识别"，拔了重插、或者换个口才好。

你换了根线，好了。**可为什么？**

线不都是 USB 线吗？插头一模一样、都能插进去、大部分时候也都能用——**"能用"和"合规"之间，隔着一整套你平时看不见的东西。** 那根差一点的线、板上那段绕远了没等长的走线，都会让高速信号在到达对端之前，**眼图悄悄往下塌**。塌到一定程度，接收端就开始认错位、要求重传，宏观上就是**掉速**；再塌一点，干脆枚举失败、"设备无法识别"。

肉眼看这根线、这块板，什么都看不出来。**你需要一个客观的、能判"过还是没过"的尺子。**

这个尺子，就是 **USB 2.0 高速信号质量测试**，核心是**一张眼图模板（eye template）**：把信号所有的跳变叠在一个比特周期里画成"眼睛"，再往上扣一张规范画好的**模板**——**眼睛只要碰到模板里那块禁区，就是 fail。** 一张图，把"这根线到底行不行"这种模糊的判断，变成一个**客观、可复现、能贴 logo 的结论**。

**读完这篇，你能搞懂：**

- ✅ USB 2.0 的三档速度、480M 的高速到底特殊在哪——**NRZI 编码、位填充、电流模式驱动**，为什么它的沿反而"不能太快"；
- ✅ 一致性测试测哪三类——**发送端眼图、接收端灵敏度、通道线缆**，和第 38 篇那套框架怎么对上；
- ✅ **我的项目到底要测哪几类**——按你在链路里是 host / device / hub / 线缆的**角色**去判定，而不是三类全都盲目做一遍；
- ✅ 怎么测——**近端 TP2、远端 TP3** 两个测试点、夹具、怎么让芯片进"测试模式"发出规定码型；
- ✅ ⚙️那几个关键仪器旋钮（带宽、时钟恢复、抖动分离、去嵌）**怎么从 SI 概念落到 USB 测试上**；
- ✅ 那张**眼图模板**上的电压、时间线是从哪儿来的、near-end 和 far-end 为什么用不同模板、一份一致性报告怎么读、哪项 fail 说明什么；
- ✅ **标准到底去哪查、怎么把这些拼成一份不遗漏的测试计划**——一张照着填的测试计划矩阵，拿到项目就能落地。

## 一、USB 是什么、480M 特殊在哪：先把主角认清

USB 你天天插，但"高速"这个词在 USB 2.0 里有特定含义，先把三档速度分清楚。

**USB 2.0 一根线上，其实跑着三档速度**（都在 D+／D− 这一对差分线上）：

- **低速 LS：1.5 Mbps**——鼠标、键盘这类。
- **全速 FS：12 Mbps**——早期 U 盘、音频设备。
- **高速 HS：480 Mbps**——U 盘、移动硬盘、摄像头，本篇的主角。

注意：**日常说的"USB 2.0"，绝大多数时候指的就是这个 480M 的高速（High-Speed）。** 它比全速快了整整 40 倍，信号也从"慢悠悠的数字电平"变成了**需要当高速信号来伺候的东西**——这才有了"一致性测试"的必要。

**高速 HS 有三个和低速/全速完全不同的地方，正是它要专门测的原因：**

**① 电流模式驱动、摆幅很小。** 低速/全速是电压模式，信号在 0 和 3.3V 之间摆；**高速换成了电流模式**——发送端是一个约 **17.78 mA** 的电流源，打到两端各 45Ω（合起来 22.5Ω）的负载上，得到一个**差分只有约 400 mV** 的小摆幅信号，围着地电位上下摆。摆幅小、速度快，**信号质量的余量就紧**，差一点就塌——这正是要测眼图的根本原因。

**② NRZI 编码 + 位填充，给时钟恢复留跳变。** USB 高速**不单独走一根时钟线**，接收端得**从数据里自己把时钟"抠"出来**（时钟恢复／CDR，就是第 32 篇讲的那套）。可要是数据里一长串都不跳变，接收端就会"跟丢"。USB 的解法有两层：先用 **NRZI 编码**（数据是 0 就翻转电平、是 1 就保持），再加**位填充**——**只要连续出现 6 个 1，就强行插进去一个 0**，逼出一次跳变。〔回链 **32 抖动 / 时钟恢复**——CDR 靠跳变锁相，位填充就是协议层面替 CDR 保证"总有跳变可用"〕

**③ 一个比特只有 2.083 纳秒。** 480 Mbps，一个比特的时间（UI，Unit Interval）＝1÷480M ≈ **2.083 纳秒**。眼图的横轴就是这一个 UI——所有跳变叠上去，中间睁开的那只"眼睛"越大，说明留给接收端判 0/1 的电压和时间余量越足。

> 一句话：**USB 2.0 的主角是 480M 的高速（HS）——电流模式、约 400mV 小摆幅、NRZI+位填充自带时钟、一个比特 2.083 纳秒。** 摆幅小、速度快、还得自己恢复时钟，所以它必须当高速信号来测眼图，不能再当普通数字电平糊弄过去。

![图2：USB 2.0 的三档速度与高速 HS 的三个关键特征。上：一根 D+/D− 差分线上跑三档——LS 1.5M（鼠标键盘）、FS 12M（早期设备）、HS 480M（U盘/硬盘/摄像头，本篇主角），日常说的 USB 2.0 多指 HS。下·HS 三个特殊点：① 电流模式驱动，约 17.78mA 电流源打到 22.5Ω→差分约 400mV 小摆幅、围着地摆（对比 LS/FS 的 0~3.3V 电压模式）；② NRZI 编码（遇 0 翻转/遇 1 保持）+ 位填充（连续 6 个 1 后强插 1 个 0）逼出跳变，供接收端时钟恢复（↔32）；③ 一个比特 UI＝1/480M≈2.083ns，就是眼图横轴一格。结论：摆幅小+速度快+自恢复时钟→必须当高速信号测眼图](/images/40/ill02_hs_signaling.png)

这三个特征里，**最根本、也最直观的一个，就是"驱动方式"的切换**——它直接决定了信号在示波器上"长什么样"。我们单独放大看一眼这两种驱动方式：

- **低速 / 全速（LS/FS）用电压模式**：驱动器把线在 **0V 和 3.3V 之间来回"拉"**，用电压高低表示 0/1，摆幅大、看着就是标准的数字方波；另配 Rpu（1.5kΩ 上拉）/ Rpd（15kΩ 下拉）做速度与连接识别。当普通数字电平伺候就够，**不必测眼图**。
- **高速（HS 480M）换成电流模式**：靠一个约 **17.78mA 的恒流源**把电流打到两端各 45Ω（并联 22.5Ω）的负载上，只得到**差分约 400mV 的小摆幅、还围着地电位上下摆**；进高速时会关掉 1.5k 上拉、两端做 45Ω 端接。

**同一对 D+/D− 线，两种完全不同的信号长相**——摆幅从 3.3V 缩到 400mV、参考点从"0/3.3V 之间"挪到"围着地摆"，余量一下子紧了。这正是 HS 必须当高速信号来测眼图、而 LS/FS 不用的根本原因。

![图3：两种驱动方式对比——LS/FS 电压模式 vs HS 电流模式，信号长相截然不同。左·电压模式（LS/FS）：推挽驱动器把 D+/D− 线在 0V↔3.3V 之间来回拉，用电压高低表示 0/1，摆幅≈3.3V 大、是标准数字方波；另有 Rpu 1.5k 上拉 3.3V、Rpd 15k 下拉 GND 做速度与连接识别；速度慢（≤12M）、余量宽松，当普通数字电平就够、不必测眼图。右·电流模式（HS 480M，本篇）：约 17.78mA 恒流源把电流打到两端各 45Ω（并联 22.5Ω）负载上，只得到差分≈400mV 小压降、围着地电位摆；高速时关掉 1.5k 上拉、两端 45Ω 端接；摆幅小、速度快（480M）、余量紧→必须当高速信号测眼图。底部结论：同一对 D+/D− 线，LS/FS 用电压模式（大摆幅），chirp 握手后切进 HS 改电流模式（小摆幅、围着地摆）——这就是它必须测眼图的根本原因](/images/40/ill02b_drive.png)

## 二、正确使用与设计要点：90Ω、等长、连接器，以及"怎么进高速"

测之前先讲清楚：这条高速链路在设计上要守什么规矩，也就是**眼图之所以会塌的那些"源头"**。

**① 差分 90Ω，走线要等长。** D+／D− 是一对**差分线**，规范要求差分阻抗做到 **90Ω**（每根对地约 45Ω）。板上走线要**等长、等宽、贴着一起走**，中间不要塞过孔、不要跨参考地的分割缝——这些都是模块三反复讲过的 SI 要点。〔回链 **27 反射 / 阻抗匹配 · 28 端接 · 30 串扰**——阻抗不连续会反射、参考地被掏会串扰，源头都在这儿；这里不再展开〕

**② 连接器和线缆是重灾区。** 板子本身往往没问题，**问题常常出在连接器的过渡和那根线缆上**——USB 线长了、料差了，插入损耗一大，信号到远端就塌。所以一致性测试专门有**远端（far-end）**这一测法，就是要看**信号过完最长的合规线缆之后**还认不认得出（后面第五节细讲）。

**③ 沿不能太陡——高速反而"怕快"。** 这点很反直觉：一般我们担心边沿太慢，但 USB 2.0 高速规定**上升/下降时间要 ≥ 500 皮秒（20%–80%）**，也就是**沿不能太陡**。为什么？沿越陡、高频分量越多，**EMI 辐射越强、串扰越凶、反射也越难压**。所以 USB 高速的驱动器是**故意把沿放缓**的——这是"够用就好、别过冲"的典型工程取舍。

**④ 怎么从"全速"切进"高速"：chirp 握手。** 设备刚插上时，大家都先按**全速**说话；然后设备发一串 **chirp K** 信号"举手"说"我支持高速"，主机若也支持就回一串 **KJKJKJ…**，双方**握手确认后一起切到高速模式**。测试时要让设备稳定停在高速状态发信号，就得懂这套握手（并常常要借"测试模式"把它固定住，见第五节）。

> 一句话：**高速链路的设计红线——差分 90Ω、走线等长贴走、别塞过孔跨分割缝；连接器和线缆是塌眼图的重灾区，所以要单测远端；沿故意做到 ≥500ps 别太陡（压 EMI）；靠 chirp 握手从全速切进高速。** 这些"源头"守不好，后面眼图就一定塌。

![图4：USB 2.0 高速的设计要点与"怎么进高速"。左·物理层：D+/D− 差分对做 90Ω（每根对地≈45Ω），走线等长等宽贴走、忌过孔/跨参考地分割缝（↔27/28/30）；连接器过渡+线缆是插入损耗重灾区→专门要测远端。中·沿反而怕快：上升/下降 ≥500ps(20-80%)，故意放缓——沿太陡→高频分量多→EMI/串扰/反射都变凶（够用就好）。右·chirp 握手进高速：插入先按全速→设备发 chirp K 举手"我支持 HS"→主机回 KJKJKJ→双方切到高速。这些源头守不好，眼图必塌](/images/40/ill03_design.png)

## 三、测什么：发送端、接收端、通道，三类各测一头

现在把第 38 篇那套通用框架**套到 USB 上**。一致性测试永远是这三大类，USB 2.0 高速一一对应：

**① 发送端 Tx——信号质量测试（本篇重点）。** 让被测设备**发出规定的测试码型**，用示波器抓下来，做成眼图，扣上模板判 pass/fail。同时量几个具体参数：**差分摆幅（400–700 mV）、上升/下降时间（≥500 ps）、抖动、EOP（包结束）宽度**等。这就是俗称的 **HS 信号质量测试（Signal Quality）**——USB 一致性里做得最多、也最能反映"这个发送端好不好"的一项。

**② 接收端 Rx——灵敏度 / squelch 测试。** 反过来，看接收端**能不能在信号变差时还认得出、在没信号时不乱认**。核心是 **squelch（静噪）门限**：规范要求**差分幅度低于 100 mV 时，接收端要当成噪声忽略掉**；**高于 150 mV 时，必须识别为有效信号**；中间是灰区。测法是用可编程信号源，把幅度、加上抖动往下压，看接收端在多差的信号下还能正确收——这就是"接收灵敏度"。

**③ 通道 Channel——线缆 / 连接器测试。** 单独考线缆和连接器本身：**插入损耗、回波损耗**够不够规范。这一类就是模块三的 **S 参数**在管（第 33 篇），要定位线缆/连接器坏在哪一段就用 **TDR**（第 39 篇）。〔回链 **33 S 参数 · 39 TDR**——通道这条路好不好，频域用 S 参数算总账、时域用 TDR 定位，本篇不重复〕

对 USB 2.0 高速来说，**发送端眼图是绝对的主角**——这也是本篇后面几节围绕讲的核心。接收端和通道各占一头，构成完整的一致性画面。

![图5：USB 2.0 一致性测试的三大类（套第 38 篇框架）。① 发送端 Tx·信号质量（本篇重点）：设备发规定测试码型→示波器抓→做眼图扣模板→判 pass/fail，同时量摆幅 400-700mV、沿≥500ps、抖动、EOP 宽度。② 接收端 Rx·灵敏度/squelch：信号源把幅度+抖动往下压，看接收端还能否正确收；squelch 门限 <100mV 忽略/>150mV 必识别。③ 通道 Channel·线缆连接器：插损/回损够不够规范，频域用 S 参数(↔33)算账、时域用 TDR(↔39)定位。三类各测一头，发送端眼图是主角](/images/40/ill04_what.png)

## 四、我的项目要测哪几类：先判定自己在链路里的"角色"

上一节把三大类摆出来了，但**真到了你自己的项目，未必三类都要你测**。拿到一个"要测 USB 2.0"的活儿，第一件事不是翻规范，而是先问一句：**我的产品在这条 USB 链路里，到底是发送方、接收方，还是提供通道的那一段？** 角色定了，要测哪几类就定了。

- **你做的是 device（外设）**——U 盘、移动硬盘、摄像头、带 USB 的模组：这个口**既要往外发、也要往里收**，所以**发送端 Tx 信号质量 + 接收端 Rx 灵敏度都要测**。
- **你做的是 host（主板 / SoC 主控口）**：同理，主机口也是**既发又收**，**Tx + Rx 都要**。
- **你做的是 hub（集线器）**：它的**上行口对主机相当于一个 device、下行口对外设相当于一个 host**，所以每一类口都按上面的规则测，**多个口都要覆盖**。
- **你做的是线缆 / 连接器**：你交付的是"那段路"，主要测**通道 Channel**——插入损耗、回波损耗（S 参数 / TDR）；Tx/Rx 是用你线缆的人去测。
- **你做的是系统整机（集成的是现成、已认证的模组和线缆）**：往往不必把 Tx/Rx/Channel 全项重跑一遍，重点是**关键链路抽测 + 系统级互操作**（插各种设备能不能正常枚举、跑满速）。

**两条判定铁律，记住就不会多测也不会漏测：**

1. **一个双向口 = 既是发送方又是接收方**，所以**绝大多数芯片级项目，Tx 和 Rx 两类都跑不掉**；只有单向的特殊场合才可能只测一头。
2. **通道 Channel 只在"那段线缆 / 连接器 / 板级走线是你设计的"时才由你测**；如果你用的是现成、已过认证的线缆，就不必替它重新做通道测试。

最后，**测试范围的宽窄，由你的目标决定**：要**贴 USB-IF logo 拿认证**，就得到授权实验室按官方 compliance 清单**跑全项**；只是**内部验证 / 排查问题**，则可以**挑关键项、重点盯 margin** 自己先测一轮。

> 一句话：**别一上来就三类盲目全测——先认清你是 host / device / hub / 线缆里的哪个角色。双向口 Tx+Rx 都要，Channel 只测你自己设计的那段；认证跑全项、自测挑关键项。角色定了，测试清单的骨架就定了。**

![图6：我的项目要测哪几类——按在 USB 链路里的角色判定。先问：我的产品是发送方、接收方，还是提供通道？五种角色对应必测类：device 外设→Tx+Rx（双向口既发又收）；host/主板→Tx+Rx；hub 集线器→上行口当 device、下行口当 host，每口都测；线缆/连接器→只做 Channel（插损/回损 S参数/TDR）；系统整机(集成现成模组)→关键链路抽测+系统级互操作，未必跑全项。两条判定铁律：①双向口=既发又收→Tx 和 Rx 通常都要测；②Channel 只在线缆/连接器/板级通道是你设计的才由你测，用现成认证线缆可不重复。范围由目标定：认证贴 logo 到授权实验室跑完整 compliance，内部验证挑关键项+盯 margin 自测](/images/40/ill04b_role.png)

## 五、怎么测：近端 TP2、远端 TP3，进"测试模式"发码型

这一节落到最实操的部分——真要测发送端眼图，仪器怎么接、码型怎么来。

**两个测试点，测两件不同的事：**

- **近端 TP2（near-end）：** 探在**发送端芯片输出口附近**、还没进线缆的地方。它考的是**发送端本身的信号质量**好不好——芯片这一关过没过。
- **远端 TP3（far-end）：** 探在**信号过完一根最长的合规线缆之后**的地方。它考的是**信号扛过最坏线缆损耗之后**、接收端还认不认得出。

同一个信号，近端眼图张得大、远端因为线缆损耗会**明显小一圈**——所以**近端和远端用的是两套不同的眼图模板**（后面第六节讲判据来源时细说为什么不同）。

**怎么让设备"乖乖发出规定的码型"——测试模式。** 你不能指望设备正常工作时刚好发出你想要的图案。USB 规范专门留了一组**测试模式（test mode）**，主机通过一条 **SET_FEATURE** 命令让设备进入，常用的有：

- **Test_Packet**——反复发一个**精心设计的测试包**，里面故意塞满各种最坏情况的比特组合（连续跳变、连续同值触发位填充等），**专门用来做眼图**。
- **Test_J / Test_K**——让线固定停在 J 态或 K 态，用来量直流电平、静态参数。
- **Test_SE0_NAK**——用于特定的收发状态测试。

做眼图信号质量，用的就是 **Test_Packet**：让设备反复吐这个测试包，示波器长时间叠加，就得到一个统计意义上的眼图。

**别忘了夹具（fixture）。** 探不能直接怼在焊盘上——要通过一个**标准的测试夹具**（USB-IF 认可的信号质量测试板）把信号引到示波器，夹具本身的损耗后面要**从结果里扣掉**（去嵌，见下一节）。

> 一句话：**近端 TP2 考发送端芯片本身、远端 TP3 考过完最长线缆后还认不认得出，两个点各用一套眼图模板；靠 SET_FEATURE 让设备进 Test_Packet 模式反复发最坏码型、经标准夹具引到示波器叠成眼图。**

![图7：USB 2.0 高速发送端眼图怎么测。上·两个测试点：TP2 近端探在发送芯片输出口（考芯片本身信号质量）、TP3 远端探在过完最长合规线缆之后（考扛过最坏损耗后接收端还认不认得出）；同一信号远端眼明显小一圈→近端/远端用两套模板。下·测试流程：主机发 SET_FEATURE→设备进 Test_Packet 模式反复吐"最坏情况测试包"（塞满连续跳变+触发位填充的比特）→经 USB-IF 标准夹具引到示波器→长时间叠加成眼图→扣模板判。另有 Test_J/Test_K 量静态电平](/images/40/ill05_howto.png)

## 六、⚙️关键仪器设置：把 SI 概念拧到 USB 这颗螺丝上

这一节是模块六的灵魂——**同样一台示波器、同样几个旋钮，前面模块三讲的是"它们是什么"，这里讲"测 USB 时具体拧到几"。** 每一条都回链对应的 SI 篇，原理不复述。

**① 带宽：至少覆盖 5 次谐波。** UI＝2.083 ns、沿约 500 ps，主要频率成分在 GHz 量级。示波器和探头的带宽**要够到信号 5 次谐波以上**，否则你还没测，仪器自己先把沿磨钝、眼图变好看了——测了个假的。做 USB 2.0 高速，通常用**带宽几个 GHz 级别**的示波器（一般建议 ≥4 GHz 起步）。〔回链 **29 上升时间 / 带宽**——带宽不够＝仪器自己在骗你〕

**② 时钟恢复（golden PLL / CDR）：眼图的横轴基准。** USB 高速没有单独时钟线，做眼图前必须**先从数据里恢复出时钟**当作水平参考——用规范约定的**软件时钟恢复（golden PLL）**，参数（环路带宽等）按 USB 规范设。这一步没配对，眼图的水平位置就是错的。〔回链 **32 抖动 / 时钟恢复**——CDR 怎么锁相在那篇讲过〕

**③ 抖动分离：把随机和确定性分开。** 一致性不只看眼图张不张得开，还要把抖动**分成随机抖动 RJ 和确定性抖动 DJ**，必要时用 **BER 外推**（浴盆曲线）推算到很低误码率下的总抖动。〔回链 **32 抖动 · 31 眼图**——RJ/DJ 分离、浴盆曲线在那两篇〕

**④ 去嵌（de-embedding）：把夹具的锅扣掉。** 信号要穿过夹具/线缆才到示波器，这段自己的损耗会让眼图**偏小**——用夹具的 S 参数把它**从结果里扣掉**，才测到芯片真实的输出。〔回链 **33 S 参数 · 38 去嵌 / 嵌入**——去嵌是把夹具"减掉"，第 39 篇还讲了它在时域里的等价操作（时域门）〕

**⑤ 加载眼图模板（eye mask）：一键判 pass/fail。** 最后把 USB-IF 规范提供的**眼图模板**加载进一致性软件，近端选近端模板、远端选远端模板，软件自动判**眼睛有没有碰禁区**。〔回链 **31 眼图**——模板怎么扣、余量怎么读在那篇〕

> 一句话：**带宽够 5 次谐波（别让仪器骗你）↔29、按规范配 golden PLL 时钟恢复（定横轴）↔32、抖动分 RJ/DJ 必要时 BER 外推↔32/31、用 S 参数去嵌把夹具扣掉↔33/38、加载近端/远端眼图模板一键判↔31。** 每个旋钮背后都是模块三那位老朋友，这里只是拧到 USB 的刻度上。

![图8：测 USB 2.0 时的六个关键仪器设置↔SI 篇映射（模块六灵魂）。① 带宽≥信号 5 次谐波（≥4GHz 级），否则仪器自己磨钝沿→测假的↔29；② golden PLL/CDR 从数据恢复时钟当眼图横轴基准，参数按规范设↔32；③ 抖动分 RJ/DJ + BER 外推浴盆曲线↔32/31；④ 用夹具 S 参数去嵌，把夹具/线缆损耗扣掉才见芯片真实输出↔33/38（时域门↔39）；⑤ 加载 USB-IF 近端/远端眼图模板一键判 pass/fail↔31。中心一台示波器，六个旋钮各连回一篇 SI。原理不复述，只讲拧到几](/images/40/ill06_knobs.png)

## 七、判据怎么来：那张眼图模板不是拍脑袋画的

很多人把眼图模板当成"规范给的一个神秘图案"，照着扣就行。但模块六要讲透的正是这一层——**模板上的每条线都是推出来的。**

**眼图模板长什么样。** 一张 USB 2.0 高速眼图模板，本质是在"一个 UI × 差分电压"这张图上画出的**禁区**：

- **中间一个六边形（keep-out）**：眼睛睁开的区域必须把这个六边形**整个罩住**、信号轨迹不许探进去——这保证了**留给接收端判决的电压余量和时间余量**都够。
- **上下两条外框线**：限制信号的**最大摆幅**，对应差分 **400–700 mV** 那个上下限——太小接收端要认错，太大可能损伤、也说明驱动没控好。

**这些数字是从哪儿推出来的？**

- **电压方向**：下限接着接收端的 squelch 门限（信号得明显高于"必须识别"的 150 mV，才有余量），上限接着驱动能力和过冲限制——**从接收端认得出、发送端不过冲，两头夹出来**。
- **时间方向**：一个 UI 是 2.083 ns，减掉**抖动预算**（RJ+DJ 占掉的那部分），剩下的才是眼睛该睁开的宽度——**模板的水平禁区就是"抖动吃掉之后必须还剩这么宽"**。〔回链 **31 眼图 · 32 抖动**——眼高＝电压余量、眼宽＝时间余量，抖动吃眼宽，都在那两篇〕

**为什么近端和远端用不同模板？** 因为信号**过完线缆一定会塌一圈**（插入损耗）。要是近端远端用同一张严模板，那所有线都过不了；所以规范承认"远端会小一点"，给远端一张**相对宽松、但仍保证接收端认得出**的模板。**近端管"发送端出厂够不够好"，远端管"扛过最坏线缆后还够不够用"**——两个模板各守一道关。

> 一句话：**眼图模板＝把"接收端认得出（squelch 150mV）+ 发送端不过冲（≤700mV）+ 抖动吃掉后眼宽还得够（UI 2.083ns 减抖动预算）"这几条，画成一张带六边形禁区的图。** 近端严、远端松（因为线缆必然塌一圈），都是推出来的，不是拍脑袋。

![图9：USB 2.0 眼图模板的结构与判据来源（回链 31/32）。左·模板长相：横轴 1 个 UI（2.083ns）、纵轴差分电压；中间六边形 keep-out 禁区必须被睁开的眼完全罩住（保证电压+时间余量），上下外框对应摆幅 400-700mV 上下限。中·数字从哪来：电压下限←接收端 squelch"必须识别"150mV+余量、上限←驱动不过冲；时间←1 UI 减抖动预算(RJ+DJ)剩下的眼宽↔31/32。右·为什么近端/远端两套模板：信号过线缆必塌一圈(插损)，近端严(管出厂质量)、远端松(管扛过最坏线缆后仍够用)。模板每条线都是推出来的](/images/40/ill07_mask.png)

## 八、把上面拼成一份不遗漏的测试计划：五步 + 一张照着填的矩阵

到这里，"测什么、我要测哪几类、怎么测、判据怎么来"都齐了。最后一步，也是最能体现工程师功力的一步——**把这些拼成一份不会漏项的测试计划**。方法就五步：

1. **定接口版本 / 速度**：先写死"USB 2.0 高速（HS 480M）"——版本和速度档不同，判据和测试项就不同。
2. **判角色 → 定测哪几类**：用上面第四节的角色判定，圈出你要测的是 Tx / Rx / Channel 里的哪几类。
3. **把该类下的必测项列全**：比如 Tx 类就要列全"眼图模板、差分摆幅、上升/下降时间、抖动、EOP 宽度"——别只测眼图漏了摆幅和沿。
4. **每项去查标准、取判据**（下面细说"去哪查"）。
5. **给每项排设备 / 夹具 / 测试模式，并定一个 margin 目标**（不是"过就行"，而是"留够多少余量才算过"）。

**第 4 步"去哪查标准"，很多人卡在这里，说清楚：**

- **三份文件，各管一段**：`USB 2.0 Specification`（**第 7 章"电气"**是电平、摆幅、时序这些参数的总源头）→ `USB 2.0 Electrical Test Specification`（**眼图模板、测试点、测试步骤、测试模式**都在这份 compliance 文档里）→ `USB-IF Compliance Program` 文档（**认证流程、要跑哪些项、在哪些授权实验室**）。
- **去哪拿**：都在 **USB-IF 官网 usb.org** 的 Documents / Compliance 区免费下载。
- **你几乎不用手画模板**：正经的**一致性测试软件**（如 Keysight U7243、Tektronix TDSUSB 等）已经把这些 limit / eye mask **内置**了——你在软件里**选对协议版本（USB 2.0 HS）+ 测试点（近端 / 远端）**，它就自动加载对应模板和判据，测完直接出 pass/fail 报告。**"标准从哪来"最终落到：选对软件里的那一项。**
- **认证 vs 自测**：贴 logo 的**认证**必须用规范指定的夹具、软件、在授权实验室按官方清单跑；**内部自测**可以简化，但判据仍要引用同一份规范，别自己拍脑袋定"差不多就行"。

把这五步的产出，落到**一张测试计划矩阵**里，每一行填满，就不会漏项：

| 测试项 | 测试点 | 标准出处 | 判据 | 设备 / 夹具 · 测试模式 | margin 目标 | 结果 |
|---|---|---|---|---|---|---|
| HS 眼图信号质量 | TP2 近端 / TP3 远端 | 电气测试规范·眼图模板 | 不碰六边形禁区；摆幅 400–700mV | ≥4GHz 示波器 + 标准夹具 · Test_Packet | 眼高/眼宽余量 ≥ ?% | ☐ |
| 差分摆幅 / 上升下降时间 | TP2 | Spec 第 7 章电气 | 400–700mV；沿 ≥500ps | 同上 · Test_Packet / Test_J·K | — | ☐ |
| Rx squelch 灵敏度 | 接收端输入 | Spec 接收器一节 | <100mV 忽略 / >150mV 识别 | 可编程信号源 + 抖动注入 | 门限余量 | ☐ |
| 通道 插损 / 回损 | 线缆 / 连接器两端 | 线缆 / 连接器规范 | 插损 / 回损曲线达标 | VNA（S 参数）/ TDR | — | ☐ |

> 一句话：**测试计划 = 定版本速度 → 判角色圈出要测的类 → 把该类必测项列全 → 每项去 usb.org 那三份文件（或一致性软件内置 limit）取判据 → 排设备/夹具/模式并定 margin 目标。一张矩阵照着填满，就漏不了项。** 这套框架换到 HDMI、MIPI、PCIe，只需替换"标准出处 + 判据 + 测试模式"三列，骨架完全一样。〔回链 **番外02《如何制定一份硬件测试计划》**——通用的测试计划方法论在那篇〕

## 九、怎么看懂一致性报告：pass/fail、margin、哪项 fail 说明什么

测完，一致性软件吐给你一份报告。会读报告，比会按开始键重要得多。

**报告的三个层次：**

- **总判 pass/fail**：整条测试项过没过。但光看这个不够。
- **每项的余量（margin）**：这才是关键。比如眼高要求 ≥某值、你测到刚好卡在线上——**虽然 pass，但 margin 只有 3%**，说明这批板子一到量产、一遇温漂就可能翻车。**margin 充足的 pass 才是真 pass。**
- **具体数值 + 眼图截图**：摆幅多少、上升时间多少、抖动多少、眼图碰没碰禁区。

**哪项 fail，说明什么（会看报告 = 会定位）：**

- **眼图碰了禁区、眼太小** → 信号质量整体不行：可能是驱动摆幅不足、或抖动过大、或（远端）线缆损耗太大。
- **摆幅超上限（>700mV）** → 驱动过强/过冲没控住；摆幅不足（<400mV）→ 驱动能力弱或负载不对。
- **上升/下降时间太快（<500ps）** → 沿太陡，EMI/串扰风险；太慢 → 带宽不足、通道损耗大。
- **抖动超预算** → 回去查时钟质量、电源噪声（PI！）、串扰；确定性抖动大常常指向**周期性干扰源**。
- **只有远端 fail、近端 pass** → 发送端本身没问题，锅在**线缆/连接器/通道**——这时候就该请出 S 参数和 TDR 定位了。

**这就是模块六的闭环**：报告告诉你哪项 fail、fail 得多离谱，你就能反推是发送端、接收端、还是通道的问题，再调用对应的工具（眼图/抖动/S 参数/TDR）去定位。

> 一句话：**别只看总 pass/fail——盯每项的 margin（卡边的 pass 是定时炸弹）；按 fail 的项反推病灶（眼太小=质量差、摆幅超限=驱动问题、抖动超预算=查时钟/电源/串扰、只有远端 fail=锅在线缆通道），再用对应工具去定位。**

## 十、一句话总结

USB 2.0 高速，是我们把整套 SI 工具第一次真刀真枪套到真实协议上的起点。把这一篇拆成八句话带走：

- **① 主角**：USB 2.0 说的就是 **480M 高速（HS）**——电流模式约 400mV 小摆幅、NRZI+位填充自带时钟、一个比特 2.083ns；摆幅小、速度快、还得自恢复时钟，**所以必须测眼图**。
- **② 测什么**：一致性测**三类**——**发送端**眼图信号质量（本篇主角）、**接收端** squelch 灵敏度、**通道**线缆插损/回损。
- **③ 我这项目要测哪几类**：**先看你在链路里的角色**——device / host / hub 是双向口，**Tx+Rx 都要**；线缆 / 连接器只测 **Channel**。角色定了，测试清单骨架就定了。
- **④ 怎么测**：**近端 TP2**（考芯片）+ **远端 TP3**（考扛过最坏线缆）各测一次、各用一套模板；靠 **SET_FEATURE 进 Test_Packet 模式**发最坏码型、经标准夹具叠成眼图。
- **⑤ 仪器设置**：全是模块三的老朋友拧到 USB 刻度——**带宽够 5 次谐波↔29、golden PLL 时钟恢复↔32、抖动分离+BER 外推↔32/31、S 参数去嵌扣夹具↔33/38、加载眼图模板一键判↔31**。
- **⑥ 判据从哪来**：那张**眼图模板不是拍脑袋**——六边形禁区+400~700mV 外框，是从"接收端认得出+发送端不过冲+抖动吃掉后眼宽还够"推出来的，近端严、远端松；**标准就在 usb.org 那三份文件里、一致性软件已内置成 limit**。
- **⑦ 测试计划怎么不漏项**：**定版本速度 → 判角色 → 列全必测项 → 查标准取判据 → 排设备/模式定 margin**，一张矩阵照着填就漏不了项。
- **⑧ 出问题怎么定位**：看报告**盯 margin**（卡边的 pass 是定时炸弹）→ 按 fail 的项**反推是发送端/接收端/通道** → 再用对应工具（眼图/抖动/S 参数/TDR）去定位。

> **一句话的一句话：从"我这项目要测哪几类"，到"怎么测、判据哪来、计划怎么排"，再到"出了问题怎么定位"——这就是模块六的闭环。**

从这一篇起，一致性测试不再是抽象方法论了。**下一篇，我们进 USB 3.x（SuperSpeed，5G/10G）**——速率再上一个台阶，就得请出 **SSC 扩频时钟、去加重（de-emphasis）、以及专门造"压力眼"考接收端的 Rx 抖动容限（jitter tolerance）**。USB 2.0 这张眼图模板是入门，3.x 才是高速一致性真正的深水区。

## 🔧 工程师速查（附录）

> 规范数值出处：**USB 2.0 Specification** 及 **USB 2.0 Electrical Test Specification**（USB-IF）。以下为科普整理，正式测试以最新版规范与一致性测试文档（Compliance）为准。

| 项 | 要点 |
|---|---|
| 三档速度 | LS 1.5 Mbps／FS 12 Mbps／**HS 480 Mbps**（日常说的 USB 2.0 多指 HS，本篇主角），都在 D+/D− 一对差分线上 |
| HS 信令 | 电流模式驱动，约 **17.78 mA** 打到 22.5Ω（45Ω‖45Ω）→ 差分约 400mV 小摆幅、围地电位摆 |
| 编码 | **NRZI**（遇 0 翻转／遇 1 保持）+ **位填充**（连续 6 个 1 后强插 1 个 0）逼出跳变，供接收端时钟恢复〔↔32〕 |
| UI（比特时间） | 1÷480M ≈ **2.083 ns**，即眼图横轴一格 |
| 差分阻抗 | **90Ω**（每根对地≈45Ω）；走线等长等宽贴走、忌过孔/跨参考地分割〔↔27/28/30〕 |
| 差分摆幅 | **400–700 mV**（p-p，量在 45Ω‖45Ω 负载）——眼图模板的上下外框 |
| 上升/下降时间 | **≥500 ps**（20%–80%）——沿故意放缓限 EMI/串扰/反射（高速反而"怕快"） |
| 进入高速 | 插入先按全速→设备发 **chirp K**→主机回 **KJKJKJ**→握手切到 HS |
| 测三类〔↔38〕 | **Tx** 发送端眼图信号质量（主角）／**Rx** 接收灵敏度 squelch／**Channel** 线缆连接器插损回损 |
| **测哪几类·角色** | 先判角色：**device / host / hub → Tx+Rx 都要**（双向口既发又收）；**线缆/连接器 → 只测 Channel**；系统整机→抽测+互操作。认证跑全项、自测挑关键项 |
| 接收 squelch | 差分 **<100mV 忽略为噪声**、**>150mV 必须识别**、100–150 灰区；断开检测远端反射叠加 **>625mV** |
| 测试点 | **TP2 近端**（考发送芯片本身）／**TP3 远端**（考过完最长合规线缆后还认不认得出），各用一套模板 |
| 测试模式 | SET_FEATURE 进 **Test_Packet**（发最坏码型做眼图）／Test_J／Test_K（量静态电平）／Test_SE0_NAK |
| 夹具 | 经 USB-IF 标准信号质量测试板引到示波器，夹具损耗后续去嵌扣掉 |
| ⚙️带宽〔↔29〕 | 覆盖信号 5 次谐波以上（常 ≥4GHz 级），否则仪器自己磨钝沿→测假的 |
| ⚙️时钟恢复〔↔32〕 | golden PLL/CDR 从数据恢复时钟当眼图横轴基准，环路参数按规范设 |
| ⚙️抖动〔↔32/31〕 | 分 RJ/DJ，必要时 BER 外推浴盆曲线推到低误码率总抖动 |
| ⚙️去嵌〔↔33/38〕 | 用夹具/线缆 S 参数扣掉其损耗，才见芯片真实输出（时域等价＝时域门↔39） |
| ⚙️眼图模板〔↔31〕 | 加载 USB-IF 近端/远端模板，软件自动判眼睛碰没碰禁区 |
| 模板结构 | 中间六边形 keep-out 禁区（保电压+时间余量）+ 上下外框（400~700mV 摆幅限） |
| 判据来源 | 电压←squelch 150mV+驱动不过冲两头夹；时间←1UI 减抖动预算剩的眼宽；近端严/远端松（线缆必塌一圈）〔↔31/32〕 |
| 看报告 | 别只看总 pass/fail→**盯每项 margin**（卡边的 pass=量产/温漂定时炸弹）；看具体值+眼图截图 |
| fail 反推 | 眼太小=质量差(摆幅/抖动/损耗)；摆幅超限=驱动过冲/偏弱；沿太快=EMI；抖动超预算=查时钟/电源PI/串扰；只远端 fail=锅在线缆通道→S 参数+TDR 定位 |
