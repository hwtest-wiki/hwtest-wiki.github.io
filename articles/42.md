# 同样一根 USB-C 线，为什么有的 100W 快充、有的只有 5V？——USB-C / USB PD：一致性测试里唯一不看眼图的一篇

先说个几乎人人都遇到过的现象。

你有两个充电器，标称都是 65W、100W 的大功率快充，线也都是 USB-C 对 USB-C。同一台笔记本，插 A 充电器，屏幕上蹭蹭显示"快速充电"；插 B 充电器，或者只是**换了一根便宜的线**，就变成"正在充电（慢）"，甚至干脆只有 5V、越用越掉电。功率明明够，为什么充不进去？

答案是：**问题几乎从来不在"电不够"，而在 CC 线上那场"供电谈判"没谈成。** 充电器和设备要先在一根叫 **CC** 的针上，用一套协议把"你能给多大、我要多大"一条条对齐，谈妥了才敢把电压从 5V 抬到 20V。谈崩了——比如线太差、协议不兼容、某一步超时——就只能退回最保守的 5V 慢充保命。

![图1：USB-C/PD 慢充翻车现场。左：同一台笔记本，插 A 充电器 65W→"快速充电"；插 B 充电器或换根便宜线→只有 5V"慢充"，功率明明够却充不进去。中：差别不在"电够不够"，而在 CC 线上那场"供电谈判"——充电器广播"我能给 20V/5A"、设备请求"我要 20V"、双方 Accept 并把 VBUS 从 5V 抬到 20V，全程一来一回。右：谈崩的三种典型原因——线太差没有 e-marker/CC 通信断、协议版本不兼容、某一步握手超时——任何一步崩，就回落到最保守的 5V 慢充。所以这一篇的一致性，测的不是眼图，是"这场谈判能不能安全谈妥"](/images/42/ill01_pain.png)

## 一、先掰开两个最常被混为一谈的概念：USB-C 是"插座"，USB PD 是"谈判"

无数人把 "USB-C" 和 "快充" 划等号，这是理解这一篇的第一个坎。**它俩根本是两层东西：**

- **USB-C（Type-C）是"接口的形状"** —— 一个 24 针、正反都能插的**连接器和物理层规范**。它只规定"这个口长什么样、有哪些针、正反插怎么认"。**USB-C 本身不保证快充，也不保证高速**——你完全可以有一个只跑 USB 2.0、只给 5V 的 USB-C 口。
- **USB PD（Power Delivery）是"供电的协议"** —— 一套**跑在 CC 针上的通信协议**，让两端**协商**出该用多高的电压、多大的电流。它才是"快充"的大脑。

一句话：**USB-C 是插座，USB PD 是插座上那场决定给多少电的谈判。** 一个是硬件形态，一个是协议行为，测法也完全不同。

理清了这个，再看 USB-C 口的**供电其实分三个层级**，一层比一层能给：

| 层级 | 靠什么 | 能给多少 |
|---|---|---|
| **① 传统 BC1.2** | 老办法，在 D+/D− 数据线上握手 | 5V，最多 1.5A（DCP/CDP） |
| **② Type-C 电流广播** | CC 针上用**电阻**告诉对方"我能给多大" | 5V，最多 **3A**（15W），不需要 PD |
| **③ USB PD 协商** | CC 针上用**协议**谈判 | 5V～**48V**、最高 **240W** |

**没有 PD、甚至没有 Type-C 电流广播时，会一路回落**——这就是慢充的根源：谈判没谈成，就退到最保守那一层。

![图2：USB-C ≠ USB PD，以及供电三层级。上：USB-C（Type-C）是"接口形状"——24 针、正反插的连接器/物理层，本身不等于快充；USB PD 是"供电协议"——跑在 CC 针上协商电压电流，才是快充的大脑。二者是两层东西、测法不同。下：USB-C 口供电分三层级，一层比一层能给——①传统 BC1.2（D+/D− 握手，5V/1.5A）→ ②Type-C 电流广播（CC 电阻，5V/最多 3A）→ ③USB PD 协商（CC 协议，5～48V/最高 240W）；谈判失败就一路回落到最保守层，这就是"慢充"](/images/42/ill02_concept.png)

## 二、CC 那两根针到底在干嘛：认插入、认方向、报电流、给线供电

USB-C 能正反插、能协商供电，秘密全在那两根 **CC 针（CC1 / CC2）** 上。在通电协商之前，光靠几个**电阻**，它就把好几件事办了：

**① 认"插上了没" + 认"哪面朝上"。** 供电的一方（Source，比如充电器）在 CC 上挂一个**上拉电阻 Rp**；用电的一方（Sink，比如手机）在 CC 上挂一个**下拉电阻 Rd = 5.1kΩ**。插上之后，被拉出电压的那根 CC，就是"真正接通的这一面"；另一根空着——**方向检测就是这么来的**，根本不需要芯片，纯电阻分压。

**② 报"我能给多大电流"。** Source 用**不同阻值的 Rp**，在还没跑 PD 协议时就先粗粒度地广播自己的供电能力，Sink 量一下 CC 上的电压就知道了：

| Rp 上拉 | 广播的能力 |
|---|---|
| **56kΩ** | Default USB（5V / 500mA 或 900mA） |
| **22kΩ** | 5V / **1.5A** |
| **10kΩ** | 5V / **3A** |

**③ 给"聪明的线"供电：Vconn + Ra。** 支持大电流或高速的**全功能线缆**，线里藏了一颗芯片叫 **e-marker**，记录这根线能扛多少电流、支持什么协议。这颗芯片要供电——**当一根 CC 被用来通信/检测方向时，另一根 CC 就改作 Vconn（约 5V）给线里的 e-marker 供电**。线端用一个 **Ra（约 800Ω～1.2kΩ）** 电阻表明"我这儿是根需要供电的线"。**没有 e-marker 的便宜线，最高只认到 3A**——这也是很多"大功率充电器充不快"的隐藏元凶。

![图3：USB-C 的 CC 针在通电协商前靠电阻办成的三件事。① 认插入+认方向：Source 挂上拉 Rp、Sink 挂下拉 Rd=5.1kΩ，被拉出电压的那根 CC 就是接通面，另一根空着——正反插纯靠电阻分压，无需芯片。② 报电流能力：Source 用不同 Rp 粗粒度广播——56kΩ=Default(500/900mA)、22kΩ=1.5A、10kΩ=3A，Sink 量 CC 电压即知。③ 给线供电：全功能/大电流线里有 e-marker 芯片记录线的能力，通信占用一根 CC 时，另一根改作 Vconn(约5V)给 e-marker 供电，线端用 Ra(约1kΩ)表明身份；没 e-marker 的便宜线最高只认 3A](/images/42/ill03_cc.png)

## 三、USB PD 协议：CC 线上一场 300kbps 的"供电谈判"

电阻只能粗粒度广播到 5V/3A。想上更高电压、上快充，就得启动 **USB PD**——在同一根 CC 针上，叠一层**数字通信**谈判。

**怎么通信？** PD 用一种叫 **BMC（双相标记编码）** 的方式，在 CC 上以 **300kbps** 的速率收发数据包——注意，这速度和跑数据的 480M、10G 完全不是一个量级，它只是**发几条命令谈判用的慢速控制通道**。

**谈什么？** Source 把自己能提供的每一档电压电流，打包成一个个 **PDO（供电数据对象）** 广播出来。PDO 分几种：

- **Fixed PDO（固定档）**：5V / 9V / 15V / 20V 这些固定电压（SPR 标准功率范围，最高 20V/5A = **100W**）。
- **EPR 扩展档**：PD 3.1（2021 年）新增，再往上加 **28V / 36V / 48V**，最高 **48V×5A = 240W**——但需要 EPR 专用线。
- **APDO / PPS（可编程电源）**：PD 3.0 起支持，电压能**以约 20mV、电流约 50mA 的小步连续可调**，让充电器直接跟着电池需要的电压走，省掉手机内部再降压的损耗、少发热——现在手机快充大量用它。

**PD 版本一句话理清：** PD 2.0（封顶 100W）→ PD 3.0（加 PPS、扩展消息、快速角色交换）→ PD 3.1（加 EPR，冲到 240W）。

**谈判长什么样？** 这是理解 PD 一致性的核心——它是一套**严格的一问一答**：

1. **Source_Capabilities**：充电器广播"我有这些档：5V/3A、9V/3A、20V/5A……"
2. **Request**：设备挑一档回过去"我要 20V"。
3. **Accept**：充电器答"可以"。
4. **PS_RDY**：充电器把 VBUS 电压真的抬到 20V、稳住后，发"电源就绪"。

**而且每一条消息，接收方都必须立刻回一个 `GoodCRC` 确认收到**——收不到 GoodCRC 就重发，重发几次还不行就报错。这套"发一条、确认一条"的机制，是 PD 可靠性的地基，也是一致性测试要盯死的地方。

> 还有个细节：PD 包用 **SOP（包起始符）** 区分"跟谁说话"——**SOP** 是充电器和设备对话；**SOP′ / SOP″** 是跟**线里的 e-marker** 对话（问这根线能扛多大）。测试时要能分清这三种，才知道是设备的问题还是线的问题。

![图4：USB PD 协议全貌。左：物理上——PD 用 BMC 双相标记编码在 CC 针上以 300kbps 收发数据包（只是慢速控制通道，与 480M/10G 数据无关）。中：供电能力用 PDO 描述——Fixed 固定档(5/9/15/20V·SPR≤100W)、EPR 扩展档(28/36/48V≤240W·需专用线)、APDO/PPS 可编程(约20mV/50mA 小步连调·省损耗少发热)；PD 版本 2.0(100W)→3.0(加PPS)→3.1(加EPR 240W)。右：SOP 区分跟谁说话——SOP=设备↔充电器、SOP′/SOP″=跟线里 e-marker 对话](/images/42/ill04_pd.png)

![图5：USB PD 一次典型供电协商的时序（一问一答，每条必回 GoodCRC）。① Source→Sink：Source_Capabilities「我有 5V/3A、9V/3A、20V/5A…」→ Sink 回 GoodCRC。② Sink→Source：Request「我要 20V」→ Source 回 GoodCRC。③ Source→Sink：Accept「可以」→ Sink 回 GoodCRC。④ Source 把 VBUS 从 5V 平滑抬到 20V、稳定后 → PS_RDY「电源就绪」→ Sink 回 GoodCRC。下方 VBUS 曲线随之从 5V 阶梯爬到 20V。任何一条消息收不到 GoodCRC 就重发，超时/重发耗尽就报错——一致性测试盯死的正是"每一步的顺序、时序、和 GoodCRC 确认"](/images/42/ill05_seq.png)

## 四、测什么三类：不看眼图，看"电气 + 协议 + 供电"

回到第 38 篇那套"三大类"框架，USB PD 一样能套，但**每一类的内容和眼图篇完全不同**——这里没有 Tx 眼图、没有 Rx 抖动容限，取而代之的是：

**① CC 电气 & BMC 信号质量。** 最贴近传统硬件测试的一类：Rp / Rd / Ra 三个电阻的**阻值和容差**对不对、CC 上 BMC 波形的**电压摆幅、上升/下降沿、位速率（300kbps）、单位间隔**符不符合规范。这一项还是要用示波器，本质是查"CC 这根线上的电信号干不干净、时序对不对"（沾一点第 29 篇边沿的边）。

**② 协议一致性（重头戏）。** 这是眼图篇完全没有的维度：**消息的顺序对不对、时序（比如收到请求后多久内必须回应）符不符合、GoodCRC 有没有正确收发、超时和重试逻辑对不对、状态机（Source / Sink / 双角色 DRP）切换对不对**。测的不是波形，是**行为**。

**③ 供电电气。** 谈妥之后，**VBUS 真的给对了吗**：各档电压电流的**精度**（比如 20V 档误差在不在 ±5% 内）、从一档切到另一档的**建立/切换时序**、**过流 OCP / 过压 OVP / 短路保护**灵不灵、掉电行为对不对。这一类关系到**安全**——给错电压是会烧设备的。

> **对照第 38 篇的"激励方式三分法"**：USB 2.0 是①进 compliance 模式发固定码型看眼图，USB 3.x 还加②PRBS 环回考抖动容限；而 **USB PD 两者都不是——它是"协议协商类"**：你不需要让它发某段码型，而是**主动扮演对端、跟它一问一答地走完整个协商流程，看它每一步答得对不对**。这就是"不看眼图"的根本原因。

![图6：USB PD 测什么三类（不看眼图版）。① CC 电气&BMC 信号质量：Rp/Rd/Ra 阻值容差、CC 上 BMC 波形的摆幅/上升沿/位速率 300kbps/单位间隔——最贴近传统硬件测试，仍用示波器(沾边29边沿)。② 协议一致性(重头戏)：消息顺序/响应时序/GoodCRC 收发/超时重试/状态机 Source·Sink·DRP 切换——测行为不测波形。③ 供电电气：VBUS 各档电压电流精度(如 20V±5%)/切换建立时序/OCP·OVP·短路保护/掉电——关系安全，给错电压会烧设备。右下角标注：对照第 38 篇三分法，PD 属"协议协商类"——扮演对端一问一答走协商，不发码型不看眼图](/images/42/ill06_what.png)

## 五、怎么测：协议分析仪 + 电子负载 + PD 测试仪，示波器退居配角

正因为重头在"协议"和"供电"，测 USB PD 的**台面和眼图篇完全不一样**。眼图篇的主角是高带宽示波器；这里主角换人了：

- **测试点在哪：** 不再是高速差分对，而是 **CC 针**（看协商）和 **VBUS**（看供电）。
- **PD 协议分析仪（主角之一）：** 像 Ellisys、Teledyne LeCroy、GRL 这类工具，**串在 CC 线上，把整场谈判的每一条消息、每一个 GoodCRC、每一步的时间戳全抓下来**，你能一眼看到"谈到哪一步崩了"。这是查协议问题的核心武器。
- **PD 测试仪 / 激励器（主角之二）：** 它能**主动扮演对端**——你测充电器（Source），它就装成各种刁钻的设备（Sink）去请求各种档位、故意超时、故意发错，看充电器扛不扛得住；你测设备（Sink），它就装成充电器广播各种能力，看设备要得对不对。**"扮演对端跑协商"，就是 PD 的"发码型"。**
- **电子负载：** 像 Chroma、Kikusui 的可编程电子负载，接在 VBUS 上**按需拉不同的电流**，看谈到某档后电压稳不稳、拉到极限时 OCP 保护跳不跳。
- **示波器（配角）：** 只在看 **CC 上 BMC 信号质量**、和 **VBUS 电压切换那一下的沿/过冲**时才上，不再是全场核心。

还有一步绕不开：**VIF 文件**——你得先声明这个产品支持哪些角色、哪些档位、哪个 PD 版本，一致性工具据此才知道该考它哪些项。

![图7：测 USB PD 的台面（示波器退居配角）。中间 DUT(被测充电器或设备)。① PD 协议分析仪串在 CC 线上，抓全场谈判每条消息+GoodCRC+时间戳→看谈到哪步崩。② PD 测试仪/激励器主动扮演对端(测 Source 就装 Sink 发各种请求/故意超时发错；测 Sink 就装 Source 广播各种能力)——"扮演对端跑协商"就是 PD 的发码型。③ 可编程电子负载接 VBUS 按需拉电流→看各档电压稳不稳、拉极限时 OCP 跳不跳。④ 示波器只在看 CC 的 BMC 信号质量、VBUS 切换沿/过冲时才上，配角。左上：先填 VIF 文件声明支持的角色/档位/PD 版本，工具据此选测试项](/images/42/ill07_setup.png)

## 六、判据从哪来：不是从 SI 预算推，是规范和安全直接"写死"

眼图篇的判据（眼高、眼宽、抖动限值）是从信道损耗和抖动预算一层层**推**出来的。USB PD 不一样——它的判据**大多是 USB-C 和 PD 规范直接写死的电气值和时序值**，背后是**互通和安全**，不需要你去推导：

| 判据类 | 典型限值（举例，具体以规范版本为准） | 出处 |
|---|---|---|
| CC 电阻 | Rp 56k/22k/10k、Rd 5.1kΩ、Ra 0.8～1.2kΩ | USB Type-C 规范 |
| BMC 信令 | 位速率 300kbps、UI / 上升沿 / 摆幅按规范窗口 | USB PD 规范 |
| 协商时序 | 如"收到消息后须在约定时间内回 GoodCRC / 回应"、各类超时门限 | USB PD 规范 |
| VBUS 电压 | 各档精度（如 5V/9V/15V/20V 的容差窗）、切换建立时间 | USB PD 规范 |
| 保护 | OCP / OVP / 短路，触发阈值与响应 | USB Type-C / PD + 安规 |

> 所以这篇的"回链"和眼图篇不同：它主要**回链到第 38 篇的方法论**（一致性=照规范逐条对照）和**安全**，而不是回链模块三的 SI 概念推导。**记住一句：眼图篇的判据是"推"出来的，PD 篇的判据是"规范和安全写死"的。**

## 七、常见翻车与调试：慢充 / 不充 / 不认 / 烧，怎么用协议分析仪揪出真凶

这一篇的调试，武器不是示波器眼图，而是**协议分析仪抓下来的那段谈判 log**——看它**卡在哪一步**，原因基本就锁定了：

- **只有 5V、慢充（最常见）**：PD 协商压根没成或没跑到高档。查：便宜线**没有 e-marker**（最高只认 3A）、CC 通信断（线里 CC 没接/接触不良）、两端 **PDO 没对上**（充电器没有设备想要的档）、或协议版本不兼容 → 回落 5V。
- **完全不认 / 不供电**：**方向检测失败**（CC 上 Rp/Rd 不对、CC 针损坏）、**Vconn 缺失**导致 e-marker 线不工作、Sink 的 Rd 没挂上。
- **能充但不稳 / 掉档**：某档 **VBUS 精度或纹波超标**、电子负载一拉大电流就掉、或 **OCP 误触发**（还没到额定就跳）。
- **协商能开始却中途断**：**时序违规**——某条消息回应超时、GoodCRC 没在窗口内返回、状态机在角色交换（DRP）时死锁。
- **最严重：烧设备**：**协商没谈妥就上了高压**、或 **OVP 没做好**——这是安全红线，一致性里对保护和时序卡得最死就是为了防这个。

调试口诀：**先看协议 log 停在哪条消息（协议层），再量那一步的 VBUS 和 CC 波形（电气层），最后对时序窗口（时序层）。** 三层一套，翻车原因基本无处可藏。

## 八、一句话总结

USB-C / USB PD，是整个模块六里**唯一一篇不看眼图的一致性测试**。

> **USB-C 是"插座形状"（24 针、正反插、CC 定向），USB PD 是"跑在 CC 针上的供电协议"，二者别混。** 供电分三层级：传统 BC1.2（5V/1.5A）→ Type-C 电流广播（CC 电阻 Rp 56k/22k/10k 报 Default/1.5A/3A）→ **USB PD 协商**（BMC 300kbps 在 CC 上一问一答：Source_Capabilities → Request → Accept → PS_RDY，每条必回 GoodCRC；PDO/EPR/PPS 覆盖 5～48V、最高 240W）。它测三类：**CC 电气 & BMC 信号质量**（电阻容差、波形时序，还用示波器）、**协议一致性**（消息序列 / 时序 / GoodCRC / 状态机，测行为）、**供电电气**（VBUS 各档精度、切换时序、OCP/OVP 保护，关乎安全）。台面主角换成 **PD 协议分析仪 + PD 测试仪（扮演对端跑协商）+ 电子负载**，示波器退居配角。判据大多是**规范和安全直接写死**的电气 / 时序值，不像眼图那样从 SI 预算推。调试就靠**协议 log 停在哪一步**：慢充多是线没 e-marker / PDO 没对上 / 协议回落，不认多是方向或 Vconn，烧设备则是保护和时序没守住。对照第 38 篇——**PD 属于"协议协商类"一致性：不发码型、不看眼图，靠扮演对端一问一答把协议走完。**

![图9：USB-C / USB PD·工程师速查——USB-C(24针/正反插/CC 定向)≠USB PD(CC 上的供电协议)；供电三层级 BC1.2→Type-C 电流广播(Rp 56k/22k/10k)→PD 协商；CC 电阻 Rp/Rd 5.1k/Ra 1k+Vconn 给 e-marker 供电；PD=BMC 300kbps，PDO/EPR/PPS 覆盖 5-48V/240W，协商序列 Source_Cap→Request→Accept→PS_RDY+GoodCRC，SOP/SOP′/SOP″；测三类 CC电气&BMC/协议一致性/供电电气；台面 协议分析仪+PD测试仪+电子负载，示波器配角；判据规范&安全写死；翻车反查慢充/不认/烧。属第38篇"协议协商类"一致性——不看眼图。一张图带走整篇](/images/42/ill09_quickref.png)

## 🔧 工程师速查（附录）

| 项 | 要点 |
|---|---|
| USB-C vs USB PD | USB-C=接口形状（24 针 / 正反插 / CC 定向）；USB PD=跑在 CC 针上的供电**协议**。USB-C 本身不等于快充 |
| 供电三层级 | ①BC1.2（D+/D− 握手，5V/1.5A）→ ②Type-C 电流广播（CC 电阻，5V/最多 3A）→ ③USB PD 协商（5～48V/最高 240W）；谈不成一路回落 |
| CC 电阻 | Rp（Source 上拉）56kΩ=Default / 22kΩ=1.5A / 10kΩ=3A；Rd（Sink 下拉）=5.1kΩ；Ra（线 e-marker）≈0.8～1.2kΩ |
| 方向检测 | 被拉出电压的那根 CC=接通面，另一根空着或作 Vconn；纯电阻分压，无需芯片 |
| Vconn / e-marker | 通信占一根 CC 时，另一根作 Vconn（≈5V）给线里 e-marker 供电；无 e-marker 的便宜线最高只认 3A |
| PD 物理层 | BMC（双相标记编码）在 CC 上，**300kbps** 慢速控制通道（与 480M/10G 数据无关） |
| PDO 类型 | Fixed 固定档（5/9/15/20V，SPR≤100W）；EPR 扩展档（28/36/48V，≤240W，需专用线）；APDO/PPS（≈20mV/50mA 小步可调） |
| PD 版本 | 2.0（≤100W）→ 3.0（加 PPS/扩展消息/快速角色交换）→ 3.1（加 EPR，≤240W，2021） |
| 协商序列 | Source_Capabilities → Request → Accept → PS_RDY；**每条消息接收方必回 GoodCRC**，收不到就重发→超时报错 |
| SOP 类型 | SOP=设备↔充电器；SOP′/SOP″=跟线里 e-marker 对话（问线的能力） |
| 测什么·三类 | ①CC 电气 & BMC 信号质量（电阻容差/波形/时序，用示波器）②协议一致性（序列/时序/GoodCRC/状态机 DRP，测行为）③供电电气（VBUS 各档精度/切换时序/OCP·OVP 保护） |
| 激励方式（↔38） | PD=**协议协商类**：不发码型、不看眼图，靠 PD 测试仪**扮演对端一问一答**跑完协商 |
| 台面工具 | PD 协议分析仪（Ellisys/Teledyne/GRL，抓谈判 log）+ PD 测试仪/激励器（扮演对端）+ 可编程电子负载（拉 VBUS）+ 示波器（配角，看 CC/VBUS 波形）；先填 VIF 文件 |
| 判据从哪来 | 大多**规范 + 安全直接写死**的电气/时序值（不像眼图从 SI 预算推）；回链第 38 篇方法论 |
| 翻车反查 | 慢充=线无 e-marker/PDO 没对上/协议回落；不认=方向或 Vconn 失败；不稳=VBUS 精度/OCP 误触发；中途断=时序超时/DRP 死锁；烧=没谈妥就上高压/OVP 缺失 |
| 调试口诀 | 先看协议 log 停在哪条（协议层）→ 再量该步 VBUS/CC 波形（电气层）→ 最后对时序窗口（时序层） |
| 标准出处 | USB Type-C Cable and Connector Spec（USB-IF）；USB Power Delivery Spec R3.1（USB-IF） |
