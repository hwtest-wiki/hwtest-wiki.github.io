# 信号完整性入门：一根走线，怎么把好好的方波啃变形？

<div class="article-meta"><span class="badge">📘 模块三 · 信号完整性 SI · 开篇</span><span class="badge">⏱ 约 16 分钟</span><span class="badge qr">🔧 含工程师速查</span></div>


先看一个让人头皮发麻的现场。

一条 **1Gbps** 的高速串行线，设计阶段仿真跑得漂漂亮亮：眼图睁得又大又亮，眼高 **400mV**、眼宽 **0.7UI**，误码率低到 **10⁻¹²**，余量满满。可板子打样回来，示波器一抓实测眼图——**眼高只剩 80mV、眼宽塌到 0.2UI**，误码率飙到 **10⁻³**，也就是每发一千个 bit 就错一个。

你把芯片换了、固件查了个遍，全没问题。最后才反应过来：**问题出在这根线本身。** 这根信号走了个直角、还打了两个过孔，让它的阻抗从 **50Ω** 一路跳到 **75Ω**，信号在这儿反弹、振铃；旁边紧挨着一根时钟线，一跳变就往它身上串噪声；再加上走了 **10cm** 的 FR4 板材，高频分量被吃掉一大截——**反射、串扰、损耗，三样一起下嘴**，把那只本来睁得好好的"眼睛"，硬生生啃闭了。

**这就是信号完整性的核心命题：在高速下，铜线不再是"理想导线"，它自己就会让信号变形、变脏。** SI 这门功课，就是要搞清楚信号怎么被啃坏的、坏到什么程度、还能不能被接收端认对。这一篇，我们先把地基打牢。

**读完这篇，你能搞懂：**

- ✅ 信号完整性到底管什么、跟前面的电源完整性有什么不同；
- ✅ 为什么低速没事、一到高速铜线就"变脸"——**导线变成传输线**是怎么回事；
- ✅ 三大杀手怎么下嘴：**反射**（阻抗不连续）、**串扰**（相邻线耦合）、**损耗**（越走越软）；
- ✅ 这三样最后怎么算总账——**眼图**为什么是 SI 的"体检报告"；
- ✅ SI 到底测些什么：示波器、眼图、TDR、S 参数……一张路线图看懂后面要学啥。

## 一、先认清：信号完整性管什么，为什么高速会"变脏"

前面两个模块讲的**电源完整性（PI）**，管的是**供电轨稳不稳**：纹波大不大、掉压多不多、阻抗低不低。而**信号完整性（SI）**，管的是**数据线上的信号干不干净**：你发一个方波出去，接收端收到的还是不是一个能认对的方波。一个管"电"，一个管"信"，是硬件测试的两大支柱。

那问题来了：同样一根铜线，为什么低速时啥事没有，一到高速就开始作妖？

关键根本**不在频率高低，而在信号的边沿有多快**——也就是**上升时间**。低速时，你发一个信号，它的电压在整根线上几乎同时变化，这根线就是一根**等电位的导线**，用欧姆定律想就够了。可当上升沿快到一定程度，信号来不及"整根线一起变"，它得像水波一样，**从发送端一路"传"到接收端**。这时候，这根铜线就不再是导线，而变成了一条**传输线**——它有了一个自己的脾气，叫**特征阻抗 Z0**（常见是单端 50Ω、差分 100Ω）。

一个好用的经验判据：**当信号在线上单程跑一趟的时间，超过它上升时间的六分之一左右，就得把这根线当传输线来对待了。** 换句话说，**不是看你时钟多少兆，而是看边沿多陡、线走多长**。哪怕是个几十兆的信号，只要边沿够陡、线够长，一样会出 SI 问题。

一旦进了传输线的世界，三个原来可以忽略的效应，全冒出来了，这就是我们要挨个测的三大杀手：**反射、串扰、损耗**。

> **一句话记住：PI 管"电稳不稳"，SI 管"信号干不干净"。信号会不会变脏，看的不是时钟多少兆，而是边沿多陡、线走多长——上升沿够快、走线够长，铜线就从"导线"变成有特征阻抗 Z0 的"传输线"，反射、串扰、损耗三个杀手随之登场。**

## 二、杀手一 · 反射：阻抗一不连续，信号就往回弹

先说最经典的杀手：**反射**。

信号在传输线上是**以波的形式传播**的。只要沿途特征阻抗一直不变，波就一路顺顺当当传到底。可一旦它遇到**阻抗突变**的地方——接头、过孔、直角拐弯、线宽变化、分支，尤其是**末端没做端接**——就有一部分能量会像回声一样**反弹回来**。

反弹多少，由一个简洁的公式决定，叫**反射系数 Γ**：

**Γ =（ZL − Z0）÷（ZL + Z0）**

其中 Z0 是传输线阻抗，ZL 是它遇到的那个负载阻抗。三种极端最好记：末端**开路**（ZL 无穷大），Γ = +1，信号**全反弹、同相**，叠上去电压冲到两倍，这就是**过冲**；末端**短路**（ZL = 0），Γ = −1，全反弹但**反相**；末端**正好匹配**（ZL = Z0），Γ = 0，**一点都不反弹**——这正是我们后面要做端接匹配的原因。

反弹回来的波，会**叠加**到原信号上，来回好几趟，波形就被啃成了**过冲、下冲、一圈一圈的振铃**，甚至中间出现一个**台阶**。这些畸变一旦冲过接收端的判决门限，或者振铃迟迟不稳，接收端就会**误判 0/1**。开头那条线阻抗从 50Ω 跳到 75Ω，就是在过孔和直角处发生了反射。

> **一句话记住：信号在传输线上以波传播，遇到阻抗突变（过孔/直角/末端不端接）就按反射系数 Γ=(ZL−Z0)/(ZL+Z0) 部分反弹——开路 Γ=+1（过冲到两倍）、短路 Γ=−1、匹配 Γ=0（不反弹）。反弹波叠加成过冲、振铃、台阶，冲过门限就误码。治它靠阻抗连续 + 端接匹配。**

## 三、杀手二 · 串扰：隔壁线一跳，你这根就被带歪

第二个杀手，来自**邻居**：**串扰（crosstalk）**。

两根挨得近的走线之间，其实是有"隐形连接"的——一根线上的**变化电场**会通过**互容**耦合过去，**变化磁场**会通过**互感**耦合过去。于是当一根线（叫它**攻击线 aggressor**）电平跳变时，就会在旁边那根线（叫它**受害线 victim**）上，凭空**感应出一个噪声尖峰**，哪怕受害线自己压根没在动。

串扰还分两个方向看：往**信号源那头**耦回去的，叫**近端串扰（NEXT）**；顺着往**接收端**跑过去的，叫**远端串扰（FEXT）**。密集排布的高速并行总线——比如 DDR 的地址数据线、一排排的高速信号——最容易中招：一堆线同时跳变，噪声叠加起来，受害线上的尖峰可能大到直接让接收端读错。

治串扰的思路也很直接：**拉开线间距**（常说的 3W 规则，线间距至少三倍线宽）、**在信号之间插地线/地平面隔离**、**缩短并行段的长度**。这些我们后面串扰专题会细讲，这里先记住它的机理：**耦合来自电磁场，距离越近、并行越长、边沿越陡，串得越凶。**

> **一句话记住：相邻走线通过互容（电场）和互感（磁场）耦合，攻击线一跳变，就在受害线上感应出噪声尖峰——回源头的叫近端串扰 NEXT、去接收端的叫远端串扰 FEXT。距离越近、并行越长、边沿越陡越严重，DDR 等密集并行线尤甚。治法：拉间距（3W）、插地隔离、缩短并行段。**

## 四、杀手三 · 损耗：走得越远、频率越高，波形越软

前两个杀手把波形啃出毛刺，第三个杀手则是**慢慢把信号"磨软、磨小"**：**损耗**。

信号沿着走线跑，能量会一路被吃掉，主要两笔账：

- **导体损耗（趋肤效应）**：频率一高，电流不再走满整个导体截面，而是全挤到导体**表面**那一薄层，等效电阻随频率的**平方根**往上涨——频率越高，铜线越"细"，损耗越大；
- **介质损耗**：PCB 板材（比如 FR4）的绝缘介质在高频电场下会**吸收能量**，损耗大小跟材料的**损耗角 tanδ** 成正比，频率越高、走线越长，吃掉的越多。

这两笔账合起来，效果就像给信号加了个**低通滤波器**：**高频分量衰减得比低频狠**。而方波的陡边沿，恰恰是靠一大堆高频分量拼出来的——高频被吃掉，边沿就**变缓**、幅度就**变小**，一个 bit 的能量还会**拖尾**到下一个 bit 上，造成**码间干扰（ISI）**：前一个符号没走干净，就污染了后一个。

所以同一条链路，**速率越高、走线越长、板材越差，损耗就越致命**。高速接口为什么要用低损耗板材、为什么长走线要加均衡（EQ），根子都在这。

> **一句话记住：损耗来自导体趋肤效应（等效电阻随 √f 上升）和介质损耗（随频率 × tanδ）——合起来是个低通，高频衰减更狠。方波的陡边沿靠高频撑着，高频被吃掉后边沿变缓、幅度变小、拖尾成码间干扰 ISI。速率越高、线越长、板材越差越严重，靠低损耗板材 + 均衡 EQ 补救。**

![图5：损耗——导体趋肤效应（电流挤到表面，Rac∝√f）+ 介质损耗（∝ f·tanδ），合成一条高频衰减的低通曲线；时域后果：上升沿变缓、幅度变小、拖尾成 ISI 码间干扰](/images/27/ill05_loss.png)

## 五、三样的总账单：眼图，一眼看出信号干不干净

反射、串扰、损耗，各有各的脾气，可它们最后都汇成同一张**总账单**——**眼图（eye diagram）**。

眼图怎么来的？很简单：把接收端这串信号，**按一个 bit 的周期（叫一个 UI，单位间隔）一段段切开，全叠在一起**。成千上万个 0/1 跳变叠上去，中间就会叠出一个像**睁开的眼睛**一样的图形。这只"眼睛"，就是信号质量的体检报告：

- **眼睛的高度（眼高）**，代表**幅度余量**——被噪声、串扰、损耗吃掉一部分后，接收端还能剩多少电压来分清 0 和 1；
- **眼睛的宽度（眼宽）**，代表**时间余量**——被抖动、码间干扰吃掉一部分后，还剩多少干净的时间窗口让接收端稳稳采样。

信号越干净，眼睛睁得越大越亮；反射的振铃、串扰的噪声、损耗的拖尾一叠加，眼睛就从上下、左右一起被**挤扁、糊住**，直到**闭合**。眼图一闭，接收端在采样那一刻就找不到一块"干净又稳定"的区域，**误码**就来了。开头那条线，眼高从 400mV 塌到 80mV、眼宽从 0.7UI 缩到 0.2UI，就是三个杀手一起挤出来的结果。

**所以眼图是 SI 最核心的一张图：它不告诉你是哪个杀手干的，但它一眼告诉你——这条链路现在到底行不行。** 后面我们会专门讲怎么测眼图、怎么从眼图倒推是谁在捣乱。

> **一句话记住：眼图 = 把信号按一个 UI 一段段切开全叠在一起，叠出一只"眼睛"。眼高 = 幅度余量（被噪声/串扰/损耗吃），眼宽 = 时间余量（被抖动/ISI 吃）。三个杀手一起挤，眼睛就从睁开到闭合，一闭合就误码。眼图是 SI 的综合体检报告。**

![图5动图：眼图从睁开到闭合——一开始眼睛又大又亮（眼高/眼宽余量足），随着反射振铃、串扰噪声、损耗拖尾逐步叠加，上下被压、左右被糊，眼睛一步步闭合](/images/27/ill05b_eye_anim.gif)

## 六、SI 到底测什么：一张路线图

搞懂了三个杀手和眼图，我们就能给整个 SI 模块画一张**路线图**了。SI 测试，无非从三个角度下手：

- **时域看波形**：用示波器直接抓信号，看**过冲、振铃、边沿**，切出**眼图**看整体余量，再做**抖动分解**看时间轴上抖了多少（后面 RJ/DJ/PJ 专题细讲）；
- **反射定位**：用 **TDR（时域反射计）**，往线里发一个陡阶跃，看反射回来的波形，就能算出**阻抗沿线怎么变、不连续点具体在哪**——哪个过孔、哪个接头出问题，一测便知；
- **频域看传输**：用 **VNA 测 S 参数**——**S21 是插入损耗**，量信号从这头到那头损失了多少；**S11 是回波损耗**，量有多少被反弹回来。频域这套，专治损耗和阻抗匹配。

对照本模块后面的篇章，你就知道每一篇在补哪块拼图：**阻抗匹配与端接**（治反射）、**上升时间与边沿带宽**（搞清高频从哪来）、**TDR 与反射定位**、**串扰专题**、**眼图怎么测**、**抖动 Jitter 分解**、**S 参数与损耗**、**均衡 EQ**。一篇篇下来，你就能把"信号为什么脏、脏在哪、怎么测、怎么治"这条链彻底打通。

这一篇是地基：**先认准三个杀手和眼图这张总账单，后面每一招都是在对付它们。**

> **一句话记住：SI 测试三条路——① 时域（示波器看过冲/振铃/边沿 + 眼图 + 抖动分解）；② 反射定位（TDR 发阶跃、定位阻抗不连续点）；③ 频域（VNA 测 S 参数：S21 插入损耗、S11 回波损耗）。后续篇章：端接匹配、边沿带宽、TDR、串扰、眼图、抖动、S 参数、均衡 EQ，逐一拆解。**

## 写在最后

从这一篇起，我们正式从"电稳不稳"的**电源完整性**，走进"信号干不干净"的**信号完整性**。它的世界观其实就一句话：**在高速下，铜线不再无辜，它自己就会啃坏你的信号。**

而啃信号的，就这三个杀手：**反射**在阻抗不连续处让信号反弹成振铃、**串扰**让隔壁线的跳变串成噪声、**损耗**把高频吃掉让边沿变软拖尾。它们各自的畸变，最后都汇成**眼图**这张总账——眼睛睁着，链路就活；眼睛闭了，数据就错。

**SI 测试的全部功夫，就是把这只眼睛量出来、看懂它、再一个杀手一个杀手地把它重新撑开。** 后面的每一篇，都是在教你对付其中一个。

> **SI 开篇一句话：高速下铜线会自己啃坏信号——反射、串扰、损耗三个杀手，把方波啃成脏波形，账全记在眼图上。测 SI，就是量眼图、抓杀手。**

| **反射** | 阻抗不连续处波部分反弹 | 过冲、下冲、振铃、台阶 | Γ=(ZL−Z0)/(ZL+Z0)；开路 +1、短路 −1、匹配 0 | 阻抗连续 + 端接匹配 |
| **串扰** | 互容（电场）/互感（磁场）耦合 | 受害线感应噪声尖峰 | 近端 NEXT / 远端 FEXT；距离近、并行长、边沿陡越凶 | 3W 间距、插地隔离、缩短并行段 |
| **损耗** | 导体趋肤（∝√f）+ 介质（∝f·tanδ） | 边沿变缓、幅度变小、ISI 拖尾 | 等效低通，高频衰减更狠 | 低损耗板材 + 均衡 EQ |

### 判断要不要按"传输线"处理

- 经验判据：**走线单程延迟 > 上升时间的 ~1/6** → 必须当传输线，考虑 Z0、反射、端接。
- 记住：**看边沿陡不陡、线长不长，不是只看时钟频率**。几十兆的信号边沿够陡、线够长照样出 SI 问题。

### 眼图怎么读

- **眼高**（幅度余量）：被噪声 / 串扰 / 损耗吃；越大越好。
- **眼宽**（时间余量）：被抖动 / 码间干扰 ISI 吃；越大越好。
- 眼睛**睁得大又亮** = 信号干净；**被挤扁 / 闭合** = 采样窗口没了 = 误码。

### SI 测什么 · 仪器对照

- **示波器**：波形（过冲/振铃/边沿）、**眼图**、抖动分解。
- **TDR（时域反射计）**：发阶跃，**定位阻抗不连续点**在哪、阻抗沿线怎么变。
- **VNA（S 参数）**：**S21 = 插入损耗**（信号损失多少）、**S11 = 回波损耗**（反射多少）。
- **误码仪 BERT**：直接测误码率 BER，是链路好坏的最终裁决。

### 常见翻车速查

- **过冲振铃严重** → 阻抗不连续 + 末端没端接；查过孔/直角/线宽突变，加端接匹配。
- **相邻线互扰** → 间距太近或并行太长；拉开 3W、插地、错开走线。
- **长线信号变软、误码** → 损耗 + ISI；换低损耗板材、缩短、加均衡。
- **仿真好实测差** → 仿真模型理想化（阻抗连续、无过孔寄生）；务必实测眼图 + TDR 对照。
