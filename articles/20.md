# 手册写着过流保护 3 安，短路那一下却冲到 8 安——MOS 还是烧了

<div class="article-meta"><span class="badge">📘 模块二 · 电源完整性 PI · 电源保护功能</span><span class="badge">⏱ 约 15 分钟</span><span class="badge qr">🔧 含工程师速查</span></div>


一颗电源芯片，数据手册上白纸黑字写着：**过流保护（OCP）3A**。你照着这个做设计，下游器件、走线都按 3A 留了余量，心想短路了顶多到 3A，保护一动就没事。

可真到输出短路那一下，你拿电流探头一抓——电流"嗖"地冲到了 **8A**，**持续了几十微秒**，电源才把自己关掉。就这几十微秒、这 8 安，把前级的 MOS 干烧了。

保护明明有，手册也写了 3A，怎么还是烧了？

因为保护这东西，**光"有"没用**。真正决定它管不管用的，是三件你必须亲手测的事：**阈值准不准**（到底在多少触发）、**响应快不快**（从超限到动作要多久）、**恢复怎么恢复**（关了之后是自己回来、还是反复打嗝、还是干脆锁死）。这一篇，咱就把四大保护、连同这三件事，一次讲清楚。

**读完这篇，你能搞懂：**

- ✅ 四大保护 **OVP / OCP / OTP / UVLO** 各自在防什么——一张图看明白；
- ✅ 为什么 **UVLO 和 OVP 一起，给输入电压圈出一个"正常工作窗口"**，还都带迟滞；
- ✅ **OCP 过流保护的三种脾气**——限流、打嗝、锁存，差别在哪、怎么选；
- ✅ **OTP 过温保护**为什么会让芯片"热一下、停一下"地循环；
- ✅ 这四类**怎么测**——阈值、响应时间、恢复行为，分别用什么仪器怎么逼出来。

## 一、先认人：四大保护各防什么

电源芯片里这四个保护，说白了就是四个"哨兵"，各盯一个量，一旦冲出安全范围就动手。先一句话认全：

- **OVP（过压保护，Over-Voltage Protection）**：盯**电压**。输出（或输入）电压冲太高，就关断或者钳位，免得下游那些娇贵芯片被高压烧穿。
- **OCP（过流保护，Over-Current Protection）**：盯**电流**。电流超了限值（最常见就是输出短路），就限流或者关断，保护 MOS、走线、电源自己。
- **OTP（过温保护，Over-Temperature Protection）**：盯**温度**。芯片结温太高，就降频或者关断，给自己降温，防止热失控烧毁。
- **UVLO（欠压锁定，Under-Voltage Lock-Out）**：盯**输入电压下限**。输入电压还没到能正常工作的门槛，就锁住不让启动，免得在"半死不活"的低压下误动作、甚至烧管子。

你发现没——前三个是"**超了上限**"就动手（压、流、温太高），UVLO 是"**没够下限**"就不干活（输入太低）。四个一起，把电源框在一个**电压不高不低、电流不爆、温度不烫**的安全区里运行。

> **一句话记住：四大保护各盯一个量——OVP 盯电压上限、OCP 盯电流上限、OTP 盯温度上限、UVLO 盯输入电压下限。前三个"超限就关"，UVLO"没够就不启"，四个一起把电源圈在安全工作区里。**

## 二、UVLO + OVP：给输入电压圈一个"工作窗口"

先把盯电压的两个放一块讲，因为它俩其实是**一对**——一个管下限、一个管上限，合起来给输入电压划出一段"**只在这个窗口里才工作**"的范围。

**UVLO 管下限。** 输入电压从 0 慢慢往上爬，**没爬到 UVLO 门限之前，芯片死活不启动**。为什么要这样？因为电压太低的时候，芯片内部的 MOS 驱动不够强、导通电阻大，硬要工作就会发热甚至烧管；而且低压下逻辑容易乱。所以干脆**锁住**，等输入够高了再放行。

**OVP 管上限。** 输入（或输出）电压要是冲得太高，超过 OVP 门限，芯片就**关断或钳位**，保护下游。

中间这段，就是芯片的**正常工作窗口**。低于 UVLO——不启动；高于 OVP——关断；只有落在中间，才正常干活。

这里有个特别重要、又特别容易被忽略的细节：**这两个门限都带"迟滞"（hysteresis）**。啥意思？拿 UVLO 举例——它"**开启**"的门限和"**关闭**"的门限**不是同一个值**。比如输入升到 **4.5V** 才启动，但启动后要一直掉到 **4.2V** 才关闭。这中间 0.3V 的差，就是迟滞。

为什么非要搞这么个迟滞？因为如果开关用**同一个**门限，那当输入电压正好在门限附近抖动（带点纹波、带点波动），芯片就会**开一下、关一下、开一下、关一下**地疯狂抖动，这叫"**振荡**"，特别伤器件。加了迟滞，开关门限拉开一段距离，就把这种临界抖动**摁住了**。OVP 同理，也带迟滞。

> **一句话记住：UVLO 管输入下限（没够就不启动）、OVP 管上限（超了就关断），合起来圈出"正常工作窗口"。两个门限都带迟滞——开启和关闭不是同一个值，就为了防止电压在门限附近抖动时，芯片疯狂开关振荡。**

## 三、OCP 过流保护：限流、打嗝、锁存，三种脾气

再说盯电流的 OCP，它是四个里**行为最多样**的一个。同样叫"过流保护"，电流超限之后它怎么动作，有**三种完全不同的脾气**，你得知道自己手里这颗是哪种：

**① 限流（Constant Current）。** 电流一到限值，电源就**死死把电流摁在这个值**，不让它再涨，同时输出电压往下掉。好处是输出还在、负载轻了能自动恢复；坏处是短路时它一直在**全电流 + 大压差**下硬扛，**自己很容易过热**——所以限流常常得配合 OTP 一起用。

**② 打嗝（Hiccup）。** 电流超限，电源**先关一下**，歇一小会儿，再**试着重启**一次；要是故障还在，就再关、再歇、再试……于是输出就"**嗝、嗝、嗝**"地一下一下试。好处是故障时平均功耗很低、芯片不发热；故障一排除，下次重启就正常恢复了。这是现在最常见的一种。

**③ 锁存（Latch-off）。** 电流一超限，电源**直接关死、锁住**，不再自己尝试恢复——必须**断电重新上电**（或拉一下使能）才能复位。好处是最安全、最果断，适合那种"出了过流必须人来检查"的场合；坏处是不能自恢复，体验上"一碰就死"。

选哪种看场合：要自恢复、低功耗，用打嗝；要绝对安全、出事就停，用锁存；要输出别断、能扛一会儿，用限流（记得管好温度）。

> **一句话记住：OCP 过流后有三种脾气——限流（摁住电流、输出压降、自己易热）、打嗝（关一下试一下、低功耗、故障除了能恢复）、锁存（直接关死、必须重新上电）。测之前先搞清楚你这颗是哪种，别拿打嗝的标准去要求锁存的。**

## 四、OTP 过温保护：热一下、停一下的循环

第三个，OTP 过温保护，盯的是芯片自己的**结温**。

芯片干活就会发热，尤其是上面说的限流硬扛、或者负载很重的时候。结温要是爬过 OTP 的门限（典型一百五十度上下），芯片就**降频或者干脆关断**，把功耗砍掉，让自己凉快下来。

这里又是**迟滞**在起作用：OTP 也不是同一个门限开关。比如**升到 160℃ 关断**，得**降到 140℃ 才恢复**。于是如果散热一直不行、故障没排除，你会看到芯片**热到 160℃→关断→降到 140℃→重启→又热到 160℃→再关断**……这么一下一下地**循环**。表现出来，就是设备"**工作一会儿、停一会儿、再工作一会儿**"，反复横跳。

碰到这种"间歇性工作"，别光怀疑软件——很可能就是 OTP 在反复救场，**根子是散热或者负载有问题**。OTP 是最后一道防线，它频繁动作，说明前面的热设计已经扛不住了。

> **一句话记住：OTP 盯结温，超过门限（典型 ~150℃）就降频/关断给自己降温，也带迟滞（如 160℃ 关、140℃ 恢复）。散热不行时会"热→关→凉→开"地循环，表现为设备间歇性工作——这是热设计告急的信号，不是软件 bug。**

![图5：OTP 过温保护循环——结温爬到关断门限就关断降温，降到恢复门限再重启，散热不足时形成"热→关→凉→开"的反复循环（迟滞）](/images/20/ill05_otp.png)

![图5动图：结温随时间爬升，触到 OTP 上门限就关断、温度回落，掉到下门限又重启升温，循环往复形成锯齿状温度曲线](/images/20/ill05b_otp_anim.gif)

## 五、怎么测：把保护一个个"逼"出来

保护功能的测试，思路和前面不一样——前面是"别让它出事"，这里是**反过来，主动制造异常，把每个保护逼到动作**，然后看它**在哪触发、多快动作、怎么恢复**。一个一个来：

1. **OVP / UVLO 用可调电源扫输入**：把输入电压从低到高、再从高到低**慢慢扫**，记下芯片**启动**的电压（UVLO 开启门限）、**关断又恢复**的电压（UVLO 关闭门限、OVP 门限）；开启和关闭两个值之差，就是**迟滞**。
2. **OCP 用电子负载拉电流 / 直接短路**：用电子负载把输出电流慢慢往上拉，看**在多少安触发**；再做一次**真短路**，用示波器单次触发抓那一下，看电流**冲到多高峰值、多久把它压住**（响应时间），以及它是**限流、打嗝还是锁存**。
3. **OTP 给芯片加热**：用热风枪、加热台或者烤箱给芯片升温（或者干脆让它满载自己热），测**在多少结温触发**、降到多少**恢复**，看那个迟滞和循环。
4. **每个都要量"响应时间"**：从超限那一刻，到保护真正动作之间的**延迟**——开头那块板烧 MOS，问题就出在这几十微秒的延迟上。响应慢，峰值就压不住。

测的时候记着三个维度：**阈值**（在哪触发）、**响应时间**（多快动作）、**恢复行为**（自恢复 / 打嗝 / 锁存 + 迟滞）。三个都对，保护才算真的靠得住。

> **一句话记住：保护要"主动逼出来"测——OVP/UVLO 用可调电源扫输入电压（记开启/关闭门限和迟滞）；OCP 用电子负载拉载 + 真短路（示波器单次触发看峰值、响应时间、是限流/打嗝/锁存）；OTP 加热升温（记触发/恢复温度）。每个都量阈值、响应时间、恢复行为三件事。**

![图6：保护功能测试连接图——可调电源扫输入测 OVP/UVLO、电子负载拉载+短路测 OCP、热风枪/烤箱加热测 OTP，示波器单次触发抓阈值与响应时间](/images/20/ill06_test.png)

## 七、写在最后

这一篇想让你记住一件事：**保护功能，"有"和"管用"是两码事。** 手册写着 OCP 3A，不代表短路那一下就真的卡在 3A——它可能冲到 8A、延迟几十微秒才动作，这中间足够烧掉器件。

所以保护这东西，**必须亲手逼出来测**：在哪触发（阈值准不准）、多快动作（响应跟不跟得上）、怎么收场（限流、打嗝还是锁存，迟滞够不够）。把这三件事量清楚，你才敢说这个电源"保护做到位了"。

下次再碰到"明明有保护却还是烧了 / 保护老是误动作 / 设备间歇性重启"，别只看手册那一行字——上可调电源、电子负载、加热台，把每个保护一个一个逼出来，看它真实的脾气。

下一篇，我们聊电源的另一种"边界本事"——**掉电保持时间（hold-up time）**：输入突然断电的那一刻，电源还能稳稳撑住多久，够不够让系统从容把数据存好、安全关机。

### 判据表 1 · 电源四大保护核心判据

| 保护 | 盯什么 | 测什么 | 判据 / 参考 |
|---|---|---|---|
| **OVP 过压** | 电压上限 | 触发门限 / 响应时间 / 恢复 | 门限在器件耐压内留余量；响应足够快不让下游过压；钳位或关断 |
| **OCP 过流** | 电流上限 | 限流值 / 响应时间 / 行为 | 短路峰值与响应在 MOS·SOA / 保险 I²t 内；明确限流/打嗝/锁存 |
| **OTP 过温** | 结温上限 | 触发/恢复温度 / 迟滞 | 典型触发 ~150℃，恢复带迟滞（如 160→140℃）；频繁动作=散热告急 |
| **UVLO 欠压** | 输入下限 | 开启/关闭门限 / 迟滞 | 开启 > 关闭（迟滞防临界振荡）；低于关闭门限可靠锁定不工作 |

### 关键概念

- **迟滞 hysteresis**：开启与关闭门限不同（UVLO/OVP/OTP 都有），防电压/温度在门限附近抖动时反复开关振荡
- **OCP 三种行为**：
  - **限流（constant current）**：摁住电流、输出电压下垂、芯片持续受热（常配 OTP）
  - **打嗝（hiccup）**：周期性关断—重试，故障时低功耗，故障除自恢复
  - **锁存（latch-off）**：一次关死，必须重新上电 / 拉使能复位
- **响应时间**：从超限到保护动作的延迟；延迟内的峰值（电流/电压）才是真正烧器件的元凶
- **SOA / I²t**：MOS 安全工作区 / 保险熔断能量——OCP 峰值与响应必须落在其内

### 测试设置

- **OVP / UVLO**：可调直流电源，输入电压从低到高、再从高到低慢扫；记 UVLO 开启门限、关闭门限、OVP 门限、各自迟滞；示波器看关断/恢复瞬间
- **OCP**：电子负载（CC 模式）逐步拉载找触发点；再做真短路，示波器单次触发抓电流峰值 + 响应时间 + 判断限流/打嗝/锁存
- **OTP**：热风枪 / 加热台 / 烤箱给芯片升温（或满载自热），热电偶 / 热像仪测结温，记触发温度、恢复温度、迟滞
- **响应时间**：触发源选被保护量（电压/电流），单次触发，测超限沿到动作沿的时间差
- **三维度**：每个保护都量 ① 阈值 ② 响应时间 ③ 恢复行为（含迟滞）

### 常见翻车与对策

- **有保护却烧器件** → 响应太慢 / 阈值太高：测真实触发点与响应时间，按 SOA·I²t 反推是否够快
- **正常工作就误跳保护** → 阈值太靠近工作点 / 无迟滞：核对门限余量，确认迟滞够大
- **设备间歇性重启** → OTP 反复动作：查散热与负载，OTP 是结果不是病根
- **低压下烧管 / 乱动作** → UVLO 门限太低或缺迟滞：抬高 UVLO 开启门限、加足迟滞
- **短路偶尔不恢复** → 锁存型需重新上电：确认是 latch-off 而非 hiccup，按设计预期选型
