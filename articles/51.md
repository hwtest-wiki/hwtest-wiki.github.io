# I2C 读不到从机、总线卡死、挂多几片就掉速、换个上拉就好了——I2C：低速、单端、开漏、共享，测上升时间 + 电平 + 时序 + 协议，不是眼图

先说个做硬件、写驱动的人几乎都踩过的场景。

你在主板上挂了一颗传感器、一片 EEPROM、或者一个 PMIC，用 I2C 去读它，结果 **读不到、返回全是 0xFF 或者 0x00、驱动报 NACK**；或者本来好好的，**多挂了几片芯片之后就开始时好时坏、掉速**；再或者，**总线突然卡死——SDA 一直被拉在低电平下不来**，整条总线上的设备全哑了；又或者，**把那颗上拉电阻从 10kΩ 换成 2.2kΩ，一切又正常了**。

做硬件的人慢慢会明白：**这类 I2C 的「读不到、掉速、卡死、换个电阻就好」，绝大多数根本不是芯片坏、不是驱动写错，而是这条物理总线——上拉电阻选得对不对、总线上挂的电容大不大、电平域匹不匹配、时序和协议满不满足规范——没配好、没测好。** 上拉太弱、或者总线上挂的设备太多、走线太长，**上升时间就慢下来**——SCL 都跳到一半电压了，从机还没认出这是个高电平，高速档自然就跑不动、只能掉速甚至误读；上拉太强、或者某颗芯片灌电流能力不够，**低电平又拉不干净**，逻辑 0 被认成 1。

I2C 到底是怎么工作的、为什么那两颗上拉电阻这么关键、它的信号和时序该怎么测、协议帧怎么解——这，就是这一篇要讲的，也是我们**从高速接口一致性，转入低速板级总线**这条新线的第一篇。

## 一、先认清主角：I2C 是什么、它和高速接口到底不一样在哪

**I2C**（Inter-Integrated Circuit，集成电路间总线，读作 I-squared-C）是飞利浦（今 NXP）发明的**板级、低速、两线、多设备共享**的同步串行总线。你身边到处都是它：主板上的**温度 / 湿度 / 光线 / 陀螺仪传感器、EEPROM、实时时钟 RTC、PMIC 电源管理芯片、触摸屏配置、显示器读 EDID、风扇 / 电压监控**——这些「不用高速、但要能对话」的小芯片，绝大多数都挂在 I2C 上。

它只用**两根线**：**SCL（串行时钟）+ SDA（串行数据）**，外加地和电源。一个**主机（Master，通常是 SoC / MCU）** 用 SCL 打节拍、发起读写；一堆**从机（Slave，各种外设芯片）** 挂在同一对线上，靠**各自的地址**被点名对话。

它和前面模块六所有高速接口，有四个根本区别——**每一个都直接决定了它怎么测**：

1. **单端 + 开漏，不是差分推挽**：高速接口几乎都是**差分对**（一对线看压差、抗噪好），发送端**推挽**（能主动拉高也能拉低）。I2C 是**单端**（对地看电平）、而且是**开漏 / 开集（open-drain）**——芯片**只能把线拉低，不能主动拉高**，靠外接的**上拉电阻**把线拉回高电平。这就带来了 I2C 的灵魂：**高电平是「松手让上拉拉上去」，上升沿快慢由「上拉电阻 × 总线电容」这个 RC 决定**，不是芯片驱动出来的。

2. **时钟单独一根 SCL，不埋在数据里、不用 CDR**：高速串行把时钟**编进数据流**，接收端靠 **CDR** 把时钟从数据边沿里恢复出来——所以要看眼图、测抖动。I2C 有**独立的时钟线 SCL**，数据 SDA 在 SCL 的节拍下采样，**根本没有 CDR、没有眼图这回事**。测它，测的是 SCL / SDA 两条**实打实的方波**：边沿多快、电平多高多低、相互时序对不对。

3. **多设备挂同一对线、靠地址寻址、wired-AND**：高速接口大多**点对点**（一对一，顶多加 Retimer）。I2C 是**一对多总线**——多颗从机（甚至多主机）挂在**同一对 SCL / SDA** 上，靠**7 位（或 10 位）地址**点名。因为都是开漏，总线是**线与（wired-AND）**：**只要有一个设备拉低，整条线就是低**。这意味着**总线电容是所有挂载设备 + 走线的总和**，挂得越多、走线越长，电容越大、上升越慢。

4. **从机能拖慢主机——时钟拉伸（clock stretching）**：这是 I2C 独有的一招。从机如果**还没准备好数据**，可以**把 SCL 摁住不放（拉低）**，主机看到 SCL 上不去，就得**等**——这叫时钟拉伸。高速接口里没有这种「接收方反过来卡住时钟」的机制。它很有用，但**主机必须支持**，否则从机一拉伸，主机以为总线出错，就误读、超时。

一句话：**I2C = 单端开漏、时钟单独一根、多设备共享寻址、还能时钟拉伸的低速板级两线总线**；它测的是**上升 / 下降时间 + 电平 + 时序 + 协议解码**，把你前面高速那套「差分对、看眼图、靠 CDR、测抖动」的思路，整个反了过来。

## 二、原理：开漏 + 上拉的 RC、START/STOP/ACK 的帧、共享总线的仲裁与拉伸

I2C 的原理，可以拆成三块，每一块都直接对应一类要测的东西。

### 2.1 开漏 + 上拉：上升时间由 RC 决定，这是 I2C 的命门

I2C 的每一根线（SCL、SDA）都是**开漏**：芯片里是一颗**只能往下拉（拉低到地）的管子**，要么拉低、要么**高阻（松手）**。松手时，谁把线拉回高电平？——**外接到 Vdd 的上拉电阻 Rp**。于是：

- **下降沿**：芯片主动拉低，又快又利索（有源驱动）。
- **上升沿**：芯片松手，靠 **Rp 通过总线电容 Cb 充电**回到高电平——这是一条 **RC 指数曲线**，快慢由 **Rp × Cb** 决定。NXP 规范给的关系是 **上升时间 t_r ≈ 0.8473 × Rp × Cb**。

这就是 I2C 一切信号问题的根：**Rp 太大（上拉太弱）或 Cb 太大（挂太多设备 / 走线太长），t_r 就超标**——SCL 还在慢悠悠往上爬，采样点到了电平还没到高，高速档就跑不动、误读；反过来 **Rp 太小（上拉太强）**，虽然上升快了，但**芯片拉低时要灌的电流大**，若灌不动，**低电平 V_OL 拉不到规定值以下**，逻辑 0 被认成 1。所以选上拉，是在「上升够快」和「低电平拉得下去」之间**卡一个窗口**——这是 I2C 独有、也是最常调的一件事。

### 2.2 START / STOP / ACK：一帧长什么样

I2C 的一次通信，由主机用几个**规定的边沿事件**框起来：

- **START（起始，S）**：SCL 为高时，**SDA 由高拉低**——宣布「我要开始了，大家听好」。
- **数据位**：SCL 为低时 SDA 可以变化，**SCL 为高的整个期间 SDA 必须稳定**（这是采样窗口）；每个时钟脉冲传 1 位。
- **地址 + 读写位**：START 后的第一个字节是**7 位从机地址 + 1 位读写方向（R/W）**；被点名的从机在**第 9 个时钟**把 SDA 拉低给一个 **ACK（应答）**，没人应答就是 **NACK**（SDA 保持高）——你「读不到从机」时，示波器上看到的往往就是**第 9 拍没有 ACK**。
- **STOP（停止，P）**：SCL 为高时，**SDA 由低放高**——宣布「本次结束」。中途不发 STOP 而再来一个 START，叫**重复起始（Repeated START，Sr）**，用于「写地址后立刻转读」不释放总线。

所以协议一层测的，就是**这些帧对不对**：START/STOP 边沿合不合规、地址对不对、第 9 拍有没有 ACK、数据字节读出来是不是预期值。

### 2.3 共享总线：wired-AND、仲裁、时钟拉伸

因为大家都开漏、都挂在一对线上，总线是**线与**：**任何一个设备拉低，线就是低；只有所有设备都松手，上拉才能把它拉高**。这带来三件事：

- **总线电容是全体之和**：每颗芯片的引脚电容、每段走线的分布电容，全加在 Cb 里——**挂得越多、线越长，Cb 越大、上升越慢**。这就是「多挂几片就掉速、时好时坏」的物理原因。
- **多主仲裁**：若有多个主机，谁在发高、总线却被别人拉低了，它就知道「我输了」，自动退出——靠线与天然实现无损仲裁。
- **时钟拉伸**：从机没准备好，就**把 SCL 摁在低电平**，主机想放高也放不上去（线与），只能等它松手——这是 I2C 让慢速从机跟上的机制，但**要求主机识别并容忍 SCL 被拖住**，否则超时误判。

## 三、速率档与电平：100k 到 5M，电平从 Vdd 派生

I2C 的规范由 **NXP UM10204** 定义，速率分几档、电平按 Vdd 派生，一张表理清：

| 速率档 | 缩写 | 最高时钟 | 信号方式 | 上升时间 t_r（max） | 备注 |
|---|---|---|---|---|---|
| **标准模式** | Sm | 100 kbit/s | 开漏 + 上拉 | 1000 ns | 最经典、最兼容 |
| **快速模式** | Fm | 400 kbit/s | 开漏 + 上拉 | 300 ns | 最常用 |
| **快速模式+** | Fm+ | 1 Mbit/s | 开漏 + 上拉（强驱动） | 120 ns | 灌流能力更强（20 mA） |
| **高速模式** | Hs | 3.4 Mbit/s | 开漏 + 电流源上拉 | 80 / 160 ns | 需主机码、SCLH 有源上拉辅助 |
| **超快速模式** | UFm | 5 Mbit/s | **推挽、单向、无 ACK** | — | 只写不读，非标准 I2C 收发 |

几个必须记住的点：

- **电平门槛按 Vdd 派生，不是固定电压**：I2C 规定 **V_IL（低电平上限）≤ 0.3 × Vdd**，**V_IH（高电平下限）≥ 0.7 × Vdd**。所以 **1.8V 域和 3.3V 域不能直接混接**——3.3V 器件要求 V_IH ≥ 2.31V，而 1.8V 器件最高才 1.8V，**永远达不到**，必须加**电平转换器**。「电平域搞错」是 I2C 最隐蔽的翻车之一。
- **低电平要拉到 V_OL ≤ 0.4V**：芯片拉低时得把线压到 0.4V（3mA 灌流下）以下，才算合格的逻辑 0；上拉太强 / 灌流不够都会顶不下去。
- **Hs 高速模式是特例**：要先在 Fm 下发一个**主机码**切换，且高速段用**电流源做有源上拉**帮 SCL 快速拉高——因为纯电阻上拉在 3.4M 下 RC 太慢。它在消费电子里其实**用得不多**，多数场合 Fm / Fm+ 就够。
- **UFm 超快速模式是另一路**：**推挽输出、单向、无 ACK**，本质是「广播式只写」（如驱动 LED），**不是能读回的标准 I2C**，测法也接近普通推挽单端信号，不在本篇 I2C 主线重点。

![图4：I2C 速率档阶梯。标准 Sm(100k·开漏上拉·t_r≤1000ns·最兼容)→快速 Fm(400k·开漏上拉·t_r≤300ns·最常用)→快速+ Fm+(1M·强驱动 20mA·t_r≤120ns)→高速 Hs(3.4M·电流源有源上拉·需主机码·t_r 80/160ns)→超快 UFm(5M·推挽单向无 ACK·只写不读非标准)。底部：①电平按 Vdd 派生非固定——V_IL≤0.3Vdd、V_IH≥0.7Vdd（3.3V 要求高电平≥2.31V，1.8V 器件够不到→必须电平转换）②低电平须 V_OL≤0.4V@3mA③越往上越吃上升时间：速率越高 t_r 上限越紧，越考验上拉与总线电容④Hs 用得少、Fm/Fm+ 覆盖绝大多数场景。都归 NXP UM10204](/images/51/ill04_versions.png)

## 四、独特之处：开漏上拉 RC + 单端无 CDR + 共享寻址 + 时钟拉伸

I2C 有四处高速接口没有、直接影响测试的独特设计：

- **① 开漏 + 上拉，上升时间 = RC**：这是 I2C 最独有的测试点。**t_r ≈ 0.8473 × Rp × Cb**——上升沿快慢不是芯片决定，是你板上那颗上拉电阻和总线总电容决定的。所以 I2C 调试里，**「量上升时间、算 / 换上拉电阻、量总线电容」是家常便饭**，这在推挽驱动的高速接口里根本不存在（那边是驱动强度、去加重）。选上拉：**Rp 最大值**由**上升时间**卡（Rp_max ≈ t_r /（0.8473 × Cb）），**Rp 最小值**由**低电平灌流**卡（Rp_min ≈ (Vdd − V_OL) / I_OL，Sm/Fm 的 I_OL 取 3mA、Fm+ 取 20mA）。常见取值 **1kΩ~4.7kΩ**，Cb 大就往小挑。

- **② 单端、双向、无 CDR、无眼图**：SCL / SDA 都是**单端对地**的电平，**没有差分、没有时钟恢复、没有眼图模板**。测它用**普通示波器看两条方波**即可，关心的是**边沿（t_r / t_f）、高低电平（V_OH / V_OL）与门槛裕量（对 0.7Vdd / 0.3Vdd）、过冲 / 振铃**，而不是抖动直方图和眼高眼宽。

- **③ 多设备共享 + 地址寻址 + wired-AND**：一对线挂多颗芯片，靠**地址点名**、**线与**共存。带来两个独特测点：**总线电容 Cb 会随挂载数量增长**（要测、要控制在 400pF 以内）；**地址冲突 / 地址错**会导致点名不应答（协议层要解码地址核对）。

- **④ 时钟拉伸（clock stretching）**：从机拖住 SCL 让主机等，是 I2C 独有的**接收方反卡时钟**机制。测试 / 调试时要专门看：**从机有没有拉伸、拉伸多久、主机认不认**——很多「偶发读错、超时」就是主机不容忍拉伸，或者拉伸太久拖垮了实时性。

![图5：I2C 四大独特。①开漏+上拉上升时间=RC：t_r≈0.8473×Rp×Cb，上升沿由板上上拉电阻 Rp 和总线电容 Cb 决定不是芯片——量上升时间/换上拉/量总线电容是家常便饭（高速推挽接口没有）。选上拉：Rp_max 由上升时间卡（≈t_r/0.8473Cb）、Rp_min 由低电平灌流卡（≈(Vdd−V_OL)/I_OL，Sm/Fm 取 3mA、Fm+ 取 20mA），常见 1k~4.7kΩ。②单端双向无 CDR 无眼图：SCL/SDA 单端对地电平，普通示波器看两条方波，关心边沿 t_r/t_f、高低电平 V_OH/V_OL 与门槛裕量、过冲振铃，不是抖动眼图。③多设备共享+地址寻址+wired-AND：一对线挂多片靠地址点名+线与，总线电容随挂载增长要控 400pF 内，地址冲突/错→点名不应答。④时钟拉伸：从机摁住 SCL 让主机等（接收方反卡时钟，独有），要看拉伸多久/主机认不认，偶发读错超时常在此](/images/51/ill05_unique.png)

## 五、测什么 · 三类：信号电气、时序、协议解码

I2C 的测试，也能归成三大类，**但主角是上升时间 + 电平 + 协议帧，不是眼图抖动**：

1. **信号电气（Signal / Electrical）**：用示波器看 SCL / SDA 两条模拟波形，量：**上升时间 t_r、下降时间 t_f、高电平 V_OH、低电平 V_OL、对门槛（0.7Vdd / 0.3Vdd）的裕量、过冲 / 下冲 / 振铃、总线电容 Cb（可由 t_r 反算或直接测）**。这一类直接暴露「上拉选得对不对、挂载电容大不大、电平域匹不匹配」。

2. **时序（Timing）**：量 SCL / SDA 之间的时间关系，对 NXP 规范表：**时钟频率 f_SCL、高电平时间 t_HIGH / 低电平时间 t_LOW、数据建立 t_SU;DAT / 保持 t_HD;DAT、START 建立 / 保持 t_SU;STA / t_HD;STA、STOP 建立 t_SU;STO、总线空闲 t_BUF**。时序不满足，采样点可能落在数据跳变上，读错位。

3. **协议解码（Protocol）**：用示波器的 **I2C 触发 / 解码**功能，或**逻辑分析仪**，把波形翻译成**帧**：**START / STOP / 重复起始、地址 + R/W、每字节的 ACK / NACK、数据内容、时钟拉伸**。这一类回答「读不到（第 9 拍无 ACK？地址错？）、读错值、总线卡死（SDA 被谁一直拉低）」这类真问题。

贯穿三类的共同度量，全是**低速单端**的东西：上升 / 下降时间、电平与裕量、setup / hold 时序、地址 / ACK / 帧——**没有眼图、没有抖动分离、没有去嵌、没有 BER 外推**。这和机内高速串行「抖动是天王山、要去嵌要 BERT」正好是两个世界。

![图6：I2C 测什么三类（对照高速：主角是上升时间+电平+协议帧，不是眼图抖动）。①信号电气：示波器看 SCL/SDA 两条模拟波形，量上升时间 t_r、下降时间 t_f、高电平 V_OH、低电平 V_OL、对门槛 0.7Vdd/0.3Vdd 的裕量、过冲下冲振铃、总线电容 Cb——暴露上拉选得对不对/挂载电容大不大/电平域匹不匹配。②时序：量 SCL/SDA 时间关系对 NXP 表——时钟频率 f_SCL、t_HIGH/t_LOW、数据建立 t_SU;DAT/保持 t_HD;DAT、START 建立保持 t_SU;STA/t_HD;STA、STOP 建立 t_SU;STO、总线空闲 t_BUF。③协议解码：示波器 I2C 触发解码或逻辑分析仪翻成帧——START/STOP/重复起始、地址+R/W、每字节 ACK/NACK、数据、时钟拉伸，回答读不到/读错/卡死。共同：全是低速单端度量，无眼图/抖动分离/去嵌/BERT](/images/51/ill06_test3.png)

## 六、怎么测 · 台面：普通示波器 + 协议解码 / 逻辑分析仪 + 量上拉与电容

I2C 的测试台面，比高速接口**朴素得多**——**不用一致性软件、不用去嵌、不用 BERT、不用高带宽夹具**，一台普通示波器加个逻辑分析仪就能干大半：

- **① 直接探 SCL / SDA**：I2C 频率低（几百 kHz 到几 MHz），**普通带宽示波器 + 无源探头**直接探**上拉电阻附近或从机引脚**的 SCL、SDA 即可，不需要专用夹具、不需要去嵌校准。两个通道同时抓 SCL、SDA，才能量它们之间的时序。

- **② 量上升 / 下降时间与电平**：示波器上量 **t_r（10%→90%）、t_f、V_OH / V_OL**，对着规范档位的限值（如 Fm 的 t_r ≤ 300ns）和门槛（0.7Vdd / 0.3Vdd）看裕量。**上升沿明显偏慢**，八成是上拉太弱或 Cb 太大。

- **③ 量 / 估总线电容、试上拉**：可以**换上拉电阻观察 t_r 变化反算 Cb**（t_r ≈ 0.8473 × Rp × Cb），或直接用 LCR / 电容表量总线对地电容，确认在 **400pF 以内**。调试时**换一颗更小的上拉**（如 10k→2.2k）看问题消不消失，是最快的判定手段——这也是「换个上拉就好了」的由来。

- **④ 协议解码 / 逻辑分析仪**：打开示波器的 **I2C 解码**（设好 SCL/SDA 通道、地址位宽、阈值），或接**逻辑分析仪**，把波形翻成 **地址 + R/W + ACK/NACK + 数据 + START/STOP**。**读不到从机**就看第 9 拍有没有 ACK、地址对不对；**总线卡死**就看是 SCL 还是 SDA 被一直拉低、谁拉的。

- **⑤ 电平域 / 供电检查**：确认**主从两端 Vdd 一致或有电平转换**（V_IH ≥ 0.7Vdd 能不能满足），确认**上拉接的是哪个电源轨、上电时序对不对**（呼应模块二 PI——上拉接错轨或上电顺序不对，会导致上电瞬间总线异常）。

![图7：测 I2C 的台面（比高速朴素得多：不用一致性软件/去嵌/BERT/高带宽夹具）。①直接探 SCL/SDA：频率低，普通带宽示波器+无源探头探上拉附近或从机引脚，两通道同抓才能量时序，不需专用夹具/去嵌。②量上升下降时间与电平：t_r(10→90%)/t_f/V_OH/V_OL 对规范限值(如 Fm t_r≤300ns)和门槛(0.7/0.3Vdd)看裕量，上升偏慢=上拉弱/Cb 大。③量估总线电容试上拉：换上拉看 t_r 变化反算 Cb（t_r≈0.8473RpCb）或直接量，确认 400pF 内；换小上拉(10k→2.2k)看问题消不消是最快判定。④协议解码/逻辑分析仪：示波器 I2C 解码或逻辑分析仪翻成地址+R/W+ACK/NACK+数据+START/STOP，读不到看第 9 拍 ACK/地址，卡死看谁拉低。⑤电平域/供电检查：主从 Vdd 一致或电平转换、上拉接对电源轨、上电时序（呼应模块二 PI）](/images/51/ill07_bench.png)

## 七、判据：NXP UM10204 那张表 —— 直接查表比对，不是眼图模板反推

I2C 的判据，全部来自 **NXP UM10204《I²C-bus specification and user manual》** 的**时序与电气特性表**——而且和高速接口很不一样：**它是「查表比对」，不是「从信道预算反推眼图模板」**。核心就这几块：

- **电平**：V_IL ≤ 0.3 × Vdd、V_IH ≥ 0.7 × Vdd、V_OL ≤ 0.4V（3mA 灌流）——**电平和门槛裕量**直接查表比。
- **边沿**：上升时间 **t_r**（Sm ≤ 1000ns / Fm ≤ 300ns / Fm+ ≤ 120ns）、下降时间 **t_f**（Sm ≤ 300ns / Fm ≤ 300ns / Fm+ ≤ 120ns）——**这条最常超**，超了就是上拉 / 电容问题。
- **时序**（以 Sm / Fm 为例）：数据建立 **t_SU;DAT ≥ 250 / 100ns**、数据保持 **t_HD;DAT ≥ 0**、START 保持 **t_HD;STA ≥ 4000 / 600ns**、STOP 建立 **t_SU;STO ≥ 4000 / 600ns**、总线空闲 **t_BUF ≥ 4700 / 1300ns**、SCL 低 **t_LOW ≥ 4700 / 1300ns** / 高 **t_HIGH ≥ 4000 / 600ns**——**逐项对表**。
- **上拉与电容**：**Cb ≤ 400pF**（Sm/Fm/Fm+），上拉 **Rp** 取值同时满足「上升时间上限」和「低电平灌流下限」两个约束。

和高速接口最大的不同：机内串行的眼图模板、抖动预算，是**从信道损耗反推出来的一条曲线**；I2C 的判据就是**规范里印死的一张时序表 + 一组电平门槛**，你拿示波器量到的数**逐格对比**，超没超一目了然——**简单、确定，但一个都不能漏**。

## 八、常见翻车 → 分层揪真凶（别一上来就怀疑芯片坏）

I2C 的「读不到、掉速、卡死、时好时坏」，最忌讳一上来就换芯片、重写驱动。正确姿势还是**分层**：

- **① 电气 / 上拉层（最高频）**：**上拉太弱或 Cb 太大 → t_r 超标 → 高速档认不出高电平**，表现为「降到 100k 就好、上 400k 就错」「多挂几片就掉速」；**上拉太强 / 灌流不够 → V_OL 拉不下去 → 逻辑 0 认成 1**；**根本没接上拉 → 线一直低 / 浮空 → 读全 0xFF 或 0x00**。先量 t_r、V_OL，试换上拉——大半问题死这层。

- **② 电平域层**：**1.8V 域和 3.3V 域直接混接、没做电平转换 → V_IH 够不到（3.3V 要 ≥2.31V，1.8V 给不了）**，表现为「单独测都对、一接到一起就读不到 / 偶发错」。查两端 Vdd、上拉接的电源轨、有没有电平转换芯片。

- **③ 协议层**：**地址填错 / 地址冲突（两颗芯片同地址）→ 点名无 ACK → 读不到**；**主机不支持时钟拉伸、从机偏偏拉伸 → 超时、误读**；**总线卡死（SDA 被某从机一直拉低，常因之前一次传输被打断、从机状态机卡在半路）→ 全总线哑**，解法是主机**发 9 个时钟脉冲 + 一个 STOP 把从机顶出来**。用解码 / 逻辑分析仪定位到底哪一帧、哪一位出错。

- **④ 负载 / 布线层**：**挂太多设备、走线太长 → Cb 超 400pF → 波形圆钝、上升慢 + 串扰**，表现为「样机好好的、批量或长线版本就出问题」。测总线电容、精简挂载 / 缩短走线 / 上总线缓冲（I2C buffer / 中继器）。

**口诀**：先量上升时间和低电平、试换上拉（电气 / 上拉）→ 查电平域和电源轨 → 解码看地址 / ACK / 拉伸 / 谁拉死总线（协议）→ 最后看挂载电容和走线（负载）。

![图8：I2C 翻车分层（别一上来换芯片/重写驱动）。①电气/上拉层（最高频）：上拉太弱/Cb 太大→t_r 超标→高速档认不出高电平（降 100k 就好/多挂几片掉速）；上拉太强/灌流不够→V_OL 拉不下→0 认成 1；没接上拉→线一直低/浮空→读全 0xFF/0x00。先量 t_r、V_OL 试换上拉。②电平域层：1.8V/3.3V 混接没电平转换→V_IH 够不到（3.3V 要≥2.31V）→单独对一起就错，查两端 Vdd/电源轨/电平转换。③协议层：地址错/冲突→无 ACK→读不到；主机不支持拉伸而从机拉伸→超时误读；总线卡死 SDA 被拉死→发 9 时钟脉冲+STOP 顶出来。解码/逻辑分析仪定位到帧到位。④负载/布线层：挂太多/走线长→Cb 超 400pF→波形圆钝上升慢+串扰（样机好批量出问题），测电容/精简挂载/上 I2C buffer。口诀：量上升时间低电平试上拉→查电平域电源轨→解码看地址 ACK 拉伸谁拉死→看挂载电容走线](/images/51/ill08_debug.png)

## 一句话总结

**I2C 是我们从高速接口一致性转入低速板级总线的第一站：它是「单端开漏 + 上拉、时钟单独一根 SCL、多设备挂同一对线共享寻址、还能时钟拉伸」的低速两线总线，把前面高速那套差分 / 眼图 / CDR / 抖动整个反了过来。它最灵魂的一件事，是那两颗上拉电阻——上升时间 t_r ≈ 0.8473 × 上拉电阻 × 总线电容，上拉太弱或挂载电容太大就上升慢、认不出高电平、掉速误读，上拉太强又拉不下低电平、把 0 认成 1，所以选上拉是在「上升够快」和「低电平够低」之间卡窗口，Rp 最大值由上升时间定、最小值由灌流定，常见 1k~4.7kΩ。它测什么三类：信号电气（上升下降时间、V_OH/V_OL、门槛裕量、总线电容）、时序（对 NXP UM10204 那张表逐项比 t_SU;DAT / t_HD;STA / t_LOW / t_HIGH / t_BUF）、协议（示波器 I2C 解码或逻辑分析仪看地址 + R/W + 第 9 拍 ACK/NACK + START/STOP + 时钟拉伸）。台面朴素——普通示波器探 SCL/SDA + 协议解码 + 换上拉试，不用一致性软件、不用去嵌、不用 BERT。判据是查规范表比对（电平按 Vdd 派生 0.3/0.7、t_r 各档上限、Cb ≤ 400pF），不是从信道预算反推眼图模板。调试永远分层：先量上升时间和低电平、试换上拉，再查电平域和电源轨，再解码看地址 / ACK / 拉伸 / 谁拉死总线，最后看挂载电容和走线。**

| **测三类** | ①信号电气（t_r/t_f/V_OH/V_OL/Cb）②时序（t_SU;DAT / t_HD;STA / t_LOW / t_HIGH / t_BUF 对表）③协议解码（地址/ACK/START/STOP/拉伸） |
| **怎么测** | 普通示波器探 SCL/SDA + I2C 触发解码 / 逻辑分析仪 + 换上拉试 + 量总线电容 + 电平域 / 电源轨检查；不用去嵌 / 一致性软件 / BERT |
| **判据** | NXP UM10204 时序 + 电平表**直接查表比对**（不是从信道预算反推眼图模板） |
| **调试分层** | 上拉 / 电气（t_r、V_OL、试换上拉）→ 电平域（1.8/3.3V、电源轨）→ 协议（地址 / ACK / 拉伸 / 谁拉死总线）→ 负载（Cb、走线） |
| **一句话** | 从高速「差分看眼图靠 CDR」走到低速「单端量上升时间 + 查表比时序 + 解码看帧」，灵魂在两颗上拉电阻 |
