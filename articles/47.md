# 同一块 Gen5 SSD，插主板这个槽跑满、插那个槽却掉到 Gen1——PCIe：一致性测试里把「嵌入参考封装 + 参考均衡」玩到极致的接口，到 Gen6 还第一个跳进了 PAM4 三只眼

先说个装机、做服务器的人几乎都踩过的坑。

你买了一块标称 **Gen5** 的固态硬盘，跑分软件一测傻眼了：插主板**靠 CPU 那个 M.2 槽**，顺序读能到**每秒十几个 GB**；插到**主板边上那个槽**，却只有**每秒几个 GB**，一查协商速率，竟然掉到了 **Gen3、甚至 Gen1**。再比如你插一块**显卡 / AI 加速卡**，设备管理器里明明是 x16 的卡，一看却跑在 **x4、x8**，或者干脆**偶发识别不到、蓝屏、系统日志里一堆 PCIe 报错（AER）**。

很多人以为是 SSD 坏了、卡坏了。但做硬件的人清楚：**这类「掉速、掉 lane、识别不到、偶发蓝屏」里，有一大半根本不是设备坏了，而是 PCIe 这条高速链路——信号眼图、链路训练、参考均衡、或者插槽到设备那段通道——没测好、没协商好。** 同一块 SSD 插不同槽速度天差地别，往往就是因为不同槽的**走线长度、过孔、连接器**不一样，通道损耗一大，链路训练就只能回落到低速档保命。

这条总线怎么从 Gen1 一路翻到 Gen6、它的眼图到底怎么测、为什么 PCIe 的测法比谁都「重」——这，就是这一篇要讲的。

## 一、先认清主角：PCIe 是什么、你主板上到处都是它

**PCIe（PCI Express，高速外设互连）** 是当今 PC、服务器、工作站里最主流的**板级点对点高速总线**。你主板上那些插槽和接口，绝大多数都是它：

- **显卡插槽**（x16）、**M.2 / U.2 固态硬盘**、**网卡 / 声卡 / 采集卡**、**服务器背板上的 SSD / AI 加速卡 / 智能网卡**——全走 PCIe。
- 结构上它是**点对点**的：一端是 **Root Complex（根复合体，通常在 CPU / 芯片组里）**，另一端是 **Endpoint（端点设备，比如那块 SSD 或显卡）**，中间可以经过 **Switch（交换器）** 扩展。

PCIe 有三个关键特征，正好对应这一篇的测试主线：

1. **串行差分 + 嵌入式时钟 → 靠 CDR**：每条 lane 是一对差分线（收发各一对，全双工），**没有独立时钟线**，时钟埋在数据里，接收端用 CDR 恢复——**所以要看眼图**，和 USB3 / DP / UFS 完全同岸。
2. **多 lane 叠加（x1 / x4 / x8 / x16）**：把多条 lane 并起来堆带宽，x16 就是 16 条 lane 一起上；链路训练时会**协商**用几条 lane、跑哪一代。
3. **另有一根参考时钟（refclk）**：除了数据里的嵌入式时钟，PCIe 还给两端的 PLL 喂一根 **100 MHz 参考时钟**——这根 refclk 怎么给（共享 / 独立），后面会讲，是 PCIe 独有的一道题。

一句话：**PCIe = 主板上连显卡 / 固态 / 网卡 / AI 卡的那条串行差分高速总线**；数据靠 CDR、天生站在「看眼图」这一岸。

## 二、原理与演进：嵌入式时钟看眼图，编码从 8b/10b 到 128b/130b 再到 PAM4

### 2.1 数据链路：嵌入式时钟 + CDR（和 USB3 / DP / UFS 同岸）

PCIe 的每条 lane 都是**串行差分**，**没有转发时钟线**，时钟信息编在数据里，接收端靠 **CDR（时钟数据恢复，↔第 32 篇）** 从数据边沿把时钟捞出来。所以它天生站在 q02 的「**嵌入式时钟 / 看眼图**」这一岸——和 USB3、DP、UFS 的 M-PHY 一模一样。测的是**眼图 + 抖动 + 参考均衡**，不是 setup/hold 时序。

### 2.2 编码三代：从 8b/10b，到 128b/130b，再到 PAM4

PCIe 的编码方式，随着代数换过三次，这是理解它速率和测法的关键：

- **Gen1 / Gen2 = 8b/10b**：每 8 bit 用 10 bit 传，**20% 开销**（和 USB3 Gen1、UFS 一样）。
- **Gen3 / Gen4 / Gen5 = 128b/130b**：每 128 bit 只加 2 bit 头，**开销降到约 1.5%**——这就是为什么 Gen3（8 GT/s）的有效带宽差不多是 Gen2（5 GT/s）的两倍还多。
- **Gen6 = PAM4 + FLIT + FEC**：见下一节，这是 PCIe 最大的一次跳变。

### 2.3 参考时钟架构：共享 / 独立 + 扩频（PCIe 独有的一道题）

除了数据里的嵌入式时钟，PCIe 还有一根 **100 MHz 参考时钟（refclk）** 喂两端 PLL。它怎么给，分三种，直接影响 CDR 和一致性测试：

- **Common Refclk（共享参考时钟）**：两端用同一根 refclk，最常见（主板插槽就是这样）。两端时钟同源，CDR 压力小。
- **SRIS（独立参考时钟 + 各自扩频）**：两端各用各的 refclk，都开扩频。用于**线缆 PCIe**（如外接 SSD、扩展坞）——板子分开、没法共享时钟时用，**对 CDR 要求更高**。
- **SRNS（独立参考时钟、不扩频）**：类似 SRIS 但不开扩频。

其中**扩频时钟（SSC）** 是为降 EMI 用的：把 refclk 在 ±0.5% 的窄带里轻微抖动，把能量摊开、降低辐射峰值。**测抖动时，示波器的时钟恢复（golden PLL）必须跟得住 SSC 的调制**，否则会把 SSC 误当成抖动测进去（↔第 32 篇）。

![图3：PCIe 三大原理骨架。上区块 数据链路：串行差分、无转发时钟、时钟埋数据靠 CDR（↔32）→"嵌入式时钟/看眼图"岸，和 USB3/DP/UFS 同路，测眼图+抖动+参考均衡。中区块 编码三代：Gen1/2=8b/10b（20% 开销）→ Gen3/4/5=128b/130b（约 1.5% 开销，效率大涨）→ Gen6=PAM4+FLIT+FEC（下一节）。下区块 参考时钟架构（PCIe 独有）：另有一根 100MHz refclk 喂两端 PLL——Common Refclk 共享（主板插槽,CDR 压力小）/ SRIS 独立+各自扩频（线缆 PCIe,CDR 要求高）/ SRNS 独立不扩频；扩频 SSC ±0.5% 降 EMI，示波器 golden PLL 必须跟得住 SSC 否则误当抖动。底部一句：PCIe 是"看眼图"岸最极致的选手](/images/47/ill03_arch.png)

## 三、版本与速率：从 Gen1 2.5 GT/s 到 Gen6 64 GT/s，每代翻倍

PCIe 的速率用 **GT/s（每秒吉传输）** 表示，几乎**每代翻倍**。主流版本对应关系（单向、每 lane）：

| 版本 | 线速率 | 编码 | 每 lane 有效带宽 | x16 单向带宽 |
|---|---|---|---|---|
| Gen1 | 2.5 GT/s | 8b/10b | ≈ 0.25 GB/s | ≈ 4 GB/s |
| Gen2 | 5.0 GT/s | 8b/10b | ≈ 0.5 GB/s | ≈ 8 GB/s |
| Gen3 | 8.0 GT/s | 128b/130b | ≈ 1.0 GB/s | ≈ 15.75 GB/s |
| Gen4 | 16 GT/s | 128b/130b | ≈ 2.0 GB/s | ≈ 31.5 GB/s |
| Gen5 | 32 GT/s | 128b/130b | ≈ 4.0 GB/s | ≈ 63 GB/s |
| Gen6 | 64 GT/s | PAM4 + FLIT + FEC | ≈ 8 GB/s | ≈ 128 GB/s |

要点：

- **GT/s 不等于 GB/s**：要先扣编码开销、再除以 8、再乘 lane 数。Gen3 起换 128b/130b，所以「GT/s 数字 ÷ 8」就差不多是每 lane 的 GB/s。
- **Gen6 的 64 GT/s 是「假翻倍」**：它的**波特率还是 32 Gbaud**（和 Gen5 一样），靠 **PAM4 每符号 2 bit** 把比特率翻上去——这样才不至于让通道带宽需求也跟着翻倍（否则主板根本扛不住）。这和示波器带宽选型的道理一致：**PAM4 用「一个符号两比特」换「不再拉高波特率」**。
- **x16 到 Gen5 已是每秒 63 GB、Gen6 达 128 GB**：这已经是消费和服务器主板上最快的板级总线，AI 加速卡、高端 SSD 全靠它喂数据。

![图4：PCIe 版本与速率阶梯（每代翻倍）。横向阶梯：Gen1 2.5GT/s(8b/10b,0.25GB/s/lane,x16≈4GB/s)→Gen2 5.0(8b/10b,0.5,x16≈8)→Gen3 8.0(128b/130b,~1.0,x16≈15.75)→Gen4 16(128b/130b,~2.0,x16≈31.5)→Gen5 32(128b/130b,~4.0,x16≈63)→Gen6 64GT/s(PAM4+FLIT+FEC,~8,x16≈128)。每档标编码方式虚线框，8b/10b(20%)/128b/130b(1.5%)/PAM4 分色。右端强调框：Gen6 64GT/s 是"假翻倍"——波特率仍 32Gbaud，靠 PAM4 每符号 2bit 翻比特率，不拉高通道带宽需求(主板扛不住)。底部一句：GT/s≠GB/s，先扣编码再÷8再×lane；x16 到 Gen6 达 128GB/s，AI 卡/高端 SSD 全靠它](/images/47/ill04_versions.png)

## 四、独特主角：Gen6 PAM4——从 NRZ 单眼，跳进三只眼 + 强制 FEC

如果说前面几篇的串行接口都在 **NRZ（两电平：0 和 1，一只眼）** 里翻速率，那 **PCIe Gen6 就是主流串行里第一个跳进 PAM4** 的——这是这一篇最该带走的反差。

- **NRZ（Gen1~Gen5）**：每个符号只有高 / 低两个电平，示波器上叠出**一只眼**，判据就是那只眼睁得够不够大。
- **PAM4（Gen6）**：每个符号有**四个电平（00 / 01 / 10 / 11）**，一个符号携带 **2 bit**，示波器上叠出**上、中、下三只眼**。好处是波特率不用翻倍就把比特率翻倍；代价是**四个电平挤在同样的电压范围里，每只眼都比 NRZ 小得多、信噪比骤降**。
- **所以 Gen6 强制上 FEC（前向纠错）**：PAM4 眼小、误码率天然高，光靠信号裕量保不住，必须在链路里加 **FEC** 边传边纠错，把误码率压回目标；同时改用 **FLIT（固定大小流控单元）** 模式配合 FEC。
- **测法跟着变**：Gen6 的一致性要**同时看三只眼**，还要评估 **PAM4 特有的电平失真（如 RLM 电平分离度）**、以及 FEC 之前的原始误码率——比 NRZ 单眼复杂了一个量级。

一句话：**PCIe Gen6 = 主流串行第一次从「一只眼 NRZ」跳到「三只眼 PAM4」，代价是眼变小、信噪比降，只能靠强制 FEC 边传边纠。** 这也是后面高速接口（未来的 UALink、CXL、乃至 224G 以太网）都在走的路——PCIe 只是先趟了。

![图5：Gen6 PAM4——从 NRZ 单眼跳三只眼+强制 FEC。左：NRZ(Gen1~5) 两电平 0/1、示波器一只眼、判眼睁够不够大。中：PAM4(Gen6) 四电平 00/01/10/11、每符号 2bit、示波器上中下三只眼；标"波特率不翻(仍 32Gbaud)、比特率翻倍；但四电平挤同样电压范围→每只眼更小、信噪比骤降"。右：所以 Gen6 强制 FEC 前向纠错(眼小误码高,边传边纠压回目标)+FLIT 固定流控单元配合。底部强调：Gen6 一致性要同时看三只眼+评 PAM4 电平失真(RLM 电平分离度)+看 FEC 前原始误码率，比 NRZ 单眼复杂一个量级。一句话：PCIe 先趟了 PAM4 这条路(未来 CXL/UALink/224G 以太网都要走)](/images/47/ill05_pam4.png)

## 五、测什么·三类：Tx 眼图（嵌参考封装）、Rx 压力（嵌参考均衡）、链路 / 协议

PCIe 的一致性测试，和 USB3 / DP 一样分三大阵营，但它把「**嵌入参考模型**」这一招用到了极致：

1. **Tx / 发送端眼图**：示波器在规定测试点（如 **TP2**）抓差分眼图，但 PCIe 独特的一步是——**要先把一个「参考封装 + 参考通道」的损耗数学嵌进去**，模拟信号到达接收端芯片引脚时的样子，再套眼图模板判读。还要验各 **preset（预设的去加重 / 预加重档，Gen5 有 11 档 P0~P10）** 下的眼。
2. **Rx / 接收端压力测试（jitter tolerance）**：用码型发生器造**校准过的压力眼**（把损耗、抖动、串扰加到规范最坏值），灌给被测接收端，测它的**抖动容限 / 误码率**（↔第 41 篇）。PCIe 这里的关键是：判据是「**接收端嵌入一个行为级 DFE（参考均衡器）之后**」还能不能正确恢复——这就是 README 里说的「**嵌 reference package + DFE**」。
3. **链路 / 协议（LTSSM + 均衡协商）**：PCIe 的链路训练由状态机 **LTSSM（链路训练与状态机）** 驱动，从检测、轮询、配置一路到 L0（正常工作）；**Gen3 起多了专门的「均衡阶段」，两端在这里协商 preset、调 TX 去加重和 RX 均衡系数**，训到最佳眼。测协议要用 **PCIe 协议分析仪** 看 LTSSM 走到哪、均衡协商成不成、有没有回落。

此外，**通道 / CEM（板级）** 也要测：PCIe 有专门的 **CEM（板卡机电规范）**，用 **CBB（一致性基板）+ CLB（一致性负载板）** 这套治具，模拟最坏情况的主板 / 板卡通道，验插槽到设备那段的插损 / 回损 / 串扰。

![图6：PCIe 一致性测试三阵营（把"嵌参考模型"玩到极致）。左 Tx/发送端：示波器在 TP2 抓差分眼→独特一步：先数学嵌入"参考封装+参考通道"损耗模拟到接收芯片引脚的样子→套眼图模板；验各 preset(Gen5 有 P0~P10 共 11 档去加重/预加重)。中 Rx/接收端压力(jitter tolerance)：码型发生器造校准压力眼(loss+jitter+串扰加满)灌被测端→判据是"接收端嵌行为级 DFE 参考均衡后"能否正确恢复=嵌 reference package+DFE(↔41)。右 链路/协议：LTSSM 链路训练(检测→轮询→配置→L0)，Gen3 起有均衡阶段协商 preset/调 TX 去加重+RX 均衡；PCIe 协议分析仪看走到哪/回落没。底部：通道/CEM 用 CBB 基板+CLB 负载板模拟最坏主板/板卡通道测插损回损串扰](/images/47/ill06_test3.png)

## 六、怎么测：进 compliance 模式 + 打 compliance pattern + 嵌参考封装 / 参考均衡

PCIe 的台面流程，和 USB3 / DP 那套「进测试模式 → 抓眼图 → 套模板」同源，但多了 PCIe 专属的几步：

- **进 compliance 模式**：PCIe 有个巧妙的机制——把设备插进 **compliance 测试槽**、或在链路训练时故意**让链路进不到 L0**，设备就会自动进入 **Polling.Compliance** 态，**循环输出规定的 compliance 测试码型**，并且可以**用热复位（toggle）在各 preset / 各代速率之间切换**，方便示波器逐档抓眼。
- **打 compliance pattern**：让被测端持续输出规范定义的一致性码型（含各 preset），示波器长时间累积眼图。
- **嵌参考封装 + 参考通道**：Tx 眼图判读前，把 **PCI-SIG 提供的参考封装 / 参考通道 S 参数**用软件嵌进测量，还原信号到接收端芯片引脚处的样子。
- **参考均衡 CTLE + 行为级 DFE**：Gen3 起，判据都是「**参考均衡之后**」的眼——把 **CTLE + DFE（↔第 34 篇）** 按规范系数挂上，把闭着的眼睁开再判。Gen4 / Gen5 通道损耗极大，**裸眼图基本全闭**，全靠这套参考均衡。
- **Gen6 三只眼**：PAM4 要同时抓上中下三只眼，评电平分离度和 FEC 前误码率。

> 一句话：**PCIe 的仪器设置 = USB3 / DP 那套嵌入式时钟测法，再叠上「嵌参考封装 + 嵌行为级 DFE」这道 PCIe 独门工序。** 你在第 41 / 44 篇学的抖动分离、参考均衡、golden PLL，这里全能复用，只是嵌的模型更讲究。

## 七、⚙️ 关键仪器设置：把模块三的 SI 概念落到 PCIe 旋钮上

PCIe 的仪器设置，是这一岸各接口里**最讲究**的一套：

- **嵌参考封装 / 参考通道（embed）↔第 33 篇的逆操作**：和「去嵌」相反，这里是**故意把规范的参考封装 / 通道损耗嵌进去**，模拟信号到接收端引脚的样子——这是 PCIe 判 Tx 眼图的前提。
- **参考均衡 CTLE + DFE↔第 34 篇**：按 PCI-SIG 给的参考均衡系数挂，判据是「均衡后」的眼；Gen4 / Gen5 必用。
- **时钟恢复 golden PLL + 跟 SSC↔第 32 篇**：用规范的黄金 PLL 环路恢复时钟，且必须跟得住扩频（SSC）的调制，否则把 SSC 误测成抖动。
- **带宽按波特率选↔第 29 / 速查 02**：按线速率约 0.7~1 倍选示波器带宽（Gen5 32 GT/s → 用 33 GHz 级示波器；Gen6 PAM4 仍 32 Gbaud，带宽需求没跟着翻倍——这正是 PAM4 的用意）。
- **抖动分离 + eye mask↔第 31 / 32 篇**：TJ 拆 RJ / DJ，套 PCIe 眼图模板；Gen6 要拆三只眼、算电平分离度。

> 一句话：**PCIe 把「嵌模型 + 参考均衡」做到了极致——判的从来不是你示波器上看到的裸眼，而是「嵌了参考封装、过了参考均衡之后」那只（Gen6 是三只）眼。**

## 八、判据从哪来：PCI-SIG 的 Base Spec + CEM Spec + 一致性测试规范

和第 38 篇讲的一样，PCIe 的判据来自标准的一致性测试规范，且由 **PCI-SIG（PCIe 标准组织）** 统一维护：

- **Base Specification（基础规范）**：定义各代速率、编码、LTSSM、均衡、电气参数、眼图模板、抖动预算——**芯片 / SerDes 层面**的符合性。
- **CEM Specification（板卡机电规范）**：定义**插槽 / 板卡（add-in card）** 的机械 + 电气要求，配 **CBB / CLB** 治具测板级通道——**板卡 / 主板层面**的符合性。
- **一致性测试规范（CTS）+ Integrators List**：PCI-SIG 定期办 **Compliance Workshop（一致性大会）**，产品测过一致性套件、进 **Integrators List（集成者名单）**，才算官方认证「能和全世界的 PCIe 设备互通」。

所以 PCIe 判据体系是**分层的**：**电气 / SerDes 归 Base Spec，板卡 / 插槽归 CEM Spec**——和 DP（PHY CTS + Link Layer CTS）、UFS（M-PHY CTS + JEDEC UFS）结构上完全一个套路。眼图模板 / 抖动预算，同样是从**信道损耗 + 抖动预算**反推而来（↔第 38 篇）。

## 九、常见翻车与调试：掉速到 Gen1 / 掉 lane / 识别不到 / 蓝屏 AER，怎么分层揪

PCIe 翻车，**先分层、别一上来就赖信号**（顺序和 USB / DP / UFS 一样）：

1. **先看链路训练层（LTSSM / 代数 / lane 数）**：设备识别没？LTSSM 训到 L0 没？跑在第几代、几条 lane？——**「掉速到 Gen3 / Gen1」「x16 卡跑成 x4」多半是通道太差、均衡协商失败、回落了**，先查这里，跟裸眼图无关。
2. **再看电气层（眼图 / 参考均衡 / 通道）**：训到高代但偶发误码、AER 报错——量差分眼（嵌参考封装 + 参考均衡后）、量插槽到设备通道的插损 / 回损 / 串扰、看 preset 选得对不对、球脚 / 连接器阻抗是否连续。
3. **参考时钟层（refclk / SSC）**：抖动异常、SRIS 场景不稳——查 refclk 质量、SSC 调制是否合规、golden PLL 是否跟得住。
4. **协议 / 均衡协商层**：链路稳但性能上不去、偶发 AER——用协议分析仪查 LTSSM 均衡阶段、preset 协商、FLIT / FEC（Gen6）行为。
5. **供电 / 完整性层（PI，↔模块三）**：显卡 / SSD 大电流突发时纹波 / 掉压导致偶发错误——PCIe 大功率设备（显卡）PI 尤其关键。

**现象速查**：掉速到 Gen3/Gen1 = 通道差 / 均衡协商失败 / 回落；x16 跑成 x4 = lane 训练 / 连接器 / 通道；训到高代仍偶发 AER = 眼图 / 参考均衡 / 通道 skew；SRIS 场景不稳 = refclk / SSC / CDR；插不同槽差别大 = 那个槽的走线 / 过孔 / 连接器损耗；大负载才偶发蓝屏 = 供电 PI。

## 一句话总结

**一句话：PCIe 是主板上连显卡 / 固态 / 网卡 / AI 卡的板级串行差分高速总线，数据靠 CDR、站在「看眼图 + 参考均衡」这一岸——而且是这一岸最极致的选手。学会了 USB3 / DP / UFS，PCIe 你已经通了大半。**

- **最极致的玩法**：一致性测试独创「嵌入参考封装 + 嵌行为级 DFE 之后再判眼图」，判的从来不是裸眼。
- **速率 / 编码演进**：Gen1 2.5 GT/s 每代翻倍到 Gen6 64 GT/s；编码从 8b/10b 到 128b/130b；Gen6 更第一个跳进 PAM4 三只眼 + 强制 FEC。
- **别家没有的坎**：一道参考时钟架构题（共享 / 独立 + 扩频）。
- **调试先分层**：先看训到第几代、几条 lane，再量嵌了参考封装、过了参考均衡之后的眼。
