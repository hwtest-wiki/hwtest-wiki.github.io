# 建立时间与保持时间：信号电平明明没问题，为什么系统偶尔就采错、还查不出规律？

<div class="article-meta"><span class="badge">📘 模块四 · 时序与时钟 · 时序基础 / 建立时间 / 保持时间</span><span class="badge">⏱ 约 18 分钟</span><span class="badge qr">🔧 含工程师速查</span></div>


一块板子，某个数据信号电平完全正常——高电平够高、低电平够低，示波器抓下来波形干干净净、边沿也利落。可系统就是**偶尔出错**：跑一万次错一次，还**没法稳定复现**，换块板子可能又好了。你测电平、测幅度、测眼图，全都合格，查了一整天毫无头绪。

**这种"电平没毛病、却偶发出错、还查不出规律"的问题，八成不是信号质量的事，而是时序的事。** 具体说：这个数据**变化的时刻**，离时钟去采它的那个**边沿太近了**，踩进了一个"不许数据变化"的窗口。这个窗口，就是由**建立时间**和**保持时间**划出来的。

数字电路里，寄存器（触发器）是靠**时钟边沿**去"拍照"数据的。要想拍清楚，数据不能在快门按下的瞬间还在动——**快门前得稳一会儿（建立时间），快门后还得再稳一会儿（保持时间）**。这段时间里数据一动，拍出来就是糊的，甚至会让触发器陷入一种"既不是 0 也不是 1"的诡异状态。

**读完这篇，你能搞懂：**

- ✅ 建立时间、保持时间到底是什么——时钟沿前后那个"不许数据变化"的窗口；
- ✅ 违反了会怎样——**亚稳态**，触发器卡在中间态、可能采错还会往下传；
- ✅ 时序怎么算——**数据到达时间**、**裕量（slack）**，一个寄存器到另一个寄存器的时序检查；
- ✅ 一个关键区别——**建立违规可以降频救，保持违规降频救不了**（新手最容易搞反）；
- ✅ 怎么测、怎么保证——STA 静态时序分析、示波器实测、总线 spec、跨时钟域同步器；
- ✅ **什么时候该验时序、判据从哪查**——器件 datasheet 的 Tsu/Th ＋ 总线规范（DDR/SPI/I2C），盯 STA 的 slack 余量；
- ✅ **偶发采错怎么定位**——碰禁区 → 是建立还是保持、要不要查跨时钟域，一张流程图带走。

## 一、建立时间与保持时间：时钟沿前后那个"不许动"的窗口

先把主角认清。数字系统里到处都是**触发器（D 触发器，DFF）**——它在**时钟的有效边沿**（通常是上升沿）把输入 D 的值"锁"进去、送到输出 Q。你可以把它想成一个**在特定时刻按快门的相机**。

要拍清楚，被拍的数据（D）在快门那一刻附近必须**站稳别动**。具体分成两段：

- **建立时间（setup time，Tsu）**：在时钟有效沿**到来之前**，数据必须已经稳定下来的**最短时间**。相当于"快门按下前，你得先摆好姿势 Tsu 这么久"。
- **保持时间（hold time，Th）**：在时钟有效沿**到来之后**，数据还必须继续保持稳定的**最短时间**。相当于"快门按下后，别急着动，再保持 Th 这么久"。

把这两段拼起来，就是时钟沿前后一个**总宽度 = Tsu + Th 的"禁区窗口"**：

> **数据的跳变，只要不落在这个窗口里，就没事；一旦数据在这个窗口内还在变化，就叫"违反时序"（timing violation）。**

注意，这个窗口**不是信号本身的属性，而是那颗触发器/芯片的属性**——是器件手册里给死的两个数（比如某触发器 Tsu=0.15ns、Th=0.08ns）。你的任务，是让**数据到达和变化的时刻**，避开每一颗被它驱动的触发器的这个窗口。

## 二、违反了会怎样：亚稳态，最难缠的偶发错误

数据在禁区里变化，触发器"没拍清楚"，会发生什么？答案是数字电路里最阴魂不散的现象——**亚稳态（metastability）**。

正常情况下，触发器的输出 Q 要么是稳稳的 0、要么是稳稳的 1。可当采样时刻数据正在跳变、能量不上不下，触发器内部会被推到一个**不稳定的平衡点**上——输出**既不是明确的 0，也不是明确的 1**，而是卡在中间某个电平上，或者缓慢地、随机地朝 0 或 1 **漂移、振荡**一段时间，最后才随机落定。

这个"卡住"带来两个大麻烦：

1. **可能采到错误的值。** 它最后落到 0 还是 1 是**随机**的，跟数据真实该是什么没关系——这就是偶发错误的来源。
2. **亚稳态会往下传播。** 如果这个还在"发抖"的输出被后面的电路当成正常信号用了，错误就会扩散，甚至让整个状态机跑飞。

亚稳态最气人的地方在于它的**统计性**：它不是每次都发生，而是**概率性**的。数据边沿离采样沿越近、时钟频率越高，撞上的概率越大。工程上用一个指标衡量它——**平均无故障时间（MTBF）**：违规越频繁，MTBF 越短，系统就越容易"没规律地抽风"。**这正是为什么这类问题偶发、难复现、难定位。**

![图3：违反建立/保持时间的后果——亚稳态。正常时触发器输出 Q 是稳稳的 0 或 1；一旦采样瞬间数据在禁区内跳变，触发器被推到不稳定平衡点，输出既非 0 也非 1，在中间电平上随机振荡/缓慢漂移一段时间才落定（且落到 0 或 1 是随机的=可能采错）。这个还在发抖的输出若被下游电路当真，错误会传播开。亚稳态是概率性的，用 MTBF 衡量——所以问题偶发、难复现](/images/35/ill03_metastable.png)

## 三、时序怎么算：数据到达时间与裕量（slack）

那怎么**提前算出**会不会违规？这就要看数据从一个触发器跑到下一个触发器，中间花了多少时间。

考虑最典型的一段路径：**发起触发器**（起点，被时钟沿触发后吐出数据）→ 一堆**组合逻辑**（与或门、加法器之类）→ **走线** → **捕获触发器**（终点，用下一个时钟沿把数据采进去）。

数据**真正到达**捕获触发器的时刻，由三段延迟加起来：

> **数据到达时间 = 发起触发器的 Clk-to-Q 延迟（Tcq）＋ 组合逻辑延迟（Tlogic）＋ 走线延迟（Troute）**

而捕获触发器**要求**数据什么时候到齐，就看它的两个窗口：

**建立时间检查（setup check）——数据不能来太晚。** 捕获用的是**下一个**时钟沿（晚一个周期 T），数据必须在那个沿之前留出 Tsu 稳定：

> **Tcq + Tlogic + Troute + Tsu ≤ T（时钟周期）**
> 换成裕量：**建立裕量 = T −（Tcq + Tlogic + Troute + Tsu）**

**保持时间检查（hold check）——数据不能变太早。** 这里换成**同一个**时钟沿：新数据不能来得太快，快到把捕获触发器**这一拍**还没采稳的旧值给冲了：

> **Tcq + Tlogic + Troute ≥ Th**
> 换成裕量：**保持裕量 =（Tcq + Tlogic + Troute）− Th**

**裕量（slack）就是"还富余多少时间"**：**正的 = 通过，负的 = 违规。** STA 工具报告里，你看到的就是每条路径的 setup slack 和 hold slack。

举个具体的数（时钟周期 T = 2ns，也就是 500MHz）：

- Tcq = 0.3ns，Tlogic = 1.0ns，Troute = 0.2ns，Tsu = 0.15ns，Th = 0.10ns。
- 建立裕量 = 2.0 −（0.3 + 1.0 + 0.2 + 0.15）= **+0.35ns**，通过。
- 保持裕量 =（0.3 + 1.0 + 0.2）− 0.10 = **+1.40ns**，通过。

还有两个"吃裕量"的家伙必须记住：**时钟偏斜（skew）**——同一个时钟到两颗触发器有先后差；**时钟抖动（jitter，就是上一模块讲的）**——边沿本身在抖。它们都会**直接从裕量里扣掉**。所以设计里 slack 不能卡在 0 附近，得留够余量对付 skew、jitter 和温度电压漂移。

![图4：数据路径与裕量。数据从发起触发器（被时钟沿触发）出发，经 Clk-to-Q(Tcq)+组合逻辑(Tlogic)+走线(Troute)到达捕获触发器。建立检查：数据到达+Tsu 必须≤时钟周期 T（数据别来太晚），建立裕量=T−(Tcq+Tlogic+Troute+Tsu）。保持检查：Tcq+Tlogic+Troute 必须≥Th（新数据别来太早冲掉这一拍），保持裕量=(Tcq+Tlogic+Troute)−Th。裕量正=过、负=违规；时钟偏斜 skew 和抖动 jitter 直接吃裕量](/images/35/ill04_datapath.png)

## 四、一个关键区别：建立违规能降频救，保持违规降频救不了

这是整篇最重要、也是新手最容易搞反的一点，单独拎出来讲。

先看两条时序检查里，**时钟周期 T 出现在哪**：

- **建立检查**里有 T：`Tcq + Tlogic + Troute + Tsu ≤ T`。数据来晚了不够时间？**把 T 加大（降低时钟频率）**，右边变大，不等式就容易满足了。**所以建立违规，降频通常能救。**
- **保持检查**里**根本没有 T**：`Tcq + Tlogic + Troute ≥ Th`。它比较的是**同一个时钟沿**上，新数据来得快不快、会不会冲掉旧值——**跟时钟周期一点关系都没有**。**所以保持违规，你把频率降到多低都没用！**

这个区别的工程意义很大：

- **建立违规**：相对好办。降频、优化组合逻辑（缩短 Tlogic）、换更快的器件，都能加裕量。
- **保持违规**：更麻烦、更危险。它意味着数据路径**太快了**——数据变化跑得比时钟这一拍采稳还快。降频无效，只能**在数据路径上人为加延迟**（插缓冲、绕线）、或者调时钟树减小 skew。而且保持违规一旦流片就**改不了**，只能靠改设计。

一句话记牢：**建立违规是"跑得太慢"（降频可解），保持违规是"跑得太快"（降频无解）。**

![图5：建立违规 vs 保持违规，以及降频的作用。左：建立检查里有时钟周期 T，数据来太晚→降频（T 变大）就能救。右：保持检查里没有 T，比较的是同一个时钟沿上新数据会不会太快冲掉旧值→降频完全无效，只能给数据路径加延迟/调时钟 skew。记住：建立违规=跑得太慢（降频可解），保持违规=跑得太快（降频无解）](/images/35/ill05_setup_vs_hold.png)

## 五、怎么测、怎么保证

时序不是测出来才知道，而是**设计阶段就要算、流片/上板后再验证**。几条主线：

**1. STA 静态时序分析（设计阶段的主力）。** 芯片/FPGA 设计里，EDA 工具会把所有寄存器到寄存器的路径**穷举**一遍，结合工艺库里的延迟和你给的时钟约束，报出每条路径的 setup slack 和 hold slack。**只要有一条是负的，就是时序违规，必须修。** 这是数字设计签核（signoff）的核心一环。

**2. 示波器实测建立/保持（板级验证）。** 板子回来，用示波器**同时抓时钟和数据**，量数据的跳变边沿离时钟采样沿有多远，直接读出实际的建立时间和保持时间余量。很多协议软件能直接给"setup/hold 测量"和相对时钟的**数据眼图**。这是排查"电平合格却偶发出错"的利器。

**3. 总线一致性 spec 里的建立/保持。** 你熟悉的每条总线，规范里都白纸黑字规定了接口的建立/保持时间：**DDR 有 tDS / tDH**（数据相对 DQS 选通），**SPI、I2C** 也各有从设备对主时钟的建立/保持要求。一致性测试就是逐项验证这些数达标。

**4. 跨时钟域（CDC）用同步器。** 当一个信号从一个时钟域进另一个（两个时钟没关系），建立/保持**几乎必然会偶尔违反**、亚稳态无法完全避免。工程上不硬扛，而是用**同步器**——最常见是**两级触发器串联**：让第一级即使进了亚稳态，也有整整一个周期去"decay 落定"，第二级再采，把亚稳态传下去的概率压到极低（MTBF 拉到几十年以上）。**跨时钟域不加同步器，是最经典的时序事故。**

再说几个必须绕开的**坑**：

- **只看电平、不看时刻**：电平全合格不代表时序合格，偶发错误先怀疑建立/保持。
- **把保持违规当建立违规去降频**：越降越以为没事，其实根本没治，还可能因为温度变化时好时坏。
- **忘了 skew、jitter、PVT**：温度/电压/工艺（PVT）让延迟漂移，时钟偏斜和抖动吃裕量，slack 卡在 0 附近迟早出事，要留够余量。
- **跨时钟域裸传信号**：不过同步器直接用，等于主动制造亚稳态。

![图6：怎么测、怎么保证时序。① STA 静态时序分析：EDA 工具穷举所有寄存器路径、报 setup/hold slack，负值即违规（设计签核核心）。② 示波器实测：同抓时钟和数据、量边沿到采样沿距离、看数据眼图。③ 总线 spec：DDR 的 tDS/tDH、SPI/I2C 的建立保持，一致性逐项验。④ 跨时钟域 CDC 用两级触发器同步器：给亚稳态一个周期落定、把传播概率压到极低。坑：只看电平、保持违规误当建立违规降频、忘了 skew/jitter/PVT、跨时钟域裸传](/images/35/ill06_measure.png)

## 六、什么时候该验时序、判据从哪查、盯什么余量

前面把"是什么、为什么、怎么算"讲透了，这一节落到工程上：**拿到一块板 / 一个设计，到底什么时候必须验时序、要验哪几类、判据又从哪查。**

**什么时候该验？** 一句话：**只要有寄存器在按时钟采样、有总线在收发数据、有信号跨了时钟域，就得验时序**——光量电平、量眼图都不够。反过来，纯组合逻辑输出、常年不变的静态配置脚，可以先放一放。

**要验哪几类（对应三个战场）：**

1. **芯片 / FPGA 内部路径**——寄存器到寄存器，靠 **STA 静态时序分析**穷举，报每条路径的 setup/hold slack。这是设计签核阶段的主力。
2. **板级总线接口**——DDR、SPI、I2C 这些收发口，用**示波器同抓时钟和数据**，实测接口的建立/保持余量。
3. **跨时钟域（CDC）**——异步信号从一个时钟域进另一个，重点验同步器有没有、recovery/removal 够不够。

**判据从哪查？这是最多人卡住的地方。** 建立/保持这些数**不是拍脑袋定的，全都有出处**：

- **器件 datasheet**：触发器 / 芯片的 **Tsu、Th、Clk-to-Q（Tcq）**，手册里给死；
- **总线一致性规范**：DDR 的 **tDS / tDH / tCK** 来自 **JEDEC**；SPI、I2C 以及各类接口的建立/保持时间，来自各自的接口规范；
- **时钟约束（SDC）**：你在 STA 工具里写的时钟周期、skew、输入输出延迟约束，就是把上面这些判据翻译给工具。

**盯什么余量？** 盯 **slack**：STA 报告里每条路径的 setup slack / hold slack，**正的过、负的违规**。关键是**别卡在 0 附近**——skew（时钟偏斜）、jitter（时钟抖动）、PVT（工艺/电压/温度漂移）都会实打实地从余量里扣，留不够，量产和高低温下就会时好时坏。

![图7：什么时候该验时序、验哪几项、判据从哪查。三类战场：①芯片内部路径（寄存器→寄存器，STA 报 slack，签核穷举）②板级总线接口（接口建立/保持，示波器同抓时钟+数据，DDR/SPI/I2C 收发口）③跨时钟域 CDC（异步跨域、两级同步器、recovery/removal）。判据来源：器件 datasheet 的 Tsu/Th/Tcq ＋ 总线规范（DDR→JEDEC 的 tDS/tDH/tCK、SPI/I2C timing）。盯余量：STA 的 slack 正=过负=违规，别卡 0 附近，给 skew/jitter/PVT 留够。口诀：电平对≠时序对，偶发难复现的错先怀疑建立/保持](/images/35/ill08_when.png)

## 七、偶发采错、难复现，怎么定位

真到了排查现场——**电平全合格、却偶发出错、还没规律**——按这条流程走，别瞎试：

**① 先量：把时钟和数据同时抓下来。** 用示波器同抓时钟和被怀疑的数据线，量**数据跳变沿离时钟采样沿有多远**，看它是不是压进了建立/保持那个禁区窗；设计阶段则直接看 STA 报告里这条路径的 slack。

**② 碰了禁区 / slack 为负，按四类分诊、对症开药：**

- **建立违规（数据来太晚）**：slack 为负、数据到得太晚。**降频、优化组合逻辑、换更快的器件**都能加裕量——这类相对好治。
- **保持违规（数据变太早）**：数据变化跑在了这一拍采稳之前。**记住降频完全无效！** 只能在数据路径上**加延迟（插缓冲、绕线）**或**调时钟树减小 skew**。
- **跨时钟域偶发**：信号跨了没关系的两个时钟域，偶发亚稳态。**加两级触发器同步器**，把亚稳态传播概率压到极低。
- **时钟本身脏**：skew、jitter 直接吃掉了余量。这时问题不在数据路径，而在时钟——**留到下一篇专门查时钟质量**，并在设计里给足余量。

**③ 改完复测**：STA 的 slack 要转正、示波器上实测的建立/保持余量要够。**④ 别卡临界点签核**：给 skew / jitter / PVT 留足余量，才算真过。

## 八、一句话总结

这一篇是整个"时序与时钟"模块的地基：

> **数字系统靠时钟边沿给数据"拍照"，拍清楚的前提是数据在快门前后都别动——快门前的最短稳定时间叫建立时间，快门后的叫保持时间，两段合成一个"不许动"的禁区窗口。** 数据在禁区里变化就是时序违规，会引发亚稳态：输出卡在中间态、可能采错还会往下传，表现为偶发、难复现的错误。算时序看裕量（slack）：数据到达时间 = Clk-to-Q + 组合逻辑 + 走线，正裕量过、负裕量违规。**记住那个关键区别：建立违规能降频救（跑得太慢），保持违规降频救不了（跑得太快）。** 保证手段是 STA + 示波器实测 + 总线 spec + 跨时钟域同步器，别忘了给 skew、jitter、PVT 留余量。

## 🔧 工程师速查（附录）

**① 两个定义**
- 建立时间 Tsu：时钟有效沿"前"，数据必须已稳定的最短时间
- 保持时间 Th：时钟有效沿"后"，数据必须继续稳定的最短时间
- 合成一个宽 Tsu+Th 的"禁区窗口"，数据跳变落窗口内=违规；Tsu/Th 是器件参数

**② 违规后果：亚稳态**
- 输出既非 0 也非 1，中间态随机振荡/漂移后才落定（可能采错）
- 会向下游传播；概率性发生，用 MTBF 衡量→偶发、难复现

**③ 时序计算（slack）**
- 数据到达时间 = Clk-to-Q(Tcq) + 组合逻辑(Tlogic) + 走线(Troute)
- 建立裕量 = T −（Tcq+Tlogic+Troute+Tsu）
- 保持裕量 =（Tcq+Tlogic+Troute）− Th
- 正=过、负=违规；skew、jitter、PVT 直接吃裕量

**④ 关键区别（最易搞反）**
- 建立检查有 T → 建立违规=跑太慢，**降频可解**
- 保持检查无 T → 保持违规=跑太快，**降频无解**，只能加延迟/调 skew

**⑤ 怎么测/保证**
- STA 静态时序分析：穷举路径报 slack，负值即违规（签核核心）
- 示波器实测：同抓时钟+数据、量边沿到采样沿、看数据眼图
- 总线 spec：DDR 的 tDS/tDH、SPI/I2C 建立保持
- 跨时钟域 CDC：两级触发器同步器，压低亚稳态传播概率

**⑥ 常见坑**
- 只看电平不看时刻；保持违规误当建立违规降频
- 忘了 skew/jitter/PVT 余量；跨时钟域裸传不加同步器

**⑦ 什么时候验·判据从哪查**
- 何时验：有寄存器按时钟采样 / 有总线收发 / 有跨时钟域 → 就得验（电平对≠时序对）
- 三类：芯片内部路径（STA slack）· 板级总线接口（示波器实测）· 跨时钟域（同步器）
- 判据来源：器件 datasheet 的 Tsu/Th/Tcq ＋ 总线规范（DDR→JEDEC tDS/tDH/tCK、SPI/I2C）＋ SDC 约束
- 盯余量：slack 正=过负=违规，别卡 0 附近，留 skew/jitter/PVT

**⑧ 偶发采错的定位流程**
- ① 示波器同抓时钟+数据，量跳变沿离采样沿多远、碰禁区没（或看 STA slack）
- ② 分诊：建立违规→降频/优化逻辑；保持违规→降频无效、加延迟/调 skew；跨时钟域→加两级同步器；时钟脏→查时钟
- ③ 改完复测 slack 转正、余量够；④ 留够 skew/jitter/PVT 余量再签核
