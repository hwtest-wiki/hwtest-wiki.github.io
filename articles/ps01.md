# 笔记本几十条电源轨、上百个 PI 测试项——一张覆盖矩阵，保证一个测试项都不漏

<div class="article-meta"><span class="badge">📘 模块二 · 电源完整性 PI · 收官总结篇 · 项目作战地图</span><span class="badge">⏱ 约 24 分钟</span><span class="badge qr">🔧 含工程师速查</span></div>


一位工程师接手一块笔记本主板的 PI 测试。他经验很足，模块二这些测试项他闭着眼都会做：VCORE 的负载线扫得漂亮、DDR 的纹波压得干净、各路 LDO 的精度调整率一项不落。他埋头测了两三周，报告攒了厚厚一摞，**每一页都是绿的**。

板子过了 EVT、过了 DVT，量产了。然后某个批次在客户手里**批量出问题**：合盖睡眠、拔掉充电器过一夜，第二天早上一开盖——**黑屏，按电源键要按两三次才能起来**。返修台上复现、拆机、抓波形，最后定位到：有一条给 EC/RTC 供电的**常开小轨（always-on）**，在「适配器拔掉、系统进 S4」这个切换瞬间有一下**掉坑**，把维持电路复位了。

复盘会上最扎心的一句话是：这条轨的这个测试项，**从来没有出现在他的测试清单里**。不是他不会测——这就是一个上电/掉电时序题，他闭着眼都会测。是他**根本没想到要测这一条轨的这一个场景**。他凭经验、凭记忆列的清单，覆盖了他「想得到」的所有轨和项，却漏掉了他「没想到」的那一条。

**这就是本篇要解决的问题：不是「每一项怎么测」，而是「怎么保证一项都不漏」。** 前者靠技术，是过去二十多篇的事；后者靠方法，是这一篇的事。经验越丰富的工程师，越容易凭脑子列清单——而**凭脑子列清单，就一定有缺口**。这一篇，我们把「不漏测」从一件靠责任心、靠记性的事，变成一件**靠一张表、靠一套流程就能保证**的事。

**读完这篇，你能搞懂：**

- ✅ 为什么「每一项都会测」不等于「不会漏测」——漏测的根子不在技术，在**没有系统化的覆盖**；
- ✅ 把模块二所有 PI 测试项收成一张**测试项全集字典**（不漏的基础：先有全集，才谈得上覆盖）；
- ✅ 从一个真实项目到一份完整测试计划的 **7 步法**（原理图/datasheet → 轨清单 → 分类 → 生成项 → 叠加系统项 → 定判据 → 排阶段）；
- ✅ 不遗漏的引擎——**覆盖矩阵**（轨 × 测试项）：怎么填、为什么「**空格 = 漏测**」是铁律、怎么靠轨型批量带列不填到累死；
- ✅ 每一类轨「天生」该测哪些项（**轨型驱动**：DVS 轨才有跳变列、多相轨才有均流列）；
- ✅ 分层 checklist：**L1 器件级 → L2 单轨级 → L3 系统级 → L4 边界级**，从下往上一层不落；
- ✅ 笔记本项目**最容易漏、也最值钱**的那几类 PI 关注点（双电源切换、S-state、PD 宽压、待机漏电）；
- ✅ 测试阶段怎么排（**EVT / DVT / PVT** 各测什么、先保什么）；
- ✅ 一张**「测试项 → 对应篇号」全景索引表**，哪一项不会测，直接翻回对应那一篇。

> 说明：这一篇守**纯 PI 电气测试**的边界——精度/纹波/瞬态/时序/保护/DVS/多相/序列这些电气项。热测、EMC、续航、跌落这些是别的模块的活，本篇只在它们**影响到 PI 电气表现**时（比如「热受限降频对轨的影响」）带一句，不越界去教热测本身。

## 一、先破个假设：「每一项都会测」≠「不会漏测」

我们花了二十多篇，练的是**招式**——每一个测试项怎么做、判据怎么定、坑在哪。招式练到位，你能把任何**单独拎出来**的测试做得又快又准。

但一个整机项目翻车，**极少是因为某一项测错了**。更常见的是：那一项你压根没测——因为它没进你的清单。就像开头那位工程师，问题那条轨的那个时序，他闭着眼都会测，可他**没想到要测它**。

**招式解决「会不会测」，阵法解决「漏没漏测」。** 这是两件完全不同的事：

| | 招式（前 20+ 篇） | 阵法（这一篇） |
|---|---|---|
| 回答的问题 | 这一项**怎么测**、怎么判 | 这个项目**要测哪些项**、一项别漏 |
| 失败长什么样 | 测了，但**测错/判错** | 每一项都测对了，但**漏了一整项没测** |
| 靠什么 | 技术、仪器、经验 | 流程、清单、一张矩阵 |
| 越熟练越…… | 越准 | 越容易**凭记忆列清单 → 出缺口** |

漏测的可怕在于：**它在测试报告里是「隐形」的。** 你测过的每一项都写着「PASS」，报告一片绿；漏掉的那一项**根本不在报告上**，你翻一百遍报告也发现不了缺口——因为缺口是「没有的东西」，不是「错的东西」。

所以对付漏测，不能靠「更仔细地检查已测项」，只能靠一件事：**在动手测之前，先把「该测的全集」摊在桌面上，然后一项一项确认「测 / 不测（为什么）」——不允许有任何一项处于「没想过」的状态。** 这就是覆盖矩阵要干的活。

## 二、不漏的地基：先有一张「PI 测试项全集字典」

你没法覆盖一张你根本没列全的清单。所以第一步，是把**整个模块二学过的所有 PI 测试项**，收进一张分组字典。这不是让你背，而是让你手边永远有一张「全集」，任何时候都能拿它去比对「我这个项目，这些项都考虑到了吗」。

按测量域把 PI 测试项分成九组：

- **A 精度类** —— 静态精度（DC 落点）、线调整率（输入变）、负载调整率（负载变）。
- **B 纹波噪声类** —— 输出纹波、开关噪声、开关节点（SW）振铃、FFT 噪声溯源（找噪声从哪来）。
- **C 动态响应类** —— 负载瞬态（阶跃拉载看过冲/下冲/恢复）、线瞬态（输入跳变）、环路稳定性（相位裕度/增益裕度）。
- **D 时序与上下电类** —— 上电时序、单调性（只升不回头）、软启动与浪涌电流、PGOOD 行为、掉电保持时间。
- **E 保护类** —— 过压 OVP、欠压 UVP、过流 OCP / 短路 SCP、过温 OTP（四大保护的门限与恢复）。
- **F 效率与阻抗类** —— 电源转换效率（轻/中/重载）、PDN 阻抗（目标阻抗、谐振点）、去耦电容有效性。
- **G 动态电压类（DVS/AVS）** —— VID 码跳变瞬态、压频序（DVFS）配合、调压建立时间、轨探头 DC 耦合测法。
- **H 多相类** —— 逐相均流、相位交错纹波抵消、负载线 AVP / Vdroop（DC + AC 斜率）、轻载减相、相间热均衡。
- **I 系统级类** —— 供电树 / Rail Map、上下电序列与依赖、交叉调整率（一轨扰动另一轨）、主 + 副 PMIC/VR 协同、总线命令解码（SVID/SPMI/PMBus）。

这九组，就是你的**测试项全集**。它是覆盖矩阵的**列**的来源——矩阵横着的那一排，就是从这张字典里挑出来的。（每一项具体怎么测，见文末第八节的「测试项 → 篇号」索引表，这里只负责「列全」。）

## 三、从项目到测试计划的 7 步

有了全集字典，怎么把它落到**你手上这个具体项目**？靠下面 7 步，把「一块陌生主板」变成「一份完整 PI 测试计划」。这 7 步是**通用工作流**，笔记本、手机、服务器主板都套得上。

**输入**：原理图（schematic）+ 平台设计指南（PDG / Platform Design Guide）+ 各电源 IC 的 datasheet + 电源态定义（S0/S3/S4/S5 等各态哪些轨该在）。

1. **建全轨清单（Rail Map）** —— 把原理图里**每一条电源轨**都列进一张表，一条不落：轨名、电压、由哪颗 IC 供、供给谁、属于哪个电源域。**这是整个不漏体系的地基**——矩阵的「行」就是这张清单。漏了一条轨，后面所有覆盖都白搭。
2. **每轨分类** —— 给每条轨贴一个「轨型」标签：**DVS 动态轨 / 定压轨 / 多相大电流轨 / Load Switch 分配轨 / 常开 AON 轨**。分类决定了这条轨「天生」该测哪些项（见第四节）。
3. **按轨型查表生成测试项** —— 用「轨型 → 必测项」的映射（第四节那张表），给每条轨自动带出它该测的列。这一步把「凭记忆想」换成「照表套」，缺口就堵住了。
4. **叠加系统级项** —— 单轨项之外，补上**跨轨**的系统项：上下电序列、轨间依赖、交叉调整、S-state 切换、双电源输入切换。这些是单看任何一条轨都测不出来的（呼应深化 03）。
5. **定判据来源** —— 每一项的「合格线」从哪来，写清楚出处：**datasheet 规格**（器件级）/ **平台规范**（如 Intel VR spec 的负载线 mΩ）/ **系统要求**（如唤醒时间）。判据没出处，测了也不算数。
6. **排优先级与测试阶段** —— 把项分到 **EVT / DVT / PVT**（第七节）：先保能开机的，再铺全量，最后产线抽测。
7. **写用例 + 报告模板** —— 每一项落成可执行用例（条件、步骤、判据、记录字段），配统一报告模板，让别人也能照着测、结果能横向比。

七步走完，你手上就有了：一张**全轨清单**、一张**覆盖矩阵**、一摞**用例**、一套**报告模板**。而其中，第 1 步（列全轨）和第 3 步（套矩阵）是「不漏」的两个命门。

## 四、覆盖矩阵：让「漏测」在纸面上现形

这是本篇的**灵魂**。前面所有铺垫，都是为了填出这一张表。

**结构**：一张二维网格。

- **行** = 第 1 步列出的**每一条电源轨**（VCORE、VGPU、VDDQ、VDD_AON……几十条，一条一行）。
- **列** = 第二节字典里的**每一个测试项**（静态精度、纹波、瞬态、DVS 跳变、均流、序列、单调、PGOOD、交叉调整、效率、保护、边界……）。
- **格子** = 这条轨的这一项，**要不要测**。

**铁律：每个格子只有两种合法状态，不允许留空——**

- **✓（测）** —— 这条轨要做这一项，后面会有对应用例和数据。
- **N/A + 原因** —— 这条轨**不做**这一项，且**写明为什么**（例：「VDD_AON 是常开轨，不做上电序列，因为它先于一切上电」「这条 LDO 是定压轨，不做均流，因为它不是多相」）。

**空格是唯一的敌人。** 一个空格，意味着这条轨的这一项，你**还没决定**要不要测——也就是**还没想过**。开头那位工程师翻车的那一格，就是一个「没想过」的空格。把矩阵铺开、要求**每一格都必须落成 ✓ 或 N/A**，漏测就从「脑子里一个看不见的疏忽」，变成「纸面上一个刺眼的空格」——**一眼可见、可以评审、可以交接、可以在开测前就补上**。

这就是覆盖矩阵的全部魔力：**它不保证你测得对，它保证你「没有一项是没想过的」。** 「想过之后决定不测」是专业判断，没问题；「压根没想到」才是事故。矩阵把这两者彻底分开。

**怎么填才不会累死？** 几十条轨 × 十几个项，几百个格子，一个个手填当然崩溃。靠**轨型批量带列**（下一节）：先给每条轨贴轨型标签，同一轨型「天生」该测哪些、该 N/A 哪些是固定的，一整类轨可以**成批**落格，最后只需要人工审那几个特殊轨。填矩阵于是变成「套模板 + 改例外」，而不是「从零想每一格」。

## 五、轨型驱动：每一类轨「天生」该测哪些

覆盖矩阵能批量填，靠的是这条规律：**测哪些项，主要由「轨型」决定，而不是由轨名决定。** 只要你给一条轨定了型，它该点亮哪些列、该 N/A 哪些列，基本就定了。下面这张「轨型 → 必测项」映射表，就是第 3 步（生成测试项）和第四节（批量填矩阵）的引擎：

| 轨型 | 典型例子 | 必测（✓） | 通常 N/A |
|---|---|---|---|
| **DVS 动态轨** | VCORE、VGPU | 精度、纹波、瞬态、**VID 跳变/建立时间**、序列、单调、PGOOD、保护、效率 | —— |
| **多相大电流轨** | VCORE（8~16 相） | 上面全部 + **逐相均流、相位交错、负载线 AVP、减相、相间热均衡** | —— |
| **定压轨** | VDDQ、各路 1.8V/3.3V | 精度、纹波、瞬态、序列、单调、PGOOD、保护、（交叉调整） | DVS 跳变、均流 |
| **Load Switch 分配轨** | 各路外设供电门控 | 导通压降、开关时序、浪涌、保护（走第 26 篇项） | 均流、DVS |
| **常开 AON 轨** | EC/RTC、VDD_AON | 精度、纹波、**待机漏电流**、掉电保持、（先于一切上电，序列另计） | 上电序列、DVS、均流 |

有了这张表，第四节那句「批量带列」就落地了：**给 VCORE 贴上「多相 DVS 轨」→ 它那一行的均流、负载线、跳变列自动点亮；给 VDDQ 贴上「定压轨」→ 它那一行的均流、跳变列自动打 N/A。** 几十条轨里，绝大多数都能这么成批处理，真正需要你逐格斟酌的，往往只有那么三五条特殊轨。

## 六、分层 checklist：L1 → L4，从下往上一层不落

覆盖矩阵管的是「广度」（每条轨每个项都想到）。还要一个东西管「深度」——每一项要测到**哪一层**才算到位。这就是分层 checklist，从器件一直查到边界：

- **L1 器件级** —— 单颗电源 IC 自己 datasheet 上的项：软启动斜率、保护门限（OVP/OCP/OTP 点）、PGOOD 逻辑、开关频率。**看一颗 IC，不看它接了谁。**（对应速查篇 q01「电源芯片测试地图」的视角。）
- **L2 单轨级** —— 这一条轨作为一个整体的电气表现：精度、纹波、瞬态、时序、效率。**看一条轨，不看别的轨。**（模块二正篇 06~26 的主战场。）
- **L3 系统级** —— 轨与轨**之间**的事：上下电序列、依赖、交叉调整、主副协同、S-state 切换。**看整机配电，单看任何一条轨都测不出来。**（深化 03 的主场。）
- **L4 边界级（PI 相关）** —— 极限条件下的电气表现：温度上下限时的精度/纹波漂移、宽压输入（USB-C PD 5~20V）下各轨行为、老化后的参数漂移。**注意：只测「边界条件下 PI 侧的电气表现」，热测/EMC 本身是别的模块的事。**

四层的关系是**从下往上支撑**：L1 器件不合格，L2 单轨一定出问题；L2 单轨没测透，L3 系统级的怪问题根本无从查起。做计划时，四层都要在覆盖矩阵里有落点——很多项目栽跟头，就栽在只做了 L1/L2，把 L3/L4 当成「应该没事吧」跳过了。

## 七、笔记本项目最容易漏、也最值钱的几类 PI 关注点

上面的框架是通用的。但**笔记本**这个品类，有几类 PI 关注点特别容易漏、一旦漏了又特别容易在客户手里爆——它们几乎都藏在 L3/L4 那两层，也几乎都是「切换/边界」场景。单独拎出来提醒：

- **双电源输入切换** —— 适配器 ↔ 电池来回切的那个**瞬间**，输入母线（VIN/VSYS）会有一下跌落或过冲，下游各轨跟不跟得住？开头那个「拔充电器就掉坑」的翻车，就是这一类。
- **电源态 S0 / S0ix(Modern Standby) / S3 / S4 / S5** —— 每个态下**哪些轨该在、哪些该关**、态与态**切换时的时序**、以及**唤醒时轨的恢复**。Modern Standby（S0ix）尤其坑：系统「看着睡了」但很多轨还半醒着，漏电、掉坑都爱在这里出。
- **USB-C PD 输入 5~20V 宽压** —— 输入电压能从 5V 一路谈到 20V，**在整个宽压范围内**各轨精度/纹波/效率都得成立，不能只在 20V 一个点测完就算。
- **热受限下的 DVS / 降频对轨的影响** —— 机器热了会降频，降频伴随 VID 下调、电流重新分配——**这个电气过程**要测（轨怎么跟随），但热测本身不归 PI。
- **电池低电量 / 掉电保持 / 快充涓流 / 待机漏电流** —— 低电量告警点的轨行为、断电瞬间的保持时间、涓流充电阶段的纹波、以及合盖一整晚的**待机漏电流**（直接决定「一晚上掉几个点电」）。

这五类，建议在覆盖矩阵里**单开一组「系统场景」列**，强制每条相关轨都对它们表态 ✓ / N/A——因为它们最不容易「凭经验想到」。

## 八、测试阶段编排：EVT / DVT / PVT 各干什么

覆盖矩阵列出了「要测的全集」，但不是所有项都在同一时间测。按产品阶段把项**排进节奏**，才不会 EVT 就想一口吃成全量、也不会 DVT 漏了该铺的边界：

| 阶段 | 目标 | PI 测什么 |
|---|---|---|
| **EVT**（工程验证） | **先保证能开机、能跑** | 点亮各轨、基本轨精度、上电序列、单调性、PGOOD、四大保护基本门限 |
| **DVT**（设计验证） | **全量铺开、把边界压出来** | 覆盖矩阵**全部** ✓ 项：纹波/瞬态/环路/效率/DVS 跳变/多相均流+负载线/交叉调整 + **温度极限 + S-state 边界 + PD 宽压** |
| **PVT / MP**（量产验证/量产） | **关键项抽测、保一致性** | 关键轨精度/纹波抽测、上电序列抽测、ORT（长时老化）、产线快测（go/no-go） |

一句话记法：**EVT 保「起得来」，DVT 保「压不垮」，PVT 保「批量稳」。** 覆盖矩阵是「测什么」，这张阶段表是「什么时候测」——两张表配合，才是一份能执行的计划。

## 十、一句话总结

**你会不会测每一项，靠的是技术；你漏不漏测任何一项，靠的是一张表。** 把模块二所有 PI 测试项收成一张字典，用 7 步把项目变成计划，再用一张「轨 × 测试项」覆盖矩阵——**规定每一格都必须是「测」或「写明原因的不测」，绝不允许空格**——漏测这件事，就从「靠责任心和记性」，变成了「靠流程和纸面就能兜住」。矩阵不保证你测得对，但它保证：**没有任何一项，是你「压根没想到」的。** 而整机 PI 翻车，十次有九次，栽的正是那个「没想到」。

### 一、笔记本 PI 测试计划 7 步速查

1. **建全轨清单 Rail Map** —— 原理图每条轨列全（轨名/电压/供电 IC/负载/电源域），一条不落。
2. **每轨分类** —— DVS 动态轨 / 多相大电流轨 / 定压轨 / Load Switch 轨 / 常开 AON 轨。
3. **按轨型查表生成测试项** —— 用「轨型 → 必测项」表批量带列（见下）。
4. **叠加系统级项** —— 序列/依赖/交叉调整/S-state/双电源切换。
5. **定判据来源** —— datasheet / 平台规范（VR spec 等）/ 系统要求，写明出处。
6. **排优先级与阶段** —— 分 EVT / DVT / PVT。
7. **写用例 + 报告模板** —— 条件/步骤/判据/记录字段，统一模板。

### 四、分层 checklist（深度：一层不落）

- **L1 器件级**：软启动斜率 / 保护门限(OVP/OCP/OTP) / PGOOD 逻辑 / 开关频率。
- **L2 单轨级**：精度 / 纹波 / 瞬态 / 时序 / 效率。
- **L3 系统级**：上下电序列 / 依赖 / 交叉调整 / 主副协同 / S-state 切换。
- **L4 边界级**：温度极限电气漂移 / PD 宽压各轨行为 / 老化漂移（只测 PI 侧电气）。

### 五、EVT / DVT / PVT 阶段分配

| 阶段 | 一句话 | 测什么 |
|---|---|---|
| EVT | 保「起得来」 | 点亮/基本精度/上电序列/单调/PGOOD/保护基本门限 |
| DVT | 保「压不垮」 | 覆盖矩阵全 ✓ 项 + 温度极限 + S-state 边界 + PD 宽压 |
| PVT/MP | 保「批量稳」 | 关键轨抽测 + ORT 老化 + 产线快测 |
