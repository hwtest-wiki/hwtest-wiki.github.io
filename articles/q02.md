## 一、第一把尺子：这个接口是「干嘛用的」

先按**用途**粗分，这决定了它长在板子的哪、跟谁打交道、测试台面大概什么样：

- **通用数据传输**：USB（2.0/3.x/4）、Thunderbolt 雷电。插各种外设、传文件，讲究**通用互通**。
- **显示 / 视频**：HDMI、DisplayPort（含笔记本内屏 eDP）。把画面从主机送到屏幕，讲究**带宽够不够跑高分辨率高刷新**。
- **板内高速 / 扩展**：PCIe。CPU 到显卡、到 NVMe SSD 的主干道，**板内点对点、速率最高的一档**。
- **存储**：UFS / eMMC（手机存储）、PCIe/NVMe（电脑 SSD）。
- **相机 / 屏幕（移动端专用）**：MIPI D-PHY / C-PHY，接手机的摄像头（CSI）和屏幕（DSI）。
- **内存**：DDR / LPDDR，CPU 边上那条**并行**高速总线（严格说不算"串行接口"，但测试思路要单拎出来讲）。
- **供电**：USB PD（跑在 USB-C 的 CC 针上）。**注意——它不传数据，只谈供电**，是模块六里的异类。
- **网络**：以太网（笔记本 RJ45 / 车载）。

> 一句话记：**USB/雷电管通用、HDMI/DP 管显示、PCIe 管板内主干、UFS/NVMe 管存储、MIPI 管手机的屏和相机、DDR 管内存、USB PD 管供电、以太网管上网。**

## 二、第二把尺子：时钟从哪来——这才是「测法」的分水岭

用途决定台面，但**真正决定"怎么测、看不看眼图"的，是时钟架构**。高速接口收数据时，接收端必须知道"每一位在什么时刻采样"，这个"节拍"从哪来，分三种，测法天差地别：

### ① 嵌入式时钟 + CDR —— 看眼图的主力军

发送端**不单独发时钟**，把时钟"埋"进数据流里（靠编码保证有足够跳变），接收端用一个叫 **CDR（时钟数据恢复）** 的电路，从数据的边沿里把节拍"提"出来。

- **代表**：PCIe、USB 3.x、SATA、DisplayPort、HDMI 2.1 FRL、以太网 SerDes……**现代大多数高速串行接口都是这一类。**
- **为什么看眼图**：既然节拍要从数据边沿里恢复，那**边沿抖不抖、眼睛睁不睁得开，直接决定能不能采对**。所以这一类的一致性核心就是**眼图 + 抖动 + 均衡**——正是第 31（眼图）、32（抖动）、34（均衡）打下的地基。模块六里 USB、HDMI、DP、PCIe 的重头都在这。

### ② 源同步 / 转发时钟 —— 有独立时钟，测法要单拎

发送端**专门发一路时钟（或选通/strobe）**，和数据并排走，接收端拿这路时钟去采数据。

- **代表**：**DDR / LPDDR**（用 DQS 选通）、**MIPI D-PHY**（专门一条时钟 lane 陪着数据 lane 走）、**HDMI 2.0 及以前的 TMDS**（3 条数据 + 1 条独立时钟通道）。
- **测法不同在哪**：不用从数据里恢复时钟，但**数据和时钟之间的建立/保持时间（setup/hold）、偏斜（skew）成了命门**——这是第 35（时序基础）、36（时钟测试）的地盘。眼图也看，但要**相对那路时钟去看**。

> ⚠️ 同一个 HDMI，跨过 2.1 就换了架构：**≤2.0 的 TMDS 是"源同步（独立时钟通道）"，2.1 的 FRL 变成"嵌入式时钟"**——这也是为什么 HDMI 2.1 的测法和老 HDMI 不一样。MIPI 也横跨两类：**D-PHY 是源同步（有时钟 lane），C-PHY 是嵌入式（3 线一组、时钟埋在符号里）**。

### ③ 协议协商类 —— 不看波形，看"行为"

有一类"接口"根本不是在比信号干不干净，而是在比**一套协议跑得对不对**——最典型就是上一篇的 **USB PD**：它在 CC 针上用 300kbps 的慢速通信一问一答谈供电，**测的是消息顺序、时序、状态机，不是眼图**。

- 这正对应第 38 篇那张**「激励方式三分法」**（下一节展开）。

![图2：两把尺子把接口归位。第一把尺子·用途：通用数据(USB/雷电) / 显示(HDMI/DP) / 板内主干(PCIe) / 存储(UFS·NVMe) / 相机屏(MIPI) / 内存(DDR·LPDDR) / 供电(USB PD) / 网络(以太网)。第二把尺子·时钟架构（决定测法）：①嵌入式时钟+CDR——不发时钟、从数据边沿恢复节拍，看眼图+抖动+均衡（PCIe/USB3.x/DP/HDMI2.1 FRL/以太网，↔31·32·34）②源同步/转发时钟——单独发一路时钟并排走，命门是 setup/hold 与 skew（DDR/LPDDR·DQS、MIPI D-PHY·时钟lane、HDMI≤2.0 TMDS，↔35·36）③协议协商类——不看波形看行为（USB PD·CC 上 300kbps，↔38）。中间提示：HDMI 跨 2.1 从源同步(TMDS)变嵌入式(FRL)、MIPI 的 D-PHY(源同步)与 C-PHY(嵌入式)分属两类](/images/q02/ill02_twolens.png)

**横着扫一眼你就能记住三件事：**

1. **绝大多数现代高速串行接口都是"嵌入式时钟 + CDR"** → 所以模块六看起来"到处都在讲眼图"，因为它们本质是同一套 SI 打法（31/32/34），换的只是速率、编码和判据。
2. **少数几个是"源同步"**（DDR、MIPI D-PHY、老 HDMI TMDS）→ 测的重心从眼图偏向**时序与 skew**（35/36）。
3. **USB PD 是唯一的"协议协商类"** → 完全不看眼图，看协议行为（42）。

## 四、挂第一张速查：激励方式三分法（↔ 第 38 篇）

上面"看眼图还是看协议"的差别，第 38 篇已经用一张**「激励方式三分法」**讲透——测一致性时，怎么"喂"信号给被测对象，分三类，直接决定台面：

- **① 固定 / 循环码型**：让被测口进 compliance 模式，发一段规定的码型（如 USB 的 Test Packet），示波器抓下来叠眼图、扣模板。**嵌入式时钟接口的发送端测试都走这条**（USB、PCIe、HDMI、DP……）。
- **② PRBS + 环回（loopback）**：造一只最坏的"压力眼"灌给接收端，让它环回，数误码率。**考接收端抖动容限走这条**（USB 3.x Rx、PCIe Rx……）。
- **③ 功能 / 实时业务**：不发码型，而是跑真实业务流。**源同步、突发式的接口（MIPI D-PHY 的 HS/LP 突发、DDR 的实际读写）常走这条**；协议协商类（USB PD）也归在"不发码型、走行为"这一侧——扮演对端一问一答跑协议。

> 一句话对上号：**嵌入式时钟接口 → 多用①②（码型/环回，看眼图）；源同步突发接口 → 多用③（实时业务）；USB PD → 扮演对端走协议。** 这就是"谁看眼图、谁看协议"的根。

![图3：激励方式三分法（↔第38篇）与接口对号入座。①固定/循环码型：进 compliance 模式发规定码型→示波器叠眼图扣模板；嵌入式时钟接口的发送端测试都走这条（USB/PCIe/HDMI/DP）。②PRBS+环回 loopback：造压力眼灌给接收端、环回数误码率；考 Rx 抖动容限走这条（USB3.x Rx/PCIe Rx）。③功能/实时业务：不发码型跑真实业务流；源同步突发接口（MIPI D-PHY HS/LP 突发、DDR 实际读写）常走这条，协议协商类（USB PD 扮演对端一问一答）也归在这一侧。底部结论：嵌入式时钟→①②看眼图，源同步突发→③实时业务，USB PD→走协议——这就是谁看眼图谁看协议的根](/images/q02/ill03_stimulus.png)

## 五、挂第二张速查：串行高速带宽选型（别把"5 倍原则"用错）

选示波器测高速接口，新手最容易踩的坑：**把第 29 篇的"5 倍原则"直接套到数据速率上**——"40Gbps？那得 5×40=200G 的示波器！"结果发现市面上根本没有、也不需要这么高带宽的示波器。

问题出在**"5 倍原则"是给"时域边沿测量"用的，不是给"串行眼图"用的**，两者的带宽逻辑完全不同：

| 场景 | 带宽怎么定 | 依据 |
|---|---|---|
| **测边沿 / 上升时间**（时域保真） | 取最高有效频率的 **~5 倍**（到 5 次谐波） | 第 29 篇：要还原方波的陡沿，需要高次谐波 |
| **串行链路眼图**（NRZ） | 约 **0.7×～1× 波特率** 就够（≈1.4× 奈奎斯特） | NRZ 主要能量在基频附近，眼图不需要到 5 次谐波 |
| **一致性判决** | **直接按规范写死** 的带宽档选 | 规范会规定用多大带宽的参考接收器 / 示波器 |

**关键点：数据速率不是频率。** 40Gbps 的 NRZ 信号，奈奎斯特（基频）是 **20GHz**，不是 40GHz——

- 按 0.7× 波特率的主流经验，**下限约 28GHz**；
- 留点裕量，选 **40～50GHz 级**实时示波器（或用等效时间采样示波器）就足够；
- **200GHz 纯属把"速率×5"这个错误公式当真了。**（真要上百 G 带宽，那是 PAM4 高阶调制 / 前沿科研的场景，不是普通 NRZ 眼图。）

> 记死这句：**边沿看 5 倍谐波，串行眼图看约 0.7 倍波特率，一致性判决按规范写死——三个场景三套带宽逻辑，别混用。**

![图4：串行高速带宽选型速查（别把5倍原则用错）。三场景三套逻辑：①测边沿/上升时间（时域保真）→取最高有效频率约5倍（到5次谐波），依据第29篇。②串行链路眼图（NRZ）→约0.7×~1×波特率就够（≈1.4×奈奎斯特），因 NRZ 能量集中在基频附近。③一致性判决→按规范写死的带宽档选。中间用 40Gbps NRZ 举例算清：数据速率≠频率，奈奎斯特=20GHz 不是40GHz；0.7×波特率下限≈28GHz，留裕量选40~50GHz 级示波器足够；"5×40=200GHz"是把速率×5的错误公式当真（百G带宽是 PAM4/科研场景）。底部口诀：边沿看5倍谐波·眼图看0.7倍波特率·判决按规范写死](/images/q02/ill04_bandwidth.png)

## 六、怎么用这张地图：拿到一个陌生接口，三步定测法

1. **先用第一把尺子定用途** → 知道它长在哪、大概什么台面（显示口找信号发生器+显示图形，存储口要能跑读写，供电口要电子负载……）。
2. **再用第二把尺子定时钟架构** → 立刻知道"看眼图还是看时序还是看协议"：
   - 嵌入式时钟 → 眼图 / 抖动 / 均衡（翻 31·32·34），发送端走码型、接收端走环回；
   - 源同步 → 加测建立/保持 + skew（翻 35·36），相对那路时钟看；
   - 协议协商类 → 协议分析仪 + 扮演对端（翻 38·42）。
3. **最后按速率量级选仪器带宽** → 用上一节的"0.7 倍波特率 / 按规范写死"定示波器档，别套错 5 倍原则。

三步走完，一个你没测过的接口，也能立刻说出"它该怎么测、翻系列哪几篇"。

## 七、一句话总结

高速接口再多、速率再吓人，**两把尺子就能全归位**：

> **第一把尺子看用途**（通用数据 / 显示 / 板内主干 / 存储 / 相机屏 / 内存 / 供电 / 网络），决定台面；**第二把尺子看时钟架构**，决定测法——**嵌入式时钟 + CDR**（现代大多数：USB/PCIe/DP/HDMI2.1/以太网）从数据边沿恢复节拍，所以**看眼图 + 抖动 + 均衡**；**源同步 / 转发时钟**（DDR、MIPI D-PHY、老 HDMI TMDS）有独立时钟，命门在**建立/保持与 skew**；**协议协商类**（USB PD）不看波形、看协议行为。再挂上两张速查——**激励方式三分法**（码型 / 环回 / 实时业务，↔38）说清"谁看眼图谁看协议"，**串行带宽选型**（边沿 5 倍谐波 / 眼图约 0.7 倍波特率 / 判决按规范写死）避免把 5 倍原则用错。看懂这张地图，模块六后面每一个接口，都只是"同一套方法 + 这个口的具体判据"。

![图5：串行高速接口全景地图·工程师速查（一图带走）。两把尺子：①用途分八类②时钟架构分三类（嵌入式+CDR→看眼图/源同步→看时序skew/协议协商→看行为）。接口归位速记：多数现代串行=嵌入式看眼图(USB3.x/PCIe/DP/HDMI2.1/以太网/UFS)；源同步看时序(DDR·DQS/MIPI D-PHY·时钟lane/HDMI≤2.0 TMDS)；USB PD=协议协商不看眼图。两张挂图：激励方式三分法(码型/环回/实时业务↔38)、带宽选型(边沿5倍谐波·眼图0.7倍波特率·判决按规范写死)。三步用法：定用途→定时钟架构→按速率选带宽。速率量级：USB2 480M·USB3.x 5-20G·USB4/雷电4 40G·雷电5 80-120G·HDMI2.1 48G·DP2.1 80G·PCIe Gen6 64GT/s·MIPI D-PHY 4.5G/lane·UFS4 23.2G/lane·DDR5 4800-8400MT/s](/images/q02/ill05_quickref.png)

**激励方式三分法（↔38）**

| 类 | 怎么喂 | 谁用 |
|---|---|---|
| ① 固定/循环码型 | 进 compliance 发规定码型→叠眼图扣模板 | 嵌入式时钟接口的发送端（USB/PCIe/HDMI/DP） |
| ② PRBS+环回 | 造压力眼灌 Rx、环回数误码率 | 接收端抖动容限（USB3.x/PCIe Rx） |
| ③ 功能/实时业务 | 不发码型跑真实业务/走协议 | 源同步突发（MIPI D-PHY/DDR）、USB PD 扮演对端 |

**串行高速带宽选型**

| 场景 | 带宽 | 依据 |
|---|---|---|
| 测边沿/上升时间 | 最高有效频率 ~5 倍（5 次谐波） | 第29篇时域保真 |
| 串行眼图(NRZ) | ~0.7×~1× 波特率（≈1.4×奈奎斯特） | NRZ 能量集中基频 |
| 一致性判决 | 按规范写死的带宽档 | 规范/参考接收器 |
| 例：40Gbps NRZ | 奈奎斯特=20GHz；下限≈28GHz；选40~50GHz 足够；**不是 200GHz** | 数据速率≠频率 |

**三步定测法**

| 步 | 做什么 |
|---|---|
| 1 | 第一把尺子定用途 → 台面 |
| 2 | 第二把尺子定时钟架构 → 看眼图/看时序/看协议 |
| 3 | 按速率量级选带宽（0.7×波特率或规范写死，别套 5 倍原则） |
