# 拆开手机主板，几十颗电源芯片到底各管什么？

<div class="article-meta"><span class="badge">📘 模块二 · 电源完整性 PI · 家族总览</span><span class="badge">⏱ 约 11 分钟</span><span class="badge qr">🔧 含工程师速查</span></div>


把一台手机拆开，盯着主板看，你会发现一个反常识的事实：**最多、最密的一类芯片，往往不是 CPU、不是内存，而是电源芯片**——大大小小几十颗，密密麻麻铺在板子上。

一台手机就一块电池，电压还在 **3.0V 到 4.2V** 之间晃来晃去。可它上面的芯片，要的电五花八门：CPU 核心要 **0.6～0.9V** 的大电流、内存要 **1.1V**、各种接口要 **1.8V**、屏幕和马达要更高的电压、射频和音频还要**特别干净**的电……

**怎么用一节晃动的电池，变出十几路、各不相同、还要又稳又干净的电？** 答案，就是那几十颗电源芯片。它们不是一种东西，而是**分工明确的几个家族**。这一篇，我们就把这张「电源 IC 家族图谱」画清楚。

**读完这篇，你能搞懂：**

- ✅ 为什么一块板要用**几十颗**电源芯片，而不是一颗搞定；
- ✅ 电源 IC 的**五大家族**——DCDC、LDO、Load Switch、PMIC、充电/电量计——各管一摊什么；
- ✅ 每个家族**各自要测什么**（纹波？PSRR？浪涌？时序？），它们怎么对应到前面讲过的测试项；
- ✅ 多路电源之间会互相影响的**交叉调整率**是什么；
- ✅ 工程师**怎么选**：什么时候用 DCDC、什么时候非 LDO 不可、什么时候只要一个开关。

## 一、为什么一块板要几十颗电源？

根子上的矛盾就一句话：**电源只有一个（电池），用电的「嘴」却有几十张，而且每张嘴的要求都不一样。**

一节锂电池，电量满是 4.2V、快没电是 3.0V，一边用一边掉。可板上的芯片各有各的「口味」：

- **CPU / GPU 核心**：要 0.6～0.9V，但**电流极大**（峰值几安培），还会瞬间猛增猛减；
- **内存 DDR**：要 1.1V 左右，且很在意纹波；
- **各种数字接口 IO**：要 1.8V；
- **射频 / 音频 / 摄像头模拟**：电压不高，但**特别怕噪声**——电源脏一点，信号就糊、音频就有底噪；
- **USB / 快充输出**：要抬到 5V 甚至更高；
- 还有**充电**（把外面的电灌进电池）、**电量计**（算还剩多少电）、**上下电时序**（谁先上、谁后上）要管。

每一种「既要这个电压、又要这种脾气（大电流？低噪声？要不要能单独开关？）」的需求，都得配一颗**合适的**电源 IC。于是一块板上，自然就长出了几十颗、分工不同的电源芯片。

> **一句话记住：一节晃动的电池要喂饱几十张「口味」各异的嘴——不同电压、不同电流、不同对「干不干净」的要求，逼出了一整套分工明确的电源芯片。**

## 二、一节电池，怎么变出十几路电？

把上面这件事画成一棵树就清楚了：**输入是一节晃动的电池，输出是十几路又稳又各异的电**，中间靠各类电源 IC 做「转换 + 稳压」。

电池电压在晃（3.0～4.2V），但每一路输出都必须**稳稳地**卡在它该在的电压上——不管电池是满还是快没电、不管负载忽大忽小。这就是所有电源 IC 的共同使命：**把不稳的输入，变成稳定的、指定电压的输出**。

区别只在于「怎么变、变出来干不干净、能扛多大电流」——而这，正好把它们分成了不同的家族。

> **一句话记住：所有电源 IC 干的是同一件事——把晃动的电池电压，转成一路路稳定的指定电压；区别在效率、噪声、电流能力，于是分出了不同家族。**

![图2：一节电池（3.0–4.2V 晃动）经各类电源 IC，分出 CPU 0.8V / DDR 1.1V / IO 1.8V / 射频干净电 / USB 5V 十几路稳定输出](/images/16/ill02_power_tree.png)

## 三、五大家族，各管一摊

电源 IC 虽多，归类起来主要是**五大家族**。记住每家「一句话定位 + 该测什么」，这张图谱就立起来了。

**① DCDC（开关电源，含 Buck 降压 / Boost 升压）——高效大电流的主力。**
靠开关 + 电感「搬运」能量，效率高（90%+）、能扛大电流。给 CPU、GPU、DDR 这些**吃电大户**供电。代价是开关会带来纹波和 EMI。
**测什么**：纹波与噪声、效率、负载/线路瞬态、环路稳定性、开关节点 SW 的振铃过冲。

**② LDO（低压差线性稳压器）——又小又干净的「清道夫」。**
本质是串联可调电阻，输出**极干净、噪声极低**，但压差全发热、只适合**小电流、小压差**。给射频、音频、传感器、PLL 这些**怕噪声**的模块供电；也常接在 DCDC 后面做「后级清洗」（DCDC 先高效降压，LDO 再把纹波滤干净）。
**测什么**：电源抑制比 PSRR、输出噪声、dropout 压差。

**③ Load Switch（负载开关）——一个「智能电闸」。**
它不变压，本质是一个**可控的开关**：让某个模块单独通电 / 断电，顺便控制上电时的**浪涌电流**和时序、断电时帮忙放电。省电（不用的模块直接断掉）、保护（短路就切断）。
**测什么**：导通电阻 Ron、上电浪涌 inrush、放电时间、通断时序。

**④ PMIC（电源管理 IC）——一颗顶一片的「集成总管」。**
把好多路 DCDC + LDO + 上下电时序 + 复位 PGOOD **集成进一颗芯片**，专门给 SoC 这种「要十几路电、还讲究谁先上电」的大块头统一供电。省板面积、省设计。
**测什么**：上下电时序、上电单调性、各路电压精度、PGOOD/复位逻辑。

**⑤ 充电 IC + 电量计——管电池进出的一对搭档。**
充电 IC 把适配器 / 快充的电，按 **CC（恒流）→ CV（恒压）** 曲线安全地灌进电池；电量计则用库仑计 / 阻抗法**估算还剩多少电**（你看到的电量百分比就靠它）。
**测什么**：充电曲线、充电精度与温度保护、电量计估算误差。

> **一句话记住：DCDC 管大电流高效供电、LDO 管小电流低噪声、Load Switch 管通断与浪涌时序、PMIC 管多路集成与时序、充电/电量计管电池进出——五个家族，各测各的。**

## 四、多路之间会互相影响：交叉调整率

当好几路电源**集成在一颗芯片里**（典型就是 PMIC），它们共享同一个输入、同一片地、同一个基准。于是会出现一个容易被忽略的现象：

**其中一路的负载突然变重，另一路的输出电压会被「带歪」一下。** 明明你没动那一路，它却跟着抖了一下。这种「一路扰动、殃及另一路」的程度，就叫**交叉调整率（Cross Regulation）**。

它在单路独立电源上不存在，却是**多路 PMIC 必须专门验**的一项：给某一路加载阶跃，同时盯着另一路的输出有没有明显波动。交叉调整率差，会让一个模块的「用电抽搐」污染到另一个模块。

> **一句话记住：多路集成电源（PMIC）里，一路加载会把另一路的电压带歪——这叫交叉调整率，是单路电源没有、多路必须专门测的坑。**

![图5：交叉调整率——PMIC 的 A 路负载突然加重，共享输入/地/基准让 B 路输出也被带着抖了一下](/images/16/ill05_cross_reg.png)

## 五、怎么选：一张决策树

知道了五大家族，选型其实就是「**看这路电的脾气，对号入座**」：

- **大电流、效率优先**（CPU / GPU / DDR、续航敏感）→ **DCDC**；
- **怕噪声、要干净**（射频 / 音频 / 敏感模拟 / PLL）→ **LDO**（或「DCDC 高效降压 + LDO 后级清洗」两级配）；
- **只是要给某个模块通 / 断电、控浪涌和时序** → **Load Switch**；
- **一颗 SoC 要十几路电 + 讲究上下电时序** → **PMIC**；
- **管电池充电与电量** → **充电 IC + 电量计**。

记住一个常见组合拳：**DCDC 打头阵负责「高效把电压降下来」，LDO 殿后负责「把最后那点纹波清干净」**——这就是为什么很多敏感模块前面，你会看到一颗 DCDC 后面又跟着一颗 LDO。

> **一句话记住：大电流找 DCDC、怕噪声找 LDO、只通断找 Load Switch、多路集成找 PMIC、管电池找充电/电量计；敏感模块常用「DCDC 降压 + LDO 清洗」两级配。**

![图6：电源选型决策树——按「大电流？怕噪声？要通断？多路集成？管电池？」一路分到对应家族](/images/16/ill06_decision_tree.png)

### 对照表 1 · 电源 IC 五大家族选型对照

| 家族 | 一句话定位 | 典型用途 | 效率 | 噪声 | 关键测试项 |
|---|---|---|---|---|---|
| **DCDC**（Buck/Boost） | 开关搬运、高效大电流 | CPU/GPU/DDR | 高（90%+） | 有纹波/EMI | 纹波、效率、瞬态、环路稳定性、SW ringing |
| **LDO** | 线性、低噪声、小压差 | 射频/音频/PLL/传感器 | ≈Vout/Vin | 极低 | PSRR、输出噪声、dropout 压差 |
| **Load Switch** | 可控开关、通断与浪涌 | 模块电源门控 | ≈1（仅 Ron 损耗） | — | Ron、inrush、放电、时序 |
| **PMIC** | 多路集成 + 时序 | SoC 统一供电 | 各路不同 | 各路不同 | 上下电时序、单调性、各路精度、PGOOD |
| **充电 IC + 电量计** | 管电池进出 | 充电 / 电量估算 | — | — | CC/CV 曲线、温度保护、电量误差 |

### 判据 · 交叉调整率（Cross Regulation）

- 定义：多路集成电源中，A 路负载变化引起 B 路输出电压的变化量（共享输入/地/基准导致）。
- 测法：给 A 路加载阶跃，示波器盯 B 路输出的偏移与恢复；只在多路/PMIC 上测，单路不存在。
- 判据：B 路偏移越小越好（具体限值看该路对精度的要求，常见要求在百分之几以内）。

### 选型决策树（文字版）

- 大电流 / 效率优先 → DCDC
- 怕噪声 / 要干净 → LDO（或 DCDC + LDO 两级）
- 仅通断 / 控浪涌时序 → Load Switch
- 一颗给十几路 + 时序 → PMIC
- 充电 / 电量 → 充电 IC + 电量计

### 常识 · 两级供电

- 敏感模拟模块常见「**DCDC 高效降压 + LDO 后级清洗**」：DCDC 负责效率、LDO 负责把纹波/噪声压到底。
- 后面每个测试专项挂到对应家族：纹波/效率/环路→DCDC；PSRR/dropout→LDO；inrush/Ron→Load Switch；时序/单调性→PMIC。
