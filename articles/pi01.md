# 同一颗 CPU，核心电压每秒跳几百次——你还按 1.0V±3% 测纹波，测的是一个根本不存在的工作点

<div class="article-meta"><span class="badge">📘 模块二 · 电源完整性 PI · 深化篇 · 动态调压 AVS/DVS</span><span class="badge">⏱ 约 29 分钟</span><span class="badge qr">🔧 含工程师速查</span></div>


一位工程师接手一块笔记本主板，测 CPU 核心轨 **VCORE** 的纹波和精度。他按 datasheet 上写的"**0.85V**"设好判据，示波器一挂，**傻眼了**：这条轨的电压根本不老实待在 0.85V——一会儿窜到 **1.05V**，一会儿又掉到 **0.6V**，来回跳，像心电图。

他第一反应是"电源坏了"。查了半天 VRM、换了探头，波形照跳不误。直到旁边做系统的同事瞟了一眼："**这不正常吗？CPU 一会儿满载一会儿空闲，电压当然跟着跳。**"

问题不在电源，在**他脑子里那个"电压是定值"的假设**。现代 CPU/SoC 为了又快又省电，**电压是随工作频率实时调的**：要跑高频（干重活）就把电压拉高保性能，降到低频（摸鱼）就把电压压低省功耗。这套机制叫 **DVS/AVS 动态调压**。所以 VCORE 不是"一个点"，而是**在一串电压档位（OPP）之间不停跳的动态变量**。

**你还拿"0.85V±3%"这把尺子去卡它，卡的是一个下一毫秒就不存在的工作点。** 真正该测的，是**每一次跳变本身**：跳得稳不稳？会不会过冲打穿芯片？稳得够不够快，赶得上换频吗？这一篇，我们就把这个被整个系列忽略的"动态电压"，从原理到实测，一次讲透。

**读完这篇，你能搞懂：**

- ✅ 为什么现代 CPU 核心电压是**动态变量**——DVS（按频率调）和 AVS（闭环给最低够用电压）到底在干什么、为什么能省电；
- ✅ 两套 PC 平台怎么调压——**x86 用 SVID/SVI 命令 VRM**、**高通骁龙 PC 用 SPMI/RPMh 调 PMIC**，命令通道不同，但 PI 该测什么**完全一致**；
- ✅ 闭环 AVS（高通 **CPR**）是怎么"**只给刚好够用的最低电压**"、省掉工艺 guardband 的；
- ✅ 电压**跳变那一下**要测的四件事——**建立时间、过冲/下冲、受控压摆率、逐 OPP 静态精度**，判据怎么定；
- ✅ 最容易漏的**压频协调时序**——为什么"升频要先加电压、降频要先降频再降电压"，错了就死机；
- ✅ 怎么测——**轨探头 DC 耦合 + 偏置** 配 **总线命令解码触发**，同屏抓"命令→跳变→稳定"三段（以及为什么这里**不能像测纹波那样用 AC 耦合**）；
- ✅ 这套动态调压比普通定压电源，**多出了哪些普通 IC 根本没有的测试项**。

## 一、先破假设：核心电压不是定值，是逐 OPP 在跳的动态变量

先把最大的误解掰过来：**现代 CPU/SoC 的核心电压，不是一个固定值，而是一个随时间不停变的动态量。**

原因是功耗。芯片的动态功耗大致按 **P ∝ C·V²·f** 走——电压 V 是平方项，最费电。要想省电，最有效的就是**在不需要性能的时候，把电压压下去**。可电压又不能随便压：频率越高，晶体管翻转越快，需要的电压越高才不出错。于是芯片厂商把"频率 + 对应最低够用电压"打包成一档一档的工作点，叫 **OPP（Operating Performance Point，工作性能点）**，也常被叫作 **P-state**：

- 干重活（高频）→ 跳到高 OPP：**高频 + 高电压**，保性能；
- 摸鱼（低频）→ 跳到低 OPP：**低频 + 低电压**，省功耗；
- 完全空闲 → 更低的档甚至进 retention（保持态）。

系统（OS 的 governor / 芯片内的功耗管理）根据负载**每秒切换几百上千次 OPP**。所以 VCORE 在示波器上看就是**一串高低不等的电压台阶，随负载来回跳**——这不是故障，这是它**正常的工作方式**。

![图1：破除"电压是定值"——上排：普通定压电源，VCORE 一条平线 0.85V±3%，测纹波/精度就够；下排：现代 CPU 核心，VCORE 是随负载在 0.6V / 0.85V / 1.05V 等 OPP 档位之间不停跳的阶梯波形，标注"每秒跳几百次"。红框圈出跳变沿，标注"真正要测的是这一下"](/images/pi01/ill01_dynamic.png)

> **一句话记住：现代 CPU/SoC 核心电压是随频率实时调的动态变量（DVS/AVS），在一串 OPP 档位间每秒跳几百次。拿"一个定值±3%"去测，测的是一个不存在的工作点——真正要测的是"跳变过程"本身。**

### 一个好记的类比：电压像"按活儿实时调的工资"

把芯片想成一个雇工：

- **活多（高频）就给高工资（高电压）**，让它有劲干快活；活少（低频）就把工资压低（低电压）省成本——这就是 **DVS（Dynamic Voltage Scaling，动态电压调节）**：按当前要跑的频率，查表给一个对应电压。
- 更聪明的老板不照死表发钱，而是**装个监工实时盯着这块硅片到底够不够快**（受温度、工艺、老化影响），**只给"刚好让它按时干完活"的最低工资**——这就是 **AVS（Adaptive Voltage Scaling，自适应电压调节）**，闭环的，能把"以防万一多给的那点余量"（工艺 guardband）也省掉。

但"调工资"这一下有讲究：**①不能抖**（一下发太多——电压过冲，可能打穿芯片）；**②不能慢**（钱还没到账就催开工——电压没稳住就换频，直接死机）；**③顺序不能错**（先涨薪还是先加活，反了要出事）。这三条，正好对应下面要测的三类东西。

![图2：类比图——雇工干活。左：DVS 按活儿多少查表发工资（高频高压/低频低压）；右：AVS 加了个"监工"实时测硅片够不够快，只发最低够用工资，省掉 guardband。下方三条红字警示：①不能抖(过冲) ②不能慢(建立时间) ③顺序不能错(压频时序)](/images/pi01/ill02_analogy.png)

## 二、两套 PC 平台怎么调压：x86 的 SVID vs 高通骁龙的 SPMI

电压要跳，得有人"下令"、有人"执行"。现在的笔记本市场，**两套方案并存**，很多人只熟 x86 那套，碰到高通骁龙笔记本就懵——其实**命令通道不同，但要测的 PI 内容完全一样**。

**x86（Intel / AMD）平台**：
- **CPU 自己算好"我下一步要跳到哪个电压"**，通过一根专用串行总线把电压码（VID）发出去——Intel 叫 **SVID（Serial VID）**，AMD 叫 **SVI2 / SVI3**。
- 接令的是**多相 VRM 控制器**（板上一颗独立芯片，带一堆 DrMOS 功率级），它把 VCORE 调到 CPU 要的电压。
- SVID 是 **3 线**（SCLK 时钟 / SDATA 数据 / ALERT），开漏，速率不高（**≤ 26.25MHz** 量级），除了发 SetVID 改压，还能用 GetReg 反过来读电流、温度等遥测。

**高通骁龙 PC 平台**：
- 调压命令走 **SPMI**（System Power Management Interface，MIPI 标准的 **2 线** 总线：SCLK + 双向 SDATA），也可由片内 **RPMh**（Resource Power Manager-hardened，一个硬件资源管理框架）统一下发。
- 接令并真正把电压调出来的是 **PMIC**（电源管理芯片），由它设 CPU 子系统（APC）的电压。
- 高通还有一手**闭环 AVS**，叫 **CPR**（Core Power Reduction）——见第三节。

两套方案的对照，是本篇最该记住的一张表：

![图3：两平台调压链对照。上路 x86：CPU →(SVID/SVI 串行总线)→ 多相 VRM 控制器 → VCORE；下路 高通骁龙 PC：功耗管理 →(SPMI / RPMh)→ PMIC → APC(CPU)电压。右侧大字标注：命令通道不同，但 PI 关注点完全一致——都要测跳变的建立时间/过冲/压摆率/顺序](/images/pi01/ill03_two_platform.png)

| | x86（Intel / AMD） | 高通骁龙 PC |
|---|---|---|
| 谁下令 | CPU 算好 VID | 功耗管理 / RPMh |
| 命令通道 | **SVID**（Intel）/ **SVI2·3**（AMD），串行 VID 总线 | **SPMI**（MIPI 2 线）/ RPMh |
| 谁执行 | **多相 VRM 控制器** + DrMOS | **PMIC** 设 APC 电压 |
| 闭环 AVS | 平台相关 | **CPR**（硅片自测，给最低够用电压） |
| **PI 关注点** | **建立时间 / 过冲 / 压摆率 / 逐 OPP 精度 / 压频顺序 / 总线解码** | **完全一致** |

> **一句话记住：x86 走 SVID/SVI 命令多相 VRM，高通骁龙走 SPMI/RPMh 命令 PMIC——命令通道和执行者不一样，但"电压跳变要测什么"两边一模一样。**

## 三、闭环 AVS：芯片自己测够不够快，只给"最低够用电压"

上面说的 DVS 是**开环**的：查一张"频率→电压"表，给一个固定电压。但这张表是**出厂按最差情况定的**——为了保证每一颗芯片、在最高温、最老化的极端条件下都不出错，表里的电压会**多留一截余量**，这截余量叫 **guardband（护带）**。大多数芯片、大多数时候根本用不上这截余量，等于**白白多耗电**。

**闭环 AVS 就是来省这截余量的。** 以高通的 **CPR** 为例：芯片内部埋了**传感器**（监视关键路径的实际速度），实时测"这块硅片在当前温度/工艺/电压下到底够不够快"，然后**动态地把电压压到刚好够用的最低点**，再经 RPMh 下发给调节器。硅片好、温度低，就少给点；硅片差、温度高，就多给点——**因材施电**，比开环那张"一刀切留足余量"的表省得多。

这里有个特别容易写错、也容易讲反的点，务必记牢：

> ⚠️ **CPR / 闭环 AVS 的目标是"给刚好够用的最低电压"，不是"追求最高电压"。** 它是往下压（省电），压到"再压就要出错"的临界点上方一点点为止。

![图4：闭环 AVS(CPR) 原理。左：开环 DVS——查表给固定电压，顶上留一大截 guardband 余量(灰色阴影，标"白耗电")。右：闭环 AVS——片内传感器实时测硅片够不够快，把电压压到"临界点 + 一点点安全裕量"，省掉大部分 guardband(绿色箭头向下"省电")。底部注：目标是最低够用电压，不是最高](/images/pi01/ill04_cpr.png)

## 四、跳变那一下，到底要测什么：四件事 + 一张矩阵

认知理顺了，回到测试。电压不再是"一个点一个限值"，而是**每一次跳变都要过关**。核心是四件事：

**① 电压跳变建立时间（settling time）。** 从收到调压命令、电压开始变，到**稳定进入新电压的容差带内**（业内常用进入 **±2%** 判定），要多久。典型是 **µs 级**。这件事是**生死线**：如果系统要升频，电压却还没稳到高档就把频率拉上去，芯片在"高频 + 不够的电压"下必然算错——**直接死机**。所以建立时间必须**赶在频率切换之前**完成。

**② 跳变过冲 / 下冲（overshoot / undershoot）。** 升压时电压容易先冲过头再回落（过冲），降压时容易先掉过头（下冲）。过冲不能越过芯片的**绝对最大电压（abs-max）**，否则会打穿；下冲不能掉破**最低工作电压**，否则同样算错。

**③ 受控压摆率（slew rate，V/µs）。** 电压变化的斜率，由 PMIC/VRM 按命令设定。**太快** → 过冲大、冲击应力大；**太慢** → 建立时间不够、误了换频时序。所以它是被"**控制**"的，要验它是否落在设定范围。

**④ 逐 OPP 静态精度。** 每一个电压档，落点准不准。VID/DAC 的步进通常是 **5mV 或 10mV** 一档（如 Intel VR13：5mV 或 10mV/step），要**逐档**验落点，不能只测一个档。

![图5：跳变一下的四个考点(放大一个电压 step)。横轴时间、纵轴电压。标注：①settling time(从命令到进入±2%带)②overshoot(升压顶部尖峰,不得越 abs-max 红线)③slew rate(上升斜率 V/µs)④目标档落点精度(DAC 5/10mV)。左侧竖排"命令沿"](/images/pi01/ill05_step.png)

把这四件事和 OPP 交叉，判据就**从"一行固定限值"变成了一张矩阵**——这是动态调压和普通定压电源在"判据形态"上最根本的区别。下面用一个**自制示例**（零涉密，数值仅示意）把这张矩阵**填满**，让你看清它实际长什么样：

| OPP 档 | 频率 | 目标电压 | 静态精度 | 过冲/下冲预算 | 建立时间 | 压摆率 |
|---|---|---|---|---|---|---|
| P-idle（空闲） | 0.8 GHz | 0.60V | ±10 mV | 下冲不破 0.55V | < 20 µs | 8~12 mV/µs |
| P-nom（标称） | 2.4 GHz | 0.85V | ±10 mV | 过冲不越 1.15V | < 15 µs | 8~12 mV/µs |
| P-turbo（睿频） | 3.5 GHz | 1.05V | ±10 mV | 过冲不越 1.15V | < 10 µs | 8~12 mV/µs |
| retention（保持） | — | 0.45V | ±15 mV | 只需单调 | 不涉换频 | 慢速可 |

**怎么读这张表**（这才是收获点）：

- **纵向**：每一档都要单独验，不能只测标称档。**睿频档（1.05V）的过冲最危险**——它离 abs-max（这里 1.15V）最近，一旦过冲越线就打穿。
- **横向**：同一档要同时过静态精度、过冲、建立时间、压摆率四关，**缺一不可**。
- **建立时间随频率收紧**：档越高、要跑的频率越急，留给电压稳定的窗口越短（这里从 20µs 收到 10µs）——所以**睿频档的建立时间是最难的一关**，也是"没稳住就换频→死机"最容易发生的地方。
- **retention 档特殊**：它不参与换频，只要求电压单调、别掉太低丢数据，判据和别的档不一样。

这张表就是你测试计划的**骨架**：拿到一颗真实芯片，把它的 OPP 表（真实值属 NDA，只在内部填）套进这个结构，一格一格验过去，就不会漏。

> **一句话记住：动态调压要测的是"每一次跳变 + 每一个档"，判据是 OPP×{静态精度、过冲、建立时间、压摆率} 的矩阵，不是一行固定限值。其中睿频档的建立时间赶不上换频 = 死机，是头号生死项。**

![图6：OPP 判据矩阵填数示例（自制零涉密）。表格：纵轴 P-idle/P-nom/P-turbo/retention 四档，横轴 频率/目标电压/精度/过冲预算/建立时间/压摆率，每格填示意数值。用红色高亮 P-turbo 行的"过冲不越1.15V"和"建立时间<10µs"两格，标注"最危险/最难的一关"](/images/pi01/ill06_matrix.png)

## 五、最容易漏的一项：压频协调时序

前面四件事测的是"一次跳变本身好不好"。还有一件事，比单次跳变更隐蔽、也更致命——**电压和频率这两件事，谁先谁后**。

规则只有两句，但反了就出事：

- **升频：先加电压，等电压稳住，再升频。** 因为高频需要高电压撑着，电压没到就升频 = 高频跑在不够的电压上 = 算错/死机。
- **降频：先降频，再降电压。** 先把频率降下来（对电压需求变低），再压电压才安全；反过来先降压，频率还在高位就会瞬间"电压不够"。

这不是测某个电压值准不准，而是测**两个事件的先后顺序和间隔对不对**——属于**时序（timing）**问题。测的时候要**同屏对齐看**：一路解调压总线命令（什么时候下的令），一路看电压轨真正响应（什么时候到位），再对照频率切换的时刻，验证顺序没错、间隔留够。

![图7：压频协调时序。上路"升频"：时间轴上 先"加压命令"→电压爬升并稳定→再"升频"，标绿✓；下方画一个反例 先升频后加压 标红✗"高频欠压→死机"。下路"降频"：先"降频"→再"降压"标绿✓。用箭头标出"顺序"和"间隔"两个要点](/images/pi01/ill07_seq.png)

> **一句话记住：升频先加压后升频、降频先降频后降压——这是"顺序对不对"的时序问题，不是"电压准不准"的数值问题。解总线命令 + 看电压响应 + 对照换频时刻，三者同屏对齐验证。**

## 六、怎么测：轨探头 DC 耦合 + 偏置，配总线命令解码触发

测法是这一篇的灵魂。动态调压的测量，和普通看纹波**很不一样**，有两个关键点。

**关键点 1：混合域——模拟波形 + 总线命令，同屏对齐。**

普通电源测试只看一路模拟波形。这里要**两路一起看**：

- **Ch1**：**电源轨探头**测 VCORE 的电压波形（看跳变过程）；
- **Ch2 / 数字**：挂在**调压总线**上（x86 = SVID/SVI，高通 = SPMI），**解码出命令**（什么时候、把哪条轨、调到哪个电压）。
- **触发**：触发在**总线的调压命令**上，或在**电压跳变的边沿**上，用单次/正常触发，把"**命令→跳变→稳定**"这三段同屏抓下来。
- **带宽**：要 ≥ 跳变速度的 **5 倍**，才能看清跳变细节；深存储抓多次跳变做统计。

这样一屏下来，你能直接量出：命令下达 → 电压开始动的延迟、建立时间、过冲、落点，全部对齐着看。

![图8：怎么测(混合域同屏)。Ch1 电源轨探头(DC+偏置)测 VCORE 跳变波形；Ch2 数字探头挂 SVID/SPMI 总线，下方解码出命令帧"SetVID 0.85→1.05V"。竖虚线把画面分三段：命令 / 跳变 / 稳定。触发标在命令帧。右上标"带宽≥跳变5×"](/images/pi01/ill08_measure.png)

**关键点 2：必须 DC 耦合 + 轨探头偏置——这里千万别用 AC 耦合。**

这点最容易和第 08 篇"测纹波"搞混，务必分清：

- 第 08 篇测**纹波**：VCORE 是个大直流（比如 0.85V）上面叠着几 mV 的小交流。为了把小交流看清，用 **AC 耦合**把那 0.85V 的直流挡掉、放大交流——**08 这么做完全正确**。
- 本篇测**跳变**：我们要测的**恰恰就是那个直流电平本身在怎么变**（从 0.85V 跳到 1.05V）。如果用 AC 耦合，它那个 ~10Hz 的高通特性会**把要测的直流台阶滤掉**，你看到的不再是一个干净的电压台阶，而是被微分成的尖脉冲——**测的东西全没了**。

所以本篇必须用 **DC 耦合**，并用**专用有源电源轨探头**加 **DC 偏置**（把 0.85V 这个基准平移到屏幕中间，再用小量程高分辨率看跳变细节）。普通 10:1 无源探头噪声太大，不合格。

> ⚠️ **和第 08 篇不矛盾**：08 测"大直流上的小交流"→AC 耦合挡直流；本篇测"直流本身怎么动"→必须 DC 耦合 + 偏置。**目的不同，耦合方式就相反，08 本身没错、不用改。**

![图9：耦合方式对照。左"AC 耦合(测纹波,第08篇)"：大直流被挡掉,只留小纹波,✓适合纹波。右"DC 耦合+轨探头偏置(本篇测跳变)"：完整保留 0.85→1.05V 的电压台阶,✓适合跳变。中间画一个反例:用 AC 耦合测跳变→台阶被微分成尖脉冲,✗"要测的内容被滤掉"](/images/pi01/ill09_coupling.png)

**测量小抄：** 示波器内置的 **Overshoot** 测过冲、**Settling Time**（进入 ±2%）测建立时间，直接读数；再配总线解码选件（SVID/SVI/SPMI），命令和波形就对齐了。

## 七、和普通电源 IC 比，多出了哪些活

把动态调压和普通定压电源 IC（定压 DCDC/LDO，即正篇讲的那些）摆一起，你会发现它**打破了普通电源测试的一个默认假设**：电压是定值。假设一破，测试项、判据形态、仪器、失效模式全变了：

| 维度 | 普通定压电源 IC | 动态调压 AVS/DVS |
|---|---|---|
| 电压性质 | **定值**（1.0V±x%） | **动态变量**（逐 OPP 跳） |
| 测试粒度 | 单轨输出一个点 | 单轨 × **每个 OPP** |
| 新增核心测试项 | —— | **跳变建立时间 / 过冲 / 压摆率 / 逐 OPP 精度 / 压频顺序 / 总线命令解码** |
| 判据形态 | **一行固定限值** | **OPP × 多列 矩阵**（含时序类判据） |
| 触发方式 | 自由 / 边沿 | **总线调压命令** 或 电压边沿 |
| 仪器 | 普通探头即可 | **有源轨探头(DC+偏置) + 总线协议解码** |
| 软件依赖 | 上电即测 | **需固件/OS 强制切 OPP**（cpufreq governor / 压测工具） |
| 典型失效（普通 IC 没有） | —— | 跳变过冲打穿、**没稳住就换频→死机**、压频顺序错 |

这张表里加粗的每一项，都是**普通电源 IC 根本不会碰到、这里新增**的——这也是读者最该带走的"增量清单"。

## 八、把一条动态轨的测试用例写出来（模板 + 实例）

道理讲完，落到活儿上——**拿到一条 DVS 轨（比如 VCORE），怎么把它的测试用例写出来？** 给你一套可以直接套的流程和模板，这也是这一篇最该带走的东西。

**第一步：从原理图 + datasheet 提取这条轨的档案。**

- 它是不是 DVS 轨？（不是就按普通定压测）
- 调压走哪条总线？（x86=SVID/SVI；高通=SPMI）执行者是谁？（VRM / PMIC）
- 有没有**远端 Kelvin 感测**？—— 有的话，**精度和纹波要探感测点/负载端，不是电感输出端**。
- OPP 有几档？各档的目标电压、abs-max、最低工作电压是多少？（真实值查平台 NDA 文档，只填内部）

**第二步：按"跳变四件事 + 压频时序"把测试项列全。** 对照第四、五节，一条 DVS 轨至少要有这些用例：

| 用例编号 | 目的 | 激励 / 前置 | 测点与仪器 | 判据 |
|---|---|---|---|---|
| DVS-01 | 逐 OPP 静态精度 | 固定在每一档 | 感测点，轨探头 DC | 各档落点在 ±LSB |
| DVS-02 | 升压跳变过冲 + 建立时间 | 强制低档→高档切换 | 轨探头 DC+偏置，总线命令触发 | 过冲不越 abs-max；settling < 换频窗口 |
| DVS-03 | 降压下冲 + 建立时间 | 强制高档→低档切换 | 同上 | 下冲不破最低工作压；settling 达标 |
| DVS-04 | 受控压摆率 | 触发一次跳变 | 轨探头，测斜率 | 落在 PMIC/VRM 设定带 |
| DVS-05 | 压频协调时序 | 跑真实负载让系统自动切档 | 轨探头 + 总线解码 + 频率标志 | 升频先加压、降频先降频，顺序/间隔正确 |
| DVS-06 | 遥测交叉核对（选做） | GetReg 读回电压/电流 | 总线解码 | 遥测值与实测一致 |

**一个填好的用例实例（自制示例，零涉密）：**

> **DVS-02 · VCORE 升压跳变（P-nom → P-turbo）**
> - 目的：验证 0.85V→1.05V 跳变的过冲与建立时间
> - 前置：强制 CPU 从 2.4GHz 睿频到 3.5GHz
> - 测点：VCORE 远端感测点，有源轨探头 1:1、DC 耦合、偏置 0.9V、全带宽
> - 触发：SVID 总线 SetVID 命令帧
> - 判据：**过冲峰值 ≤ 1.15V（abs-max）**；**建立时间（进入 1.05V±2%）< 10µs**；且必须**早于频率切到 3.5GHz 的时刻**

## 九、常见坑

- **拿定压思路测动态轨**：把跳来跳去的 VCORE 当成"电源不稳/故障"，其实是它正常在跳 OPP。先搞清这条轨是不是 DVS 轨。
- **只测一个电压档**：动态轨要**逐 OPP** 测过冲/建立时间/精度，只测一个档等于漏测大半。
- **用 AC 耦合测跳变**：直流台阶被高通滤成尖脉冲，测的东西全没了（详见第六节，和第 08 篇目的不同）。
- **不解码总线、只看波形**：不知道命令什么时候下的，就量不准"命令→跳变"的延迟，也验不了压频顺序。
- **建立时间只看"到了没"，不看"赶没赶上换频"**：稳定进容差带的时刻，必须早于频率切换的时刻，否则高频欠压死机。
- **精度探错点**：有远端感测的轨，探电感输出端会把线阻压降算进去——**要探感测点/负载端**。
- **把 CPR/闭环 AVS 理解反**：它是"往下压到最低够用"省电，不是"追求最高电压"。
- **SVID / SVI / SPMI / AVSBus 混为一谈**：SVID 是 Intel、SVI2/3 是 AMD、SPMI 是 MIPI（高通用）、AVSBus 是 PMBus 的开放标准——通道不同、别混。
- **软件没配合**：不跑压测/不强制切 OPP，轨根本不跳，你可能测了个"假静态"。

---

## 一句话总结

现代 CPU/SoC 的核心电压不是定值，而是**随频率实时跳变的动态变量**（DVS 按表调、AVS 闭环给最低够用电压省 guardband）；x86 用 **SVID/SVI 命令多相 VRM**、高通骁龙 PC 用 **SPMI/RPMh 命令 PMIC**——命令通道不同，PI 该测的**完全一致**。真正要测的是**跳变过程本身**：逐 OPP 的**建立时间、过冲/下冲、受控压摆率、静态精度**，外加**压频协调时序**（升频先加压、降频先降频后降压，错了就死机）。测法上要**混合域**看——**轨探头 DC 耦合 + 偏置**抓电压台阶、**总线解码触发**对齐命令，同屏拿下"命令→跳变→稳定"。**别用 AC 耦合**（那是测纹波的，会把要测的台阶滤没）。相比普通定压电源，判据从"一行限值"升级成"OPP×测试项矩阵"，多出一整套普通 IC 没有的动态测试项。

## 🔧 工程师速查（附录）

### 动态调压核心概念

| 概念 | 一句话 | 关键点 |
|---|---|---|
| **DVS**（动态电压调节） | 按当前频率查表给对应电压 | 开环；表里含 guardband 余量 |
| **AVS**（自适应电压调节） | 闭环给"最低够用电压" | 省 guardband；高通实现 = **CPR** |
| **OPP / P-state** | 频率 + 对应电压 打包成一档 | 每秒切换几百次，VCORE 逐档跳 |
| **guardband** | 出厂为最差情况多留的电压余量 | AVS 就是来省它的 |
| **CPR**（Core Power Reduction） | 片内传感器测硅片速度，压到临界+微裕量 | 目标=最低够用，不是最高 |

### 两平台调压链对照

| | x86（Intel/AMD） | 高通骁龙 PC |
|---|---|---|
| 命令通道 | **SVID**(Intel) / **SVI2·3**(AMD)，3 线串行，≤26.25MHz，SetVID/GetReg | **SPMI**(MIPI 2 线, ≤26MHz) / **RPMh** |
| 执行者 | 多相 VRM 控制器 + DrMOS | PMIC 设 APC 电压 |
| 闭环 AVS | 平台相关 | **CPR** |
| VID/DAC 步进 | VR13：5mV 或 10mV/step | PMIC DAC 设点 |
| 开放对照标准 | **AVSBus**（PMBus 1.3 Part III，≤50MHz，改压 <640ns，含压摆率命令） | 同 |

### 跳变四件事 + 判据

| 测试项 | 判据要点 | 头号风险 |
|---|---|---|
| 建立时间 settling | 进入 ±2% 的用时，须 < 换频窗口 | **没稳住就换频 = 死机** |
| 过冲/下冲 | 过冲不越 abs-max、下冲不破最低工作压 | 过冲打穿芯片 |
| 受控压摆率 slew | 落在 PMIC/VRM 设定带 | 太快过冲、太慢误时序 |
| 逐 OPP 静态精度 | 每档落点在 ±LSB（5/10mV 量级） | 只测一档 = 漏测 |

### 压频协调时序（顺序判据，不是数值）

- **升频**：先加电压 → 稳住 → 再升频（反了 = 高频欠压死机）
- **降频**：先降频 → 再降电压（反了 = 瞬时欠压）
- 验法：解总线命令 + 看电压响应 + 对照换频时刻，**三者同屏对齐**
