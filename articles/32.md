# 抖动 Jitter：时钟边沿平均只差了 2ps，为什么误码率还是不达标？——不把抖动拆开，你根本治不了它

<div class="article-meta"><span class="badge">📘 模块三 · 信号完整性 SI · 抖动 Jitter</span><span class="badge">⏱ 约 26 分钟</span><span class="badge qr">🔧 含工程师速查</span></div>


一条 5Gbps 的串行链路，UI 是 200ps。板子回来测时钟质量，工程师抓了一段边沿，仪器报出一个**抖动均方根值（RMS）只有 2ps**——这数看着漂亮极了，才 UI 的百分之一。他松了口气，觉得时序余量稳稳的。

可一挂上误码率测试仪，**误码率死活压不到 10⁻¹² 的指标**。回头再仔细看眼图，眼睛的水平开口被交叉点那两簇又厚又糊的东西啃掉了一大块——**眼宽被抖动吃掉了差不多 0.2UI，也就是 40ps**。

问题就出在这：**他只看了均方根 2ps 这个"平均"，却没看抖动的"最坏"。** 抖动里有一部分是**随机的**，它服从高斯分布、**没有上限**——你测的时间越久、跑的位数越多，它的峰峰值就越大。按 10⁻¹² 这么低的误码率去算，这 2ps 的随机抖动，峰峰值要放大到差不多 28ps；再加上链路里那些**确定性的**抖动，一共 40ps。**平均值骗了他，最坏值才是真相。**

**这就是抖动测试的核心：光测一个总数没用，你得把抖动拆开——拆成随机的和确定的，分开描述、分开计算、分开去治。** 这一篇，我们就从"抖动是什么"开始，一步步搞懂这套拆解的功夫。

**读完这篇，你能搞懂：**

- ✅ 抖动到底是什么——**边沿偏离理想位置的误差（TIE）**，怎么把眼宽啃窄；
- ✅ **什么时候该测抖动、要测哪几项**——发送端抖动、接收端容限、参考时钟，一次分清；
- ✅ 为什么必须**拆开**——随机抖动**高斯无界**，只看均方根会骗你；
- ✅ 抖动家族——**RJ 随机** vs **DJ 确定**（里头还有 ISI、PJ 周期、DCD 占空比）；
- ✅ 总抖动怎么算——**双狄拉克模型**：TJ(BER) = DJ + 2·Q·RJ，按误码率折算；**判据/mask 从哪查**、怎么盯余量；
- ✅ 怎么测、怎么分离——**TIE → 直方图 / 浴盆曲线 / 频谱 → 抖动分解**；
- ✅ 怎么按类型**对症下药**——RJ、PJ、DCD、ISI 各有各的治法，附一张**判读流程图**。

## 一、抖动是什么：边沿偏离理想位置的那点晃动

先给个最朴素的定义：**抖动，就是信号的边沿，在时间轴上偏离它"本该在的位置"的那点误差。**

理想情况下，一个 5Gbps 的信号，边沿应该**每隔正好一个 UI（200ps）出现一次**，像节拍器一样准。可现实里，每个边沿都会**早一点或晚一点**到，忽前忽后地抖。把"第 n 个边沿的实际时刻"减去"它理想应该出现的时刻"，得到的这串误差，有个专门的名字叫 **TIE（Time Interval Error，时间间隔误差）**。TIE 就是抖动分析的**原始数据**——后面所有的直方图、浴盆曲线、频谱，都是从这一串 TIE 算出来的。

**抖动是怎么啃眼宽的？** 回到上一篇的眼图。眼图是把成千上万个 UI 叠在一起，理想情况下所有边沿都精确落在同一个位置，交叉点应该是**细细的一条线**。可一旦边沿在抖，这些边沿就在时间轴上**左右散开**，交叉点那一簇就从一条线摊成**又宽又厚的一团**。这一团往左右一摊，眼睛的水平开口——眼宽——就从两侧被啃窄了。**所以你在眼图上看到的"交叉点又糊又厚"，量出来就是抖动的大小。眼宽还剩多少，就是被抖动吃剩的时序余量。**

## 二、什么时候该测抖动、要测哪几项：先分清项目里谁要做、做哪三项

和眼图一样，抖动分析也不是每条线都要做。拿到一个项目，先分清：**这条链路要不要做抖动测试？要做的话，测哪几项？**

**谁要做抖动测试？** 判断标准和眼图一脉相承——**高速、串行、靠 CDR 从数据里恢复时钟**的链路，眼宽本来就紧、误码率要求又低（10⁻¹² 级），时序余量差一点点就翻车，就必须把抖动拆开测。典型的就是 **PCIe、USB3/USB4、SerDes、以太网** 这些。反过来，**低速、单端的总线（I2C / SPI / UART）一般不做抖动分解**——它们速率低、边沿富余，看**建立/保持时序**就够了。

**要做的话，测哪三项？这是最容易漏的地方。** 抖动测试不是"抓一段信号量个数"这么简单，一个完整的接口，抖动通常要测三类，各查一件事：

- **① 发送端抖动（Tx jitter）**：量芯片**发出去**的信号有多抖——拆出 TJ / RJ / DJ，按指定误码率折算，对一致性规范的 **mask / TJ 上限**判合格。这是"我发的信号干不干净"。
- **② 接收端抖动容限（Rx jitter tolerance）**：反过来考察接收端的**忍耐力**——人为往信号里**注入**规定量的抖动、扫不同频率，看接收端的 **CDR 还能不能正确恢复数据**。这是"别人发脏一点，我还认不认得"，考察的是 CDR 的跟踪能力。
- **③ 参考时钟抖动（reference clock jitter）**：很多高速接口对**参考时钟**单独有要求——量它的**相位噪声 / 周期抖动**，判据在接口规范和时钟器件手册里。参考时钟脏，整条链路的抖动都跟着遭殃。

**所以拿到项目先照这张清单点一遍：** 有 PCIe / USB3 / SerDes 这类高速串行口 → 发送端抖动、接收端容限、参考时钟**三项都要按规范测**，一项都不能漏；只有低速单端总线 → 不做抖动分解，回去看时序。**先把"测哪几项"定清楚，测试计划才不会缺项。**

![图8：什么时候测抖动、测哪三项。顶部——谁要做：高速·串行·靠CDR恢复时钟·眼宽紧/误码率严→PCIe、USB3/4、SerDes、以太网。三类测试项：①发送端抖动Tx（量TJ/RJ/DJ@BER、对一致性mask、看发出的信号多抖）②接收端抖动容限Rx（注入抖动扫频率、看CDR还能不能正确恢复、考察忍耐力）③参考时钟抖动refclk（相位噪声/周期抖动、接口单独有spec、查规范+datasheet）。底部：低速单端I2C/SPI/UART不做抖动分解，看建立/保持时序即可](/images/32/ill08_when.png)

> **一句话记住：高速串行、靠 CDR 的链路（PCIe/USB3/SerDes/以太网）才做抖动测试，且要测三项——发送端抖动（TJ/RJ/DJ 对 mask）、接收端抖动容限（注入抖动看 CDR 能不能扛）、参考时钟抖动（相位噪声，查规范+手册）；低速单端总线看时序即可。**

## 三、为什么必须拆开：随机抖动是高斯无界的，只看均方根会骗你

开头那个工程师栽的跟头，值得掰开细讲，因为它是抖动测试里**最常见、也最致命**的一个坑。

抖动按"脾气"分，就两大类：**随机的**和**确定的**。它俩对误码率的影响，完全是两套逻辑。

**随机抖动 RJ（Random Jitter）——高斯分布、没有上限。** 它来自器件里那些根本消不掉的物理噪声：电阻的热噪声、半导体的散粒噪声、时钟源的相位噪声。这类噪声服从**高斯分布**，它有个要命的特性：**理论上没有上限**。高斯分布的尾巴无限延伸，意味着**只要你测得够久、跑的位数够多，早晚会撞上一个特别大的抖动值**。所以 RJ **不能用峰峰值来描述**——你测 1 分钟得到的峰峰值，和测 1 小时得到的，根本不是一个数，测得越久越大。RJ 只能用**均方根值 RMS（也就是高斯分布的 σ）**来描述，这个数是稳定的。

**问题就在这：均方根 2ps，不代表最坏只有 2ps。** 高斯分布里，超过 1 个 σ 的概率不小，超过 3 个 σ、7 个 σ 的概率越来越小，但**永远不为零**。你要求误码率做到 10⁻¹²，意思就是"一万亿个 bit 里最多错一个"，那你就必须考虑那些**一万亿分之一才出现一次的、特别大的抖动**。数学上，误码率越低，对应要考虑的 σ 倍数越多：

- 误码率 10⁻¹²，对应大约 **±7 个 σ**（总共约 14 个 σ 的峰峰摆动）；
- 误码率 10⁻⁹，对应大约 ±6 个 σ。

所以 RMS 是 2ps 的随机抖动，**按 10⁻¹² 算，峰峰值 = 14.07 × 2ps ≈ 28ps**。看，2ps 一下子变成 28ps——**这不是仪器骗你，是你只看了平均、没按误码率去折算最坏。**

**确定抖动 DJ（Deterministic Jitter）——有界、有具体成因。** 和 RJ 相反，DJ 是有明确来源的（电源纹波、串扰、码型、阻抗不连续……），它**有上限、有边界**，用**峰峰值**来描述就够了，测多久都是那么大，不会一直涨。

**看明白这个区别，就懂了为什么必须拆开：** RJ 要按误码率放大、用 σ 描述，DJ 直接用峰峰值——两者的算法完全不同。你要是把它们混在一起测个总峰峰值，测短了会**严重低估**（没撞上 RJ 的大尾巴），测长了又不稳定；只有**先拆开、再按各自的规则合起来**，才能算出一个在指定误码率下**可信、可复现**的总抖动。

![图3：为什么必须拆开。左：随机抖动RJ——高斯钟形分布、尾巴无限延伸（无界），累积样本越多峰峰越大，只能用RMS(σ)描述；标注10⁻¹²→±7σ、10⁻⁹→±6σ；RMS 2ps → 峰峰14.07×2≈28ps。右：确定抖动DJ——有界、有明确边界，用峰峰值描述、测多久都一样。中间警示：只看均方根2ps会严重低估最坏情况](/images/32/ill03_split.png)

> **一句话记住：随机抖动 RJ 高斯无界，样本越多峰峰越大，只能用均方根 σ 描述，必须按误码率折算（10⁻¹²≈14σ，2ps→28ps）；确定抖动 DJ 有界，用峰峰值描述。两者算法不同，必须拆开。**

## 四、抖动家族：一棵树看清 RJ、DJ 和它的小弟们

把抖动这一大家子画成一棵树，你就再也不会搞混了。最顶上是**总抖动 TJ（Total Jitter）**，往下分两大支：

**左支：随机抖动 RJ。** 前面讲过了——高斯、无界、热噪声/散粒噪声/相位噪声，用 σ 描述。这一支通常没有再往下分。

**右支：确定抖动 DJ。** 有界、有成因，它下面还藏着好几个"小弟"，各有各的脾气：

- **ISI / 数据相关抖动 DDJ（Data-Dependent Jitter）**：这个我们其实上一篇就见过了——就是**码间干扰**。走线损耗让高频衰减，连续翻转的位和连续不变的位，到达接收端的边沿位置不一样，于是**边沿抖动跟着码型走**：这一位前面是 `000` 还是 `010`，边沿到的时刻就差一点。它跟码型强相关，所以叫数据相关抖动。**治它的办法就是上一篇说的均衡和降损耗。**

- **周期抖动 PJ（Periodic Jitter）**：来自某个**固定频率的周期性干扰**，最典型的就是**电源纹波**（开关电源的开关频率）、**串扰**（邻线的固定频率信号）、**PLL 的参考杂散**。它的特征是**周期性**——在频谱上会冒出一个**尖峰**，尖峰的频率直接告诉你干扰源是谁。这是 PJ 最好用的定位线索。

- **占空比失真 DCD（Duty Cycle Distortion）**：上升沿和下降沿走得不对称，或者判决**阈值偏了**，导致高电平和低电平的宽度不相等，边沿位置系统性地偏移。差分信号里正负两条不平衡，也会引起 DCD。

- **有界不相关抖动 BUJ**：像串扰这种，有界、但跟本路数据码型不相关，单列一类。

**为什么要认清这些小弟？因为它们各有各的"指纹"和治法：** ISI 跟码型走、PJ 在频谱上是尖峰、DCD 让高低电平不等宽。测的时候把它们分离出来，一看是哪种占主导，就知道该去查损耗、查电源、还是查占空比——**抖动分解的价值，就是把"时序余量不够"这个笼统的病，诊断到具体的病灶。**

![图4：抖动家族树。顶=总抖动TJ；左支=随机抖动RJ（高斯/无界/热噪声散粒噪声相位噪声/用σ描述）；右支=确定抖动DJ（有界/用峰峰值），下分：ISI-DDJ数据相关抖动（跟码型走、损耗引起→均衡治）、PJ周期抖动（电源纹波/串扰/PLL杂散、频谱上尖峰）、DCD占空比失真（上升下降不对称/阈值偏）、BUJ有界不相关（串扰）。旁注：各有指纹，分离后对症下药](/images/32/ill04_family.png)

> **一句话记住：TJ 总抖动 = RJ 随机（高斯无界、σ）+ DJ 确定（有界、峰峰）。DJ 里还有 ISI（跟码型走、损耗引起）、PJ（周期性、频谱尖峰、电源/串扰）、DCD（占空比/阈值不对称）。认清指纹，才能对症。**

## 五、总抖动怎么算：双狄拉克模型，按误码率折算

拆开之后，怎么把它们合成一个"在指定误码率下的总抖动"？工程上最常用的是一个简洁的模型——**双狄拉克（Dual-Dirac）模型**。

它的思路很直白：**把所有确定抖动 DJ，近似成两个相距 DJ（峰峰值）的尖峰（两个狄拉克函数）；再把随机抖动 RJ 的高斯分布，分别扣在这两个尖峰上。** 这样合出来的总抖动分布，就是两个"带高斯拖尾的峰"。于是在某个误码率下，总抖动的算法就变成一个漂亮的加法：

**TJ(BER) ＝ DJ(峰峰值) ＋ 2 × Q(BER) × RJ(RMS)**

这里的 **Q(BER)**，就是把误码率翻译成"要考虑多少个 σ"的那个系数（和上一篇眼图里 Q 因子是同一套换算）：

| 误码率 BER | Q 值 | 总抖动里的 RJ 项系数 2Q |
|---|---|---|
| 10⁻⁹ | 6.0 | 12.0 |
| 10⁻¹² | 7.03 | 14.07 |
| 10⁻¹⁵ | 7.94 | 15.88 |

拿开头那条链路套一下：假设仪器分离出来 **RJ 的 RMS ＝ 2ps、DJ 的峰峰 ＝ 12ps**，要求误码率 10⁻¹²：

**TJ ＝ 12ps ＋ 14.07 × 2ps ＝ 12 ＋ 28.1 ≈ 40ps。**

40ps 正好是 0.2UI——**这就是为什么眼宽被啃掉了 0.2UI，误码率压不下去。** 你看，**只要你把 RJ 和 DJ 分开测出来，就能不用真跑到 10⁻¹²，直接算出总抖动够不够、眼宽还剩多少**——这跟上一篇用 Q 因子和浴盆曲线外推误码率，是同一套省时间的思路。

**这个公式也直接告诉你该去治谁：** 如果 TJ 超标，看是 **DJ 项大**（那就去治确定性成因：损耗、电源、串扰、占空比），还是 **RJ 项大**（RJ 被 2Q 放大了 14 倍，杀伤力最猛，得去治时钟源相位噪声、降底噪）。**RJ 每降 1ps，在 10⁻¹² 下总抖动就降 14ps**——所以低噪声的时钟源、干净的电源，往往是压总抖动性价比最高的一手。

**算出 TJ 之后，拿什么判合格？判据从哪查？** 和眼图的 mask 一样，抖动的判据**不是自己定的，全在接口的一致性规范里**：PCIe 查 PCI-SIG 的 Base/CEM 规范、USB 查 USB-IF 的 Compliance、以太网查 IEEE 802.3 对应速率条款——里面直接给出指定误码率下的 **TJ 上限、RJ/DJ 的分项限值、以及接收端抖动容限的模板（jitter tolerance mask，一张"注入抖动的频率—幅度"曲线）**。参考时钟的抖动/相位噪声要求，则在接口规范加上**时钟器件的 datasheet**。做测试计划时，第一步就是**把这条链路对应的规范章节、TJ 预算和容限模板找出来**，照着它列测试项、设判据，才不会漏项、也不会用错标准。

**同样别只看过没过，要盯"余量%"。** 把算出的 **TJ@BER 和 UI 时序预算**一比，看还剩多少余量。刚好压线过不等于安全——**量产的批次波动、温度漂移、电源老化都会再吃掉一截抖动余量**。经验上余量低于约 10%~15% 的链路，实验室过了、量产照样偶发误码。**看余量、留裕度，比单看一次 pass 靠谱得多。**

![图5：总抖动双狄拉克模型。DJ近似成相距DJ峰峰的两个尖峰，各扣一个RJ高斯拖尾；公式TJ(BER)=DJ峰峰+2·Q·RJ_RMS。Q表：10⁻⁹→6/10⁻¹²→7.03/10⁻¹⁵→7.94。算例：RJ_RMS 2ps + DJ_pp 12ps @10⁻¹² → 12+14.07×2≈40ps=0.2UI。旁注：RJ被2Q放大14倍、杀伤最猛，RJ每降1ps总抖动降14ps](/images/32/ill05_dualdirac.png)

> **一句话记住：双狄拉克模型 TJ(BER) = DJ峰峰 + 2·Q·RJ_RMS。Q 把误码率翻成 σ 倍数（10⁻¹²→7.03，2Q=14.07）。RJ 被放大 14 倍、最凶——2ps 的 RJ 加 12ps 的 DJ，@10⁻¹² 就是 40ps=0.2UI。分开测就能外推总抖动，不用真跑到 10⁻¹²。判据（TJ 上限/容限 mask）查接口规范（PCI-SIG/USB-IF/IEEE 802.3）+ 时钟手册，别只看 pass，要盯余量%。**

## 六、怎么测、怎么分离：从 TIE 到直方图、浴盆、频谱

抖动分析的全部原料，就是那串 **TIE**（每个边沿相对理想时钟的误差）。现代示波器和误码仪，会用**恢复时钟（CDR）**建立"理想边沿"的参照，量出每个边沿的 TIE，然后从这串数据里，用三张图把 RJ、DJ 各分量拆出来：

**① 直方图（Histogram）——一眼看是随机还是确定为主。** 把所有边沿的 TIE 统计成一张分布直方图。**形状会直接告诉你成分**：如果是一个漂亮的**单峰高斯钟形**，说明 RJ 占主导；如果分布**分裂成两个（或多个）峰、或者边界很硬（有界）**，说明 DJ 占主导（比如 DCD 会把它劈成两个峰）。这是最快的定性判断。

**② 浴盆曲线（Bathtub）——分离 RJ 和 DJ 的定量利器（承接上一篇）。** 还记得上一篇的浴盆曲线吗？把采样点横扫，画出误码率随位置的浴盆。**它的形状能把 RJ 和 DJ 分开**：浴盆**底部平坦的那一段宽度 ≈ DJ**（确定抖动是有界的，在这段里误码率极低）；而**两侧陡壁的斜率，由 RJ 决定**——沿着高斯的规律往下延伸，就能**外推到 10⁻¹²**，量出那个误码率下的总抖动和真实眼宽，不用真的跑那么久。

**③ 频谱 / 相位噪声——专抓周期抖动 PJ、定位干扰源。** 把 TIE 做一次傅里叶变换，看它的频谱。**周期抖动 PJ 会在频谱上冒出一个明显的尖峰**，尖峰的频率就是干扰的频率——**这是定位 PJ 来源的金钥匙**：尖峰在开关电源的开关频率上，就是电源纹波耦合进来了；在某条邻线的信号频率上，就是串扰；在 PLL 参考频率的整数倍上，就是参考杂散。而随机抖动 RJ 在频谱上是**平平的一片本底噪声**，没有尖峰。

**④ 一键抖动分解（Jitter Decomposition）。** 高端示波器和误码仪（BERT）能把上面这套自动化：给它一段信号，它直接吐出 **TJ、RJ、DJ，以及 DJ 里的 PJ、DCD、ISI 各占多少**。你要做的，是**看懂这份分解报告**——哪个分量最大，就往哪个方向去查。

**别忘老规矩：** 测抖动对仪器**本身的抖动底噪**要求极高（你要测 2ps 的 RJ，仪器自己的抖动就得远小于 2ps），而且**带宽、探头、地线**那一套还是得守（呼应第 29、31 篇），否则仪器自己的噪声会叠进去，让你**高估**抖动。

![图6：怎么测怎么分离抖动。原料=TIE（每个边沿相对CDR恢复时钟的误差序列）。三张图：①直方图——单峰高斯钟形=RJ主导，双峰/有界=DJ主导（DCD劈成两峰）；②浴盆曲线——底部平坦段宽≈DJ，两壁斜率由RJ定、按高斯外推到10⁻¹²；③频谱——PJ是尖峰（频率=干扰源:电源纹波/串扰/PLL杂散），RJ是平坦本底。④示波器/BERT一键抖动分解直接吐TJ/RJ/DJ/PJ/DCD/ISI。提醒：仪器底噪要远小于被测抖动](/images/32/ill06_measure.png)

> **一句话记住：抖动分析原料是 TIE。直方图看形状（单峰高斯=RJ、双峰/有界=DJ）；浴盆曲线底宽≈DJ、两壁斜率定 RJ 并外推到 10⁻¹²；频谱抓 PJ 尖峰、按频率定位干扰源（电源/串扰/PLL）。高端仪器一键分解成 RJ/DJ/PJ/DCD/ISI。仪器底噪要远小于被测抖动。**

## 七、怎么对症下药：认准分量，各治各的

抖动分解的终极目的，是**对症下药**。哪个分量大，就走哪条治法，别乱撒网：

**RJ 大（随机、被 2Q 放大 14 倍、最凶）：** 去查**噪声源**。换**低相位噪声的时钟源 / PLL**，给时钟和 PLL 供电做**干净的低噪声电源**（LDO、好的去耦），降低板级底噪。因为 RJ 在总抖动里被放大 14 倍，**它每降 1ps，10⁻¹² 下总抖动降 14ps**，是性价比最高的一手。

**PJ 大（周期、频谱有尖峰）：** 先看**尖峰频率**定位来源。是**开关电源频率** → 加强电源滤波、把开关噪声压下去、给敏感时钟单独供电；是**邻线串扰频率** → 回到第 30 篇，拉间距、加护地线；是**PLL 参考杂散** → 换参考、优化环路。PJ 的好处是**有频率线索，定位最直接**。

**DCD 大（占空比 / 阈值不对称）：** 校正**判决阈值**、调占空比校正电路；差分信号检查正负两条的**平衡与去偏斜（de-skew）**。

**ISI / DDJ 大（跟码型走、损耗引起）：** 这就是上一篇的活——**上均衡**（发端预加重、收端 CTLE/DFE）、**降损耗**（缩短走线、用低损耗板材、走内层），把被损耗压软的边沿补回来。

**还有一招是"跟踪"而不是"消除"：** 接收端的 **CDR 能跟踪掉一部分低频抖动**（它的时钟会跟着信号慢慢漂），所以**低于 CDR 环路带宽的那部分抖动，接收端自己能容忍**，不必死磕——这就是"抖动容限（jitter tolerance）"测试要考察的东西。测的时候要**看接收端到底看到多少抖动**，而不是只看发端的绝对值。

**一个完整的排查动作：** ① 抓一段信号做**抖动分解**，看 TJ 超标多少；② 看是 **RJ 大还是 DJ 大**；③ DJ 大就再往下看是 **PJ、DCD 还是 ISI**（频谱找尖峰、直方图看形状、码型看相关性）；④ 按分量**对症治**——RJ 治时钟电源、PJ 按频率堵源头、DCD 校阈值、ISI 上均衡；⑤ 复测，确认 10⁻¹² 下的 TJ 退回眼宽预算以内。把这套动作画成一张流程图，拿到分解报告照着走就行：

## 🔧 工程师速查（附录）

### 一、抖动是什么

| 名词 | 含义 |
|---|---|
| 抖动 Jitter | 信号边沿在时间轴上偏离理想位置的误差 |
| TIE（时间间隔误差） | 第 n 个边沿实际时刻 − 理想时刻；抖动分析的原始数据 |
| 对眼图的影响 | 让交叉点从细线摊成又厚又宽的一团，从两侧啃窄眼宽 |
| UI 参照 | 5Gbps → UI=200ps；抖动常用 ps 或 UI 比例表示 |

**★ 谁测、测哪三项**：高速·串行·靠 CDR 的链路（PCIe/USB3/SerDes/以太网）才做抖动分解；低速单端（I2C/SPI/UART）看时序即可。要测三项——① 发送端抖动 Tx（TJ/RJ/DJ@BER 对 mask）② 接收端抖动容限 Rx（注入抖动看 CDR 能不能扛）③ 参考时钟抖动 refclk（相位噪声，查规范+手册）。

### 二、抖动家族

| 类别 | 特性 | 描述量 | 成因 / 治法 |
|---|---|---|---|
| **RJ 随机抖动** | 高斯分布、**无界**、样本越多峰峰越大 | **RMS（σ）** | 热噪声/散粒噪声/相位噪声；治时钟源、降电源底噪 |
| **DJ 确定抖动** | **有界**、有明确成因 | **峰峰值** | 下分 ISI / PJ / DCD |
| — ISI / DDJ | 跟码型走 | 峰峰 | 损耗/码间干扰；上均衡、降损耗 |
| — PJ 周期抖动 | 周期性、**频谱尖峰** | 峰峰 | 电源纹波/串扰/PLL 杂散；按频率堵源头 |
| — DCD 占空比失真 | 高低电平不等宽 | 峰峰 | 阈值偏/上升下降不对称/差分不平衡；校阈值、de-skew |

### 三、总抖动算法（双狄拉克）

- **TJ(BER) ＝ DJ(峰峰) ＋ 2 × Q(BER) × RJ(RMS)**
- Q 值：10⁻⁹→6.0（2Q=12）｜10⁻¹²→7.03（2Q=14.07）｜10⁻¹⁵→7.94（2Q=15.88）
- 算例：RJ_RMS 2ps + DJ_pp 12ps @10⁻¹² → 12 + 14.07×2 ≈ **40ps = 0.2UI**
- 要点：RJ 被 2Q **放大 ~14 倍**，最凶；**RJ 每降 1ps，10⁻¹² 下 TJ 降 14ps**。
- **判据/容限从哪查**：TJ 上限、RJ/DJ 分项、接收端抖动容限 mask 在接口一致性规范（PCI-SIG / USB-IF / IEEE 802.3）；参考时钟抖动/相位噪声在规范 + 时钟器件 datasheet。做计划先找到规范章节、TJ 预算和容限模板。
- **盯余量%**：TJ@BER 对 UI 时序预算的余量；余量 < 约 10%~15% 的链路，批次波动/温漂/老化会再吃一截，量产易偶发误码。

### 四、怎么测 / 分离

- **原料 = TIE**（相对 CDR 恢复时钟的每个边沿误差）。
- **直方图**：单峰高斯 = RJ 主导；双峰/有界 = DJ 主导（DCD 劈两峰）。
- **浴盆曲线**：底部平坦段宽 ≈ DJ；两壁斜率由 RJ 定，按高斯**外推到 10⁻¹²**。
- **频谱**：PJ 是尖峰（频率=干扰源）；RJ 是平坦本底。
- **一键抖动分解**：高端示波器/BERT 直接吐 TJ/RJ/DJ/PJ/DCD/ISI。
- **仪器底噪要远小于被测抖动**；带宽、探头、地线仍守老规矩，否则高估。

### 五、对症下药

- **RJ 大**：换低相噪时钟/PLL、干净电源降底噪（性价比最高，被放大 14 倍）。
- **PJ 大**：看尖峰频率——电源纹波→滤波/单独供电；串扰→3W/护地线；PLL 杂散→换参考。
- **DCD 大**：校判决阈值、占空比校正、差分 de-skew。
- **ISI 大**：均衡（预加重/CTLE/DFE）+ 降损耗（上一篇 & 下一篇 S 参数）。
- **CDR 跟踪**：低于 CDR 环路带宽的低频抖动接收端能容忍（抖动容限 jitter tolerance），看接收端实际看到多少。

### 六、常见坑

- ☐ 只看 **RMS/均方根** 就下结论 → 漏掉 RJ 按误码率放大后的最坏值，严重低估。
- ☐ 用**总峰峰值**描述随机抖动 → 测越久越大、不可复现。
- ☐ 忽略**仪器自身底噪** → 测小抖动时被仪器噪声叠加、高估。
- ☐ 只看**发端**抖动绝对值，忘了 **CDR 能跟踪**低频部分（抖动容限）。
- ☐ 频谱有尖峰却不去查频率 → 放过了 PJ 最好用的**定位线索**。

> 说明：本文数值为通用工程近似（UI=200ps、Q↔BER 换算、双狄拉克 TJ 公式、RJ_RMS 2ps / DJ_pp 12ps / TJ 40ps 均为业界通用模型与示意值），具体项目以接口标准（PCIe/USB/以太网等）、器件手册与一致性测试规范为准；不涉及任何机密资料。
