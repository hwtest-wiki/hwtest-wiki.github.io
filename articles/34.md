# 均衡 EQ：高频被走线吃掉、眼图都快闭死了，怎么几乎零成本把眼睛重新睁开？——不是把信号放大，是把频响"整平"

<div class="article-meta"><span class="badge">📘 模块三 · 信号完整性 SI · 均衡 EQ / 预加重 / 去加重 / CTLE / DFE</span><span class="badge">⏱ 约 16 分钟</span><span class="badge qr">🔧 含工程师速查</span></div>


先接着上一篇的翻车现场往下说。

一条 SerDes 链路，跑到 8Gbps 以上，工程师量完 S 参数发现：信号能量所在的高频段，插入损耗掉了十几 dB。反映到眼图上，**眼睛几乎闭死**——高频被吃掉后，边沿变钝、拖尾一大片，一个位的能量"糊"到了后面好几个位上，采样点根本分不清是 0 还是 1，误码率爆表。

按老思路，救它只有两条路：**换损耗更低的高速板材**（贵，整板成本翻倍），或者**把走线做得更短**（布局根本挤不下）。两条路都又贵又难。

于是高速设计师换了个思路：**既然信道把高频压下去了，那我就在两头各加一个"反着来"的整形器，专门把高频抬回来，让信号从发端出去、到收端进来这一整条路的频响，重新变平。** 这个整形器，就是**均衡器**。它可以做在**发送端**（预加重 / 去加重），也可以做在**接收端**（CTLE、DFE），几乎不增加什么硬件成本，却能把闭死的眼睛重新睁开。

![图1：翻车现场——一条高速链路，S21 在信号能量所在的高频段掉了十几 dB，收端眼图几乎闭死（拖尾一大片、0/1 分不清、误码爆表）。换低损板材、缩短走线又贵又难。用均衡：在发端/收端加"反着来"的整形器，几乎零硬件成本，就把闭死的眼睛重新睁开](/images/34/ill01_scene.png)

**读完这篇，你能搞懂：**

- ✅ 均衡到底在干什么——**不是放大信号，是"整平"频响**，把信道吃掉的高频补回来；
- ✅ 码间干扰（ISI）与**游标**——拖尾是怎么糊眼的，pre / main / post 游标是什么；
- ✅ 发端**预加重 / 去加重**（TX FIR）——为什么"压低频"和"抬高频"是一回事；
- ✅ 收端 **CTLE**——一个模拟高通，简单低功耗，但会连噪声一起放大；
- ✅ 收端 **DFE**——用判决过的历史位精确减掉拖尾，不放大噪声，但会错误传播；
- ✅ 组合拳与**自适应**、以及怎么测均衡效果、有哪些坑。

## 一、均衡是什么：信道是低通，均衡是"反着来"的整形器

要理解均衡，先把上一篇的结论摆出来：**一条高速走线，本质是个低通滤波器。** 低频、直流几乎不损，频率越高损得越狠——这就是 S21 那条往下掉的曲线。

信号是方波、是跳变，它的**边沿、细节全靠高频成分**撑着。高频被信道压下去，边沿就钝、拖尾就长，眼睛就塌。

那怎么补？**做一个频响正好"反过来"的东西**：信道在高频往下掉多少 dB，我就在均衡器上让高频**往上抬多少 dB**，两条曲线一叠加，**整条链路（信道 + 均衡）的频响就重新变平了**。频响一平，边沿就利落、拖尾就短，眼睛就睁开。

这里有个最容易搞错的点，一定要说清楚：

> **均衡不是"把信号整体放大"，而是"整形"——相对压低低频、抬高高频，让各个频率的增益拉平。** 你要是把所有频率一起放大，信道的斜坡还在，眼睛照样闭；均衡的关键是**只补被吃掉的那部分（高频），让频响平坦**。

![图2：均衡的核心原理。左：信道 S21 是一条随频率往下掉的低通曲线（高频损耗大）。中：均衡器做一条正好相反、高频往上翘的高通/峰化曲线。右：两条一叠加，整条链路的频响被"拉平"到接近 0dB，高频重新补齐——时域上边沿变利落、拖尾消失、眼睛睁开。均衡不是整体放大，是把被信道压下去的高频相对抬回来](/images/34/ill02_channel.png)

![动图：同一条链路，把均衡强度从 0 慢慢加大——收端眼图从几乎闭死（拖尾糊成一团、看不清 0/1），随着高频被一点点补回来，眼高、眼宽逐渐张开，最后重新睁成一只干净的眼睛。这就是均衡"整平频响、消掉拖尾"在时域上的直观效果](/images/34/ill02b_open_anim.gif)

还要记住均衡的**天花板**：它**只能重整已有信号的形状，不能凭空造信息**。信道里除了损耗，还有**噪声和串扰**。抬高频的同时，落在高频的噪声也会被一起考虑进来。**如果信噪比已经太差，眼睛是被噪声糊死的、不是被损耗糊死的，那再怎么均衡也救不回来。** 均衡治的是"损耗/拖尾"，不是"噪声"。

## 二、码间干扰与游标：拖尾是怎么糊眼的

均衡到底在时域上消什么？答案是**码间干扰（ISI，Inter-Symbol Interference）**。这是理解后面所有均衡手段的钥匙。

想象你只发**单独一个位**（一个窄脉冲）进这条链路，在收端看它到达时的波形。理想情况下，它应该只在**自己这个位的采样时刻**出现、其它时刻都是 0。可经过低通信道，这个脉冲被"抹开"了：它的能量会**拖到前后好几个位的时间里**。

我们在每个位的采样时刻去看这个脉冲响应，得到几个"抽头"值，行话叫**游标（cursor）**：

- **主游标（main cursor）**：脉冲落在**自己这个位**采样点上的主值——这是我们真正想要的。
- **后游标（post-cursor）**：脉冲拖尾**泄漏到后面几个位**采样点上的值——低通信道的拖尾主要在这里，通常是 ISI 的大头。
- **前游标（pre-cursor）**：脉冲还没到主位、就**提前泄漏到前一个位**采样点上的值——一般较小。

单看一个位没事，可**真实数据是一长串位连着发**。当你在某个位的采样时刻做判决时，看到的不只是它自己的主游标，**还叠加了前面几个位甩过来的 post-cursor、后面位漏过来的 pre-cursor**——这些别人家的拖尾全加在了你头上，把 0 和 1 的电平搅在一起。**这就是 ISI，眼图闭合的直接元凶。**

所以均衡的任务，用游标的语言说，就是：**把 main cursor 之外的那些 pre / post 游标，尽量削平到 0。** 谁削 pre、谁削 post、在发端削还是收端削，就分出了下面三种均衡手段。

![图3：码间干扰与游标。上：发单个脉冲经过低通信道，收端波形被抹开——采样点上除了想要的主游标（main），还在前一位漏出前游标（pre-cursor）、在后面几位拖出后游标（post-cursor）。下：连续发一串位时，相邻位的 pre/post 游标全叠加到当前采样点上，0/1 电平被搅浑=ISI，眼睛闭合。均衡就是想办法把 main 之外的游标削到 0](/images/34/ill03_cursor.png)

## 三、发端预加重 / 去加重：为什么"压低频"和"抬高频"是一回事

第一招做在**发送端（TX）**，叫**预加重（pre-emphasis）**或**去加重（de-emphasis）**。这俩名字听着像两回事，其实是同一件事的两种说法。

先看数据的规律：**一连串相同的位（比如 0000 或 1111），是低频成分**，信道损得少、到收端还挺足；**而位与位之间的跳变（0→1、1→0），是高频成分**，信道损得狠、到收端就蔫了。均衡要相对抬高的，正是这个跳变。

有两种等价的做法：

- **预加重**：把**跳变后的那个位**幅度**加大**（给高频加把劲）；
- **去加重**：把**连续没跳变的位**幅度**压低**（把足足的低频压下去）。

你品一下：**"抬高跳变" 和 "压低连续位"，落到收端的相对关系是一模一样的**——都是让跳变（高频）相对连续位（低频）更突出。所以工程上这两个词经常混用，量化时用一个 dB 数表示，比如 **3.5dB、6dB** 的去加重。

TX 均衡的实现，是一个很短的 **FIR 滤波器**（有限冲激响应），就几个**抽头（tap）**：一个主抽头，加一个 post-cursor 抽头（把当前位反相一点点叠到下一位，去 post 拖尾），有的还加一个 pre-cursor 抽头。调这几个抽头的权重，就能定制发出去的波形。

**TX 均衡的好处**：它在信号**还没被信道糟蹋、也还没混入接收噪声之前**就做整形，所以**不会放大接收端的噪声**。**代价**：发送端的**总摆幅是有限的**——你"抬高"跳变，实际上往往是靠"压低"其它位换来的，等于牺牲了一部分低频幅度，整体信号能量并没有变大。

![图4：发端预加重/去加重（TX FIR）。左：连续相同位是低频、信道损得少；位间跳变是高频、损得狠。中：预加重=把跳变位抬高，去加重=把连续位压低，两者对收端等价——都让高频相对突出。右：用几抽头 FIR 实现（主抽头+post/pre 抽头），发波形被预先"反向整形"。优点是在混入接收噪声前整形、不放大噪声；代价是发送总摆幅有限](/images/34/ill04_tx.png)

## 四、收端 CTLE：一个模拟高通，简单但会放大噪声

第二招做在**接收端（RX）**，叫 **CTLE（连续时间线性均衡，Continuous-Time Linear Equalizer）**。

它本质就是一个**模拟滤波器**，增益曲线在**高频往上翘**（低频有点衰减、高频有峰化 peaking），这条上翘的曲线，正好去补信道那条下滑的曲线。两者一叠，收端看到的频响就平了。它是**连续时间**的——不需要时钟、不需要判决，信号一进来就实时地被它整形。

**CTLE 的好处**：结构简单、**功耗低**、是纯模拟的连续处理，还能**同时补 pre 和 post 拖尾**（因为它是线性滤波、对整个频响起作用）。几乎每条高速链路的接收端都会先放一级 CTLE 打底。

**CTLE 的软肋**：它是**线性放大**——**抬高高频的时候，落在高频的噪声和串扰，也被一起放大了**。所以 CTLE 的峰化不能开太猛：开太少，高频补不够、眼还是闭；开太猛，信号是补上来了，但噪声也放大了，**信噪比反而变差**。这就是后面要说的"过均衡"。

![图5：收端 CTLE（连续时间线性均衡）。CTLE 是个模拟滤波器，增益曲线在高频往上翘（峰化），正好补信道 S21 的下滑，两条一叠频响变平。优点：纯模拟连续处理、低功耗、pre/post 拖尾都能补。软肋：线性放大——抬高频时把落在高频的噪声/串扰一起放大了，所以峰化开太猛会让信噪比反而变差（过均衡）](/images/34/ill05_ctle.png)

## 五、收端 DFE：用判决过的位精确减拖尾，不放大噪声

第三招也在接收端，叫 **DFE（判决反馈均衡，Decision Feedback Equalizer）**，它和 CTLE 的思路完全不同——**它是非线性的**。

DFE 的想法特别巧：**当前这个位受到的 post-cursor 拖尾，是前面几个位甩过来的。而前面那几个位，我已经判决完了、明确知道它们是 0 还是 1。** 那我把这些**已知的历史位**，各自乘以它们对应的 post-cursor 拖尾系数，**反过来从当前信号里减掉**，不就把拖尾精确抵消了吗？

这个思路的**杀手锏**在于：**它减掉的是"已知的确定量"，不是放大信号，所以完全不会放大噪声。** 这正好补上了 CTLE 最大的短板。

但 DFE 有它自己的三个**限制**，测试和调试时必须心里有数：

1. **只能消 post-cursor，消不了 pre-cursor。** 因为它靠的是**已经判决过的历史位**，而 pre-cursor 是还没到的未来位漏过来的，DFE 拿不到、管不了（pre-cursor 得靠 TX FIR 或 CTLE 去处理）。
2. **依赖判决正确，会错误传播（error propagation）。** 万一某个位判错了，这个错误的历史位会被反馈进去、算错拖尾，可能把后面的位也带错——错误会像多米诺一样传下去。
3. **高速下反馈环时序极紧。** "判决→反馈→减掉→再判决下一位"必须在**一个位的时间内**闭环完成，速率越高越难做，往往要用展开（unrolled / speculative）结构硬扛。

所以现代高速接口，**从来不是只用一招，而是打组合拳**：

> **TX FIR（发端预削 pre/post）+ RX CTLE（模拟打底、抬整体高频）+ RX DFE（精确消 post-cursor、不放大噪声）**，三级串起来，各司其职。PCIe 4.0/5.0、USB、112G SerDes 全都是这套。

而且这些抽头、增益不是手调死的，是**自适应（adaptation）**的：芯片用 **LMS** 之类的算法，一边收数据一边自动调 TX 抽头、CTLE 峰化、DFE 系数，**朝着"眼睛最大、误码最小"的方向收敛**。你在链路训练（link training）阶段看到的，就是它在自动找最优均衡参数。

![图6：收端 DFE 与组合拳。上：DFE 用已经判决出的历史位（已知 0/1）乘 post-cursor 系数，反馈相减，精确抵消后游标拖尾——减的是确定量，不放大噪声。限制：只能消 post（消不了 pre）、判错会错误传播、高速反馈环时序紧。下：现代 SerDes 打组合拳——TX FIR + RX CTLE + RX DFE 三级串联，各司其职，再用 LMS 自适应自动调参，朝眼睛最大、误码最小收敛](/images/34/ill06_dfe.png)

## 六、怎么测均衡效果、有哪些坑

均衡好不好，测试工程师主要看这几件事：

**1. 均衡前后的眼图对比。** 最直观的验收：不加均衡时眼睛闭死，加上（或打开芯片里的）均衡后，眼睛重新睁开、眼高眼宽达标。很多示波器和一致性软件能**加载 CTLE/DFE 的行为模型**，对采集到的波形做"软件均衡"，直接呈现均衡后的眼图——这样即使探针探到的是均衡前的点，也能看到 RX 均衡后的效果。

**2. 发端预加重档位。** 一致性测试会规定发送端要在**几个预设档位（preset）**下分别测试。比如 PCIe 定义了一组 TX preset（不同的 pre/post 抽头组合），USB 也有对应的去加重档位。测试就是逐档验证发出的波形符合模板。

**3. 接收端的抖动容限 / 均衡能力。** 用一台能"造烂信号"的仪器（往信号里注入可控的 ISI、抖动、噪声），喂给被测接收端，看它的均衡能不能把这么烂的信号还救回来、误码率还达标——这叫**接收机测试（RX jitter tolerance / stressed eye）**。

再说几个**必须绕开的坑**：

- **过均衡（over-equalization）**：CTLE 峰化开太猛、TX 加重加太多，会把高频**抬过头**——出现高频过冲、并把噪声一起放大，眼睛不但没变好，反而更糟。**均衡不是越强越好，要匹配信道的损耗量。**
- **DFE 错误传播**：误码率本来就高的链路，DFE 可能越纠越乱。调试时如果发现误码成串出现，要怀疑这里。
- **只在收端补、忽略发端**：如果发端信号本身就烂（摆幅不够、上升时间太差），光靠收端均衡很吃力。**好的均衡是发端收端配合。**
- **拿均衡当万能药**：再强调一遍，均衡治损耗、治拖尾，**治不了噪声**。信噪比不够时，先去查噪声、串扰、参考电平，而不是一味加大均衡。

## 七、一句话总结

上一篇 S 参数让我们看清：高速走线是个低通，高频被吃掉、眼图会塌。这一篇的均衡，就是那味**解药**：

> **在发端和收端做一个"反着来"的整形器，把信道压下去的高频相对抬回来，让整条链路的频响变平、拖尾消失、眼睛重新睁开。** 发端用预加重/去加重（FIR），收端用 CTLE（模拟高通、简单但放大噪声）加 DFE（用判决位精确减 post 拖尾、不放大噪声但会错误传播），三招打组合拳、再自适应调参。记住两句话：**均衡是"整平"不是"放大"；均衡治拖尾、治不了噪声。**

到这里，模块三「信号完整性」就全部讲完了——从反射、串扰、阻抗，到边沿、带宽、TDR，再到眼图、抖动、S 参数，最后到均衡。**你已经能完整地看懂一条高速链路"信号干不干净"的整套体检和急救逻辑了。**

![图7：本篇速查卡。均衡=把信道压下去的高频相对抬回来、整平频响（不是放大、治不了噪声）；ISI 与游标（main/pre/post，post 是拖尾大头）；TX 预加重/去加重 FIR（不放大噪声、摆幅有限）；RX CTLE（模拟高通、低功耗、放大噪声）；RX DFE（判决位精确消 post、不放大噪声、会错误传播）；组合拳 TX FIR+CTLE+DFE 自适应；坑：过均衡、DFE 错误传播、拿均衡治噪声](/images/34/ill07_quickref.png)

## 📋 工程师速查：均衡 EQ

**① 均衡是什么**
- 信道=低通（高频损得狠）；均衡=做一条相反的高频上翘曲线去补，让频响变平
- 关键：是"整平/整形"，不是整体放大；只补被吃掉的高频
- 天花板：治损耗/拖尾，**治不了噪声**；信噪比太差救不回来

**② 码间干扰 ISI 与游标**
- 单脉冲经低通被抹开 → 主游标 main（想要的）+ 前游标 pre + 后游标 post（拖尾大头）
- 连续位时，相邻位的 pre/post 全叠到当前采样点 = ISI，眼睛闭
- 均衡任务：把 main 之外的游标削到 0

**③ 发端 TX 预加重/去加重（FIR）**
- 预加重=抬跳变位；去加重=压连续位；对收端等价（相对抬高频），用 dB 记（如 3.5/6dB）
- 几抽头 FIR 实现（主+post/pre 抽头）
- 优点：混入接收噪声前整形，不放大噪声；代价：发送总摆幅有限

**④ 收端 CTLE（连续时间线性均衡）**
- 模拟高通/峰化，高频上翘补信道下滑；连续、低功耗、pre/post 都能补
- 软肋：线性放大 → 把高频噪声/串扰一起放大；开太猛=过均衡

**⑤ 收端 DFE（判决反馈均衡）**
- 用判决过的历史位×post 系数，反馈相减，精确消 post 拖尾；减确定量、**不放大噪声**
- 限制：只消 post 消不了 pre；判错会错误传播；高速反馈环时序紧

**⑥ 组合拳与自适应**
- 现代 SerDes：TX FIR + RX CTLE + RX DFE 三级串联；LMS 自适应自动调参（链路训练）
- PCIe 4.0/5.0、USB、112G 全用这套

**⑦ 怎么测 & 坑**
- 测：均衡前后眼图对比（可加载 CTLE/DFE 模型软件均衡）、TX preset 逐档、RX 抖动容限（压力眼）
- 坑：过均衡（抬过头+放大噪声）、DFE 错误传播、只补收端不管发端、拿均衡当万能药治噪声
