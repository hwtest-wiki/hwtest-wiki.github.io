# 复位与上电时序：电压全到位、复位也拉高了，芯片却偶发起不来——复位，是不是松手太早了？

<div class="article-meta"><span class="badge">📘 模块四 · 时序与时钟 · 复位与上电时序 · 模块收官</span><span class="badge">⏱ 约 17 分钟</span><span class="badge qr">🔧 含工程师速查</span></div>


一块板子，上电后偶发开不了机。你拿万用表一路一路量——**电压全对**，一个不差。你又用示波器看复位脚——**复位确实从低拉到了高**，时序图上"复位释放"这一下也发生了。可板子就是**十次里偶尔坏一两次**：要么彻底不启动，要么启动了却"跑飞"——进了错误状态、外设初始化乱套、跑一会儿死机。换块板子，可能又好了。

你查电、查复位电平、查程序，全都对，一整天毫无头绪。

**这种"电全对、复位也拉高了、却偶发起不来"的问题，八成不是电平的事，是复位'松手'的时刻不对。** 复位这只手，撒得**太早**了——在它松手的那一瞬间，芯片赖以工作的地基还没打好：**电压可能还没爬到稳定工作区、时钟可能还没起振锁定**。复位一松，芯片就开始按时钟一拍一拍往下跑，可这时候喂给它的时钟还是乱的、电还是晃的，它自然就跑歪了。

打个比方：**复位就像赛跑的发令枪，上电时序和时钟就是赛道。** 你得等赛道铺好、跑道线画清楚，才能鸣枪。枪响太早，运动员冲出去时脚下还是坑——摔是迟早的事。

**读完这篇，你能搞懂：**

- ✅ 复位到底在干什么——为什么每次上电都必须先把芯片"按住"；
- ✅ 复位松手的**三个前提**——电压稳、时钟稳、复位保持够时长，缺一个都不能松；
- ✅ **上电复位（POR）** 怎么和上电时序挂钩——VDD 爬到阈值才开始计时、延时后才释放；
- ✅ 复位时序违规的几种典型翻车——松手太早、脉冲太窄、多域乱序、**异步释放引发亚稳态**；
- ✅ 这一摊**怎么测**——多通道示波器把电源、时钟、复位、PGOOD 摆到同一时间轴上看关系；
- ✅ **什么时候该重点盯复位时序**——按系统类型（多路电源 / 带 PLL / 多复位域）决定测哪几项、判据从哪查；
- ✅ 真遇到「上电起不来」，**照一条症状分诊流程**快速定位是四种翻车里的哪一种。

## 一、复位在干什么？为什么每次上电都要先"按住"芯片

先说清楚复位这件事本身。

一颗芯片内部，有成千上万个寄存器、一堆状态机。**刚上电的那一刻，这些寄存器里装的是什么？是随机的。** 有的是 0、有的是 1，谁也说不准——就像一屋子人刚被吵醒，乱糟糟没个头绪。要是让芯片带着这一屋子随机值直接开跑，它根本不知道自己该干嘛，状态机可能一上来就卡在某个死角。

**复位（reset）就是那声"全体立正"。** 复位信号一有效（通常是拉低，低电平复位），芯片内部所有寄存器、状态机被强制拉回一个**确定的初始状态**——程序计数器指到起始地址、外设配置清零、状态机回到 IDLE。这样等复位一松开，大家都从**同一个确定的起点**开始跑，行为才是可预期、可复现的。

所以每次上电，流程都是：**先按住（复位有效）→ 等一切就绪 → 松手（复位释放）→ 芯片从确定初态开跑。** 这一篇的全部关键，就在"等一切就绪"这几个字——**到底要等什么、等多久，才能松手。**

## 二、复位松手的三个前提：电压稳、时钟稳、保持够时长

复位这只手到底什么时候能松？记住三个前提，**缺一个都不能松**：

**前提一·电压稳了。** 芯片有个最低工作电压。电压还在往上爬、没进到规定的工作范围时，内部逻辑根本不保证正确——这时候松复位，等于让人在半睡半醒时去做算术。所以复位必须**等各路电压都爬到工作区、并稳定下来之后**才能释放。这就是上电复位要跟着上电时序走的原因。

**前提二·时钟稳了。** 数字芯片是靠时钟一拍一拍往下跑的。可晶振从上电到稳定起振，要**几毫秒**；如果用了锁相环（PLL）倍频，PLL 从开始到**锁定（lock）**还要**几十微秒到几毫秒**。在时钟起振、锁定之前，喂给芯片的时钟频率是飘的、边沿是乱的。这时候松复位，芯片按一个"还没长稳"的时钟去跑——上一篇讲的抖动、频率不稳全砸下来，能不出事？所以复位还得**等时钟起振并锁定之后**才能松。

**前提三·复位低电平保持够时长。** 芯片手册会规定一个**最短复位保持时间**（reset pulse width，比如"至少拉低 1ms"或"至少 N 个时钟周期"）。复位脉冲太窄，内部还没来得及把所有寄存器都拉到初态，就被松开了——复位等于没做干净。所以复位有效的时间必须**≥ 手册要求**。

**把三个前提合起来看**：复位释放的时刻，必须落在"电压稳"和"时钟稳"这两件事都完成**之后**，同时复位本身已经保持了足够长。这三条里但凡有一条没满足就松手，就是开头那块板偶发起不来的根因。

## 三、上电复位 POR：复位是怎么跟上电时序挂上钩的

前面说复位要等电压稳。可"等"这件事，谁来管？答案是 **上电复位电路（POR，Power-On Reset）**。

POR 的活儿很简单：**盯着电源电压，等它爬过一个阈值、再多等一小段延时，才把复位松开。** 具体分两步：

1. **监测电压过阈值**：VDD 从 0 往上爬，POR 电路一直把复位摁在低电平。等 VDD 爬过设定的阈值电压（比如 3.3V 系统里设在 2.9V 左右），POR 认为"电基本够了"。
2. **再延时一段（td）才释放**：过了阈值不等于立刻松手——POR 还会再等一个固定延时（几百微秒到几十毫秒不等），把时间留给电压彻底稳定、留给时钟起振锁定。延时到了，才把复位拉高、正式松手。

这样，复位释放就**自动跟上了上电时序**——电爬得慢，复位就松得晚；电没爬够，复位干脆不松。很多芯片内部自带简易 POR，但它的阈值和延时不一定够精确、也不一定管得了外部时钟稳没稳。所以对复杂系统，工程师常**外挂一颗专用复位芯片（复位 IC / 电压监控器 supervisor）**：阈值精准、延时可选，有的还能同时监控好几路电压，任何一路掉出窗口就立刻重新复位。

> 顺带说清楚一个容易混的点：**上电复位（POR）**管的是"上电时把复位摁住、稳了再松"；还有**掉电复位（BOR，Brown-Out Reset）**——运行中电压要是跌破下限，BOR 会立刻重新把芯片摁进复位，防止它带着"电不够"的状态乱跑。两个是一对：一个管上电醒来、一个管掉电别乱跑。

![图4：上电复位 POR 的工作波形。上：VDD 从 0 爬升，越过 POR 阈值电压 Vth（如 3.3V 系统设 ~2.9V）。中：POR 在 VDD 过阈值后，再等一段固定延时 td（几百 us ~ 几十 ms）。下：td 结束才把复位从低拉高（释放）。这样复位释放自动跟上电压爬升——电慢则复位晚、电不够则不释放。掉电时 BOR 反向：电压跌破下限立即重新拉低复位](/images/37/ill04_por.png)

## 四、复位时序违规的四种典型翻车

把前面的前提反过来，就是四种最常见的翻车姿势：

**① 松手太早（最常见）。** 复位在时钟锁定之前、或电压稳定之前就释放了。芯片带着乱时钟/晃电压开跑，进错状态、初始化乱套。偶发、难复现——因为每次上电时钟起振快慢有随机性，有时候刚好赶上、有时候没赶上。**开头那块板就是这一型。**

**② 复位脉冲太窄。** 复位有效时间短于手册要求，内部没复位干净就松了。表现为部分寄存器是初态、部分是随机值，行为诡异。

**③ 多复位域顺序错。** 大系统里往往有好几个复位域（内核、外设、DDR、PHY 各有各的复位），它们之间有先后依赖关系。释放顺序错了，比如外设复位先松、它依赖的时钟域复位却还没松，外设就抓瞎。

**④ 异步复位、异步释放 → 亚稳态。** 这一条最隐蔽，也最能呼应前面。复位信号如果是异步的（不跟任何时钟对齐），有效时没问题；可**释放的那一下**，如果正好卡在某个时钟的有效沿附近，就会**违反触发器的恢复/移除时间（recovery/removal，本质就是复位版的建立/保持时间）**——触发器进入**亚稳态**，输出既不是 0 也不是 1，随机落定后还往下传。结果就是：明明复位电平、时序图都对，芯片却偶发起不来。**解法是"异步复位、同步释放"**：复位可以异步立即生效（安全第一），但释放要用两级触发器同步到时钟域，让释放沿避开采样点。这正是第 35 篇建立/保持思想在复位上的直接应用。

![图5：复位时序违规的四种典型翻车。① 松手太早：复位在时钟锁定前释放→带乱时钟开跑（最常见、偶发难复现）；② 脉冲太窄：复位有效时间 < 手册要求→没复位干净；③ 多复位域乱序：外设复位先于其依赖的时钟域复位释放→抓瞎；④ 异步释放亚稳态：释放沿卡在时钟有效沿附近→违反 recovery/removal 时间→触发器亚稳态。解法：异步复位+同步释放（两级同步器把释放沿对齐时钟域）](/images/37/ill05_violations.png)

## 五、怎么测、怎么保证

**怎么测——把四条信号摆到同一时间轴上看。** 这一摊的测法核心就一句话：**用多通道示波器，同时抓电源、时钟、复位、PGOOD，看它们的时间关系对不对。** 具体：

- **抓上电全过程**：示波器设上升沿/掉电触发，一次抓下从 0 上电到系统跑起来的全过程；
- **量四条关键线**：主要电源 rail（尤其最后一路稳的）、主时钟（看它什么时候起振/幅度稳）、复位脚、PGOOD；
- **验三件事**：复位释放点是不是在**所有电源稳之后**？是不是在**时钟起振锁定之后**？复位低电平**保持时长**够不够手册要求？
- **复位释放点相对时钟锁定的裕量**：别卡着线释放，要留余量。时钟锁定信号（PLL 的 lock）如果引得出来，一起抓最直观。

**怎么保证——设计上就把时序焊死：**

- **专用复位 IC / 电压监控器**：阈值准、延时够，把复位释放牢牢压在电压稳之后；多路监控的，任一路掉窗立即复位；
- **让复位延时覆盖时钟锁定时间**：POR 的延时 td 要设得比"晶振起振 + PLL 锁定"还长，确保松手时时钟已稳；
- **PGOOD 菊花链联动复位**：像第 18 篇讲的，用各路电源的 PGOOD 串成上电链，最后一路 PGOOD 才去触发复位释放——电没全好，复位就不松；
- **看门狗（watchdog）兜底**：万一还是启动到错误状态，看门狗超时没被喂狗就强制重启，给系统一次重来的机会；
- **异步复位、同步释放**：跨时钟域的复位释放用两级同步器，避开亚稳态。

![图6：怎么测 + 怎么保证。左（怎么测）：多通道示波器同抓 电源rail / 主时钟 / 复位 / PGOOD 四条线，验复位释放点在①电源稳之后②时钟起振锁定之后，且③复位保持 ≥ 手册要求、释放留裕量。右（怎么保证）：专用复位IC(阈值准/延时够/多路监控)、POR延时td覆盖晶振起振+PLL锁定、PGOOD菊花链联动复位释放、看门狗超时兜底重启、异步复位+同步释放避亚稳态](/images/37/ill06_measure.png)

## 六、什么时候该重点盯复位时序？判据从哪查

复位时序每块板都要过一遍，但**投入多大精力、重点盯哪一条前提**，得看系统长什么样。先问自己三个问题——几路电？有没有 PLL？几个复位域？——答案就决定了你该把火力压在哪、判据去哪份文档里查。

- **多路电源 SoC**：现在的主控动辄五六路电（核、IO、DDR、PHY、模拟各一路）。重点盯**复位释放是不是压在了"最后一路电也稳"之后**，别哪路电还没爬到位就松了手。设计上用 **PGOOD 菊花链**串起来——每路电好了给一个 PGOOD，全好了才去触发复位释放。
- **带 PLL / 晶振倍频**：只要系统时钟经过晶振起振、PLL 倍频锁定，就必须盯 **POR 延时 td 有没有盖住"晶振起振 + PLL 锁定"的总时间**。这是**松手太早**这一型最高发的场景——td 设短了，复位抢在 PLL 锁定前释放，芯片带着还在爬的乱时钟开跑。
- **多复位域 / 跨时钟域**：大 SoC 里内核、DDR、PHY 各有各的复位域，且有依赖关系。重点验**释放顺序**：被依赖的域（比如时钟、DDR 控制器）先松，依赖它的外设后松。跨时钟域的复位释放，一律用**异步复位 + 同步释放**避亚稳态。

**判据从哪查？** 就三个地方，别拍脑袋：

1. **器件 datasheet**：最短复位脉宽（reset pulse width）、POR 阈值电压 Vth、POR / 复位 IC 的延时 td、PLL 锁定时间（lock time）——这些都是芯片手册里的硬指标；
2. **上电时序预算**：各路电源的爬升时间、上电顺序要求，来自电源设计和系统上电时序表；
3. **接口 / 平台规范**：像 DDR、PCIe 这类接口，对参考时钟就绪、复位释放的先后有明确规定。

查到判据只是第一步，签核时还得**盯余量**：复位释放点相对时钟锁定要留余量、复位保持时长相对手册最短脉宽要留余量——**别卡在临界点上签字**，临界点就是偶发问题的温床。

![图8：什么时候该重点盯复位时序 + 判据从哪查。按系统类型分诊：①多路电源 SoC→复位释放压在全 rail 稳之后、PGOOD 菊花链联动；②带 PLL/晶振倍频→POR 延时 td 必须盖住晶振起振+PLL 锁定（"松手太早"最高发）；③多复位域/跨时钟域→验释放顺序（依赖域先松）、跨域用异步复位同步释放。判据从哪查：器件 datasheet（最短复位脉宽/POR 阈值 Vth/延时 td/PLL 锁定时间）+ 上电时序预算 + 接口规范；签核盯余量、别卡临界点](/images/37/ill08_when.png)

## 七、上电起不来了，照这条流程定位

真在现场遇到"电压全对、复位也拉高，系统却偶发起不来"，别东一榔头西一棒子。照一条流程走：**先把电源、时钟、复位、PGOOD 摆到同一时间轴，再按症状分诊到四种翻车里的某一种。**

**第一步——先看波形、别只量电平。** 万用表量出来电压全对，只能证明"稳态电平对"，证明不了"上电那一瞬间的先后顺序对"。必须用多通道示波器一次抓下上电全过程，四条线摆一起看。

**第二步——按症状对根因：**

- **启动即跑飞**（一上电就进错状态、初始化乱套）→ 八成是**带乱时钟开跑**：查复位释放和时钟锁定的先后，把 POR 延时 td 加长、盖住锁定；
- **行为诡异、部分寄存器像是半初态** → 查**复位脉宽**：是不是复位有效时间比手册最短脉宽还短，没复位干净，把脉宽拉够；
- **某个外设单独抓瞎** → 查**多复位域释放顺序**：是不是这个外设的复位先于它依赖的域释放了，调整成依赖的域先松；
- **偶发、难复现，怎么都抓不到规律** → 重点怀疑**异步释放踩亚稳态**：复位释放沿卡在时钟有效沿附近，改成异步复位 + 同步释放，用两级同步器把释放沿对齐时钟域。

**第三步——改完复测留余量。** 改了设计（加长 td、拉够脉宽、调释放顺序、加同步器）之后一定要重测：症状消失、复位释放相对时钟锁定的余量转正、**反复上电几十次都稳**，才算真解决——复位问题最怕的就是"测一次好了"就收工，它偏偏是偶发的。

## 八、模块四小结：时间"对不对"，是数字系统的命根

到这里，模块四「时序与时钟」三篇就凑齐了，正好是一条链：

- **第 35 篇 建立/保持**：数据要在时钟采样沿前后的"禁区窗"外——**单次采样的时刻对不对**；
- **第 36 篇 时钟测试**：时钟本身要准、要稳（频率/占空比/抖动/相位噪声）——**那个采样沿本身靠不靠谱**；
- **第 37 篇 复位与上电时序**：开机时电压、时钟、复位要按对的先后就位——**系统一开始能不能正确醒过来**。

三篇合起来就一件事：**数字系统里，光信号电平对、光电压达标不够，"时间对不对"才是命根。** 很多"偶发、难复现、查不出规律"的疑难杂症，最后都落在时序上。

## 九、一句话总结

复位不是简单拉个电平，它是系统"正确醒过来"的发令枪：

> **复位松手的时刻，必须在电压稳、时钟起振锁定之后，且复位已保持够手册要求的时长——三条缺一不可。** 松手太早（尤其抢在时钟锁定前），芯片就在地基没打好时开跑，表现为偶发起不来、启动跑飞、难复现。用 POR / 复位 IC 把释放压在电压稳之后、让延时覆盖时钟锁定，用 PGOOD 联动、看门狗兜底，跨时钟域用"异步复位、同步释放"避开亚稳态。测的时候，多通道示波器把电源、时钟、复位、PGOOD 摆到同一时间轴，验复位释放点的先后与裕量。
