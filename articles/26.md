# Load Switch 专项：一颗小开关，凭什么要单独测四项？

<div class="article-meta"><span class="badge">📘 模块二 · 电源完整性 PI · 收官</span><span class="badge">⏱ 约 16 分钟</span><span class="badge qr">🔧 含工程师速查</span></div>


一颗 Load Switch，数据手册上白纸黑字写着导通电阻 **Ron = 20mΩ**。你拿它给一个 **3A** 的模块供电，心里一算：压降 = 3A × 20mΩ = **60mV**，发热 = 3A² × 20mΩ = **0.18W**，小意思，稳得很。

板子回来一实测：满载时，这颗开关两端的压降竟然有 **200mV**！反推回去，它实际的 Ron 已经不是 20mΩ，而是奔着 **66mΩ** 去了——足足是手册值的三倍多。这多掉的 140mV，正好把后级芯片的供电压到了欠压门槛边上，偶发起不来。

你回头翻手册才发现：那个 20mΩ，是 **25℃、VIN=5V** 下的**典型值**；而你的板子上，这颗开关贴在一堆发热器件旁边、结温奔到 **85℃**，VIN 还只有 3.3V——这两个条件一叠加，Ron 翻倍，太正常了。

**这就是 Load Switch 最容易踩的坑：你以为它只是根"导线加开关"，随便用；可它的每一项参数都跟工况强相关，手册上的漂亮数字，换到你的板子上全得打个折。** 这一篇，就把这颗小开关该测的四项，一项项测给你看。

**读完这篇，你能搞懂：**

- ✅ Load Switch 到底是什么、跟 DCDC / LDO 有什么不同，为什么值得单独测；
- ✅ **Ron 导通电阻**：为什么手册值靠不住，怎么在**你的工况**下把它量准；
- ✅ **软启动斜率**：怎么用一个电容设定上电快慢，又怎么测这条上升斜率和涌流；
- ✅ **输出放电（QOD）**：关断后输出多久归零，残压不清会惹什么祸；
- ✅ **通断时序**：使能到输出的延迟、多路开关怎么排上下电顺序；
- ✅ 一张 Load Switch 完整测试清单，照着测不漏项。

## 一、先认清：Load Switch 是个啥，为什么要单独测

第 16 篇讲电源 IC 家族时提过它：**Load Switch，负载开关，本质就是一颗受控的 MOS 开关**，外加一点驱动和保护电路。它不像 DCDC 那样搬运变压、也不像 LDO 那样卡压差稳压——它**只做一件事：通、断**。给它一个使能信号（EN 拉高），它就把输入接到输出；EN 拉低，它就断开。

那既然只是"通断"，为什么不直接用一颗 MOS、甚至一根导线，非要用它、还要单独测？因为一颗合格的 Load Switch，把"通断"这件事做得很讲究，额外带了三样本事：

- **软启动**：上电时让输出电压**缓慢爬升**，而不是"啪"地怼上去——直接压住了上一篇讲的**上电浪涌**；
- **输出放电**：关断的同时，把输出上残留的电压**主动泄放掉**，不留尾巴；
- **可控时序**：使能到输出之间有确定的延迟，多颗还能**串起来排上下电顺序**。

正因为它把这些都集成进去了，它就不再是"一根导线"，而是一个**有参数、会因工况变脸**的器件。所以测它，就得盯四项：**Ron、软启动斜率、输出放电、通断时序**。这四项，正好对应它"接得通不通、开得稳不稳、关得干不干净、来得准不准时"。

> **一句话记住：Load Switch 是一颗"智能电闸"——只管通断，但集成了软启动、输出放电、时序控制。它不是导线，是有参数、随工况变脸的器件，必须盯四项：Ron、软启动斜率、输出放电、通断时序。**

## 二、Ron 导通电阻：手册那个漂亮数字，换到你板子上就变

Ron，导通电阻，是 Load Switch 导通时输入到输出的那点电阻。它直接决定两件事：**压降**（Ron × 电流，白白吃掉的电压）和**发热**（电流² × Ron，白白烧掉的功率）。而且——**这颗开关的 Ron，是直接串进你上一篇量的那条 PDN 阻抗里的**，它高一分，整条供电路的直流阻抗就抬一分。

开头那个坑就出在这儿：手册的 Ron，永远标在一个**特定条件**下，通常是 **25℃、某个较高的 VIN**。可你的真实工况往往是"高温 + 低 VIN"，这俩都让 Ron 变大：

- **温度越高，Ron 越大**：MOS 导通电阻是正温度系数，结温从 25℃ 到 85℃，Ron 涨个五成到一倍很常见；
- **VIN 越低，Ron 越大**：VIN 低，栅极驱动电压就低，MOS 沟道开得没那么透，Ron 跟着上去（尤其低压 Load Switch 特别敏感）。

所以测 Ron，**不能只在常温、空载条件下点一下**，要在**你板子真实的 VIN、真实的负载电流、真实的温度**下测。方法本身不难，就是**四线开尔文法**：用电子负载灌一个已知电流 I，同时用**另一对表笔**紧贴开关的输入、输出两个焊盘量压降 ΔV（避开走线和接触电阻的账），Ron = ΔV ÷ I。

这里的关键跟第 08 篇量微小电压一脉相承：**压降才一两百毫伏，表笔位置、接触电阻稍微差一点，Ron 就量飞了**。一定要开尔文接法、探到器件的引脚根部。

> **一句话记住：Ron 决定压降和发热，还直接串进 PDN 直流阻抗。手册 Ron 是 25℃/高 VIN 的典型值，高温+低 VIN 会让它翻倍——必须在真实工况下、用四线开尔文法（灌已知电流、贴引脚量压降，Ron=ΔV÷I）测，才是你板子上的真 Ron。**

## 三、软启动斜率：用一个电容，设定上电有多快

上一篇（第 19 篇）我们讲过：开机那一下，一堆空电容看着像短路，会涌进几十上百安的浪涌电流。压它的一大利器，就是 Load Switch 的**软启动**——让输出电压**慢慢爬**上去，而不是瞬间怼满。电压爬得慢，dV/dt 小，按 I = C × dV/dt，涌流自然就小。

Load Switch 是怎么控制这个"慢"的？绝大多数带软启动的型号，都留了一个引脚（常叫 **CSS、CT 或 dV/dt**），**外面挂一颗小电容**：这颗电容越大，内部给输出充电的斜率就越缓，输出电压上升就越慢、软启动时间越长。等于说，**你用一颗电容，就能设定这颗开关"开门"的快慢**。

那测什么？两件事一起看：

- **上升斜率 / 软启动时间**：示波器抓输出电压，量它从 0 爬到目标电压用了多久、斜率 dV/dt 是多少，跟你选的电容对不对得上；
- **涌流峰值**：同时挂电流探头，看这条斜率下，涌流峰值压到了多少——这才是软启动的**目的**。

一个常见翻车：软启动电容选小了，斜率太陡，涌流没压下来；或者选太大，上电太慢，导致后级**上电超时、时序错位**。所以软启动斜率不是越缓越好，是要在"涌流够小"和"上电够快、不误时序"之间**卡一个平衡**，测的就是这个平衡点。

> **一句话记住：Load Switch 的软启动靠一颗外挂电容（CSS/dV/dt 引脚）设定输出上升斜率——电容越大、爬得越慢、涌流越小。测两样：示波器量 VOUT 上升斜率/软启动时间，电流探头量对应的涌流峰值。斜率不是越缓越好，要在压涌流和不误上电时序之间取平衡。**

![图3动图：软启动斜率对比——CSS 电容小则 VOUT 陡升、涌流尖峰高；电容大则 VOUT 缓升、涌流峰值明显压低；同时标注软启动时间不能太长以免误时序](/images/26/ill03b_slew_anim.gif)

## 四、输出放电（QOD）：关断后，输出得干净利落地归零

第三样本事，最容易被忽略：**输出放电**，英文常写 **QOD（Quick Output Discharge）**，或者叫 output discharge。

问题是这样的：Load Switch 一关断，输入和输出就断开了。可输出端往往并着不少电容，还有后级模块本身的容性——**开关一断，这些电容上的电荷没地方去，输出电压会"挂"在那里，慢慢地、经过负载电阻拖很久才掉下来**。这个**残留电压**会惹麻烦：

- **下次开机时序错乱**：这一路还没放干净、残着 1 点几伏，另一路已经先起来了，两路电压的上下电顺序就乱了——正好破坏我们下一节要讲的时序；
- **后级芯片处于"半死不活"的欠压区**：既没完全断电、又没正常供电，容易进**闩锁（latch-up）**或者异常状态；
- **反复插拔 / 快速重启**时，残压没清，行为变得**不可复现**，最难查。

带 QOD 的 Load Switch，一关断就在**输出到地之间接一个内部放电电阻**，主动把输出电压快速拉到 0。测法很直接：示波器抓关断瞬间的输出电压，量它**从关断到跌到 10%（或某个门限）用了多久**——带 QOD 的可能几百微秒到几毫秒就干净了，不带的可能拖几十毫秒甚至更久。

要不要 QOD、放电要多快，取决于你的时序要求。**要重启快、时序严的场合，务必选带 QOD 的、并实测这个放电时间。**

> **一句话记住：Load Switch 关断后，输出电容上的电荷若没处泄，会残留一个"拖尾电压"，导致下次上电时序错乱、后级进闩锁、行为不可复现。带 QOD（输出放电）的型号会内部接放电电阻主动泄放——测法是示波器量"关断到输出跌到 10%"的时间，时序严的场合务必选带 QOD 并实测。**

## 五、通断时序：来得准不准时，多路怎么排队

最后一项，**时序**。这是 Load Switch 相较于"一根导线"最值钱的本事——它能让供电"**按你排的顺序、按你定的节奏**"上电、下电。要测两个层面。

**第一层：单颗的通断延迟。** 从 EN 使能信号变化，到输出电压真正响应，中间有一段**确定的延迟**（turn-on delay / turn-off delay）。测法：示波器一个通道抓 EN、一个通道抓 VOUT，量两者之间的时间差。这个延迟由芯片内部逻辑加上你的软启动电容共同决定，**要测出来、并确认它落在你系统时序预算之内**。

**第二层：多路的上下电顺序。** 一块板子上十几路电，谁先谁后、间隔多少，是有讲究的（第 18 篇专门讲过上下电时序和单调性）。Load Switch 是实现这个顺序的常用手段：

- **菊花链（daisy-chain）**：把前一颗开关的"输出好了"信号（PGOOD 或 VOUT 经分压）接到后一颗的 EN——前一路稳了，才放后一路，天然排成先后顺序；
- **统一由 MCU / PMIC 控 EN**：用一个控制器按预设时刻逐个拉高各路 EN。

不管哪种，测的都是**多路 VOUT 同屏抓波形**，确认上电顺序对、间隔够、下电顺序也对（通常下电反过来）。这里最容易和上一节的**输出放电**联动翻车：某一路关断后残压没放干净，就会让"下电顺序"名存实亡——所以时序和 QOD 得一起看。

> **一句话记住：Load Switch 时序测两层——① 单颗：示波器同抓 EN 与 VOUT，量通断延迟，确认在时序预算内；② 多路：菊花链（PGOOD 串 EN）或 MCU/PMIC 控 EN 排上下电顺序，多路 VOUT 同屏抓波形验顺序与间隔。注意残压（QOD）没放净会让下电顺序失效，两者要一起看。**

## 六、把四项收成一张清单

Load Switch 看着简单，真要测对，四项一个都不能少，而且每一项都得**在真实工况下**测，别信手册那个理想数字。收成一张清单：

- **Ron 导通电阻**：真实 VIN / 负载 / 温度下，四线开尔文法测压降，Ron = ΔV ÷ I；核压降和发热，别忘了它进 PDN 直流阻抗。
- **软启动斜率**：示波器量 VOUT 上升斜率 / 软启动时间，电流探头量涌流峰值；斜率在"压涌流"和"不误时序"间取平衡，对应调 CSS 电容。
- **输出放电 QOD**：示波器量关断后 VOUT 跌到 10% 的时间；时序严 / 重启快的场合务必选带 QOD 并实测。
- **通断时序**：单颗量 EN→VOUT 延迟；多路菊花链 / 控制器排上下电顺序，多路 VOUT 同屏验顺序与间隔，和 QOD 联动看。

这四项测下来，你才算真正"驯服"了这颗小开关——它接得通（Ron 小）、开得稳（斜率压住涌流）、关得净（QOD 无残压）、来得准（时序对）。

> **一句话记住：Load Switch 四项一个不能漏——Ron（接得通）、软启动斜率（开得稳）、输出放电（关得净）、通断时序（来得准）；每项都在真实工况下测，手册典型值只是参考。**

### 常见翻车速查

- **Ron 实测远大于手册** → 手册是 25℃/高 VIN 典型值；你的工况高温+低 VIN，Ron 翻倍属正常，务必按真实工况测。
- **软启动没压住涌流** → CSS 电容太小、斜率太陡；加大电容放缓 dV/dt，但别大到上电超时误时序。
- **关机后残压不掉、下次上电乱** → 没选带 QOD 的型号，或输出容量太大放电慢；换带 QOD 型号并实测放电时间。
- **多路上电顺序错乱** → 菊花链 PGOOD/EN 没串对，或某路残压未清；同屏抓多路 VOUT 复核，QOD 一起查。
- **压降量不准** → 表笔没开尔文接、没贴引脚根部；一两百毫伏的压降对接触电阻极敏感。

### 关键公式 / 数值

- 压降：ΔV = IOUT × Ron；发热：P = IOUT² × Ron。
- 涌流：I_inrush ≈ C_OUT × dV/dt（软启动就是压 dV/dt）。
- Ron 反推：Ron = ΔV(实测压降) ÷ IOUT。
- 例：手册 Ron 20mΩ@25℃/5V；实测 3A 下压降 200mV → 实际 Ron ≈ 66mΩ（高温低压工况）。
