# 测试计划与测试用例：测了两周的芯片，量产为什么还批量重启

<div class="article-meta"><span class="badge">📘 实战番外</span><span class="badge">⏱ 约 23 分钟</span><span class="badge qr">🔧 含工程师速查</span></div>


一颗常见的降压电源芯片，测试台上测了整整两周——输出精度、纹波、效率、瞬态，一项项测下来数据都挺漂亮，报告签字，放行量产。结果货到客户现场，批量开机重启。

追根因，问题不在芯片本身，而在一项**从头到尾没人碰过**的功能：过流保护。datasheet 里明明白白写着限流点和短路保护，测试报告里却干脆没有这一项——不是测挂了，是压根没进过测试范围。

你可能也见过另一种更隐蔽的情况：同一颗芯片、同一个"测输出纹波"的任务，交给三个人，测出三个差出一倍的结果，谁也说不清到底哪个对。原因很简单，"怎么测"从来没写清楚，全凭各自发挥——探头怎么接、带宽限没限、在什么负载下测，三个人三套做法。

这两件事，其实是同一类问题的两张脸：**会测每一项，不等于能把一颗芯片测明白。** 前一个缺的是**系统性**——一份能兜住全局、让你一眼看出"还差哪项没测"的测试计划；后一个缺的是**可复现性**——一条条写死了条件、换谁来测都是同一个结果的测试用例。

这一篇，我们就拿一颗大家都能买到、datasheet 写得相当规范的降压芯片 **TI TPS54331**（3.5–28V 输入、3A 输出、集成高边 MOS）当靶子，手把手走一遍：从零挖出测试项、写出一份测试计划，再把它展开成能直接执行的测试用例。方法本身跟这颗芯片没关系，学会了，换任何一颗 IC 都能照着套。

读完这篇，你能带走：

- ✅ 拿到一颗陌生芯片，怎么从 datasheet + 应用场景**把测试项挖全、不漏**
- ✅ 怎么写一份**工程实战版测试计划**（范围 / 项清单 / 条件矩阵 / 判据 / 放行标准）
- ✅ 怎么把一个测试项**展开成一条条可复现的测试用例**，判据从 datasheet 哪来
- ✅ 怎么用一张**追溯矩阵**证明自己没漏项、可复现、可追溯
- ✅ 一套**空白模板 + TPS54331 填好范本**（文末 Excel 可下载），换任何 IC 直接套

## 一、先分清：测试计划 vs 测试用例，谁是谁

很多人写测试文档，一上来就甩一张 Excel，把"测什么"和"怎么测"糊在一起。其实这是两层东西，分不清就容易两头都做不好。

打个比方：

- **测试计划 = 打仗前的作战地图。** 它回答"我要打哪些目标、为什么打、按什么顺序、要多少兵力、打到什么程度算赢"。它统领全局。
- **测试用例 = 每个目标的战术动作卡。** 它回答"这一个目标，具体怎么打、什么天气打、打中的标准是多少、怎么记战果"。它能照着执行。

落到测这颗 TPS54331：

- 计划层：这颗芯片我要测哪些项（精度、纹波、效率、瞬态、启动、过流、过温……）、每项为什么测、优先级、在什么输入 / 负载 / 温度下测、用什么仪器、测到什么标准算过、不过怎么办。
- 用例层：拿"负载调整率"这一项，展开成——Vin=12V、负载从空载到 3A、25℃、四线开尔文接法、判据是 Vout 变化 ≤ 多少 mV（来自 datasheet）、步骤 1234、记录表格。

关系一句话就能说清：**计划决定"测什么、测到什么程度算过"，用例决定"具体怎么测、怎么算过"。一个测试项，配上不同的测试条件，会展开成多条测试用例。**

为什么非得分开？回头看开头翻车那颗芯片，坑就在这：如果只有一堆零散用例（测了纹波、测了精度），没有一份统领的计划去清点"这颗芯片一共该测哪些项、哪些是必测",你根本不知道自己漏了"过流保护"这一整项——测到哪算哪，漏了也没人发现。

> **一句话记住：** 测试计划是"测什么、测到什么程度算过"的作战地图，测试用例是"具体怎么测、怎么算过"的动作卡；先有计划兜住全局不漏项，再用用例保证每一项测得可复现。

## 二、测试项从哪来：四个抽屉，挖到不漏

写计划的第一步，也是最容易翻车的一步：**把测试项挖全**。漏项往往不是能力问题，是方法问题——没有一套系统的挖项方法，全凭经验和记忆，迟早会漏。

给你四个"抽屉"，对着一颗芯片逐个过一遍，基本不会漏。以 TPS54331 为例：

**抽屉①：datasheet 的电气特性表——每一行规格，就是一个测试项。**
这是最硬、最不该漏的来源。翻到 TPS54331 datasheet 的 Electrical Characteristics（电气特性）那张大表，从上往下一行行看：参考电压 0.8V、开关频率 570kHz、高边 MOS 导通电阻、静态电流、峰值限流点（3.5–5.8A）、UVLO 阈值、过温点（165℃）……每一行有数值、有 min/typ/max，就是一个可测、可判的测试项。厂家把规格写给你了却不测，等于考题发到手里交白卷。（开头漏掉的过流，就藏在这张表里——逐行扫一遍根本漏不掉。）

**抽屉②：应用场景——这颗电源到底给谁供电，决定测试侧重。**
同一颗 TPS54331，带不同负载，重点完全不同：

- 给 MCU / CPU 核供电 → 瞬态响应、纹波是重点（负载突变会不会导致复位）
- 给 RF / ADC 供电 → 纹波噪声、开关频率杂散是重点（脏电源直接毁性能）
- 给一堆数字外设 → 上电时序、负载调整率是重点

datasheet 不会告诉你你的应用侧重，这一条得你结合板子来定。

**抽屉③：保护与异常特性——正常能工作不算完，得测它扛不扛得住。**
EN / UVLO 使能与欠压锁定、逐周期过流限制、短路 hiccup（打嗝）、过温关断、输出过压……这些平时不出场，一出场就是救命的，恰恰最容易漏——因为正常功能测试里根本碰不到它们。必须专门列一类"异常与保护"项，逼自己去测。

**抽屉④：标准 / 客户 / 可靠性要求——datasheet 之外的硬约束。**
工作温度范围、EMC / EMI、上电时序要求、寿命 / 老化、客户规格书里额外的验收项。这些来自你的项目和行业，不在芯片手册里，得单独收集。

方法很朴素：拿张纸或开个 Excel，四个抽屉一字排开，对着 datasheet 和你的应用，一项一项往抽屉里扔，扔完就是一份测试项清单。这份清单，就是下一步测试计划的骨架。

![图2：挖测试项的四个抽屉——datasheet 电气特性表 / 应用场景 / 保护与异常 / 标准与客户，汇成一张测试项清单](/images/b02/ill02_sources.png)

> **一句话记住：** 测试项别靠脑补，对着四个抽屉挖——datasheet 每行规格、你的供电场景、所有保护与异常、标准与客户要求；其中 datasheet 那张电气特性表逐行扫，是最不该漏的一类。

## 三、写测试计划：工程实战版模板，用 TPS54331 填一遍

测试项挖全了，接下来把它们组织成一份**测试计划**。网上的模板动辄几十页，一堆管理流程。我们做的是工程实战版：**技术为主干，管理要素一句话带过**。一份真能指导你测一颗芯片的计划，核心就这七块：

**① 目的与范围。** 一句话说清：验证 TPS54331 在本项目应用条件下（比如 Vin=12V 典型、输出 3.3V / 2A）的功能、性能和保护是否满足 datasheet 与项目要求。范围要划边界——测什么、明确不测什么（例如"本次不含 EMC 认证，另行安排"），免得后面扯皮。

**② 被测对象与依据文档。** DUT = TPS54331 + 你的应用电路（列出关键外围：电感、输出电容、决定 Vout 的分压电阻）。依据 = TPS54331 datasheet（**写清版本 / 日期**）、应用原理图版本、相关标准、客户规格书。判据全部溯源到这些文档，**不拍脑袋**。

**③ 测试项清单（分类 + 优先级）。** 把第二步挖的项按类目组织，正好对应《电源器件测试地图》(q01) 里 DCDC 那一列：

- 基本功能：能不能正常启动、输出电压对不对
- 静态性能：输出精度、线 / 负载调整率、效率、静态电流
- 动态性能：负载瞬态响应、纹波与噪声、开关节点波形
- 保护与异常：过流 / 短路 hiccup、UVLO、过温、输出过压、EN 控制
- 启动与时序：软启动时间、上电 / 掉电、浪涌
- 环境与可靠性：高低温、长时间老化

每项标优先级：

- **P0 必测（放行红线）**：输出精度、纹波、过流保护、启动——漏一个都不许签字
- **P1 重要**：效率、瞬态、UVLO、温度
- **P2 补充**：极限边界、老化

（📌 每一类具体怎么测、判据怎么读，本系列都有对应篇，速查区列了回链；这里只讲"怎么把它们组织进计划"。）

**④ 测试条件矩阵——工程实战里最值钱的一块。** 一个项不是测一次就完。TPS54331 输入能从 3.5V 到 28V、负载从 0 到 3A、温度可能 -40 到 85℃，三个维度一乘，组合直接爆炸。计划里要**明确列出条件矩阵，并说清怎么裁剪**：

- 输入 Vin：取 min 3.5V / typ 12V / max 28V 三个角点（不是每 1V 测一遍）
- 负载：空载 / 半载 1.5A / 满载 3A / 过载（触发保护）
- 温度：常温 25℃ / 高温 85℃ / 低温（按需）

裁剪原则就一句：**不做全组合，抓关键角点和最坏情况**。效率测 Vin 三点 × 负载扫描；精度测角点组合；瞬态测 typ 输入下的负载跳变。矩阵列清楚，测起来才不会又漏又重。

**⑤ 测试环境、仪器与工装。** 可编程直流电源（给 Vin）、电子负载（拉载 / 瞬态）、示波器（带宽够，纹波要 20MHz 限带 + 同轴 / 弹簧地）、高精度万用表、温箱、测试夹具。列清型号和关键设置——这直接决定用例能不能复现（开头第二个坑，三个人测纹波三个结果，一半就出在探头接法和带宽没统一）。

**⑥ 样品与通过标准。** 样品数量 ≥ 3（尽量覆盖不同批次），写明板子状态。通过 / 退出标准就一条最关键：**P0 项全部通过、无未关闭的 P0 缺陷，才允许放行**——把这条写进计划，开头那种"过流没测就签字"的事故根本发生不了，因为放行前必须对照 P0 清单逐项打勾。

**⑦ 风险与排期（一句话带过）。** 样品不足、温箱排队、仪器带宽不够……列出来、留个备用方案即可，管理细节不展开。

我把上面整理成了一张**空白测试计划模板**（文末 Excel 可下载），并用 TPS54331 填了一份范本。这里先给你看计划里最核心的"测试项 + 条件 + 判据来源"长什么样：

| 编号 | 测试项 | 类别 | 优先级 | 关键条件 | 判据（来源） |
|---|---|---|---|---|---|
| TP-01 | 输出电压精度 | 静态 | P0 | Vin=12V，空载 / 满载，25℃ | Vout 在标称 ±2% 内（参考 0.8V±1% + 分压电阻 1% 推算，具体查 datasheet） |

## 四、把测试项，展开成测试用例

计划是"测什么"，用例是"怎么测、怎么算过"。一条合格的测试用例，要让**任何一个人拿着它，都能测出同一个结果**——这正是开头第二个坑（三人测纹波三结果）的解药。

一条测试用例，至少要有这些字段：

- **用例 ID**：如 TC-04-01
- **关联测试项**：TP-04 过流保护（能追回计划）
- **判据依据**：datasheet 第 X 节 / 标准第 X 条（判据从哪来，可追溯）
- **优先级**：P0 / P1 / P2
- **前置条件**：DUT 状态、连线、仪器已校准、环境温度
- **测试条件**：写死每一个数值——Vin、负载、温度、探头接法、仪器带宽
- **测试步骤**：编号，具体到能照做（"1. 设电源 12V；2. 电子负载 CC 模式从 0.5A 起、每步 0.5A 加载……"）
- **预期结果 / 判据**：带数值、带来源（"达 3.5–5.8A 触发逐周期限流，Vout 下垂但不失控；短路时进 hiccup，短路解除后自动恢复"）
- **实测记录**：留空给执行时填（数值 + 波形截图编号）
- **Pass / Fail + 备注**

**核心方法：一个测试项 × 条件矩阵 = 多条用例。** 拿 TP-02 负载调整率来说，条件矩阵是 Vin(3.5 / 12 / 28V) × 温度(25 / 85℃)，那它就展开成 3×2 = 6 条用例（TC-02-01 到 TC-02-06），每条锁定一组条件。既覆盖全，又每条都能复现。

**好用例 vs 烂用例**，差别全在"能不能复现"：

- ✗ 烂用例："测输出纹波，OK。"——Vin 多少？负载多少？带宽限了没？探头怎么接？三个人做，一个用 10:1 探头拖一大段地线量出 50mV，一个用 20MHz 限带同轴贴测量出 15mV，差出三倍，谁对？
- ✓ 好用例："Vin=12V，负载 3A，温度 25℃，示波器 20MHz 带宽限制，探头用弹簧地 / 同轴贴测输出电容两端，测 Vout 纹波峰峰值，判据 ≤ 30mV。"——谁来测都是这个数。

同样，我把**空白用例卡片模板**和几条 **TPS54331 填好的代表性用例**（精度、纹波、瞬态、过流 hiccup、软启动各一条）都放进了文末 Excel，你改改条件就能套用。

## 五、不漏、可复现、可追溯：一张追溯矩阵 + 一次评审

计划和用例都写好了，怎么证明"真的没漏、真的靠谱"？靠两样东西。

**① 追溯矩阵（Traceability Matrix）——防漏的终极武器。** 做一张表，把这条链子串起来：

**datasheet 规格 / 需求 → 测试项 → 测试用例 → 判据来源 → 报告结果**

每一个 datasheet 规格，都能往右找到对应的测试项、用例、判据和最终结果；反过来，每一条用例，都能往左追到它为什么测。这样一眼就能看出：**有哪个规格，右边一栏是空的？空的就是漏测。** 开头漏的过流保护，在追溯矩阵里就是——datasheet 写了限流 3.5–5.8A，右边"测试项"一栏却空着，签字前扫一眼就抓出来了。

**② 评审——别一个人闷头测。** 计划和用例写完，拉上开发、项目、资深测试一起过一遍。开发能告诉你哪里是设计薄弱点要重点测，项目能告诉你客户最在意什么。评审花半天，比返工两周划算太多。

再配一份**覆盖性自检清单**，放心里过一遍：

- datasheet 电气特性表每一行，都有对应用例了吗？
- 所有保护（过流 / UVLO / 过温 / 过压），都单独有用例了吗？
- 每条用例的条件，写到"换人能复现"了吗？
- 每个判据，都能指回 datasheet 或标准吗？

![图8：追溯矩阵把"规格 ↔ 测试项 ↔ 用例 ↔ 判据来源 ↔ 结果"串成一条链，右边空格一眼看出漏测](/images/b02/ill08_trace.png)

> **一句话记住：** 追溯矩阵把"规格 ↔ 项 ↔ 用例 ↔ 判据 ↔ 结果"串成一条链，哪个规格右边是空的就是漏测；写完拉上开发和项目评审，半天评审省两周返工。

## 六、换任何 IC 都能套：把方法变成你自己的

回头看，我们全程拿 TPS54331 举例，但你有没有发现——**方法本身跟 TPS54331 一点关系都没有**。换成一颗 LDO、一颗 PMIC、一颗 Load Switch，甚至一颗完全不同的接口芯片，流程一模一样：

1. **挖项**：四个抽屉（datasheet 每行规格 / 应用场景 / 保护异常 / 标准客户）
2. **写计划**：目的范围 → 分类分级项清单 → 条件矩阵 → 仪器 → P0 全过才放行
3. **展开用例**：每项 × 条件矩阵 = 多条可复现用例，条件写死、判据溯源
4. **追溯**：一张矩阵串起规格 ↔ 项 ↔ 用例 ↔ 判据 ↔ 结果，扫空格防漏

拿到一颗你从没测过的芯片，别慌，也别急着上仪器。先花半天，把它的 datasheet 按这四步过一遍——你会比"直接开测"的人测得更全、更快、心里更有底。

（📌 具体每一类项怎么测、判据怎么读，配合《电源器件测试地图》(q01) 和对应各篇一起用：地图帮你定位"这颗芯片该测哪些"，本篇帮你把"该测的"落成计划和用例。）

> **一句话记住：** 挖项四抽屉、计划七要素、用例条件写死、追溯矩阵扫空格——这套流程与具体芯片无关，任何 IC、任何模块都照着套。

## 写在最后

回到开头那两件事。一颗电源芯片测了两周、签字放行，量产却批量重启——不是测试员不会测，而是缺了一份**能兜住全局的测试计划**，让"过流保护"这一整项从头到尾没进过视野。三个人测同一个纹波测出三个结果——也不是谁手艺差，而是缺了**能复现的测试用例**，条件全靠各自发挥。

一颗芯片测得好不好，从来不只是"每一项会不会测"。**更是你能不能把一颗陌生的芯片，系统地测全、测得可复现、测得可追溯。** 会挖项、会写计划、会展开用例、会做追溯——这四件事加起来，才是让开发信服、让项目放心、让你签字时心里有底的专业度。

这条**实战番外**支线，专门聊这些"技术之外、但同样决定你专不专业"的实战功夫，以后会不定期加餐。

### 测试计划模板（工程实战版 · 七块）
1. 目的与范围（测什么 / 不测什么，划边界）
2. DUT 与依据文档（芯片 + 原理图 + **datasheet 版本号** + 标准 + 客户规格）
3. 测试项清单（分类 + 优先级 P0 / P1 / P2）
4. 测试条件矩阵（输入 × 负载 × 温度，抓角点不做全组合）
5. 环境 / 仪器 / 工装（型号 + 关键设置）
6. 样品与通过标准（**P0 全过、无未关闭 P0 缺陷才放行**）
7. 风险与排期（一句话带过）

### 挖测试项 · 四个抽屉
| 抽屉 | 挖什么 | 对 TPS54331 举例 |
|---|---|---|
| ① datasheet 电气特性表 | 每行规格 = 一个测试项 | Vref 0.8V / fsw 570kHz / 限流 / UVLO / OTP |
| ② 应用场景 | 给谁供电定侧重 | 供核 → 瞬态；供 RF → 纹波 |
| ③ 保护与异常 | 平时不出场的救命项 | 过流 hiccup / UVLO / 过温 / 过压 / EN |
| ④ 标准 / 客户 / 可靠性 | datasheet 之外的约束 | 温度范围 / EMC / 时序 / 老化 |

### 测试用例卡片 · 必填字段
用例 ID ｜ 关联测试项 ｜ 判据依据(datasheet / 标准) ｜ 优先级 ｜ 前置条件 ｜ 测试条件(数值写死) ｜ 测试步骤(可照做) ｜ 预期结果 / 判据(带来源) ｜ 实测记录 ｜ Pass / Fail

> **一个测试项 × 条件矩阵 = 多条用例**（如负载调整率 Vin 三点 × 温度两点 = 6 条）。

### 好用例 vs 烂用例
| | 烂用例 | 好用例 |
|---|---|---|
| 写法 | "测纹波，OK" | "Vin12V / 满载 3A / 25℃ / 20MHz 限带 / 同轴贴测输出电容，纹波 ≤ 30mV" |
| 后果 | 三人测三结果差 3 倍 | 换人测同一个数 |
| 关键 | 条件靠发挥 | 条件写死 + 判据溯源 |

### 追溯矩阵（防漏神器）
**datasheet 规格 → 测试项 → 用例 → 判据来源 → 报告结果**
- 每个规格往右都要有对应，右边空格 = 漏测
- 写完拉开发 + 项目评审；再走一遍覆盖性自检清单

### 常见踩坑
- [ ] 只测 datasheet 给的数、漏了保护类（过流 / 过压最常漏）
- [ ] 用例条件不写全（探头接法 / 带宽 / 温度）→ 不可复现
- [ ] 判据拍脑袋、不查 datasheet 版本
- [ ] 条件矩阵做全组合 → 测不完；或只测 typ → 漏边界
- [ ] 没做追溯矩阵 → 漏没漏全凭运气
- [ ] P0 没定义、没卡放行 → 漏项也照签字
