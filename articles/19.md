# 稳态才吃 2 安，一开机却涌进 120 安——MOS 烧了、保险也跳了

<div class="article-meta"><span class="badge">📘 模块二 · 电源完整性 PI · 软启动与上电浪涌</span><span class="badge">⏱ 约 15 分钟</span><span class="badge qr">🔧 含工程师速查</span></div>


一块板子，正常跑起来稳态电流就 **2A**。你照着这个选了保险、选了前级的 MOS 开关，留足了余量，按理说稳得很。可一插上电源、电源开关一合——"**啪**"一下，保险跳了，前级那颗 MOS 还烧了。

你换上新的 MOS、换上保险，拿示波器加个电流探头再抓一次开机的瞬间。屏幕上一看就傻了：**开机那一下，电流尖峰直接冲到了 100 多安**，虽然只持续了零点几毫秒，但就这一下，把 MOS 和保险全干掉了。

稳态明明才 2A，哪来的 100 多安？

这股只在开机那**一瞬间**出现、稳态时压根看不到的大电流，就叫**上电浪涌（inrush current）**。今天这篇，就讲清楚它从哪来、有多大危害，以及工程上怎么用**软启动（soft-start）**把它压下去。

**读完这篇，你能搞懂：**

- ✅ 开机那一下**几十上百安的涌流到底从哪来**——为什么一堆电容是"罪魁祸首"；
- ✅ 一个能算的公式 **I ＝ C × dV/dt**，让你估出自己板子的涌流有多凶；
- ✅ 涌流大了会**惹哪些祸**——烧 MOS、跳保险、把母线拉趴下让旁边设备一起复位；
- ✅ **软启动**到底在"软"什么——以及 NTC、Load Switch、浪涌限流 IC 四种压涌流的手段；
- ✅ 这一下**怎么测**——电流探头 + 单次触发，盯涌流峰值、持续时间和母线跌落。

## 一、几十安的涌流，到底从哪来？

先解决最反直觉的问题：稳态才 2A，开机怎么会冲出 100 多安？

答案就俩字：**电容**。

板子上电源入口、各级输出，到处都并着**大大小小的电容**——输入端的体电容（bulk cap）动不动就几百上千微法。而电容有个天生的脾气：**两端电压不能突变**。开机前，这些电容全是 **0V**；电源开关一合，输入电压"啪"地一下就上来了，可电容还停在 0V——这一瞬间，一个 0V 的大电容，看上去就**跟一根短路线差不多**。电源为了在极短时间内把它充到位，就得灌进去一股**巨大的瞬时电流**。

这股电流有多大，是能算的，公式很简单：

**I ＝ C × dV/dt**

电流 = 电容值 × 电压的变化速率。电容越大、电压上升得越陡（dV/dt 越大），涌流就越凶。

举个能让你心里有数的例子：一个 **1000µF** 的输入电容，如果电源让它在 **0.1ms** 内就冲到 **12V**，那这一下的涌流是：

**I ＝ 1000µF × 12V ÷ 0.1ms ＝ 120A**

看到没——就这么一个电容，开机冲一下就是 120 安。板子上电容还不止一个，几个一叠加，涌流轻轻松松就是稳态电流的**二三十倍**。这就是开头那块"稳态 2A、开机 120A"的板子，涌流的真正来源。

> **一句话记住：上电浪涌（inrush）= 开机瞬间给一堆 0V 大电容充电的冲击电流。电容电压不能突变，开机那一刻大电容近似短路，I ＝ C × dV/dt——电容越大、电压升得越陡，涌流越大，常是稳态电流的几十倍。**

## 二、涌流大了会惹什么祸？

你可能会想：不就一下子嘛，零点几毫秒就过去了，能有多大事？事还真不小，涌流惹的祸有三类：

**第一·烧器件。** 开机这一下大电流，全都从前级的**保险、MOS 开关、连接器、PCB 走线**上挤过去。保险有个 **I²t（熔断能量）**指标，涌流又大、持续又有那么一会儿，能量一够它就熔断；MOS 则有**安全工作区（SOA）**，瞬间又是大电流又扛着压差，工作点冲出 SOA，当场就击穿——这就是开头烧 MOS 的原因。

**第二·把母线拉趴下，殃及邻居。** 你的板子往往不是独享一个 12V 母线，旁边还挂着别的设备。你这一下猛抽 100 多安，**12V 母线被瞬间拉低**（电源内阻 + 线阻上掉一大截压），结果旁边那台正常工作的设备，供电电压"咣"地跌破它的欠压门限，**跟着复位、甚至死机**。明明是你开机，邻居遭了殃，这种问题最难查。

**第三·上电直接失败。** 涌流太大，可能瞬间触发上游电源自己的**过流保护（OCP）**，电源以为短路了，直接打嗝（hiccup）或关断，你的板子**反而上不了电**，一开机就"啪嗒啪嗒"地反复重启。

> **一句话记住：涌流三大祸——① 大电流烧保险（I²t 超限）/ 烧 MOS（冲出 SOA）；② 把共用母线瞬间拉低，害得旁边设备一起复位；③ 触发上游电源过流保护，自己反而起不来。**

## 三、软启动：不让电压"一下子"冲上来

知道了涌流来自 **I ＝ C × dV/dt**，对策其实就藏在这个公式里：**电容 C 一般不好减（它是为了滤波、储能，必须有），那就只能去压 dV/dt**——别让电压"啪"地一下冲上来，而是让它**慢慢地、平滑地爬**上去。

这就是**软启动（soft-start）**：人为地把上电的**电压斜率压缓**，dV/dt 一小，涌流 I 就跟着小下来了。

还是上面那个例子：1000µF 的电容，硬上电时 0.1ms 冲到 12V，涌流 120A；要是用软启动，把这个上电过程**拉长到 10ms**（dV/dt 缓了 100 倍），涌流立马降到：

**I ＝ 1000µF × 12V ÷ 10ms ＝ 1.2A**

**120A 直接变 1.2A**，差了一百倍——软启动的威力就这么直接。说白了，软启动就是拿**一点上电时间**，换**一个平顺、不伤器件的开机过程**。

> 你可能发现了：这跟上一篇讲的"上电斜率 ramp rate 不能太快"其实是**一件事的两面**——斜率太快，一边会让 POR/单调性出问题，一边就是这里说的涌流过大。软启动，正是主动去管这个斜率。

> **一句话记住：软启动 = 主动把上电电压斜率 dV/dt 压缓，让电压平滑爬升而不是猛冲。因为 I ＝ C×dV/dt，dV/dt 缓 100 倍，涌流就小 100 倍（例：120A → 1.2A）。本质是拿一点上电时间，换一个不伤器件的平顺开机。**

## 四、软启动怎么实现？四种常见手段

软启动不是一种电路，而是一类思路。工程上压涌流，常见有这么四招，从简单到讲究：

**① 芯片自带的软启动脚（SS）。** 大多数 DCDC、LDO 都有个软启动引脚，外面挂一颗小电容（或者芯片内部就集成了）。上电时，这颗电容给内部基准**缓慢充电**，于是输出电压跟着基准**一点点爬上来**，涌流天然就被压住了。软启动时间，基本由这颗电容定。这是最常用的一招。

**② NTC 热敏电阻（最便宜）。** 在电源入口串一颗 **NTC**（负温度系数热敏电阻）。开机冷态时它**阻值高**（比如 10Ω），把涌流硬生生限住；通电后它自己发热、阻值掉到 **1Ω 以下**，稳态时损耗就小了。便宜、简单，但缺点也明显：稳态一直有点发热损耗，而且**热机后马上断电再开**，NTC 还没凉下来、阻值还低，限流就**失效**了。

**③ Load Switch / 理想二极管控压摆率。** 用一颗**负载开关（Load Switch）**或理想二极管芯片，在它的栅极控制脚上挂个电容，**专门控制 MOS 开通的快慢（slew rate）**——让导通过程拉长，输出电压缓升，涌流被压平。很多带 dV/dt 控制脚的 Load Switch 就是干这个的。

**④ 浪涌限流 IC / 热插拔控制器（最讲究）。** 像**热插拔（hot-swap）控制器**这类专用芯片，不光控斜率，还**实时限流 + 看 MOS 的 SOA**，涌流一旦超过设定值就主动压住，保证 MOS 不冲出安全区。带电插拔的板卡、服务器背板上，基本都靠它。

选哪种，看成本和场合：要省钱用 NTC，要可靠、可重复开关就用软启动脚 / Load Switch / 热插拔 IC。

> **一句话记住：压涌流四招——① 芯片软启动脚（外挂小电容缓升基准，最常用）；② NTC 热敏电阻（冷态高阻限流，最便宜，但有发热、热机再开失效的坑）；③ Load Switch 控压摆率（栅极电容拉长开通）；④ 浪涌限流 / 热插拔 IC（限流 + 守 SOA，最可靠）。**

## 五、怎么测：抓住开机那一下的电流尖峰

涌流和上电时序一样，**只在开机那一瞬间出现、而且只来一次**。万用表那点采样速度根本抓不住，你得用**示波器 + 电流探头**，把"开机那一下"的电流波形完整抓下来。

具体这么干：

1. **加电流探头**：用电流探头（霍尔式或电流互感器式）卡在电源入口的总线上，量进板的总电流；电压探头同时量输入电压（或 12V 母线）；
2. **设单次触发**：开机只发生一次，用**单次（Single）触发**，触发电平设在电流明显抬头的地方，电源一合就把这一下冻住；
3. **读三个关键数**：① **涌流峰值** —— 尖峰冲到多少安，和保险 I²t、MOS 的 SOA、连接器额定去比；② **持续时间 / 能量** —— 尖峰有多宽，宽乘高大致就是冲击能量，对保险熔断很关键；③ **母线跌落** —— 看输入电压被这一下拉低了多少、会不会跌破旁边设备的门限。
4. **顺便看软启动时间**：输出电压从 0 爬到稳定花了多久，在不在手册/设计窗口内——太短说明软启动没生效，太长可能拖累开机速度。

冷热两种状态都要测：尤其用 NTC 的方案，一定要测**热机后立刻重新开机**那一下（NTC 还热着、阻值低），看涌流会不会失控。

> **一句话记住：涌流只在开机一瞬间来一次。必须用示波器 + 电流探头 + 单次触发抓下来，读三个数——涌流峰值（比保险/MOS 余量）、尖峰持续时间/能量（对保险 I²t）、母线跌落（会不会害邻居）；再核一下软启动时间。NTC 方案务必补测"热机再开"。**

## 七、写在最后

这一篇想让你记住一句话：**稳态电流多大，决定不了开机那一下有多凶。** 一块稳态才 2A 的板子，开机涌流冲到上百安，把 MOS、保险全干掉——根子就在那一堆电容、和那个陡得吓人的 dV/dt。

而对策也藏在同一个公式 **I ＝ C×dV/dt** 里：电容动不得，就去压 dV/dt——这就是**软启动**。拿一点上电时间，换一个平顺、不伤器件、也不殃及邻居的开机过程。

下次再碰到"一上电就跳保险 / 烧 MOS / 旁边设备莫名复位"，别只盯稳态电流了——上电流探头、单次触发，抓住开机那一下的电流尖峰，答案就在那个尖峰里。

下一篇，我们继续把电源讲扎实，聊聊电源的"安全气囊"——**过压、过流、过温、欠压锁定这四类保护（OVP / OCP / OTP / UVLO）到底在防什么、又该怎么测**。

### 判据表 1 · 上电浪涌与软启动核心判据

| 项 | 测什么 | 判据 / 参考 |
|---|---|---|
| **涌流峰值 I_inrush** | 开机瞬间进板的电流尖峰 | < 保险额定 / MOS 的 SOA 限值 / 连接器额定；留余量 |
| **涌流持续时间 / 能量** | 尖峰宽度，估 I²t 冲击能量 | < 保险熔断 I²t；< MOS SOA 在该脉宽下的允许值 |
| **母线跌落 ΔV** | 涌流把输入/共用母线拉低多少 | 不跌破本板及同母线其它设备的欠压门限 |
| **软启动时间 t_ss** | 输出从 0 爬到稳定的时长 | 在手册/设计窗口（典型 ms~几十 ms）；够缓压住涌流又不拖慢开机 |
| **上电斜率 dV/dt** | 输出电压上升速率 | 由 I=C·dV/dt 反推：满足涌流上限对应的最大斜率 |
| **NTC 热机复测** | 热机后立即重开的涌流 | NTC 未冷却、阻值低时涌流仍在限值内（否则方案不可靠） |

### 关键概念与公式

- **上电浪涌 inrush**：开机瞬间给一堆 0V 电容充电的冲击电流，常达稳态电流几十倍
- **核心公式 I ＝ C × dV/dt**：涌流 = 电容 × 电压变化率；压涌流主要靠压 dV/dt（软启动）
- **I²t（熔断能量）**：保险的熔断判据，涌流大且久就会熔断
- **SOA（安全工作区）**：MOS 同时扛大电流 + 高压差的允许边界，涌流冲出 SOA 即击穿
- **软启动 soft-start**：主动压缓上电斜率，让输出平滑爬升以限制涌流
- **hiccup（打嗝）**：上游电源过流后反复尝试重启的保护行为

### 压涌流四种手段对照

| 手段 | 原理 | 优点 | 坑 / 适用 |
|---|---|---|---|
| **软启动脚 SS** | 外挂电容缓充基准、输出缓升 | 最常用、可重复 | 软启动时间由电容定 |
| **NTC 热敏电阻** | 冷态高阻限流，热后降阻 | 最便宜、简单 | 稳态发热；**热机再开失效** |
| **Load Switch 控 slew** | 栅极电容拉长 MOS 开通 | 集成、可控 dV/dt | 选带 dV/dt 脚的型号 |
| **浪涌限流 / 热插拔 IC** | 限流 + 守 MOS SOA | 最可靠、带保护 | 成本高；带电插拔/背板用 |

### 测试设置

- **仪器**：示波器 + **电流探头**（霍尔式可测含直流的涌流；互感器式带宽高但不过直流）；电压探头量输入/母线
- **接法**：电流探头卡电源入口总线量进板总电流；电压探头量输入电压或共用母线
- **触发**：**单次（Single）触发**，触发源选电流（或上电使能），电平设在电流抬头处；开机一次抓全
- **读数**：① 涌流峰值 ② 尖峰持续时间/能量（估 I²t）③ 母线跌落 ΔV ④ 软启动时间 t_ss
- **冷热都测**：NTC 方案必须补测"热机后立即重开"那一下

### 常见翻车与对策

- **稳态有余量却跳保险/烧 MOS** → 没算涌流：上电流探头测峰值，按 I²t/SOA 重选器件或加软启动
- **开机害旁边设备复位** → 母线被涌流拉低：限涌流（软启动/Load Switch）或加大母线去耦/降内阻
- **一开机反复重启（打嗝）** → 涌流触发上游 OCP：减小 dV/dt、延长软启动时间
- **NTC 方案偶发失效** → 热机再开 NTC 没凉：改用软启动脚 / Load Switch / 热插拔 IC
